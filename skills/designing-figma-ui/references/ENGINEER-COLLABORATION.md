## 1. エンジニアのためのFigma基本操作

### 権限の種類

| 権限 | できること |
|------|-----------|
| Editor（編集者） | Design / Prototype / Inspect タブすべて使用可。画像書き出しも可 |
| Viewer（閲覧者） | Inspectタブのみ。画像書き出しはExportタブから（後述） |

**Editorとして招待されていない場合**: デザイナーに相談するか、`Duplicate to your drafts` で自分のDraftsに複製して編集権限を得ることができる（元ファイルとのリンクが切れるため、通常は避けるのが望ましい）。

### 基本的な移動・選択操作

```
画面の移動:     Space を押しながらドラッグ
選択:           V（Moveツール）で通常選択
深いレイヤーを選択: ⌘/Ctrl + クリック（入れ子を無視して直接選択）
1階層下を選択:  ダブルクリック
```

### コメントの確認

- `C` キーでコメントモードに切り替え
- デザインに添付されたコメントを確認・返信できる
- ピン留めアイコンをクリックするとコメントスレッドを常時表示可能

## 2. Inspectタブの活用

Inspectタブは、選択したオブジェクトの実装に必要な情報を表示する。**Viewer権限でも利用可能**。

### 表示される情報

| 項目 | 内容 |
|------|------|
| Typography | フォント名、ウェイト、サイズ、行高、字間 |
| Colors | カラー値（HEX / RGB）とスタイル名 |
| Layout | 幅・高さ・XY座標・Padding |
| Code | CSS / iOS / Android向けのコードスニペット |

### テキスト・カラーの読み方

```
名前が付いているテキスト（テキストスタイル適用済み）:
  Ag  Light/Heading/S
  Font: Roboto / Weight: 500 / Size: 14px / Line height: 16px
  → スタイル名が "Light/Heading/S" と判明。デザインシステムの定義と照合できる

名前がないテキスト（スタイル未適用）:
  同じ情報が表示されるが、スタイル名がない
  → デザイナーに確認推奨（スタイル適用漏れの可能性）
```

**テキストやカラーに名前が付いていない場合の対応:**
- スタイルの適用漏れ
- デザイン作業中（未定義）
- スタイルが意図的に未定義

いずれもデザイナーへの確認が適切。

### Viewer権限でのAuto Layout制約確認

Auto Layoutが適用されているFrameに限り、Inspectタブからレイアウト制約（Resizingプロパティ）を確認できる。Auto Layout未使用の場合はコードスニペットから挙動を推測する必要がある。

## 3. Designタブの活用

**Editor権限が必要**。Inspectタブより多くの情報を確認できる。

### 主要情報

| 項目 | 確認できること |
|------|--------------|
| コンポーネント情報 | バリアント・プロパティ（State / Rounded等）の確認。`⇒` アイコンで元コンポーネントへジャンプ |
| Constraints | 画面サイズや文字数が変わったときの挙動。`Left and right`なら横幅に追従するなど |
| Export設定 | @1x / @2x / @3x などの書き出し設定が確認できる |

### Constraints情報の実装への活用

```
Constraints: Left and right / Top
  → 実装: 幅100%、上端固定。画面サイズが変わると幅が変わる

Constraints: Center / Center
  → 実装: 水平・垂直中央揃え。画面サイズが変わっても中央を維持

Constraints: Right / Bottom
  → 実装: 右端・下端固定のアブソリュート配置

Constraintsが指定されていない場合:
  → デザイナーに確認を推奨（デザインの意図が不明確）
```

### 画像の書き出し（Editorの場合）

右パネル下部のExportセクションから@2x / @3x等を選択して書き出せる。

```
PNG / JPG:  ビットマップ画像（写真・グラデーション）
SVG:        ベクター画像（アイコン・シンプルなイラスト）
PDF:        ベクターのまま保存
WebP:       Webに最適化
```

**クリップボードにコピー（チャット・スライド貼り付け用）:**

| Mac | Windows |
|-----|---------|
| `⌘ + shift + C` | `ctrl + shift + C` |

## 4. Prototypeタブの活用

Prototypeタブで画面間のつながりとアニメーションを確認できる。

### 確認できること

| 項目 | 内容 |
|------|------|
| Flows一覧 | 定義されたFlow（ユーザーシナリオ）の一覧 |
| 画面遷移の矢印 | 画面同士のつながりをビジュアル表示 |
| インタラクション詳細 | 矢印クリックでTrigger・Action・Animation設定を確認 |
| プロトタイプの再生 | 再生ボタンで実際の動作を確認 |

### Viewer権限でのプロトタイプ確認

```
手順:
  1. InspectタブのFlowsにある目玉アイコンをクリック
     → 画面同士のつながりが表示される
  2. 画面をつなぐ矢印をクリック
     → 画面遷移時のアニメーションやタイミングを確認できる
  3. 再生ボタンをクリック
     → 実際のインタラクションを体験できる
```

## 5. 画像の書き出し

### Editor権限がある場合

Design タブの最下部 Export セクションから書き出す。

```
1. 書き出したいオブジェクトを選択
2. Design タブ下部の Export + ボタンをクリック
3. スケール（@1x / @2x / @3x）とファイル形式を選択
4. Export N layers をクリックして保存
```

### Viewer権限がある場合（Exportタブ）

```
ツールバーの Export タブを選択
  → 選択オブジェクトを @2x / @3x 等で書き出せる
```

### 書き出し解像度の選択指針

| 解像度 | 用途 |
|--------|------|
| @1x (1倍) | CSSピクセル基準（Web開発向け） |
| @2x (2倍) | Retinaディスプレイ / iPhone標準解像度 |
| @3x (3倍) | iPhone Plus / Proシリーズの最高解像度 |

**8ptグリッドと書き出しの関係:**
サイズが8の倍数であれば、@0.75x（Android mdpi相当）書き出し時も整数サイズになり、表示が鮮明になる。

### 複数オブジェクトの一括書き出し

```
File > Export frames to PDF
  → 選択ページ内のすべてのFrameをPDF1ファイルとして書き出し
  → テキストリンクが含まれていればPDFでもクリック可能
```

## 6. PdM向けFigma活用

### 資料作成

デザインファイルと同じFigmaファイル内に専用ページを作成して資料を作る。

```
推奨手順:
  1. Pages + ボタンで新しいページを作成（例: "Specs" / "Presentation"）
  2. F キーでFrameを作成（資料の1ページ = 1 Frame）
  3. デザイン成果物をコピーして資料に貼り付け
     → コンポーネントを使用している場合、デザイン変更がリアルタイム反映される
  4. Text ツールで説明文を入力
  5. Pen ツールで矢印・線を追加

注意: 誤ってデザインを変更しないよう、専用ページで作業する
```

### コメントとフィードバック

```
コメントの追加:
  C キーでコメントモードに切り替え → 対象箇所をクリック → コメント入力

スレッドのピン留め:
  コメントを作業中も表示し続けるにはピン留めアイコンをクリック

コメント解決:
  対応済みのコメントはResolveして整理する（デザイナーと合意の上で）
```

### フローチャート作成

Autoflowプラグインを使うと、画面間の矢印（フローチャート用）を素早く作成できる。

```
Autoflow の使い方:
  1. ⌘ / (Quick actions) で "Autoflow" を検索・実行
  2. 始点のオブジェクトを選択
  3. Shift + 終点のオブジェクトをクリック
  4. 矢印が作成される（オブジェクト移動に追従）

注意: Autoflowの矢印はプロトタイプのインタラクションとは別物
      説明・仕様書用途の矢印
```

### テキストリンクの活用

Figmaファイル内の特定の画面へのリンクを資料に貼り込める。

```
手順:
  1. リンクしたい画面（Frame）を選択
  2. Share → Copy link でURLを取得
  3. テキストを選択して ⌘ K（Create link）でリンクを設定
```

### FigJam

ワークショップ・仕様作成・アイデア出しに特化したFigmaの別ファイル形式。

| 機能 | 用途 |
|------|------|
| 図形・矢印ツール | ダイアグラム・フローチャート作成 |
| 付箋・スタンプ | ブレインストーミング |
| ウィジェット（投票・タイマー） | ワークショップ進行 |
| Figmaオブジェクトの貼り付け | デザイン成果物を議論に持ち込む |
| URLプレビュー | Webサイトやドキュメントの参照 |
| 音声通話 | オンラインリモートセッション |

## 7. デザイン確認用ショートカット集

### 基本ナビゲーション

| 操作 | Mac | Windows |
|------|-----|---------|
| コメントモードに切り替え | `C` | `C` |
| 選択ツール（Moveツール）に切り替え | `V` | `V` |
| 画面の移動 | `Space` + ドラッグ | `Space` + ドラッグ |
| 選択中の画面にズームイン | `shift + 2` | `shift + 2` |
| すべての画面を表示 | `shift + 1` | `shift + 1` |
| コマンド検索 | `⌘ + /` | `ctrl + /` |
| パネルを隠す（全画面表示） | `⌘ + \` | `ctrl + \` |
| ショートカット一覧を表示 | `⌘ + shift + ?` | `ctrl + shift + ?` |

### 選択操作

| 操作 | Mac | Windows |
|------|-----|---------|
| 入れ子を無視して直接選択 | `⌘ + クリック` | `ctrl + クリック` |
| 1階層下のオブジェクトを選択 | ダブルクリック | ダブルクリック |
| 子要素を選択 | `enter` | `enter` |
| 親レイヤーを選択 | `shift + enter` | `shift + enter` |
| 次のレイヤーを選択 | `tab` | `tab` |
| 前のレイヤーを選択 | `shift + tab` | `shift + tab` |
| レイヤー表示を折りたたむ | `option + L` | `alt + L` |

### 測定・確認

| 操作 | Mac | Windows |
|------|-----|---------|
| オブジェクト間の距離を測定 | オブジェクト選択後 `option` + 他オブジェクトにマウスオーバー | オブジェクト選択後 `alt` + 他オブジェクトにマウスオーバー |
| 画像をクリップボードにコピー | `⌘ + shift + C` | `ctrl + shift + C` |
| テキストリンクを作成 | `⌘ + K` | `ctrl + K` |
| グリッドの表示/非表示 | `ctrl + G` | `ctrl + shift + 4` |
