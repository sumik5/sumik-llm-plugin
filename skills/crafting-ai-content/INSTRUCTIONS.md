## AIコピーライティング

AIに「キャッチコピーを作って」と普通に頼むと、無難で安全な答えしか返ってきません。これはAIが「批判されにくい、当たり障りのない表現」を優先するためです。

本スキルは、**AIのプロンプトに「心理的仕掛け」を組み込む**ことで、人の心を動かすコピーを生成する15のテクニックを提供します。

### 15テクニックの分類

- **心理トリガー系**: 好奇心・損失回避・意外性など、人間の本能的反応を利用
- **構造・ストーリー系**: 物語性・具体性・信頼性を高める構造
- **表現・トーン系**: 臨場感・親近感を演出する文体

このスキルは **プロンプトエンジニアリング** に特化しています。AIが生成した文章から「AI臭さ」を除去する作業は `writing-effective-prose` スキルを参照してください。

---

### 基本原則: 「普通に聞いたら普通の答え」問題

AIは学習データから「最も安全で一般的な表現」を選択します。その結果:

- ❌ 「アクセスアップの5つの方法」→ 誰でも思いつく
- ❌ 「効果的な時間管理術」→ 心に響かない
- ❌ 「おすすめの副業」→ クリックしたくならない

プロンプトの「仕掛け」: 心理学的トリガーをプロンプト内で明示的に指示することで、AIに「意図的に人間の感情を動かす表現」を選ばせます。

### カテゴリ別ガイド

#### 心理トリガー系

| テクニック | 活用する心理効果 | 強み |
|-----------|---------------|------|
| ①禁止命令 | カリギュラ効果（禁止されるとやりたくなる） | 好奇心を強制的に刺激 |
| ③問いかけ | 自分事化（質問されると考えずにいられない） | 受動的読者を能動的に変える |
| ④意外性 | 認知的不協和（常識が覆されると注目する） | 飽和市場でも目立つ |
| ⑤感情ワード | 感情的共鳴（理屈より感情が行動を促す） | 記憶に残りやすい |
| ⑩秘密 | 希少性の原理（限定情報は価値が高いと感じる） | 情報商材との相性◎ |
| ⑪時間軸 | 損失回避バイアス（得るより失う痛みが2倍） | 今すぐ行動させる |

**使い方のコツ:** 組み合わせ例: 「禁止命令 + 時間軸」→ 「今すぐ読むのをやめてください（手遅れになる前に）」

#### 構造・ストーリー系

| テクニック | 提供する価値 | 強み |
|-----------|------------|------|
| ②数字 | 具体性・測定可能性 | 曖昧さを排除、信頼性向上 |
| ⑥ストーリー | 共感・記憶定着 | 理屈より心に残る |
| ⑦ターゲット限定 | セグメンテーション | 「自分向けだ」と思わせる |
| ⑧失敗談 | 人間味・親近感 | 完璧主義を嫌う層に刺さる |
| ⑨比較 | 判断基準の提示 | 選択を楽にする |
| ⑬自虐 | ハードル下げ | 初心者向けコンテンツに最適 |
| ⑭データ風 | 権威性・論理性 | B2B、教育系との相性◎ |

#### 表現・トーン系

| テクニック | 演出効果 | 強み |
|-----------|---------|------|
| ⑫擬音語 | シズル感・臨場感 | 無機質な情報を体感に変える |
| ⑮呼びかけ | 対話感・親密さ | 一方通行の情報発信を双方向に |

### 使い方ガイド

#### 複数テクニックの組み合わせ

| 組み合わせ | 生成例 | 効果 |
|----------|-------|------|
| 禁止命令 + ターゲット限定 | 「ブログ初心者は読まないでください（月10万円稼ぎたい人だけどうぞ）」 | 好奇心 × 限定感 |
| 失敗談 + 数字 + ストーリー | 「7回落ちた私が、8回目でTOEIC900点取れた『たった1つの変化』」 | 共感 × 具体性 × ドラマ |
| 意外性 + データ風 | 「調査で判明。『毎日勉強』が実は成績を下げる科学的理由」 | 衝撃 × 信頼性 |

#### やりすぎ注意

- ❌ 感情ワード過多: 「衝撃!禁断の究極暴露!崩壊する常識!」
- ❌ 中身との乖離: 「人生が激変する方法」→ 内容は普通のTips
- ❌ 全てに緊急性: 毎回「今すぐ!」は信頼を失う
- **重要原則: コピーはコンテンツの質を超えられない**

### ユーザー確認の原則（AskUserQuestion）

以下の場合、AskUserQuestionで選択肢を提示してください:

1. **トーンの選択**: フォーマル（B2B）vs カジュアル（SNS）
2. **ターゲット層の特定**: 初心者向け（⑬自虐、⑦限定が有効）vs 専門家向け（②数字、⑭データ風が有効）
3. **媒体の確認**: SNS（短文・擬音語◎）、ブログ（ストーリー◎）、広告（緊急性◎）

**確認不要**: プロンプトの基本構造（テンプレートそのまま使用）、10個生成する指示（デフォルト）

## AIデザインクリエイティブ

AI画像生成ツールを使って広告・マーケティングクリエイティブを制作するための実践ガイド。

**対象媒体**: SNS投稿画像・バナー広告・ポスター・Webサイトビジュアル・サムネイル

**AIツールの役割**: AIは60〜70点のたたき台を瞬時に生成。最終的な品質向上（残り30〜40点）には、デザイン基礎知識と人間による調整が必要。

### ユーザー確認（AskUserQuestion）

確認すべき場面:
1. **プロンプト生成前**: 対象媒体（SNS投稿/バナー広告/ポスター/Web広告）
2. **プロンプト生成前**: ターゲット層（年代・性別・関心）
3. **デザイン調整時**: ブランドカラー/フォント指定の有無

確認不要: デザイン4原則の適用・文字チェックの必要性・1クリエイティブ1メッセージの原則

### 1. プロンプト設計の基本構造

**「誰に/何を/どう見せたいか」の3要素**を必ず含めます。

基本テンプレート（詳細は `references/PROMPT-TEMPLATES.md`）:
```
# タイトル・企画: [イベント名/商品名]
# 目的: [バナークリック / イベント参加 / 商品購入]
# ターゲット: [対象者の属性・課題・興味関心]
# メインコピー: [最も目立たせるキャッチコピー]
# サブコピー: [補足情報]
# CTA: [ユーザーに促したい行動]
# アイキャッチ: [メインビジュアルの説明]
# レイアウト: [視線誘導: Z型/F型、要素の配置]
# 全体の印象: [トーン&マナー]
```

**高度な手法**: YAMLフォーマット（精密コントロール）・Vibe Design（雰囲気伝達で創造性を引き出す）
→ `references/PROMPT-TEMPLATES.md` に全75パターン収録

### 2. デザイン4原則（広告クリエイティブ版）

| 原則 | 広告への適用 | プロンプト指示例 |
|------|-----------|--------------|
| **近接** | 商品名と価格を近接、CTAは関連情報直下 | `"メインコピーとサブコピーを一体のテキストブロックとして配置"` |
| **整列** | テキスト要素を一貫した揃えで統一 | `"全テキスト要素は左揃えで統一、右側にアイキャッチを配置"` |
| **反復** | シリーズ広告で同一フォント・色・スタイルを使用 | `"このブランドカラーを全シリーズで一貫して使用"` |
| **コントラスト** | メインコピーはサブコピーの2〜3倍のサイズ | `"メインコピーを他テキストより3倍大きく、アクセントカラー使用"` |

**視線誘導**: Z型（バナー・LP向け）、F型（情報量多め向け）

→ 詳細: `references/DESIGN-FUNDAMENTALS.md`

### 3. 色彩・フォント要約

| 色 | 与える印象 | 広告での活用 |
|----|-----------|------------|
| 赤 | 情熱・緊急 | セール・緊急性訴求CTA |
| 青 | 信頼・誠実 | 企業広告・B2B・金融 |
| 緑 | 自然・健康 | 環境・ヘルスケア |
| 黒 | 高級感・洗練 | ラグジュアリー・フォーマル |

**配色バランス**: ベースカラー70%・メインカラー25%・アクセントカラー5%

| フォント種別 | 印象 | 適した場面 |
|------------|------|----------|
| 明朝体（セリフ） | 格調・品格 | 高級品・和風 |
| ゴシック体（サンセリフ） | 現代的・汎用性 | Web広告・テック系 |

**フォントルール**: 1デザインで2種類まで、見出しと本文にコントラストをつける

### 4. AI画像生成ワークフロー

1. **手描きラフ → AI変換**: レイアウトの位置指定だけでもAIに伝えられる
2. **i2i活用**: 既存画像からスタイル変換・要素追加・モックアップ作成
3. **9バリエーション生成**: 1プロンプトで複数の方向性を同時探索（A/Bテスト仮説立案に有効）
4. **ハイブリッドワークフロー**: AI生成（たたき台）→文字修正→テキスト追加→フォント調整→サイズ最適化

→ ツール操作詳細: `references/TOOL-ADJUSTMENTS.md`

### 5. 頻出5課題と解決策

| 課題 | 主な解決策 |
|-----|-----------|
| 文字の破綻・読みにくさ | デザインツールで文字を消去し改めて追加（AIは文字を「図形」として認識） |
| 細部の不自然さ（指・影） | 再生成 or AI消去ツールで削除、「8K品質」を明示 |
| ブランドイメージとのズレ | カラーコード（`#RRGGBB`）指定、参照画像添付でi2i再生成 |
| 情報の優先順位の混乱 | 1クリエイティブ1メッセージの原則を徹底 |
| 複数画像生成時の一貫性欠如 | i2i参照機能・共通カラーコード・テンプレート方式を採用 |

### 6. 品質チェックリスト

- [ ] 文字は正しく読めるか（崩れた漢字・意味不明な文字列がないか）
- [ ] 人物の指・手・顔のパーツが自然か
- [ ] デザイン4原則（近接・整列・反復・コントラスト）が守られているか
- [ ] 視線誘導はZ型またはF型で自然か
- [ ] ブランドカラーが正確に反映されているか
- [ ] AI生成画像の商用利用が許可されているツールを使用しているか
