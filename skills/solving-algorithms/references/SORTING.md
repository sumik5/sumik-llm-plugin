# ソートアルゴリズムリファレンス

競技プログラミングにおけるソートアルゴリズムの選択・実装ガイド。
初等的ソート（O(N²)）と高等ソート（O(N log N)）の特性と使い分けを解説。

---

## アルゴリズム比較表

| アルゴリズム | 時間計算量（最悪） | 時間計算量（平均） | 空間計算量 | 安定性 |
|------------|-----------------|-----------------|----------|--------|
| 挿入ソート | O(N²) | O(N²) ※ほぼ整列済みはO(N) | O(1) | 安定 |
| バブルソート | O(N²) | O(N²) | O(1) | 安定 |
| 選択ソート | O(N²) | O(N²) | O(1) | **不安定** |
| シェルソート | O(N^2) | O(N^1.25) ※数列依存 | O(1) | **不安定** |
| マージソート | O(N log N) | O(N log N) | O(N) | 安定 |
| クイックソート | O(N²) | O(N log N) | O(log N) | **不安定** |
| 計数ソート | O(N + k) | O(N + k) | O(k) | 安定 |

> k = 要素の最大値

---

## 安定性の概念

**安定なソート**: 同じキー値を持つ要素が、ソート後もソート前の相対順序を維持するもの。

```
入力: [3H, 5S, 3D, 1S]  (数字で整列)
安定: [1S, 3H, 3D, 5S]  → 3H が 3D より前（入力順を維持）
不安定: [1S, 3D, 3H, 5S] → 3H と 3D の順序が逆転
```

**安定性の判定基準**:
- **隣接要素のみ交換** → 安定（挿入・バブル）
- **離れた要素を交換** → 不安定（選択・シェル・クイック）

## 初等的ソート（O(N²)）

### 挿入ソート（Insertion Sort）

**核心アイデア**: 未ソート部分の先頭要素を、ソート済み部分の適切な位置に「挿入」する。
手持ちのトランプを並べ替える操作と同じ。

**擬似コード**:
```
insertionSort(A, N):
  for i = 1 to N-1:
    v = A[i]
    j = i - 1
    while j >= 0 and A[j] > v:
      A[j+1] = A[j]
      j--
    A[j+1] = v
```

**実装のポイント**:
- 要素を「交換」ではなく「後方へ移動」することで安定性を保つ
- `A[j] > v` に等号を含めると安定性が崩れるので注意

**典型的なユースケース**:
- データがほぼ整列されている場合（O(N)に近い動作）
- N ≤ 1,000 程度の小さいデータ
- シェルソートの最終ステップ

---

### バブルソート（Bubble Sort）

**核心アイデア**: 配列末尾から「逆順の隣接ペア」を探して交換を繰り返す。最小値が泡のように先頭へ浮き上がる。

**擬似コード**:
```
bubbleSort(A, N):
  flag = true
  while flag:
    flag = false
    for j = N-1 downto 1:
      if A[j] < A[j-1]:
        swap(A[j], A[j-1])
        flag = true
```

**実装のポイント**:
- `flag` による早期終了で、整列済みデータはO(N)で処理可能
- 比較演算子に等号を使うと（`<=`）安定性が崩れる

**典型的なユースケース**:
- 反転数（転倒数）の計算（交換回数 = 反転数）
- 教育目的・動作確認用途

---

### 選択ソート（Selection Sort）

**核心アイデア**: 未ソート部分の最小値を「選択」し、未ソート部分の先頭と交換する。

**擬似コード**:
```
selectionSort(A, N):
  for i = 0 to N-1:
    minj = i
    for j = i to N-1:
      if A[j] < A[minj]:
        minj = j
    swap(A[i], A[minj])
```

**実装のポイント**:
- 比較回数は常に `(N²-N)/2` で一定（データに依存しない）
- 交換回数は最大 N-1 回（バブルソートより少ない）
- 離れた位置との交換のため **不安定**

**典型的なユースケース**:
- 交換コストが高い場合（書き込み回数を最小化したい場合）
- N ≤ 1,000 程度の小さいデータ

---

### シェルソート（Shell Sort）

**核心アイデア**: 間隔 `g` を徐々に縮めながら挿入ソートを繰り返す。大きな `g` で大まかに整列 → 小さな `g` で仕上げ。

**擬似コード**:
```
shellSort(A, N):
  Gを生成（例: {1, 4, 13, 40, 121, ...} ← g_{n+1} = 3*g_n + 1）
  for each g in G (降順):
    insertionSort(A, N, g)  // g間隔の挿入ソート

insertionSort(A, N, g):
  for i = g to N-1:
    v = A[i]
    j = i - g
    while j >= 0 and A[j] > v:
      A[j+g] = A[j]
      j -= g
    A[j+g] = v
```

**実装のポイント**:
- 最後は必ず `g = 1`（通常の挿入ソート）で終了する
- 数列の選び方で計算量が大きく変わる
  - `g_{n+1} = 3*g_n + 1` → O(N^1.25) が期待値
  - 2の冪乗（1, 2, 4, 8, ...）は非効率（偶数インデックスと奇数インデックスが最後まで比較されない）

**典型的なユースケース**:
- N ≤ 100,000 程度の中規模データ
- 単純な実装でO(N²)を超えたい場合

## 高等ソート（O(N log N)）

### マージソート（Merge Sort）

**核心アイデア**: 分割統治法。配列を2分割して再帰的にソートし、2つのソート済み配列をマージする。

**擬似コード**:
```
mergeSort(A, left, right):
  if left + 1 < right:
    mid = (left + right) / 2
    mergeSort(A, left, mid)
    mergeSort(A, mid, right)
    merge(A, left, mid, right)

merge(A, left, mid, right):
  L = A[left..mid] + [INFTY]    // 番兵を追加
  R = A[mid..right] + [INFTY]   // 番兵を追加
  i = j = 0
  for k = left to right-1:
    if L[i] <= R[j]:
      A[k] = L[i++]
    else:
      A[k] = R[j++]
```

**実装のポイント**:
- 番兵（INFTY）を末尾に置くことで境界チェックを省略できる
- `L[i] <= R[j]` で等しい場合はLを優先 → 安定性の保証
- 外部メモリ O(N) が必要（インプレースマージは実装が複雑）

**典型的なユースケース**:
- 安定なO(N log N)ソートが必要な場合
- 反転数の計算（merge中に計数）
- N ≤ 500,000

---

### クイックソート（Quick Sort）

**核心アイデア**: ピボット要素を選び、「以下」と「より大きい」の2グループに分割して再帰する。

**擬似コード（Lomuto partition scheme）**:
```
partition(A, p, r):
  x = A[r]           // ピボット = 末尾
  i = p - 1
  for j = p to r-1:
    if A[j] <= x:
      i++
      swap(A[i], A[j])
  swap(A[i+1], A[r])
  return i + 1

quickSort(A, p, r):
  if p < r:
    q = partition(A, p, r)
    quickSort(A, p, q-1)
    quickSort(A, q+1, r)
```

**実装のポイント**:
- partition 内で離れた要素を交換するため **不安定**
- 追加メモリ不要（インプレースソート）
- ピボット選択がパフォーマンスの鍵：
  - 末尾固定 → 既ソートデータで最悪 O(N²)
  - ランダム選択 / 中央値の中央値法 → 実用上ほぼO(N log N)

**典型的なユースケース**:
- 安定性が不要で最速を優先する場合
- 実際の言語標準ライブラリのソートの基礎（introsort等）

---

### 計数ソート（Counting Sort）

**核心アイデア**: 各値の出現数をカウントし、累積和で各要素の出力位置を決定する。

**擬似コード**:
```
countingSort(A, B, k):
  C = [0] * (k+1)
  for j = 0 to n-1:
    C[A[j]]++
  for i = 1 to k:
    C[i] += C[i-1]          // 累積和
  for j = n-1 downto 0:     // 逆順で安定性を保つ
    B[C[A[j]]] = A[j]
    C[A[j]]--
```

**実装のポイント**:
- 逆順（後ろから）処理することで安定性を保証
- k が大きすぎるとメモリ・時間ともに非現実的
  - 目安: k ≤ 10,000 程度
- 整数限定（実数や文字列には使えない）

**典型的なユースケース**:
- 要素の範囲 k が小さい場合（0 ≤ a_i ≤ 10,000 など）
- n が非常に大きい場合（n ≤ 2,000,000 など）
- 基数ソートの内部ソートとして使用

## 反転数（Inversion Count）

### 定義

数列 A = {a₀, a₁, ..., aₙ₋₁} において、**aᵢ > aⱼ かつ i < j** である組 (i, j) の個数。

- 反転数 = バブルソートにおける隣接交換の回数
- 数列の「乱れ度」を定量化する指標

### 計算方法

**素朴な実装 → O(N²)**: バブルソートをそのまま実行して交換回数をカウント。
n ≤ 200,000 では TLE になる。

**高速計算 → O(N log N)**: マージソートの `merge` 関数に計数処理を追加。

```
merge(A, left, mid, right) → 反転数を返す:
  L = A[left..mid], R = A[mid..right]
  i = j = 0, cnt = 0
  for k = left to right-1:
    if L[i] <= R[j]:
      A[k] = L[i++]
    else:
      A[k] = R[j++]
      cnt += (n1 - i)    // Rの要素がLの残り要素より前に来るケース
  return cnt
```

**アイデア**: マージ時に R の要素 R[j] がLの残り要素より前に移動するとき、L の残り要素数（n1 - i）だけ反転が生じる。

## 最小コストソート（Minimum Cost Sort）

### 問題設定

n個の荷物を交換によってソートする。荷物 i と荷物 j を交換するコストは `w_i + w_j`。
総コストの最小値を求めよ。

### 解法：サイクル分解

**ステップ1: サイクルを見つける**

各要素の「現在の位置 → ソート後の位置」の対応をグラフとして考えると、互いに独立したサイクルに分解できる。

例: W = {4, 3, 2, 7, 1, 6, 5} → サイクル: (4→7→5→1)、(3→2)、(6)

**ステップ2: 各サイクルのコストを計算**

サイクルの要素集合 S、要素数 n、サイクル内最小値 min_s、全体最小値 min_all について:

- **戦略A（サイクル内最小値を媒介）**:
  ```
  cost_A = Σwᵢ + (n - 2) × min_s
  ```
  ※ 各要素1回必ずコスト + 最小値が(n-2)回追加コスト

- **戦略B（全体最小値を外から借りる）**:
  ```
  cost_B = Σwᵢ + min_s + (n + 1) × min_all
  ```
  ※ min_all をサイクルに一時挿入してから最後に取り出す

- **採用**: `min(cost_A, cost_B)` を各サイクルに適用し、全サイクルの合計を答えとする

**判定条件**: 全体最小値 min_all がサイクル内最小値 min_s より小さく、かつ十分に小さければ戦略Bが有利。

**実装の概要**:
```
入力全体の最小値 s を求める
各サイクルを訪問済みフラグで列挙:
  for each 未訪問の開始点 i:
    サイクルを辿り、要素の合計 S, サイクル長 n, 最小値 m を求める
    ans += min(S + (n-2)*m,  m + S + (n+1)*s)
```

## ソートアルゴリズム選択フロー

```
要素の範囲が狭い整数？ (a_i ≤ 10,000)
  └─ YES → 計数ソート O(N+k)
  └─ NO
        N ≤ 1,000?
          └─ YES → 挿入ソート（ほぼ整列済み） / バブルソート（反転数が必要）
          └─ NO
                安定性が必要？
                  └─ YES → マージソート O(N log N)
                  └─ NO → クイックソート O(N log N) 平均
                             ※ピボット選択に注意
```

## チェックポイント（競技プログラミング向け）

| 状況 | 対策 |
|------|------|
| バブルソートがTLE | N ≤ 200,000 → マージソートで反転数計算 |
| クイックソートがTLE | 昇順入力で最悪O(N²) → ランダムピボット or マージソートに切替 |
| 計数ソートのメモリ不足 | k が大きすぎる → 座標圧縮で k を縮小 |
| 安定性が必要 | 選択ソート・クイックソートを避ける |
| シェルソートの数列設計 | `g = 3g+1` の数列（Knuth）を推奨 |
