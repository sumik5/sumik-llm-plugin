## AIコピーライティング

AIに「キャッチコピーを作って」と普通に頼むと、無難で安全な答えしか返ってきません。これはAIが「批判されにくい、当たり障りのない表現」を優先するためです。

本スキルは、**AIのプロンプトに「心理的仕掛け」を組み込む**ことで、人の心を動かすコピーを生成する15のテクニックを提供します。

### 15テクニックの分類

- **心理トリガー系**: 好奇心・損失回避・意外性など、人間の本能的反応を利用
- **構造・ストーリー系**: 物語性・具体性・信頼性を高める構造
- **表現・トーン系**: 臨場感・親近感を演出する文体

このスキルは **プロンプトエンジニアリング** に特化しています。AIが生成した文章から「AI臭さ」を除去する作業は `writing-effective-prose` スキルを参照してください。

### 15テクニック一覧

| # | テクニック | 心理効果 | 使用場面 |
|---|-----------|---------|---------|
| ① | 禁止命令 | カリギュラ効果 | 好奇心を刺激したいとき |
| ② | 数字で具体性 | 具体性の原理 | 信頼性を高めたいとき |
| ③ | 問いかけ | 自分事化 | 読み手を当事者にしたいとき |
| ④ | 意外性（常識の逆） | 認知的不協和 | 固定観念を崩したいとき |
| ⑤ | 感情ワード | 感情的共鳴 | インパクトを出したいとき |
| ⑥ | ストーリー風 | 物語への共感 | 人間味を出したいとき |
| ⑦ | ターゲット限定 | 特別感・所属欲求 | ニッチ層に刺さるコピーが欲しいとき |
| ⑧ | 失敗談 | 親近感・信頼性 | 完璧主義を避けたいとき |
| ⑨ | 比較構造 | 選択の錯覚 | 判断を促したいとき |
| ⑩ | 秘密・裏技 | 希少性の原理 | 特別情報をアピールしたいとき |
| ⑪ | 時間軸・緊急性 | 損失回避バイアス | 行動を促したいとき |
| ⑫ | 擬音語 | シズル感・臨場感 | 五感に訴えたいとき |
| ⑬ | 自虐 | 親しみやすさ | ハードルを下げたいとき |
| ⑭ | データ風 | 権威への服従 | 論理的説得をしたいとき |
| ⑮ | 呼びかけ | 対話的関係性 | 距離を縮めたいとき |

### カテゴリ別ガイド

#### 心理トリガー系

| テクニック | 活用する心理効果 | 強み |
|-----------|---------------|------|
| ①禁止命令 | カリギュラ効果（禁止されるとやりたくなる） | 好奇心を強制的に刺激 |
| ③問いかけ | 自分事化（質問されると考えずにいられない） | 受動的読者を能動的に変える |
| ④意外性 | 認知的不協和（常識が覆されると注目する） | 飽和市場でも目立つ |
| ⑤感情ワード | 感情的共鳴（理屈より感情が行動を促す） | 記憶に残りやすい |
| ⑩秘密 | 希少性の原理（限定情報は価値が高いと感じる） | 情報商材との相性◎ |
| ⑪時間軸 | 損失回避バイアス（得るより失う痛みが2倍） | 今すぐ行動させる |

**使い方のコツ:** 組み合わせ例: 「禁止命令 + 時間軸」→ 「今すぐ読むのをやめてください（手遅れになる前に）」

#### 構造・ストーリー系

| テクニック | 提供する価値 | 強み |
|-----------|------------|------|
| ②数字 | 具体性・測定可能性 | 曖昧さを排除、信頼性向上 |
| ⑥ストーリー | 共感・記憶定着 | 理屈より心に残る |
| ⑦ターゲット限定 | セグメンテーション | 「自分向けだ」と思わせる |
| ⑧失敗談 | 人間味・親近感 | 完璧主義を嫌う層に刺さる |
| ⑨比較 | 判断基準の提示 | 選択を楽にする |
| ⑬自虐 | ハードル下げ | 初心者向けコンテンツに最適 |
| ⑭データ風 | 権威性・論理性 | B2B、教育系との相性◎ |

#### 表現・トーン系

| テクニック | 演出効果 | 強み |
|-----------|---------|------|
| ⑫擬音語 | シズル感・臨場感 | 無機質な情報を体感に変える |
| ⑮呼びかけ | 対話感・親密さ | 一方通行の情報発信を双方向に |

### 組み合わせと注意点

| 組み合わせ | 生成例 | 効果 |
|----------|-------|------|
| 禁止命令 + ターゲット限定 | 「ブログ初心者は読まないでください（月10万円稼ぎたい人だけどうぞ）」 | 好奇心 × 限定感 |
| 失敗談 + 数字 + ストーリー | 「7回落ちた私が、8回目でTOEIC900点取れた『たった1つの変化』」 | 共感 × 具体性 × ドラマ |
| 意外性 + データ風 | 「調査で判明。『毎日勉強』が実は成績を下げる科学的理由」 | 衝撃 × 信頼性 |

**やりすぎ注意**:
- ❌ 感情ワード過多: 「衝撃!禁断の究極暴露!崩壊する常識!」
- ❌ 全てに緊急性: 毎回「今すぐ!」は信頼を失う
- **重要原則: コピーはコンテンツの質を超えられない**

### AskUserQuestion が必要な場面

1. **トーンの選択**: フォーマル（B2B）vs カジュアル（SNS）
2. **ターゲット層の特定**: 初心者向け（⑬自虐、⑦限定が有効）vs 専門家向け（②数字、⑭データ風が有効）
3. **媒体の確認**: SNS（短文・擬音語◎）、ブログ（ストーリー◎）、広告（緊急性◎）

## AIデザインクリエイティブ

AI画像生成ツールを使って広告・マーケティングクリエイティブを制作するための実践ガイド。

**対象媒体**: SNS投稿画像・バナー広告・ポスター・Webサイトビジュアル・サムネイル

**AIツールの役割**: AIは60〜70点のたたき台を瞬時に生成。最終的な品質向上（残り30〜40点）には、デザイン基礎知識と人間による調整が必要。

### プロンプト設計の基本構造

**「誰に/何を/どう見せたいか」の3要素**を必ず含めます。

基本テンプレート（詳細は [`PROMPT-TEMPLATES.md`](references/PROMPT-TEMPLATES.md)）:
```
# タイトル・企画: [イベント名/商品名]
# 目的: [バナークリック / イベント参加 / 商品購入]
# ターゲット: [対象者の属性・課題・興味関心]
# メインコピー: [最も目立たせるキャッチコピー]
# サブコピー: [補足情報]
# CTA: [ユーザーに促したい行動]
# アイキャッチ: [メインビジュアルの説明]
# レイアウト: [視線誘導: Z型/F型、要素の配置]
# 全体の印象: [トーン&マナー]
```

### AI画像生成ワークフロー

1. **手描きラフ → AI変換**: レイアウトの位置指定だけでもAIに伝えられる
2. **i2i活用**: 既存画像からスタイル変換・要素追加・モックアップ作成
3. **9バリエーション生成**: 1プロンプトで複数の方向性を同時探索
4. **ハイブリッドワークフロー**: AI生成（たたき台）→文字修正→テキスト追加→フォント調整→サイズ最適化

### 頻出5課題と解決策

| 課題 | 主な解決策 |
|-----|-----------|
| 文字の破綻・読みにくさ | デザインツールで文字を消去し改めて追加（AIは文字を「図形」として認識） |
| 細部の不自然さ（指・影） | 再生成 or AI消去ツールで削除、「8K品質」を明示 |
| ブランドイメージとのズレ | カラーコード（`#RRGGBB`）指定、参照画像添付でi2i再生成 |
| 情報の優先順位の混乱 | 1クリエイティブ1メッセージの原則を徹底 |
| 複数画像生成時の一貫性欠如 | i2i参照機能・共通カラーコード・テンプレート方式を採用 |

## HTMLスライド作成

1スライド = 1 HTMLファイル（1280×720px）のプレゼンデッキを生成する。**ユーザーへの全連絡は日本語で行う。**

### ワークフロー概要

| Phase | 内容 | 成果物 |
|-------|------|--------|
| 1 | ヒアリング | 出力先・スタイル・テーマ・内容・枚数を確定 |
| 2 | カスタムテンプレート読み込み | `references/templates/` のHTML解析（なければスキップ） |
| 3 | デザイン決定 | カラーパレット・フォント・アイコン（ユーザー確認必須） |
| 4 | スライド構成設計 | スライドマップ（全スライドの型・レイアウト） |
| 5 | HTML生成 | `001.html`〜`NNN.html` |
| 6 | print.html 生成 | 全スライドを iframe で並べた一覧ページ |
| 7 | チェックリスト確認 | 制約適合確認・修正 → [SLIDEKIT-CHECKLIST.md](references/SLIDEKIT-CHECKLIST.md) |
| 8 | PPTX変換（任意） | `/pptx` スキルへ橋渡し |

### 必須制約

| ルール | 値 |
|--------|-----|
| スライドサイズ | `width: 1280px; height: 720px` |
| CSS フレームワーク | Tailwind CSS 2.2.19 via CDN |
| アイコン | Font Awesome 6.4.0 via CDN |
| フォント | Google Fonts（JP 1本 + Latin 1本） |
| JavaScript | **完全禁止**（Chart.js 等も不可） |
| 外部画像 | デフォルト禁止（明示的ユーザー承認が必要） |
| ルート DOM | `<body>` → single wrapper `<div>` |

### 15レイアウトパターン概要

DOM構造・コンポーネントスニペット → [`SLIDEKIT-PATTERNS.md`](references/SLIDEKIT-PATTERNS.md)

| # | パターン | 用途 |
|---|---------|------|
| 1 | Center | 表紙・締め |
| 2 | Left-Right Split | 章区切り・概念+詳細 |
| 3 | Header-Body-Footer (HBF) | コンテンツスライド（デフォルト） |
| 4〜7 | HBF + 2/3/N-Column | 比較・プロセスフロー |
| 8〜15 | HBF + 特殊レイアウト | タイムライン・KPI・ファネル等 |

**ルール**: 同一パターンを3枚以上連続させない。

## Google Slides生成

非構造化テキスト（議事録・企画書・提案書・メモ等）から Google Apps Script（GAS）用の `slideData` 配列を生成する。

### スライドタイプ選定ガイド

| 内容の性質 | 推奨タイプ |
|-----------|----------|
| 2つの対象を比較 | `compare` |
| 時系列・プロセス | `process` または `timeline` |
| フロー・泳道図 | `diagram` |
| カード状の複数項目 | `cards` |
| 表形式データ | `table` |
| 進捗・達成率 | `progress` |
| 一般的な箇条書き | `content` |

### テキスト制限（厳守）

| 対象 | 制限 |
|------|------|
| `title.title` | 全角35文字以内 |
| `section.title` | 全角30文字以内 |
| 各パターンの `title` | 全角40文字以内 |
| 箇条書き要素 | 各90文字以内・改行禁止 |

**禁止**: `■` `→` / 箇条書き文末の句点「。」 / 改行文字（`\n`）
