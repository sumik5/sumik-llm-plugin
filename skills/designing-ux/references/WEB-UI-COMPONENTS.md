# WebサイトのUIコンポーネント・グラフィック設計

> UIガイドラインの詳細は `GUIDELINES-*.md`（CONTROLS, FORMS, NAVIGATION 等）を参照。タイポグラフィ理論は `GRAPHICS-TYPOGRAPHY-AND-TYPESETTING.md` を参照。
> 本ファイルは**Webサイト制作の実践的なUIコンポーネント設計ポイント**に特化する。

---

## 1. ナビゲーション系コンポーネント

### 1.1 ヘッダー

ユーザーの視線は左上から始まるため、ヘッダーは最初に目が留まる場所。情報を詰め込みすぎず、優先順位を明確にすることが基本原則。

**ヘッダーに含める要素（優先順位付き）**

| 優先度 | 要素 |
|--------|------|
| 必須 | ロゴ・ブランド名、グローバルナビゲーション |
| 推奨 | タグライン（コンテンツ概要）、検索フォーム |
| 任意 | SNSリンク、言語切り替え |

**固定ヘッダー（Sticky Header）の設計**

- ファーストビュー：ロゴ・タグラインを大きく見せてブランド認知を高める
- スクロール後：ヘッダーをコンパクト化（テキストラベルを削除しアイコンのみに）してコンテンツ領域を確保
- 固定領域が大きすぎるとコンテンツの閲覧を妨げる — コンパクト設計が必須

**画面サイズ別ナビゲーションの変化**

| パターン | 特徴 | 向いているケース |
|---------|------|----------------|
| ドロップダウン | ボタンから下方向に展開 | 項目数を問わない、世界観を作り込みたい場合 |
| スライドイン | 横から押し出すように展開、縦幅100%を占有 | 項目が多い、タブレットで開いたままにしたい場合 |
| メニューバー | 下部固定でアイコン+ラベルを常時表示 | 項目が少ない（4〜5項目）、頻繁に使われる機能が決まっている場合 |

---

### 1.2 フッター

ページを一通り見終えたユーザーが次の行動をとる場所。3つの役割から用途を設計する。

| 役割 | 内容 | 実装例 |
|------|------|--------|
| **他ページへの遷移** | グローバルナビと同様のリンク一覧（ファットフッター/サイトマップ） | カテゴリでグルーピングしたテキストリンク |
| **コンタクト** | 問い合わせ・資料請求への誘導 | CTAボタン、簡易フォーム |
| **関連情報** | 関連ページ・外部メディア（SNS）へのハブ | バナー、SNSリンク |

**設計のポイント**

- 著作権表示・プライバシーポリシー・利用規約は基本必須
- 中〜大規模サイトではファットフッターを簡易サイトマップとして活用
- 情報の種類・ページ階層でグルーピングし、適切なラベルを設置
- フッターはデザインの遊び・世界観を完結させる場としても有効

---

### 1.3 ナビゲーションのパターン

**グローバルナビゲーション vs ローカルナビゲーション**

| 種類 | 役割 | 配置 |
|------|------|------|
| グローバルナビ | 第2階層（主要カテゴリ）への導線 | ヘッダー等の共通エリア |
| ローカルナビ | 同一カテゴリ内のコンテンツ間移動 | グローバルナビ近くに設置 |

**現在地表示**：閲覧中のページをデザインで差別化し、現在位置を視覚的に示すこと（アクティブ状態のスタイル）。

**メガドロップダウン（メガメニュー）**

- 通常のドロップダウンに大きなパネルを使い、カテゴリ分け・アイコンで情報をグルーピング
- 項目が多い大規模サイトで情報を俯瞰させるのに効果的
- グローバルナビゲーションと組み合わせて使用

**パンくずリスト（Breadcrumb Navigation）**

- サイト内の現在位置と経路をテキストリンクで示す
- ヘッダー・ページタイトルの近くに横向きに配置
- 階層が深いECサイト・大規模サイトで特に有効
- 文字サイズは小さくなりがちだが、上位カテゴリへの素早いアクセスを提供

## 2. インタラクション系コンポーネント

### 2.1 ボタン

ボタンはユーザーが能動的に操作するパーツ。「ボタンらしさ」と「一貫性」が基本要件。

**ボタンらしさのデザイン要素**

- テキストリンクより一段上の視覚的差別化（枠線、背景色、アイコン、余白）
- サイズ・角丸・シャドウの違いでクリック率が変化する
- サイト内のボタンスタイルは統一（複数スタイルの混在は混乱を招く）

**ホバーインタラクション**

| 表現 | 効果 |
|------|------|
| 色のトーン変化（暗→明 or 明→暗） | クリック可能であることを明示 |
| 背景色とラベル色の反転 | コントラストの視覚的変化 |
| ボーダー追加、オブジェクト拡縮 | 立体感・インタラクティブ性の演出 |

**ボタンスタイル3種**

| スタイル | 特徴 | 使いどころ |
|---------|------|-----------|
| 立体ボタン | シャドウ・グラデーションで奥行きを表現 | ゲームサイト、子ども向け、エンタメ |
| フラットボタン | 装飾を排除した平面的デザイン | モダンな一般Webサービス |
| ゴーストボタン | 背景透明・外枠のみ | 写真上への配置、優先度の低いアクション |

> ゴーストボタンと通常のボタンを組み合わせることで、アクションの優先度の差を視覚的に表現できる。

---

### 2.2 タブ・アコーディオン・モーダル

限られたスペースに情報を収める段階的表示の手法。

| パターン | 特徴 | 使用場面 |
|---------|------|---------|
| **タブ / トグルボタン** | 同一エリアで情報を切り替え | ユーザーの目的が異なる複数カテゴリを同じページに並べる場合（例: Art / Music / Event） |
| **モーダル（ダイアログ）** | メインコンテンツの上に子ウィンドウを表示。メインは操作不可 | 重要な確認（取消不可操作）、画像の拡大ビュー、補足情報 |
| **アコーディオン / インライン** | ページ内に割り込むようにエリアを展開。メインは引き続き操作可能 | 項目が多いナビゲーション、モバイルでのコンテンツ折りたたみ |

**注意点**
- スクロールよりクリックはハードルが高い → すべてのユーザーに必ず見てほしい情報はタブで出し分けない
- モーダル表示時は背景を暗くし、トランジションエフェクト（フェード等）を加えて文脈変化をユーザーに明示
- スマートフォンでのアコーディオンはスクロール量を削減し、ユーザビリティ向上に効果的

---

### 2.3 フォーム

コンバージョン率を直接左右する重要なUI。ユーザーが素早く入力を終えられるよう設計する。

**フォームの3要素**

1. **入力フィールド**（テキストフィールド・ラジオボタン・チェックボックス等）
2. **ラベル**
3. **アクションボタン**

**プレースホルダの正しい使い方**

- 入力欄に薄い文字で入力例を表示（例: `mail@example.com`）
- 省スペースで理解しやすい補助手段として有効
- ⚠️ 注意: 入力時に消えるため、項目ラベルや注意文言をプレースホルダに含めてはならない

**フィードバック（バリデーション）のデザイン**

テキストフィールドのステータス別にスタイルを定義する。

| ステータス | 視覚的表現 |
|-----------|-----------|
| 通常（Idle） | デフォルトスタイル |
| フォーカス中（Focus） | 枠線の強調・色変化 |
| 入力済（Filled） | 正常状態の明示 |
| エラー（Error） | 赤枠・エラーメッセージ表示 |
| 無効（Disabled） | グレーアウト |

## 3. ビジュアル系コンポーネント

### 3.1 ヒーローイメージ

ファーストビューを占有する大きなビジュアル。サイト全体の第一印象を決定づける最重要要素。

**設計原則**

- 画像の上にナビゲーション・ロゴ・キャッチコピーを重ねるため、ごちゃついた写真は可読性を下げる
- ビジュアルにフォーカスする分、他の要素は装飾を抑えて必要情報に絞る

**カラーオーバーレイ**

写真の上に半透明の色を重ねる技法。
- UIと写真のコントラスト確保（青い空の上に青いボタンは見えない）
- コーポレートカラーを活用してブランド訴求と視認性を両立

**スクロール誘導**

- 下向き矢印・マウスアイコン等でスクロールを促す
- フラットナビゲーション（ドット・線）で下にコンテンツがあることを示す

---

### 3.2 写真を使ったデザイン

**トーンによる印象コントロール**

| トーン | 特徴 | 向いているコンテンツ |
|--------|------|-------------------|
| 高彩度・温色 | シズル感、活気、食欲 | 食品、エンタメ |
| 低彩度・グレイッシュ | 上品さ、落ち着き | ラグジュアリー、ナチュラル系 |
| 高コントラスト | 力強さ、鮮明さ | スポーツ、技術系 |
| ソフト・淡い | 柔らかさ、親しみやすさ | 子ども、ヘルスケア |

**画像最適化のポイント**

- Retinaディスプレイ対応: 表示サイズの2倍以上のピクセル数を使用
- 画質とファイルサイズのバランスを維持（圧縮ツールを活用）

**切り抜き（マスク）の活用**

| 手法 | 効果 |
|------|------|
| 被写体の輪郭に沿った切り抜き | 躍動感・活発さ、コラージュ的な世界観 |
| 角丸・円形マスク | 柔らかく親しやすい印象 |
| 特殊形状マスク | ブランドシンボルの強化 |
| タイポグラフィマスク | タイトル文字の形に写真を切り抜いてキービジュアル化 |
| グラデーションマスク | 複数画像のなじみ、背景との融合 |

---

### 3.3 イラストを使ったデザイン

写真では表現しにくい独自の世界観を構築できる。アニメーションと親和性が高い。

| 手法 | 特徴 |
|------|------|
| 手書き・水彩調 | 温かみ、ナチュラル、親しみやすさ |
| ベクターグラフィックス（SVG） | Retinaディスプレイ対応、シャープ・モダン・IT系に多い |
| 3Dグラフィックス | 先進的、技術力のアピール |
| キャラクター | ブランド人格の形成、競合との差別化、専門分野への敷居を下げる |

**イラストをパーツとして使う際の注意点**

- テキストの可読性を妨げていないか確認
- 導線（ナビゲーション・CTA）を阻害しないように配置
- 発注前にレイアウトを決め、必要なイラスト素材を明確にする

---

### 3.4 インフォグラフィック

複雑な情報・数値を視覚化して直感的に伝える手法。言語に依存せず多くの層にリーチできる。

**グラフの種類と用途**

| グラフ | 用途 |
|--------|------|
| 円グラフ | 構成比・割合の表示 |
| 棒グラフ | 項目間の量比較（項目が多い場合も有効） |
| 折れ線グラフ | 時系列の推移比較 |
| レーダーチャート | 複数軸での特性評価（5〜8項目） |
| 積み上げ棒グラフ | 総量に占める内訳の違いの比較 |

## 4. Webタイポグラフィの実践ポイント

> タイポグラフィ理論（書体の分類・組版原則等）の詳細は `GRAPHICS-TYPOGRAPHY-AND-TYPESETTING.md` を参照。
> 本セクションはWeb固有の実践ポイントに限定する。

### 4.1 Web向け書体選択

- 印刷と異なりWebでは**スクリーン解像度の制約**があるため、明朝体は線が潰れやすくゴシック体が主流
- 見出しと本文に同一書体ファミリーを使い、ウェイトの違いで情報階層を表現

### 4.2 Webフォント

| 方法 | メリット | デメリット |
|------|---------|-----------|
| システムフォント | 追加読み込みなし、高速 | OS・ブラウザ間で表示差あり |
| Webフォント（Google Fonts等） | 欧文書体を自由に指定可能 | 読み込み時間の増加 |
| 画像化 | 任意のフォント表示 | 修正困難、テキスト検索不可 |

### 4.3 カーニングとトラッキング

- **カーニング**: 字形に応じた個別の文字間隔調整 → ロゴ・タイトル画像作成時に適用
- **トラッキング**: 文字列全体を等間隔で調整 → CSSの `letter-spacing` プロパティで設定
- 情緒的な演出にはトラッキングを広めに、ただし広げすぎると意味が頭に入りにくくなる

### 4.4 ジャンプ率

見出しと本文のフォントサイズ比率。

| ジャンプ率 | 印象 | 適したコンテンツ |
|-----------|------|----------------|
| 高い（大きな差） | 躍動感・スキャン性・情報の優先順位が明確 | 一枚長ページ、ニュースメディア、記事サイト |
| 低い（差が小さい） | 落ち着き・じっくり読む体験 | 小説・詩・ブランドコンセプトページ |

## 5. デザインガイドラインの構成要素

複数デザイナーが関わるプロジェクトや継続的に改善されるWebサービスで必須。

| 定義項目 | 内容 |
|---------|------|
| **カラーパレット** | 使用カラーのHex値・役割（ベースカラー、アクションカラー等）の定義 |
| **タイポグラフィ** | 見出し・本文・キャプション別のフォントサイズ・書体・カラー |
| **アイコンライブラリ** | サイズ・線幅・塗りのルール定義（同じ意味のアイコンが複数存在する状態を防ぐ） |
| **コンポーネントデザイン** | ボタン・フォーム等の通常時・ホバー・無効時等のステータス別スタイル |
| **レイアウトパターン** | カラム数・エリア構成・グリッドシステムの定義 |

**Atomic Designの考え方（参考）**

| レベル | 名称 | 内容 |
|--------|------|------|
| Lv1 | Atoms（原子） | それ以上分割できない最小要素（フォント・カラー等） |
| Lv2 | Molecules（分子） | Atomsの組み合わせ（テキストフィールド＋ボタン＝検索フォーム） |
| Lv3 | Organisms（有機体） | Moleculesの組み合わせ（ヘッダー・フッター等） |
| Lv4 | Templates | コンテンツを入れる前のワイヤーフレームに近いテンプレート |
| Lv5 | Pages | ユーザーが実際に目にする最終デザイン |

> 詳細な設計思想は `building-design-systems` スキルを参照。

## 6. イメージ別UI・グラフィック表現の判断テーブル

| イメージ | 配色 | 書体 | グラフィック | 写真/イラスト |
|---------|------|------|------------|-------------|
| **キュート・ポップ** | 高彩度・暖色・多色 | 丸みのあるゴシック、手書き風 | 幾何学パターン、ラフな輪郭 | 手書きイラスト、大胆なトリミング（斜め・円形） |
| **エレガント・高級** | モノトーン＋アクセントカラー1色 | 細いセリフ体・細ウェイトゴシック | 大理石等のテクスチャ、洗練された余白 | 高品質写真、大判スライド |
| **ビジネス・信頼** | 落ち着いた青・紺・グレー | 読みやすいゴシック、統一感 | 整然としたグリッド、図表 | 表情が見える人物写真、ムービー |
| **オーガニック・ナチュラル** | 落ち着いた彩度、アースカラー | セリフ体、手書き風 | テクスチャ（布・紙）、有機的ライン | 素材・工程の写真、水彩調イラスト |
| **クール・先端** | ダーク背景＋ブランドカラー単色 | 細いコンデンス体、サンセリフ | 製図・ワイヤフレーム風の細線、3DCG | 製品写真、カラーオーバーレイ加工 |
| **和** | 日本の伝統色（くすみ系）、墨 | 縦組み対応書体、明朝体 | 和柄（亀甲・市松）、植物モチーフ、墨筆 | 落ち着いたトーンの写真、版画風・墨絵 |
