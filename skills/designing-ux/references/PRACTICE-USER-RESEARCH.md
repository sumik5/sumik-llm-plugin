# ユーザーリサーチ 実践ガイド

ユーザーが気づいていない本音を「感情曲線インタビュー」「弟子入りインタビュー」で収集し、「親和図法」で分析することで、本質的な欲求を発見する手法。

---

## 仕事としての始めやすさ

| こっそり練習 | 一部業務でトライアル | クライアント巻き込み |
|---|---|---|
| ★★ | ★★ | ★★★（★★★★） |

**こっそり練習**: 知り合いや同僚に「ちょっとお話を聞かせてください」とお願いし、カフェなどで気軽に実施できる。失敗しても問題ない環境から始める。まずは「友達」から。

**一部業務でトライアル**: ランチタイムを活用し、同僚・先輩をユーザーとしてインタビュー。分析結果を後日フィードバックすることでプロセスを「体感」してもらう。

**クライアント巻き込み**: 結果だけでなく「調査と分析の過程」をオープンにすることが鍵。仮説段階のユーザー調査から始めて、成果の一部を先に見せることでご理解を得る。

---

## 概要

ユーザーリサーチの目的は「事前に想定できないことを話してもらう」こと。普通に聞くだけでは本音は出てこない。表面的な発言（キラキラデータ）と、本人も気づいていない本音（ダークサイドデータ）を区別して収集・分析する。

**2段階の構成**:
- **前半戦（データ収集）**: 感情曲線インタビュー、弟子入りインタビュー
- **後半戦（データ分析）**: 親和図法

---

## 5-1 データ収集：感情曲線インタビュー

### 概要

ユーザー自身に感情曲線（感情の起伏を表すグラフ）を描いてもらいながら、思い出話をしてもらう手法。過去のことを振り返りながら話すのは、話す側にも聞く側にも楽しいため、本音が引き出しやすい。

### 進め方

1. 紙とペンを渡し、時間軸と感情（テンション）の上下でグラフを描いてもらう
2. グラフを描きながら、その時のことを話してもらう
3. グラフに直接メモを書き込むか、手元の付箋に書き出す

### 3つのコツ

**① 折れ点で何があったのかを聞き出す**

感情の上がり下がりが起きたポイントには何かが潜んでいる。
- グラフが下がっている箇所 → 課題のヒント
- グラフが上がっている箇所 → 思わぬ価値を感じているかも
「その時何があったの？」「どうして（テンションが）上がった（下がった）の？」と聞く。

**② 前後の部分も聞き出す**

ユーザーはWebサイトを突然訪れるわけではない。きっかけ・感想など、体験の前後にある時間も重要なデータ。

**③ ありのままの行動を聞き出す**

「思ったこと」と「取った行動」を別データとして扱う。
- 「思ったこと」 → 本人による脚色があるかもしれない
- 「取った行動」 → ウソがない

### 実施のポイント

- 最初は30分程度の短いインタビューから始める
- インタビュー時間（オン）とそれ以外（オフ）の切り替えを意識する
- 「ちょっとやってみる」→「ちょっと休む」を繰り返す

## 5-2 データ収集：弟子入りインタビュー

### 概要

ユーザーを「師匠」に見立て、インタビュアーが「弟子」として教えを請うインタビュー方法（Master/Apprentice Model）。事前に用意した質問をするのではなく、その場で新しい質問を作りながらユーザーに教えてもらう。

感情曲線インタビューの後に、特に気になった部分を深掘りする形で実施するのが効果的。

### 4つのコツ

**① 「はい/いいえ」で答えられる質問はしない**

❌ 「○○しますか？」「○○だと思いますか？」
✅ 「○○について**どう**思いますか？」「○○について**教えてください**」

相手が「うーん……」と考え込んでも、辛抱強く待つ。そこに本人も気づいていない部分が潜んでいる。

**② 関係なさそうなことでもひとまず聞いてみる**

後の分析で「実は関係があった」と分かることがある。ユーザー自身には関係があるから話してくれる、という姿勢で聴く。

**③ 脱線しすぎないようにコントロールする**

明らかに無関係な場合は笑顔で「それって……この調査に関係あるんですかね？」と聞く。話が本筋に戻るか、意外な関係が明らかになるか、どちらかが分かる。

**④ 最後に「何か話しておきたいことはありますか？」**

終盤は根掘り葉掘り聞きたくなるが、最後は「これだけは話しておきたいということはありますか？」と締めくくる。

### 教えてリスト（Webサイト調査の場合）

- そのWebサイト（サービス）を使ってみようと思ったきっかけ
- 利用期間・頻度
- 目の前で実際に使ってもらいながら、感じたことをそのまま話してもらう
- 不思議に思ったことだけでなく、理由が推察できるところでも「えっどうして？」と聞く

感情曲線インタビューと組み合わせる場合は「一番テンションが高かった・低かったところで何が起きていたのかをとことん教えて欲しい」という形で深掘りする。

## 5-3 データ収集の練習手順

### 共通手順

**リクルーティング**
- 知り合いに調査協力を依頼
- データの利用目的を事前に説明し承諾を得る
- 練習ならば仕事と無関係なテーマでもよい

**スタート**
- 最初は30分程度の短いインタビュー
- 一人目は必ず知人・同僚など近い関係の人から
- オン/オフの切り替えを意識する

### 記録のコツ

**とにかく全てを記録する**

❌ 初心者が犯しやすいミス：「大事なことだけ記録しよう」として選別・要約した記録しか残さない

→ 何が重要かはデータ分析の段階で判断する。この段階では「とにかく全て記録して持って帰る」。不要な情報は後で捨てればよい。

**一度やったらすぐ振り返る**

インタビューが終わったら直後に記録を見直す。階層構造が平坦すぎれば深掘りできていない証拠。次のインタビューの改善ヒントを見つける。

## 5-4 データ分析の心構え

収集した定性データはユーザーが分析していない「生データ」のため、そのまま真に受けない。

**ユーザーの言いなりにならない**

例：「カートに保存している商品の価格が下がったら通知して欲しい」
→ この要求の背景に潜む「本質的な欲求」を探る。背景（行動文脈）によっては、ユーザーの要求がビジネスの成果に繋がらない場合もある。

**本当の解決策を探る**

「ユーザーは、本当に望んでいることを、自分で意識していないから言語化もできていない」という前提に立つ。不満の仕組みを解き明かすことで、ユーザー自身が気づいていない解決策を導き出す。

## 5-5 親和図法（KJ法）

### 概要

インタビューで集めたデータから「暗黙知」を発見するための分析手法。付箋と模造紙を使ったアナログな作業が最も効率的・効果的。

**重要な認識**: 親和図法は「単なる情報の整理」ではなく「発想をして、課題を作るための手法」。名詞に注目した分類法ではなく、**動詞・形容詞に注目した課題創出法**。

### 4ステップ

```
STEP 1: データの「切片」を作る
   ↓
STEP 2: 切片で小グループを作り、見出しを貼り出す
   ↓
STEP 3: 小グループを大グループ化し、図として整える
   ↓
STEP 4: 文章に起こし、解釈と説明ができるようにする
```

### STEP 1: 切片を作る

インタビューデータ（文章）を、付箋1枚に収まる分量に抜き出して書き込む。

**ルール**:
- 事実のみを書く（感情や心理を代弁しない）
- 要約しない（後で分析するから）
- 大事かどうかの判断はしない（全て書き出す）
- 顔を近づけなくても読める文字サイズで書く
- 全ての付箋を模造紙などの広い平面に貼る

### STEP 2: 小グループと見出し

**最重要の作業**。ここの質が分析の質を決める。

小グループを作る = 複数の付箋間の「隠れた共通点」をチームで「想像・発想する」こと。

❌ **表面的な分類の例（NG）**:
見出し「ECサイトを見る」として、ECサイトに関する切片を集める → これは単なる情報整理

✅ **課題創出の例（OK）**:
見出し「明確な目的があってサイトを訪れていない」として、表面的には関係なさそうな切片でも隠れた共通点があれば同じグループに入れる → 「なぜ目的もないのに来るのか？」という発見につながる

**コツ**:
- 1グループは10枚以下を目安（大きすぎたら思い切ってバラす）
- 付箋1枚のグループがあってもよい
- 正しく作ろうとせず、小さなグループをすばやく多く作る
- 見出しはユーザーに共感できるよう豊富な情報を含める

### STEP 3: 大グループ化と図化

小グループの見出しにのみ着目して、さらにグループを作る。

小グループと全く同じように「分類せず創出する」ようにグループを作る。見出しが上手く書けていないとこのステップが機能しない → その場合はどんどん見出しを貼り替える。

グループ間に関係性が見つかったら図解してみる。

### STEP 4: 文章化

図として整えた後、「文章に起こす」ことで親和図法の威力が発揮される。

**文章化の目的**: 分析に参加していない人でも「説明なしに」理解できる状態にする。

文章化のルール:
- 専門用語・業界用語は廃す
- A3サイズの用紙に太いペンで書く
- ユーザーの利用状況・具体的な行動・価値や不満の所在を漏れなく盛り込む

**2種類の記述**:
- 黒字（説明的）: 実際の分析結果・事実
- 色字（叙述的）: 分析者による推察

## 5-6 親和図法 練習手順

1. **テーマと目的を設定する**: 例「ECサイト利用におけるユーザーの行動」を調査し、「ライトユーザーをヘビーユーザー化する施策を考える」
2. **付箋に切片を書く**: インタビューデータを1文ずつ付箋に書き起こす
3. **小グループを作る**: 隠れた共通点を発見しながらグループ化する（素早く・多く）
4. **見出しをつける**: 共通点を端的かつ豊かに表現する
5. **大グループ化**: 見出し同士の隠れた共通点を見つける
6. **文章化**: 全体の文脈を一読で理解できる文章を書く
7. **振り返り**: 文章化できたかを確認。できない部分＝追加調査が必要な部分

**「分析上手は動かし上手」**: 付箋を動かした手数・捨てた枚数の多さが品質の指標。

**「時間で区切る」**: 完璧に分析しきることは不可能。ある程度で次工程に進み、不足部分を後から補う方が効率的。

## 5-7 ツールキット

### 付箋

| 色 | 用途 |
|---|---|
| 黄色 | 切片 |
| オレンジ | 見出し（小グループ） |
| 青 | 見出し（大グループ） |
| ピンク | アイデア等 |

推奨サイズ: 75mm × 75mm の正方形タイプ。強粘着タイプが「行ったり来たり」のストレスを軽減する。

### 模造紙

推奨サイズ: 四六判（1091×788mm）。壁に貼ったまま放置するのはセキュリティ上問題のある場合があるため、模造紙を台紙として使い移動・折り畳みができるようにする。

折り目に付箋が貼られると剥がれやすいため、先に折り目をつけておき、折り目を避けて付箋を貼る。

### ペン

水性／角芯の太いペン（POPが書けるくらいの太さ）。誰からでも等しい距離感で読めることが重要。油性ペンは室内では臭いに注意。

## Light Case / Heavy Case

### Light Case（ライト級）

| 項目 | 内容 |
|---|---|
| 期間 | 2週間程度 |
| 体制 | 3名程度 |
| 調査対象 | 社内メンバー5名程度（ランチタイム活用） |
| 手法 | 感情曲線インタビュー + 社内ワークショップ（親和図法） |
| 目的 | 提案のための基礎データ収集 |
| ポイント | まず無償の「お試し調査」で成果の一部を先に見せ、正式受注に繋げる |

### Heavy Case（ヘビー級）

| 項目 | 内容 |
|---|---|
| 期間 | 2か月 |
| 体制 | 3名 + クライアント30名 |
| 調査対象 | 一般ユーザー10名（調査パネル経由） |
| 手法 | 感情曲線 + 弟子入りインタビュー + 合同ワークショップ（4チーム） |
| 目的 | クライアント協働でユーザーの「喜び」と「ガッカリ」を発見 |
| ポイント | コミュニケーション設計・チーム編成などの「チームデザイン」が成功の最大要因 |
