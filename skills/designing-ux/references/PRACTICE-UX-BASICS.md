## 仕事としての始めやすさ

| こっそり練習 | 一部業務でトライアル | クライアント巻き込み |

| ★★★ | ★★★ | ★★ |

UX視点の思考習得は一人でできるため最も始めやすい。チームへの啓蒙は関係者の理解が前提となる。

## UXとは何か

### UXの本質：「モノ」ではなく「コト」

UX（User Experience）とはユーザー体験のこと。製品・サービスという「モノ」ではなく、それを取り巻く環境全体という「コト」のデザインを指す。

**コーヒーショップの例で理解する**

コーヒーショップの商品（モノ）はコーヒーだが、実際の価値は：
- 店員の対応・雰囲気
- 椅子の座り心地・内装
- 立地・客筋・滞在時間
- ブランドとの継続的な関係

これらすべてを含む「体験全体（コト）」がUXである。

### UXの時間軸モデル

UXは製品・サービス利用中だけでなく、前後の時間にも広がっている。

| 時間軸 | 説明 | 例 |
|--------|------|-----|
| **利用前（予期的UX）** | 認知・期待・検討 | SNSで話題を見て期待が高まる |
| **利用中（一時的UX）** | 実際の使用体験 | 操作感・対応・完了感 |
| **利用後（エピソード的UX）** | 振り返り・比較 | 「あっちにしとけばよかった」 |
| **利用全体（累積的UX）** | 長期的な記憶・関係 | ブランドへの愛着・習慣化 |

**重要な示唆**: UX全体をコントロールするのは事実上不可能。だからこそユーザーを深く理解し、設計で貢献できる部分に集中することが大切。

### UI・UX・ユーザビリティの関係

```
UX（体験全体）
  └── ユーザビリティ（使いやすさ）
        └── UI（画面・操作インターフェース）
```

現場で「UXが悪い」と言われるとき、実際はユーザビリティの問題を指していることが多い。本来のUXはUIよりはるかに広い概念。

**Web制作者の実務での意味合い**

| よく使われる意味 | 本来の意味 |
|----------------|-----------|
| 「UIの使いやすさ」 | ユーザーの体験全体 |
| 「画面のデザイン品質」 | 利用前・中・後の全体設計 |

## UXデザインとは何か

UXデザインとは、UXを「設計」することである。「デザイン」は見た目の美しさではなく**設計・計画**を意味する。

### UXデザインの基本スタンス

1. **ユーザーを中心に置く**: ビジネスの都合からではなく、ユーザーから発想する
2. **ユーザーに学ぶ**: ユーザーを理解することを継続的に行う
3. **設計で貢献する**: UIを含む接点の設計でユーザー体験を向上させる

---

## 実務での現実と突破口

### よくある2つの壁

**壁①：タイミング問題**

プロジェクトに参加するときには、すでにスケジュール・予算・方針が決まっており、UXデザインを取り入れにくい。

→ 「早く声をかけてもらうこと」が究極の解決策だが、実績なく声はかからない。

**壁②：役割範囲問題**

「それはあなたの仕事じゃない」「ユーザーのことは分かっている」という反応が返ってくる。

→ 担当領域の外に出るような主張は最初は控え、自分が自然に口を出せる範囲から始める。

## 実務での取り組み方：3段階アプローチ

### Level 1：こっそり練習

ビジネス側の承認不要。設計者の判断でできる範囲。

**今日からできること**
- 担当するWebサイトやアプリを「ユーザーになりきって」使ってみる
- ユーザビリティの問題を書き出し、要因を考える
- 身近な人に使ってもらって観察する（非公式でOK）
- UX関連の書籍・資料で知識を積む

### Level 2：一部業務でトライアル

既存業務の延長として、少しずつ取り入れる段階。

**実践例**
- サイト評価依頼に対し、認知的ウォークスルーで問題発見
- ワイヤーフレーム設計にペーパープロトタイピングを組み込む
- 気軽に知り合いに話を聞いてチームで共有する
- 「当たり前だと思っていたのにユーザーは違う理解をしていた」を見つけて共有する

### Level 3：クライアントを巻き込む

関係者の理解と協力を得て、プロジェクトの公式プロセスとする段階。

**巻き込みのコツ**
- 成功事例・実績を先に作ってから提案する
- クライアントが既に困っていることの解決策として提案する
- 費用・期間への影響を最小化した形で最初の提案をする

```
   ↓

```

## UXデザインの価値を周囲に伝えるエンジン

周囲にUXデザインの価値を広げるための最強の共感ポイント：

> **「こんなにもユーザーのことを理解していなかったのか！」**

この「ユーザー無知の知」体験が、UXデザインへの協力者を生む。

**具体的な反応パターン**
- 「これはマズい。早急に改善しなければ」（リスク感覚）
- 「競合に差をつけるチャンスだ」（機会感覚）
- 「もっと良いものを作りたい」（プロダクトへの愛）

**実践のコツ**: まず自分が「ユーザー無知の知」を体験し、その体験をチームで共有する機会を作る。

## Light Case / Heavy Case

### Light Case（こっそり練習〜一部トライアル）
- 自分でWebサイトをユーザー視点で観察
- 身近な人に気軽に使ってもらって気づきを得る
- チーム内で共有して「ユーザーが理解していると思っていたことが実は違う」を発見する

### Heavy Case（クライアント巻き込み〜組織導入）
- 正式なユーザーリサーチやユーザビリティ評価の実施
- チーム全員でのワークショップ
- 組織的なUX導入プログラムの設計と実行
