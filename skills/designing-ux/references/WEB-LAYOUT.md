## 1. Webサイトの種類とデザイン方向性

サイト種類によってデザインの方向性・優先事項が大きく異なる。制作前に種類を確認し、デザイン方針を決定する。

| サイト種類 | 主目的 | デザインの優先事項 | 主なコンテンツ |
|-----------|--------|-----------------|---------------|
| **コーポレートサイト** | 企業の信頼性・ブランド訴求 | 情報整理・堅実さ・読みやすさ | 事業紹介・会社概要・採用情報 |
| **ECサイト** | 商品購入の促進 | 商品写真の見やすさ・検索性・信頼感 | 商品一覧・カテゴリナビ・決済フォーム |
| **メディアサイト** | 情報・記事の提供 | 大量コンテンツの整理・文字の読みやすさ | 記事一覧・カテゴリ・検索 |
| **プロモーションサイト** | 商品・キャンペーンの認知向上 | 視覚的インパクト・世界観表現・SNS拡散 | ビジュアル・動画・SNSリンク |
| **ランディングページ（LP）** | コンバージョン（購入・登録）の獲得 | CVボタンの反復・説得力・スクロール導線 | キャッチコピー・ベネフィット・CTA |

### サイト種類の判断基準

```
AskUserQuestion で確認すべき場面:
- 企業全体の発信 → コーポレートサイト
- 商品の販売 → ECサイト（購入機能あり）またはショップサイト（なし）
- 記事・情報提供 → メディアサイト（ニュース/オウンドメディア）
- 特定商品の訴求・キャンペーン → プロモーションサイトまたはLP
- 広告からの流入を受けてCVさせる → LP
```

## 2. レイアウトの基本概念

### 整列・整理・整頓

| 概念 | 意味 | 実践例 |
|------|------|--------|
| **整列** | 要素を列に並べて秩序をつくる | 右揃え・左揃え・中央揃え・上下揃えで視覚的な統一感を生む |
| **整理** | 必要・不要を論理的に振り分ける | 掲載すべきコンテンツとそうでないものを取捨選択する |
| **整頓** | 機能性を考慮して最適に配置する | 取り出しやすい（アクセスしやすい）配置で使いやすさを実現する |

### レイアウトの本質

- レイアウト = **「何をどう伝えるか」を形にする作業**
- 見出し・本文・図版などの構成要素に優先度を付け、大きさ・配置・余白を調整して伝えたいメッセージを形にする
- センスではなく「緻密な設計」によって、かっこいい・高級感・親しみやすさといった印象を作り出せる

## 3. Webレイアウト固有の特徴

### 印刷物との根本的な違い

| 観点 | 印刷物 | Webページ |
|------|--------|----------|
| ページサイズ | 固定 | **可変**（ウィンドウサイズに依存） |
| 閲覧デバイス | 紙 | PC・タブレット・スマートフォン等 |
| 操作 | 読むのみ | クリック・スクロール・入力など |
| 更新性 | 印刷後は不変 | 動的に更新される |
| 求められるスキル | 読みやすさ | **使いやすさ** + 読みやすさ |

### 可変幅とレスポンシブWebデザイン

- Webページの基本仕様は**可変幅（full-width）**。ブラウザのウィンドウサイズに合わせてコンテンツが伸縮する
- **レスポンシブWebデザイン**: デバイスのスクリーンサイズに応じてレイアウトを柔軟に変化させる手法。現在の主流
- 固定幅（pixel指定）は特定条件下で有効だが、左右に余白が生まれたりコンテンツが隠れるリスクがある

### ブロック構造

Webページの基本構造はHTML/CSSによる**ブロックの積み重ね**:

```
┌─────────────────────┐
│      ヘッダー         │  グローバルナビゲーションを最上部に配置
├─────────────────────┤
│                     │
│    コンテンツ領域      │  メインコンテンツ・サイドバー等
│                     │
├─────────────────────┤
│      フッター         │  補助的なリンク・コピーライト等
└─────────────────────┘
```

- テキストは「横書き・左から右・上から下」へ流れる（英語・日本語共通）
- ページは垂直スクロールが基本。「巻物」のように積み重なった構造
- すべての構成要素は**ボックス**として扱われ、グリッドベースで設計する

### レイアウトスタイルの種類

| スタイル | 特徴 | 適したサイト種類 |
|---------|------|----------------|
| シングルカラム | スマートフォンとの差異が少ない・読みやすい | ブログ・記事・LP |
| マルチカラム（2〜3列） | 情報量を多く見せられる・ナビゲーションをサイドに置ける | EC・コーポレート・ポータル |
| タイル（カード型） | 同等情報の一覧表示・モジュール化しやすい | メディア・EC商品一覧 |
| フルスクリーン | 視覚的インパクト・世界観表現 | プロモーション・エンタメ |

## 4. モバイルとデスクトップの考慮

### デバイスごとの特性

| デバイス | スクリーン | 主なレイアウト |
|---------|-----------|--------------|
| デスクトップPC | 横長・大 | マルチカラム・ワイドレイアウト |
| ノートPC | 横長・中 | マルチカラム（2列が多い） |
| タブレット | 中・縦横両対応 | 2列またはシングルカラム |
| スマートフォン | 縦長・小 | **シングルカラム** |

### 可変レイアウト設計の原則

- **デスクトップファースト vs モバイルファースト の対立より「可変レイアウト設計」が効率的**
- PCでは2〜3カラムのサイドナビゲーションが有効でも、スマートフォンではシングルカラムへ再配置が必要
- 最初から可変レイアウトで設計することで、デバイス固有の問題を事前に防げる

### ナビゲーション設計

- グローバルナビゲーションは**全ページ共通で最上部**に配置（最もスタンダード）
- 大規模サイト（数百〜数千ページ）はローカルナビゲーションをサイドに固定してユーザビリティを向上
- 階層構造を持つサイトは「現在地」を示すUIパンくずリスト等が必須
- モバイルでは「ハンバーガーメニュー」等によるナビゲーションの折りたたみが一般的

## 5. レイアウト4原則

### 原則① 拡げる（揃える）

**要素を揃えることで秩序をつくり、信頼感・安定感を生む**

- Webページのデフォルトは左揃え。右側に余白が集中するのが基本
- **中央揃え**: サイトタイトル・大見出しなど、大きな要素は中央に配置すると安定する
- レスポンシブ設計では、ワイドスクリーンは左揃え、スマートフォンは中央揃えに切り替えることがある
- 斜め揃えは難易度が高いが、動的なサイトで視覚的インパクトを出せる

| 揃えの種類 | 効果 | 使いどき |
|-----------|------|---------|
| 左揃え | 標準・読みやすい | 本文テキスト・リスト |
| 中央揃え | 安定感・フォーマル | 見出し・LP・コーポレート |
| 右揃え | 変化・個性 | アクセント要素 |

### 原則② 近づける・反復する

**関連要素を近づけ、同種要素を反復することで可読性と規則性を生む**

**近づける（グループ化）**:
- 関連性のある情報同士を集めてグループをつくる
- 区切り線・囲み線・余白の差で情報をグループ化する
- 「整然と並んでいるだけ」の情報は読み取りにくい。近づける/離す調整で改善する

**反復する（モジュール化）**:
- 見出し・画像などをグループ化した「モジュール」を水平垂直に繰り返す
- 規則性 → 安定感・視線誘導 → 視認性向上
- CMSやブログに最適。大量コンテンツでも一貫性を確保できる

### 原則③ 差をつける・動きをつける

**対比によってメッセージを強調し、モーションで体験を豊かにする**

**差をつける（対比）**:
- 見出しを本文より過度に大きくする → 強いメッセージ発信
- 重要要素の周囲に大きな余白をとる → 注目を集める
- 縦書きと横書きの混在 → 視覚的なアクセント

**動きをつける（モーション）**:
- 印刷物にはない Webレイアウト固有の表現手段
- スクロールエフェクト（パララックス・フェードイン等）
- トランジション・マイクロインタラクション
- 動きは「さりげない演出」が基本。大掛かりなアニメーションは必要最低限に
- 設計プロセスを先行させ、実装（JavaScript）との連携が重要

### 原則④ 余白・ジャンプ率

**余白とサイズ比率でメッセージの重みをコントロールする**

**余白（ホワイトスペース）**:

| 余白の量 | 印象 | 適したサイト |
|---------|------|-------------|
| 大きい（ビッグスペース） | ゆったり・高級・上品 | ラグジュアリーブランド・ポートフォリオ |
| 中程度 | バランスよく読みやすい | コーポレート・ブログ・LP |
| 小さい（余白少） | 情報密度高・一覧性高 | ニュースサイト・ポータル・EC |

**ジャンプ率**（見出しと本文のサイズ比率）:
- **高いジャンプ率** = 見出しが本文に対して非常に大きい → 強いメッセージ・スポーツ紙的インパクト
- **低いジャンプ率** = 見出しと本文の差が小さい → 落ち着いた・品のある印象

## 6. 目的別レイアウトパターン判断テーブル

| サイト種類 | 推奨カラム数 | 重点レイアウト要素 | 視線誘導パターン |
|-----------|------------|-----------------|---------------|
| **メディアサイト** | 2〜3列（またはシングル） | カードグリッド・モジュール反復・カラム数と余白の統一 | F字型（スキャン読み） |
| **エンタメ・プロモーション** | フルスクリーン | ビジュアル優先・動画・モーション・大胆な大きさ差 | 感情的なビジュアルで引き込む |
| **コーポレートサイト** | 1〜2列 | Z字型・ヒーローイメージ・CTA配置・動画背景 | Z字型（ロゴ→ナビ→CTA） |
| **ショップサイト（EC機能なし）** | シングルまたはタイル | 余白大・ボックスレイアウト・フォーカルポイント | 商品へ視線を誘導 |
| **ECサイト** | グリッド + F字型の使い分け | 商品写真の見せ方・サイドバー固定・動画活用・グリッド→F字型の段階切り替え | グリッド（ラインナップ）→F字型（一覧・比較） |
| **ランディングページ** | シングルカラム | CVボタン反復・セクションの差別化・パララックス・ナビゲーション固定 | 縦スクロール一本道 |
| **ブログサイト** | シングルまたは3カラム固定 | 余白・タイポグラフィ・読みやすさ優先・ミニマル | F字型（記事一覧）→線形読み（記事本文） |

### 目的別の補足ポイント

**メディアサイト**:
- カードグリッドが最も普及。カテゴリ別の色分け・ホバーエフェクトで情報を見つけやすくする
- 文字量が多い場合は文字サイズ・行間・余白を広くとりシングルカラムで集中を促す
- 重要記事は写真サイズ・見出し強調で視覚的に優先順位をつける

**ECサイト**:
- 商品一覧はグリッドレイアウト（直感的）、詳細比較はF字型レイアウト（スキャンしやすい）を段階的に使い分ける
- サイドバー固定でカテゴリ・フィルタ常時表示 → 迷わず目的商品へ
- 画面分割（2分割）で異なるターゲットへの入り口を同等に提供できる

**ランディングページ**:
- CVボタン（CTA）を要所に反復設置。スクロール離脱を防ぐ
- セクション間の背景・余白・色で区切りを明確にする
- パララックス等のスクロールエフェクトはLPとの親和性が高い

## ユーザー確認の原則（AskUserQuestion）

判断分岐がある場合、推測せずに確認する。

**確認すべき場面**:
- サイト種類（コーポレート/EC/メディア/プロモーション/LP）の特定が曖昧な場合
- モバイル/デスクトップのどちらを優先設計するか
- レスポンシブデザインの採用範囲（全ページ/一部/なし）
- 固定レイアウト vs 可変レイアウトの選択
- ナビゲーション方式（グローバル上部固定/サイドバー/ハンバーガー）

**確認不要な場面**:
- レスポンシブデザインの採用自体（現在の標準ベストプラクティス）
- ヘッダー→コンテンツ→フッターのブロック構造（Web標準）
- 見出しと本文を視覚的に差別化すること（ジャンプ率の基本）
