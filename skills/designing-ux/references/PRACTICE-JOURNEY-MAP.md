# カスタマージャーニーマップ 実践ガイド

ユーザーの体験全体（利用前・利用中・利用後）を時系列で可視化し、課題を発見・改善するための手法。

---

## 仕事としての始めやすさ

| こっそり練習 | 一部業務でトライアル | クライアント巻き込み |
|---|---|---|
| ★★ | ★★ | ★★★（★★★★★） |

**こっそり練習**: ユーザー調査データや分析結果があれば始められる。なければ PRACTICE-USER-RESEARCH.md のユーザー調査から着手する。

**一部業務でトライアル**: 「カスタマージャーニーマップ」という言葉がポピュラーになったため、上司・先輩の理解を得やすい。論拠の弱いジャーニーは「思い込みの強い仮説」に見えるため、実データを十分集めてから実施する。スコープを「そもそもライン」を越えすぎないよう上手にコントロールする。

**クライアント巻き込み**: ワークショップにクライアントを参加させるか、少なくとも見学してもらう。完成版の前に「仮段階のラフ」を打ち合わせに持ち込み、クライアントが持っている知見・データを提供してもらうアプローチが有効。

---

## 概要

カスタマージャーニーマップ（CJM）は、特定のペルソナが製品・サービスを利用する前後を含めた体験全体を1枚のマップとして可視化したもの。

**2つの主な目的**:
1. 課題発見: ユーザーの行動を俯瞰し、Webサイトや製品・サービスの改善点を発見する
2. 合意形成: クライアントも含めた関係者全員がユーザー目線で議論できる共通の土台を作る

---

## 6-1 カスタマージャーニーマップが求められるわけ

### Webサイトありきでユーザーは行動していない

IT業界やWeb制作の関係者は「自社Webサイトありき」で考えがち。しかしユーザーは複数のメディア・お店・デバイスを行き来しながら暮らしている。

**心構え**:
- 「ユーザーは自社のWebサイトだけを優先して突然来るわけではない」
- 「ユーザーはその場面でインターネットを使うのだろうか？」

自社Webサイトを含むユーザー行動の全体像・前後関係を見つめることで、本当の課題が見えてくる。

### マーケティングとの接点

カスタマージャーニーマップはマーケティングの文脈でも活用が増えており、Web制作側への依頼も増加している。施策立案との連携・インフォグラフィック的な表現が求められる場合もある。

---

## 6-2 カスタマージャーニーのための調査

定量的なアンケートやアクセス解析だけでは、個々のユーザーにとってリアリティのある行動・本音・感情は把握しきれない。

定量データ + インタビュー（感情曲線・弟子入り）で精度を高める。

### 調査項目の例（ECサイト）

| 調査項目 | 利用前 | 利用中 | 利用後 |
|---|---|---|---|
| 状況 | 訪問前の状況は？ | 訪問時の状況は？ | 訪問後の状況は？ |
| タッチポイント | 接触したメディア・人は？閲覧した他サイトは？ | どのサイト経由で来訪？ | 接触したメディア・人は？ |
| デバイス | どのデバイスを使ったか？ | どのデバイスで閲覧したか？ | どのデバイスを使ったか？ |
| 時間 | 利用した時間帯は？ | 利用時間帯は？ | 利用した時間帯は？ |
| 場所 | 利用した場所は？ | 閲覧場所は？ | 利用した場所は？ |
| 目的 | 商品を買って何をしたいのか？ | 何をしたいのか？ | 購入後に何をしたいか？ |
| 心理・感情 | その時どんな気持ちだったか？ | どんな気持ちだったか？ | その時どんな気持ちだったか？ |

「利用前・利用中・利用後」の時間軸に沿ってデータを収集する。

## 6-3 カスタマージャーニーマップを作る

### STEP 1: 旅の主人公（ペルソナ）を設定する

まず簡易ペルソナを作成する。Webサイトがテーマでも、普段のライフスタイルや日々の暮らしぶりが分かるよう具体的に書く。

ペルソナの作り方は PRACTICE-SCENARIO-METHOD.md の「4-1 ペルソナ作成」を参照。

### STEP 2: 調査データを付箋に書き時系列で並べる

定性データ（インタビュー・アンケート自由回答・フィールドワークメモ）を付箋に書き、ホワイトボードや模造紙に時系列で並べる。

一度貼った付箋でも、前後関係やつながりがおかしければ何度でも貼り直す。

### STEP 3: ジャーニーを構築する

以下の順序で層（レーン）を作りながらジャーニーを描く。

```
① ステップ・タッチポイント・行動・思考・感情の区分けを作る

② 「行動」の枠に調査データの行動にあたる付箋（黄色）を貼っていく
   → ペルソナの人物が行動する様子を想像しながら

③ 「思考」の枠に調査データの思考にあたる付箋（緑色）を貼っていく
   → 同様にペルソナになりきって

④ 行動・思考の流れが見えてきたら「ステップ」を付箋（青色）に書いていく

⑤ ステップごとに登場する人・お店・Webサイトなどの「タッチポイント」を
   付箋（オレンジ色）に書いていく

⑥ 最後にペルソナの気持ちの起伏を想像しながら曲線で「感情」を描く
   曲線の山や谷にその気持ちを付け足す
```

### STEP 4: 複数人のワークショップ形式で進める

一人でも作れるが、複数人での実施が理想。理由:
- 調査データに含まれる行動・思考は多岐に亘り、一人の知識・経験では判断が難しい
- 複数の知見を活用することで分析の客観性・精度が上がる
- クライアントを巻き込むことで「自分事」としてとらえてもらえる

大きな会議室の壁面や窓ガラスなど広いスペースを使う。

## 6-4 "現状の姿"から課題を解決する

### 課題点の発見

"現状の姿"のカスタマージャーニーマップを俯瞰すると、主人公が困っている姿が見えてくる。

**2種類の課題**:
1. **明白な課題**: 感情曲線が下がっている箇所 → その時またはその前後に何が起きているかを分析する
2. **見えにくい課題**: 感情曲線は下がっていないが、もっと良い体験ができたはずの部分（例: 電話で即解決できるのにメールを使っている）

課題点が見つかったら、手がかりとなる付箋を赤丸で囲み、課題を文章にした別の付箋を追加して可視化する。

### 課題解決案の検討

1. 感情曲線が下がっている部分を上げるにはどうすればよいかを考える
2. 改善案を追加し、感情曲線がどの程度変化するか線を引き直す
3. ジャーニーの始まりから終わりまでを通して見渡し、主人公の気持ちをイメージして判断する

**注意**: 100%ユーザーの満足を追求すると企業側に利益がなくなる場合もある。ユーザーのニーズと企業側のニーズのバランスをとる。

## 6-5 ツールキット

### CJMの基本フォーマット

| レーン | 内容 |
|---|---|
| ステップ | 体験の段階（例: 認知→検討→購入→利用→継続） |
| タッチポイント | 接触するメディア・人・場所・デバイス |
| 行動 | ペルソナが取る行動 |
| 思考 | ペルソナが考えていること |
| 感情（感情曲線） | テンションの上下（曲線で描く） |
| 現状課題 | 課題点の文章化 |
| 改善案 | 課題に対する解決案 |

フォーマットに決まりはなく、プロジェクトのテーマに応じてカスタマイズする。例: 家電製品なら「候補メーカー機種」レーンを追加するなど。

### テンプレートの活用

完成品から清書するのは時間がかかるため、以下の順序で進める:
1. まずホワイトボード・模造紙で付箋を使ったラフ版を作成
2. ラフ版を写真撮影して関係者と共有
3. 必要な場合のみPowerPoint等で清書する

アイコン（タッチポイントを図解化したもの）を使うと直感的に理解しやすい。

## Light Case / Heavy Case

### Light Case（ライト級）

| 項目 | 内容 |
|---|---|
| 期間 | 1週間程度 |
| 体制 | ディレクター×1 + デザイナー×1（社内インタビュー2名） |
| 調査 | 社内インタビュー（サービス利用経験者） |
| 目的 | 初めてCJMを作るクライアントへの叩き台提示 |
| 成果物 | 簡易ペルソナ + ラフカスタマージャーニーマップ |
| ポイント | 予算をかけずに「今後どのような調査をすべきか」の検討材料を提供する |

### Heavy Case（ヘビー級）

| 項目 | 内容 |
|---|---|
| 期間 | 2か月 |
| 体制 | ディレクター・CDr・リサーチャー・HCD専門家（+調査会社経由インタビュー5名） |
| 調査 | 定量アンケート + インタビュー対象者選定 + インタビュー実施 |
| 目的 | "現状の姿"CJMから"あるべき姿"CJMまで描く |
| 成果物 | 現状CJMワークショップ + あるべき姿CJMワークショップ |
| ポイント | テーマの「商材」ならではのリアリティを持たせるためにCJMをカスタマイズ |

## クライアントを巻き込むポイント

### ワークショップへの参加

可能であればクライアント側の担当者にワークショップへの参加・見学を依頼し、意見をもらう。完成品だけを見せると「他人事」に感じられやすいため、作成プロセスを「自分事」としてとらえてもらうことが重要。

### 仮段階のラフを打ち合わせに持ち込む

受託制作でクライアントをワークショップに呼べない場合:
1. まず「仮段階のカスタマージャーニーマップ（ラフ）」を作成する
2. 打ち合わせに持ち込み「このCJMを精査するために、貴社が持っているファクトやデータをいただけますか？」とお願いする
3. クライアントが提供したデータがCJMに反映されることで、納得感・当事者意識が高まる

最終成果物はインフォグラフィック形式にして他部署・上位層と共有しやすい体裁にすると、次につながりやすい。

## チェックリスト

### 調査
- [ ] 利用前・利用中・利用後の時間軸でデータを収集したか
- [ ] 定量データと定性データ（インタビュー）を組み合わせているか

### マップ作成
- [ ] ペルソナ（主人公）が具体的に設定されているか
- [ ] 全レーン（ステップ・タッチポイント・行動・思考・感情）が埋まっているか
- [ ] 感情曲線が描かれているか
- [ ] 課題点が可視化（赤丸・付箋追記）されているか

### 課題解決
- [ ] "現状の姿"の課題点が文章化されているか
- [ ] 改善案が検討されているか
- [ ] ユーザーニーズと企業ニーズのバランスが考慮されているか

### クライアント対応
- [ ] 調査・分析の過程をオープンにしているか（結果だけの提示になっていないか）
- [ ] クライアントを巻き込む機会を設けているか
