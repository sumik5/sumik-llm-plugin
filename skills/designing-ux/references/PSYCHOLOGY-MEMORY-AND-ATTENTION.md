# 記憶と注意力

注意力と記憶力の強みと弱みを理解し、それらを支援・強化するUIを設計するための認知心理学的基盤。

---

## 短期記憶と長期記憶

### 長期記憶の特性

長期記憶は、特定の場所に保存された情報の塊ではなく、脳全体に分散した**ニューロン活動パターン**として実現される。記憶が作成されるとは、そのパターンが将来再活性化されやすくなるようニューロンが変化することである。

| 特性 | 内容 |
|------|------|
| 分散保存 | 数百万のニューロンにまたがるネットワークとして存在。一部を切除しても完全消去にはならず、細部が曖昧になる |
| 容量 | 事実上無制限。860億個のニューロンが生み出す組み合わせは膨大 |
| 記録形式 | 高度に圧縮・抽象化。特徴の組み合わせとして記録され、ビットマップ的な精密コピーではない |
| 長期記憶の種類 | 意味的（事実・関係）・エピソード的（出来事）・手続き的（行動の配列） |

**UIへの示唆**: 長期記憶は不正確なため、ユーザーに正確な操作を「覚えておく」よう要求してはならない。UIは常に外部記憶として機能すべきである。

---

### 短期記憶（作業記憶）の特性

作業記憶とは「貯蔵庫」ではなく、**ある特定の時間に意識にのぼっているものすべての集合体**である。知覚からの入力と、長期記憶から再活性化された内容が統合される場所。

#### 容量の限界

| 研究 | 結論 |
|------|------|
| Miller (1956) の仮説 | 7 ± 2 項目（後に過大評価と判明） |
| 後続研究の修正値 | **4 ± 1 項目**（3〜5項目）が実際の容量 |
| 最新の見解 | 項目数よりも「特徴量」で測定すべき。注意を多く払った項目ほど詳細に保持される |

**重要**: 関連する項目は「かたまり（チャンク）」として1項目扱いになるため、見かけ上の容量は変動する。

#### 揮発性（すぐ消える）

新しい情報に注意を向けると、それまで保持していた情報が失われる。この揮発性は「目的自体」にも適用され、以下のような現象を生む：

- 別の部屋に行った瞬間、用件を忘れる
- 電話後、直前にしていたことを忘れる
- 計算中に気が散ると最初からやり直しになる

## 注意と作業記憶

### 注意の集中性と選択性

人間の知覚は周囲の情報を選択的に取り込む。特定の目標を持つと、それに関連する情報だけにフィルタリングされる。また、以下のものは目標とは無関係に注意を引く：

- **動き**（特に自分に向かってくるもの）
- **危険の予兆**
- **人の顔**

### 作業記憶の容量とUIデザイン

| UIパターン | 作業記憶への影響 | 設計指針 |
|-----------|----------------|---------|
| モード | 現在のモードを忘れると「モードエラー」が発生 | モードを避けるか、現在のモードを常時明示する |
| 検索結果 | 結果を見ると検索語への注意が失われる | 検索結果ページに検索語を常に表示する |
| 動作の要求 | 複数の依頼が重なると最初のもの以外が忘れられる | 1ページ・1メールにつき1つの主要アクションに絞る |
| 指示 | 長い指示は作業記憶の容量を超える | 作業完了まで指示全体を参照可能な状態に保つ |
| ナビゲーション階層 | 深い階層では自分の位置と目的を忘れる | 階層は浅く広く。3階層以上ではパンくずリストを表示 |

## 長期記憶とUI

### エラーを起こしやすい

長期記憶は経験を正確に記録しない。高度に圧縮・抽象化されるため、記憶した詳細が脱落したり、別の記憶と混同されたりする。

- 例：メニューの場所がどこにあったか覚えていない
- 例：「最初のペットの名前」が複数ある場合、どちらを登録したか覚えていない

### 感情に影響される

強い感情を伴った記憶ほど鮮明で持続する。日常的な繰り返し作業の記憶は薄れやすい。

### 時間的に変容する

記憶は想起のたびに再構成され、細部が変化・脱落する。特に「一般的な概念に合わない部分」が消えやすい。これは証言の信頼性問題にも関わる。

### UI設計への影響

| 問題領域 | 悪い設計 | 良い設計 |
|---------|---------|---------|
| パスワード | 複雑な制限（数字のみ6〜10桁等）で「覚えやすい番号に」と矛盾した指示 | 制限を緩和し、ユーザーが覚えられる形式を自由に設定させる |
| セキュリティ質問 | 選択肢が限定されており、自分に当てはまるものがない | 自分で質問を作成できるようにする |
| 一貫性 | 機能によって操作が異なり、例外が多い | 操作の一貫性を保ち、長期記憶への負荷を最小化する |

## 注意力と短期記憶の限界

### 目標集中と道具の透明性

ユーザーは目標達成に注意力の大半を使い、**道具（アプリ・UI）にはほとんど注意を払わない**。道具はユーザーに意識させることなく機能すべき「透明な存在」であるべきだ。

> UIが自己主張をすると、ユーザーの作業効率と目標達成を妨げる。

**設計指針**: アプリケーションは目立たない影のような存在として、ユーザーが目標に集中できるよう支援する。

---

### 変化の見落とし（Change Blindness）

ユーザーは目標と関係のない変化に気づかない。

- **注意を払わないことによる見落とし（Inattentional Blindness）**: タスクに集中していると、明らかな変化すら気づかない（「見えないゴリラ」実験）
- **変化の見落とし（Change Blindness）**: 画面上の変化に気づかず、選択ミスが生じる

**設計指針**:
- 重要な変化は視覚的・聴覚的に目立つ形で示す（ハイライト、バイブレーション等）
- エラーメッセージは初回表示時に注意を引く演出を加える

---

### 情報追跡（Information Scent）

ユーザーは目標達成時、画面の表示内容を**文字どおりに受け取り**、現在の目標に合致する「匂い」のする言葉に引き寄せられる。

| 問題パターン | 例 | 設計指針 |
|------------|-----|---------|
| 曖昧なラベル | ATMで「支払い」と「振込」が重複して見える | ユーザーの目標と逐語的に合致するラベルを使う |
| 誤解を招くラベル | 予約キャンセル確認ダイアログの「OK」と「Cancel」 | 「はい」「いいえ」のような明確な動詞を使う |
| 目標の匂い不足 | ホームページから目標への経路が不明確 | 各決断の瞬間に、目標達成への「匂い」が明確な選択肢を提示する |

---

### 目標→実行→評価サイクル

ユーザーは常に以下のサイクルで行動する：

```
目標設定 → 達成するための行動を選択・実行 → 効果を評価 → 繰り返す
```

このサイクルは複数の階層（大目標→中目標→小目標）で同時進行する。

**UIでの支援**:
| フェーズ | UIの役割 |
|---------|---------|
| 目標 | 達成までの経路を第1ステップから明確に示す |
| 実行 | コンセプトをタスクベースにし、何をすべきかを推測させない |
| 評価 | 進捗状況フィードバックを提供。目的達成に役立たないタスクは中止可能にする |

---

### 目標達成後の後始末忘れ

タスクの主要目標を達成すると、関連情報が**直ちに短期記憶から消える**。その結果「後始末」を忘れやすくなる。

| 後始末忘れの例 | 設計による解決策 |
|--------------|----------------|
| ヘッドライトつけっぱなし | 自動消灯またはアラート |
| コピー機に原稿置き忘れ | タスク完了時に自動排出または通知 |
| 特殊モードのまま放置 | タスク完了後、自動的に通常モードに戻す |
| ログアウト忘れ | 一定時間後の自動ログアウト |

## 認識と想起

### 認識の容易さ vs 想起の困難さ

| 比較軸 | 認識 | 想起 |
|--------|------|------|
| 定義 | 知覚入力がニューロンパターンを再活性化 | 知覚入力なしにニューロンパターンを再活性化 |
| 速度 | 瞬時（1秒以下） | 遅く、失敗しやすい |
| 信頼性 | 高い | 低い。間違った記憶を想起することがある |
| 進化的背景 | 生存に直結（捕食者の即時識別） | 歴史や事実の暗記には向かない |

> GUIの誕生はこの非対称性に基づく。「見て選ぶほうが、思い出して打ち込むより楽」

---

### UI設計パターン（認識ベース）

#### サムネイル表示

ユーザーが以前に見た画像は、小さなサムネイルでも認識できる。サムネイルにより、より多くの選択肢・データ・履歴を一度に提示できる。

- 例: 写真管理アプリのスライド一覧
- 例: ブラウザの閲覧履歴のサムネイル

#### 頻度順・目立ち度による表示

| 機能の使用頻度 | 表示方針 |
|--------------|---------|
| 多くのユーザーが使う機能 | 目立つ位置に表示。ユーザーは「見て認識」できる |
| 少数・パワーユーザー向け機能 | 詳細パネル・右クリックメニュー・ショートカットキーに隠す |

#### 現在位置の視覚的手がかり

視覚的認識は高速かつ信頼性が高い。以下の手法でユーザーの現在位置を瞬時に伝える：

- サイト全体の一貫した視覚スタイル（「今どのサイトにいるか」の認識）
- セクションごとの微妙なスタイル変化（「今どのセクションにいるか」の認識）
- パンくずリスト（3階層以上のナビゲーション）
- デスクトップのワークスペース固有の背景画像

#### 認証情報の想起しやすさ

認証情報を「思い出しにくくする」設計はセキュリティを低下させる（ユーザーが紙に書いたり、推測されやすいパスワードを使ったりするため）。

| 問題のある設計 | 改善された設計 |
|--------------|--------------|
| 「数字のみ6〜10桁」のような強制的な制限 | ユーザーが覚えやすい形式で自由に設定させる |
| 選択肢が固定されたセキュリティ質問 | 自分で質問を作成できるオプションを提供 |
| パスワードのヒント登録なし | 自分だけわかるヒントを登録できるようにする |
| 知識認証のみ | 生体認証（指紋・虹彩等）を選択肢として提供 |

## クイックリファレンス

### 作業記憶を考慮したUI判断基準

| 状況 | 推奨アプローチ |
|------|--------------|
| モードが必要な場合 | 現在のモードを常時・明確にフィードバック |
| 複数の情報を表示 | 3〜5項目以内に収める。関連するものはグループ化 |
| 複数ステップの作業 | 全ステップを完了まで参照可能な状態に保つ |
| 深い階層構造 | 3階層以上は避ける。不可能な場合はパンくずリスト必須 |
| キャンセル系ダイアログ | 「はい/いいえ」など、行動を明示する動詞を使う |
| 検索機能 | 検索結果ページに検索語を常に表示する |

### 認識 vs 想起 の判断基準

| ユーザーに求めること | 認識 | 想起 |
|------------------|------|------|
| UIの負荷 | 低い（推奨） | 高い（避ける） |
| 実現方法 | 選択肢を見せて選ばせる | コマンドや文字列を打ち込ませる |
| 適切な場面 | ほぼすべての操作 | 検索クエリなど、ユーザーが何を入力すべきか明確な場合のみ |
