# Experiencability（体験可能性）詳細リファレンス

## フレームワークの概要

須賀佑紀・津崎将氏が提唱する体験設計フレームワーク。従来のUI/UX設計が「ユーザーがUIに接触している時間」だけに着目していた問題を解決する。

**Experiencability（体験可能性）**: サービスとユーザーの関係全体を俯瞰し、UIとの接触時間を超えた体験を設計する概念。

---

## 可制御環境 vs 不可制御環境

### 可制御環境（UIとの接触時間）

デザイナーが直接コントロールできる環境。

**特性**:
- ユーザーが画面を見て操作している
- UIデザイン・UXデザインが主な手法
- フィードバックが即座に得られる

**設計手法**: UIデザイン・インタラクションデザイン・マイクロインタラクション・オンボーディング

### 不可制御環境（UIとの非接触時間）

デザイナーが直接コントロールできない環境。

**特性**:
- ユーザーがアプリ・サービスを使っていない時間
- 日常生活の文脈の中にいる
- サービスの「存在感」だけが頼り

**設計手法**: 世界観デザイン・ブランディング・コミュニティ設計・口コミ設計・Push通知戦略

```
1日24時間のうち、ユーザーがアプリを開いている時間は数分〜数十分。
残りの23時間以上が「不可制御環境」。
この時間にサービスを「思い出してもらえるか」が体験全体を左右する。
```

---

## 世界観デザインで期待を獲得する

### リソースが限られた組織の強み

資金・技術力・ブランド認知では大企業に勝てない。しかし「世界観の設計」はリソースに依存しない。

**原則**: 優れた世界観は「このサービスを使った後の自分」を想像させる。「使ってみたい」ではなく「なりたい自分に近づける気がする」体験を事前に提供する。

**世界観を構成する要素**:
| 要素 | 説明 |
|------|------|
| ビジョン | サービスが目指す未来の状態 |
| トーン＆マナー | 視覚的・言語的な一貫したスタイル |
| ユーザーの「なりたい姿」 | サービスを使うことで得られる自己像 |
| コミュニティ感 | 同じ価値観を持つ人々とつながる感覚 |

## 5つのイネブラ（ユーザー循環システム）

イネブラ = 「能える（いのる）」から派生。ユーザーがサービスに引き込まれる5つのフェーズ。

### 流入イネブラ

**定義**: 新規ユーザーが初めてサービスを知り、使い始めるフェーズ。

**設計課題**:
- 世界観の魅力でサービスへの「期待」を作る
- 「使ってみたい」という感情的動機を醸成する
- 最初の体験（ファーストインプレッション）で期待を裏切らない

**設計手法**:
- SNS・口コミ・メディアでの世界観伝達
- App Store/Google Playのスクリーンショット・説明文の最適化
- ランディングページでのビジョン提示

### 送流イネブラ

**定義**: 初回使用から「習慣的な使用」へ移行させるフェーズ。

**設計課題**:
- 学習コストを最小化する（オンボーディング最適化）
- 最初の「aha moment（ゆっくり確認する瞬間）」まで確実に届ける
- 離脱ポイントを特定・除去する

**設計手法**:
- プログレッシブ・ディスクロージャー（少しずつ機能を開示）
- 体験型チュートリアル（教えるのではなく体験させる）
- 最小限の登録フロー

### 還流イネブラ

**定義**: 一時的に離脱したユーザーを呼び戻すフェーズ。

**設計課題**:
- 離脱の原因を分析し、改善する
- 戻ってきた時の「再開しやすさ」を設計する
- 通知・リマインダーの適切な設計

**設計手法**:
- リエンゲージメントメール・プッシュ通知
- 「前回の続き」の明示
- サービスの変化・新機能の告知

### 定着イネブラ

**定義**: 継続的な使用を安定化させるフェーズ。

**設計課題**:
- 内発的モチベーションを育てる（外部刺激への依存を減らす）
- 使うことが「当たり前」になるルーティン化
- 代替サービスへの乗り換えコストを高める

**設計手法**:
- ユーザーのデータ・実績の蓄積と可視化
- カスタマイズ・パーソナライズ機能
- コミュニティ・社会的繋がりの強化

### ロイヤルイネブラ

**定義**: 熱狂的な支持者・アドボケートへと転換するフェーズ。

**設計課題**:
- 「このサービスの良さを伝えたい」という自発的推奨を引き出す
- ロイヤルユーザーが活躍できる場を設計する
- ブランドへの感情的結びつきを深める

**設計手法**:
- コミュニティ運営・UGC促進
- ロイヤルユーザー限定機能・特典
- 共同創造（サービス改善への参加）

## 感性的イネブラ vs 実存的イネブラ

### 感性的イネブラ

UIデザイン・世界観・美しさ・ブランドが生み出す体験的な引力。

**特性**:
- 「このサービスの雰囲気が好き」「使っていると気分がいい」
- 言語化しにくい感情的な価値
- 模倣されにくい（世界観は固有）

**例**: Airbnbの「Belong Anywhere」世界観、Notionのミニマルで知的な雰囲気

### 実存的イネブラ

ポイント・特典・機能的利便性が生み出す合理的な引力。

**特性**:
- 「このサービスを使うとXXポイントがたまる」
- 言語化・比較可能
- 模倣されやすい（競合が同じ手法を使える）

**例**: ポイント還元、特典付き会員プログラム、割引キャンペーン

### 使い分けの原則

| 状況 | 推奨 |
|------|------|
| 差別化が必要な場面 | 感性的イネブラを優先（模倣されにくい） |
| 短期的コンバージョンが必要 | 実存的イネブラを活用 |
| 長期的ロイヤリティ構築 | 感性的イネブラが効果的 |
| リソースが限られている | 感性的イネブラ（コスト非依存） |

**注意**: 実存的イネブラのみに依存すると、より高い特典を提供する競合に常に負けるリスクがある。

## ExperiencabilityのIf X then Y ルール

1. **If** UIデザインだけに集中している **then** 可制御環境の外（不可制御環境）でのExperiencabilityも設計する
2. **If** ユーザー獲得コストが高い **then** 世界観設計による期待獲得を検討する（資金力に依存しない手法）
3. **If** 新規ユーザーの離脱率が高い **then** 送流イネブラ（オンボーディング・aha moment）を見直す
4. **If** Push通知でリエンゲージメントしている **then** 頻度とストレスのバランスを管理し、内発的動機を優先する
5. **If** ポイント・特典だけでリテンション設計している **then** 感性的イネブラ（世界観・コミュニティ）で補完する
6. **If** ロイヤルユーザーを増やしたい **then** 彼らが「布教できる場・機会」を設計する
7. **If** ユーザーの定着を設計する **then** 実存的イネブラから感性的イネブラへの段階的移行を目指す
8. **If** 差別化戦略を考える **then** 感性的イネブラ（世界観）で模倣困難な価値を構築する
