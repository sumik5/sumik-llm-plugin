## 1. 制作プロセスの一般モデル

実務デザイナー（164名）の事例から抽出した共通パターン。線形ではなく、各段階間を往復する。

```
① 着想（Concept）
    ↓   ↑
② エスキース（Sketch）
    ↓   ↑
③ 検証（Verify）
    ↓   ↑
④ 提案（Present）
    ↓   ↑
⑤ 完成（Finalize）
```

### 各段階の実践内容

| 段階 | 主な活動 | 実務での典型例 |
|------|---------|--------------|
| ① 着想 | ブリーフ解釈・方向性設定・制約確認 | 「インテリアになじむ」「掛け算のデザイン」など言語化 |
| ② エスキース | スケッチ・ラフ・メモ感覚の試作 | サムネール、展開図の手書き、カラーバリエーション検証 |
| ③ 検証 | 複数方向を並走・比較・絞り込み | 「全く別の切り口」も並行して検討する |
| ④ 提案 | クライアントへの提示・フィードバック取得 | 縮小サンプル・実寸サンプルどちらも準備するケースも |
| ⑤ 完成 | 微調整・最終確認・ブラッシュアップ | 「希望通りに仕上がるまで何度も実サンプルを行う」 |

### 共通パターン（複数デザイナーの実践から）

- **並走検討**: ③検証段階では必ず複数方向（A案・B案）を同時に進める
- **エスキース優先**: 完成品を目指さず、まず手を動かしてイメージを固める
- **物理検証**: パッケージや印刷物は実物サンプルで確認する（デジタルと質感が異なる）
- **柔軟な後戻り**: どの段階でも「①着想」に戻ることを恐れない

## 2. 発想法の原則

### 2.1 アイデアの3つのスタート（描く・書く・つくる）

3つのアプローチを**並行して**走らせることで、アイデアの可能性が広がる。

| アプローチ | 手法 | 特性 |
|-----------|------|------|
| **描く**（イメージ発想） | スケッチ・サムネール | 細部にこだわらず骨格だけを考える。小さく速く |
| **書く**（コトバ発想） | キーワード・マインドマップ | 連想を放射状に広げる。言語化することで思考が整理される |
| **つくる**（カタチ発想） | プロトタイピング・試作 | 有効性がすぐわかる。予想外の発見や気づきが生まれる |

> ひとつのアプローチに固執せず、行き詰まったら別のアプローチに切り替える。

### 2.2 発酵の原則（Fermentation Principle）

アイデアは意識して生み出すだけでなく、「脳に任せて熟成させる」プロセスが有効。

```
1. 仕込み  → 課題を深く理解し、情報をインプット・思案を巡らす
2. 発酵    → 意識から離す。別の作業をする、一晩寝る
3. 出荷    → 脳の情報がつながり、アイデアが自然に浮かぶ
```

**適用条件**: 十分な「仕込み」が前提。情報インプットなしに放置しても発酵は起きない。

### 2.3 仮説思考（「もし？」のチカラ）

デザインは「もし」「もしかして」「こうしたらどうか」を一つひとつ試すプロセス。

```
A? → 検証 → 行き止まり
B? → 検証 → 行き止まり
C? → 検証 → 検証 → 検証 → OK
D? → 検証 → 行き止まり
E? → 検証 → 行き止まり
```

- **寄り道・後戻り・方針転換を許容する**。決まった答えがある問題ではない
- **アブダクション思考**: 仮説を立てて試す。証明ではなく探索

### 2.4 制約による発想解放

「制限がある」ことで逆に発想が自由になるパラドックス。

> 「一本の線で描く、ただし複雑に」という条件を与えると、答えが無数に生まれる。
> 無制約の「自由」は実は狭い空間に閉じ込められている。

**実践パターン**:
- 素材・色・形などの表現ルールを意図的に設定する
- 制約は「排除するもの」ではなく「思考を導くもの」として使う

### 2.5 パスの許容（無理に出さない判断）

- 「発酵」を待つことも選択肢。焦って出したアイデアより、熟成したアイデアの方が後から論理的に強い
- イテレーション（「あーでもない、こーでもない、これかーー？→始めに戻る」）を設計に組み込む

## 3. プロダクトデザインの発想→試作→検証サイクル

デジタルUIに限らず、プロダクト・パッケージデザインに共通する原則。

### 3.1 アイデアの育て方（絞るのではなく育てる）

- **初期段階では捨てない**: どんなアイデアもとりあえず記録する（ブレストのルールと同じ）
- **「大きく育つタネ」は最初あり得ないように見える**: 批判や却下をせず、広げる段階と絞る段階を分ける
- **掛け算の思考**: 2人のデザイナーが関わるなら「足し算」ではなく「掛け算」になるよう設計する

### 3.2 試作品思考（検証プロトタイプ）

- **「完成品」を作ろうとしない**: まず「確認するための試作」を作る
- **物理サンプルの重要性**: 画面上と実物は異なる。特に素材・印刷・立体物は実物確認が必須
- **反復を設計に組み込む**: 「希望通りに仕上がるまで何度も」が前提の計画を立てる

### 3.3 フィードバック駆動の反復プロセス

- クライアントや第三者からフィードバックを得るタイミングを意図的に設計する
- フィードバックは「完成度が高いもの」ほど細部の意見、「粗いもの」ほど方向性の意見になる
- → 欲しいフィードバックの種類に合わせて試作品の完成度を調整する

## 4. 身体感覚とデザイン思考の接続

グラフィックデザインは抽象概念ではなく、人間の身体感覚から始まる。

### 4.1 身体の天地左右感覚とレイアウトの関係

| 身体感覚 | デザインへの対応 |
|---------|----------------|
| 天（上）・地（下）感覚 | レイアウトの重心設定。上下中心より少し上が自然なバランス |
| 水平・垂直への安心感 | グリッドシステムの根拠。安定・信頼の印象 |
| 斜めへの不安・緊張感 | 動き・注意喚起・躍動感の表現に使う |

### 4.2 「見る→描く→知る」の循環

- 見るだけでは理解できない。**描くことで初めてわかる**ことがある
- 手を動かす（スケッチ・試作）ことが思考を促進する（構築主義的学習）
- デザインを「描くこと」としてではなく「**組み立てること**」として捉える

### 4.3 抽象と具象の往復運動

```
抽象（コンセプト・キーワード）
     ↕  往復
具象（スケッチ・試作・サンプル）
```

- どちらか一方に偏ると行き詰まる
- 「アーティストのように大胆に発想し、科学者のように論理的に分析し、シェフのように丁寧に定着する」

## 5. デザイナーのマインドセット

### 5.1 疑う姿勢（既存の当たり前を問い直す）

- 既成概念をとりはらう。「デザインを知らない、を知ることから学びが始まる」
- 日常で目にするデザインは「ほんの一部」。多様な側面・広さ・深さを発見する
- 「引き算のデザイン」: 足し算ではなく、本当に必要なものだけを選ぶ

### 5.2 日常観察力（半径500mからのインスピレーション）

プロが日常的に行っていること（164名の共通パターン）:

| カテゴリ | 具体的実践 |
|---------|----------|
| 日常生活 | 料理・散歩・愛好品への観察眼（色・形・質感） |
| 収集活動 | パッケージ・レコード・書籍・フライヤー収集 |
| 体験活動 | 美術館・ギャラリー・展示会（特に地方・海外） |
| 自己表現 | 個人展示・自主制作・異分野コラボレーション |
| 身体活動 | 運動（散歩・ピラティス等）で心身のバランスを保つ |

> 「日常のあらゆるものにアンテナを張る」ことが創造性の源泉。

### 5.3 不安との共存（プロフェッショナルの不確実性管理）

- 「デザインの方向性が決まらない」は異常ではない。**悩むことはデザイン思考の正常な状態**
- 苦しい時が成長の瞬間（「筋トレと同じ」）
- 年齢を重ねるほど段差は低くなるが、常に「わくわくする遊び心」を持ち続けることが成長の鍵
- **ウソをつかない**: まごころ・やさしさ・本当のことを伝える。恋愛と同じ
