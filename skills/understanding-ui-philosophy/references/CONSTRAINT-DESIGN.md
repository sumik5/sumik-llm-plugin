# 制約のデザイン 詳細リファレンス

## 制約の逆説

**「制約が自由な行動を促す」** ─ 菅俊一の核心思想。

直感に反するように見えるが、選択肢の多さ・自由度の高さは必ずしも良い体験を生まない。判断の余地を与えられると、人は「正しい選択をしなければ」というプレッシャーを感じる。

**折りたたみ傘の原則**: 通常の傘と折りたたみ傘を同じ傘立てに入れると、どちらを使えばいいか一瞬迷う。でも折りたたみ傘専用の小さな傘立てに分けると迷いが消える。制約が判断コストを消失させる。

> 「制約を作ることで、むしろ人はより能動的になれる」 ─ 菅俊一

## 前提条件のデザイン（菅俊一）

行動の「前提条件」を設計することで、ユーザーが無意識に正しい行動を取るよう導く。

### 2つのアプローチ

| アプローチ | 特性 | 例 |
|-----------|------|-----|
| **阻害要因の除去** | 「できない」を「できる」にする | パスワード入力を自動化する |
| **促進要因の付加** | 「する方が自然」にする | デフォルトをONにする |

### アフォーダンスとシグニファイア

**アフォーダンス（J.J. Gibson）**: 環境がある行動の「可能性」を持っているという性質。
**シグニファイア（D.A. Norman）**: その可能性をユーザーに「知覚させる」デザイン要素。

```
アフォーダンス: ドアノブは「回す・押す・引く」という行動の可能性を持つ
シグニファイア: 縦型バーが付いたドア = 「押す」というシグナルを発する
```

**UIでの応用**:
- ボタンらしい形（角丸・背景色・影）= タップ可能なシグニファイア
- 入力フィールドの下線 = テキスト入力を促すシグニファイア
- スワイプヒント（画面端にはみ出る要素）= 横スクロールのシグニファイア

## 知覚のデザイン

人が無意識に検知してしまう性質を意図的にデザインの中に組み込む。

### 静的情報の「時間」と「動き」

**脳内時間生成**: 矢印・行列・文章には、静止していても「動き」と「時間」が内包されている。

- 矢印の溜まり部分で目が留まり、先端方向に視線が向く
- 行列（並んだ人々）の「目的地への移動」を脳が補完する
- ポスターに描かれた人物の視線は「画面外の何か」を示し、鑑賞者の注意を引く

**実践原則**:
- 視線誘導は画面の内外を超えて機能する
- 方向を持つグラフィック要素（矢印・人物の向き・動物の姿勢）は強力な注意誘導ツール
- 文章の「読み方向（左→右、上→下）」は無意識の視線ガイドになる

### 視覚的導線のパターン

| 要素 | 誘導の仕組み | 強度 |
|------|------------|------|
| 矢印 | 先端方向への視線 | ★★★ |
| 人物・動物の視線 | 視線の先への注意 | ★★★ |
| 輝点・光源 | 光の方向への引き寄せ | ★★☆ |
| テキストの読み方向 | F字・Z字パターン | ★★☆ |
| 空白・余白 | 白い部分への注意（図地反転） | ★☆☆ |
| 色のグラデーション | 高彩度→低彩度への流れ | ★☆☆ |

## 精神的相互作用と引力設計（萩原俊矢）

UIとコンテンツを分離して設計する「器とコンテンツ」モデルへの根本的な問い直し。

### コンテンツとUIの不可分性

従来の考え方:
```
コンテンツ（テキスト・画像・動画）
    ↑
UIがコンテンツを包む「器」
```

萩原が提唱する考え方:
```
コンテンツ ⟷ UI（相互作用・不可分）
コンテンツがUIを規定し、UIがコンテンツの体験を変える
```

**精神的相互作用**: ユーザー・コンテンツ・UIの三者が互いに影響を与え合い、体験が生まれる。UIはコンテンツの「受け皿」ではなく、コンテンツとの「対話」を設計するもの。

### 引力設計

コンテンツへの「引力（gravity）」をUIで設計する。ユーザーが次のコンテンツへ自然に向かってしまう、不可抗力としての引き付ける力。

**引力の生み方**:
1. **不完全な露出**: コンテンツを少し見せる（次のカードがはみ出る）→「続きが見たい」
2. **モーションの連続性**: 前のインタラクションの勢いが次のコンテンツへ向かう
3. **サイズと重力感**: 次のコンテンツがわずかに大きく見える角度・位置
4. **音と視覚の先行**: 次のコンテンツの音や動きが先行して感知される

## 「ご自由にどうぞ」問題

何もしない状態（デフォルト状態）に制約がない場合、ユーザーは「正しい行動」を探して迷う。

### パンとの比較

皿に無造作に置かれたパン vs 一人分ずつ個包装されたパン。前者では「どれを取っていいか」「取りすぎてはいけない」という心理的負荷が生まれる。後者では迷いが消える。

**UIへの応用**:

| 「ご自由にどうぞ」型 | 制約を設けた型 | 心理的効果 |
|-----------------|-------------|---------|
| 全ての設定をデフォルト無選択 | 最初にレコメンド選択肢を提示 | 判断コスト削減 |
| 複数の同等の選択肢 | 「おすすめ」のハイライト | 選択の手がかりを提供 |
| 全ての機能を均等に表示 | 使用頻度に応じた優先表示 | 認知負荷の軽減 |
| 完全に空のフォーム | プレースホルダー・例示入力 | 入力の促し |

## 行動誘導の設計パターン

### パターン1: デフォルト活用

人は変更コストを嫌う（現状維持バイアス）。デフォルト値がそのまま採用されやすい。

```
悪いデフォルト: メールマガジン受信「オフ」(ユーザーに不利)
良いデフォルト: プライバシー設定「オン」(ユーザーの利益に合致)
```

倫理的な原則: デフォルト設定はユーザーの利益を最大化するよう設計する。

### パターン2: ステップ分割

複雑なタスクを小さなステップに分割し、各ステップで1つの判断だけを求める。

```
悪い例: 一画面に全てのフォーム項目
良い例: ステップごとに分割（Step 1/3 → Step 2/3 → Step 3/3）
```

### パターン3: コミットメント段階設計

小さなコミットメント→大きなコミットメントへと段階的に誘導。一貫性の原理を利用。

```
段階1: メールアドレス入力（低ハードル）
段階2: 名前入力（中ハードル）
段階3: 支払い情報（高ハードル）
```

### パターン4: 制限による促進

「残り3点」「今だけ」等の制限を示すことで、行動を促す。ただしウェルビーイング観点から過剰な利用は避ける。

## 制約デザインのIf X then Y ルール

1. **If** ユーザーに複雑な判断を強いている **then** 選択肢数を減らすか、「おすすめ」を明示する
2. **If** フォームの入力ハードルが高い **then** 最初に最低限の項目だけを求め、段階的に情報収集する
3. **If** デフォルト設定がある **then** ユーザーにとって最も利益になる選択肢をデフォルトにする
4. **If** 視線誘導が必要 **then** 矢印・人物視線・方向を持つグラフィック要素を活用する
5. **If** 「ご自由に」型UIになっている **then** 制約（区画・個包装・順番）を設けて判断コストを消す
6. **If** コンテンツとUIを独立して設計している **then** コンテンツの性質がUIデザインを変えうるか再検討する
7. **If** 静的な画像・テキストで視線誘導する **then** 脳が動きと時間を補完する性質を利用する
8. **If** 複数の同等な選択肢がある **then** 何らかの基準（人気・推奨・デフォルト）で優先度を付ける
