# Fluid Interfaces 詳細リファレンス

## 概念の起源

AppleのHuman Interface Guidelinesが提唱した設計思想。インターフェイスが「流れる（fluid）」とは、ユーザーの意識の連続性を断絶しないことを指す。William Jamesの心理学概念「意識の流れ（stream of consciousness）」から着想を得ている。

> 「意識は一瞬たりとも止まらず、川のように連続して流れ続ける」 ─ William James

UIがこの「流れ」を遮断する（アニメーション待ち・遷移中のインプット無効・コンテキストの完全切替等）ことで、ユーザーは「機械を使っている」という疎外感を体験する。

## 道具の透明性（Tool Transparency）

渡邉恵太が提唱する概念。優れた道具の条件は「使っていることを意識させないこと」。

- ハンマーを振る時、上手い大工はハンマーの重さや素材を意識しない。意識するのは釘だけ
- UIも同様に、ユーザーがUIそのものでなく**タスク**に集中できる状態が理想
- UIを意識させてしまうデザインは「道具の透明性」を損なっている

### 透明性を損なう典型的なパターン

| パターン | 問題 | 解決策 |
|---------|------|--------|
| アニメーション完了まで操作ブロック | 「機械の都合」を押し付ける | 中断可能なアニメーション設計 |
| 遷移完了を明示的に示すローディング | 処理の「存在」を意識させる | シームレスな遷移 |
| モーダルで全画面を覆う | 現在のコンテキストを断絶する | ハーフモーダルで背景を残す |
| 操作完了の過剰な確認ダイアログ | 判断を割り込ませる | 取り消し可能な即時実行 |

## 自己帰属感（Sense of Agency）

インプットに対してダイレクトなフィードバックを感じること。「私が操作している」という感覚の基盤。

### 慣性スクロールとバウンスの哲学的意味

iPhoneの「慣性スクロール（momentum scrolling）」と「バウンス」は単なるビジュアルエフェクトではない。指を離した後も動き続け、端でバウンスするUIは「物理世界の中にある物体に直接触れている」という錯覚を生む。これが自己帰属感の核心。

**設計原則**:
- タッチの圧力・速度に比例した応答
- 物理法則（慣性・弾性・摩擦）を模倣した動き
- フィードバックのラグゼロを目指す（遅延は自己帰属感を即座に損なう）

### Haptic Feedback

触覚フィードバックは自己帰属感の強力な強化剤。視覚情報と触覚情報が一致することで、「本物感」が増す。

## Fluid Interfacesの7原則（詳細版）

### 原則1: 応答性（Responsivity）

UIは常にインプットを受け付ける。「処理中だからインプットを無視する」は禁止。

**実装指針**:
- アニメーション実行中のタッチイベントを処理する
- ネットワークリクエスト中もUIを操作可能に保つ
- バックグラウンド処理をUIの応答性と分離する

### 原則2: 中断可能性（Interruptibility）

進行中のアニメーションや遷移はいつでも中断・方向転換できる。

**実装指針**:
- ドラッグ中に方向を変えたらアニメーションも方向を変える
- スクロール中にタップしたら即座に停止する
- 春のような弾力で位置に戻る（スナップバック）

### 原則3: 連続性（Continuity）

画面遷移でユーザーのコンテキストを断絶しない。

**ハーフモーダル（Half-Modal）設計**:
- 全画面遷移ではなく画面の半分を覆うシート型UI
- 背後のコンテキストが見えることで「どこにいるか」が常に分かる
- マルチプラットフォーム変換: スマートフォン→タブレットではフローティングモーダルに

**Hero Animation（共有要素トランジション）**:
- リスト項目→詳細画面の遷移で要素が「飛んでいく」アニメーション
- ユーザーは「同じものが大きくなった」と認知し、空間的連続性を感じる

### 原則4: 到達可能性（Reachability）

主要操作は片手の親指で届く位置に配置する。

**縦長ディスプレイの問題**:
- 画面上部に主要ナビゲーションを置くと片手操作が困難
- ハーフモーダル・ボトムシート・Floating Action Button（FAB）が下部に集中するのはこの理由

### 原則5: 統合性（Integration）

ビジュアルデザインとモーションデザインは同時に設計する。

**アンチパターン**: 静的デザインを完成させてから「後でアニメーションを付ける」アプローチ。後付けアニメーションは常に浮いた印象になる。Fluid体験はビジュアルとモーションが不可分。

### 原則6: 低遅延（Low Latency）

意識の流れに追随するには、デザインだけでなく低遅延実装が必須。

**知覚できる遅延の閾値**:
- < 10ms: 即座に反応していると感じる
- 10-100ms: 遅延を感じ始める
- > 100ms: 「機械が処理している」と感じる
- > 300ms: ストレスを感じる

### 原則7: マルチプラットフォーム一貫性（Cross-platform Consistency）

同一サービスを複数デバイスで提供する場合、UIの形態はデバイスに最適化しつつブランドの一貫性を保つ。

| デバイス | ハーフモーダルの変換形 |
|---------|-------------------|
| スマートフォン | ハーフモーダル（画面下からスライドアップ） |
| タブレット | フローティングモーダル（中央ポップオーバー） |
| デスクトップ | インスペクターパネル（サイドバー） |

## Material Engagement Theory（MET）

水野勝仁が紹介する哲学的フレームワーク。道具と人間の関係についての理論。

**核心**: 道具はヒトの認知・思考・身体感覚を変容させる。タッチスクリーンの普及は「指で直接情報に触れる」という新しい認知様式を生んだ。人間とコンピュータは双方向に影響を与え合い「共進化」する。

**デザインへの示唆**:
- 新しいインタラクションパターンを導入する時、短期の使いやすさだけでなく「習熟後にどんな認知変容が起きるか」まで設計する
- AIアシスタントやAR/VRインターフェイスはこの「共進化」の最前線

## 実践チェックリスト

### Fluid Interfacesの観点でUIを評価する

- [ ] アニメーション中もタッチイベントを受け付けるか？
- [ ] ハーフモーダルでコンテキストを保持しているか？
- [ ] 主要操作が画面下部にあるか？
- [ ] ビジュアルとモーションを同時設計したか？
- [ ] レイテンシを計測してフィードバック遅延を最小化したか？
- [ ] タッチへの物理的応答（慣性・バウンス・snap）が自己帰属感を生むか？
