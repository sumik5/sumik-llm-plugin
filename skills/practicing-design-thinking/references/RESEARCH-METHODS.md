# リサーチ手法リファレンス

UXデザインにおける調査・モデル化手法の実践ガイド。
利用文脈把握（①フェーズ）とユーザー体験モデル化（②フェーズ）の10手法を収録。

## 手法選択ガイド

| 手法 | フェーズ | 目的 | 工数 |
|------|---------|------|------|
| エスノグラフィ | ① | 利用文脈の全体把握 | 大 |
| 観察法 | ① | 行動の記録・分析 | 中 |
| コンテクシュアル・インクワイアリー | ① | 作業プロセスの深掘り | 中 |
| 個人面接法（インタビュー） | ① | 経験・価値観の聞き出し | 中 |
| フォトエッセイ | ① | 内的価値観の可視化 | 小〜中 |
| EFM法 | ① | エピソード単位のUX評価 | 中 |
| ペルソナ法 | ② | ユーザーモデルの作成 | 中 |
| ジャーニーマップ | ② | 体験の時系列可視化 | 中 |
| KA法 | ② | 体験価値の構造化 | 大 |
| 上位・下位関係分析 | ② | 本質的ニーズの導出 | 中 |

## 調査手法

### エスノグラフィ

**目的**: ユーザーの行為が行われている現場をたずね、利用文脈と体験価値を把握する。仮説発見・機会探索につながる体系的な質的調査。

**適用段階**: ①利用文脈把握

**実施ポイント**:
- **3段階で進める**: 調査計画 → フィールドワーク実施 → 事実の分析
- **調査計画**: 調査フォーカス（対象ユーザーの対象行為の範囲）を設定し、対象者を体系的に選出。フォーカスは対象行為の周辺を含めやや広めに設定する
- **フィールドワーク**: 観察とインタビューを組み合わせる（トライアングュレーション）。1〜2名調査後にチームで振り返り、焦点を随時調整する
- **分析**: 得られた事実から行為の全体像・本質的ニーズ・暗黙の価値観を体系的に整理。AEIOUフレームワーク（Activity/Environment/Interaction/Object/User）が整理に役立つ
- **観察の心得**: 「問い」を持ってフィールドに臨む。「なぜこの人はこうしているのか？」という問いが観察の焦点を作る。仮説検証の目で見ると確認バイアスに陥るので注意

**注目すべきポイント**:
- ユーザーがやっていること／**やっていないこと**／やりたくてもできていないことの3つに着目
- 「我慢・あきらめ・補完」は潜在ニーズのサイン
- 業界の常識にとらわれた見落とし（業界の非常識ニーズ）にも着目する

**よくある失敗と対策**:
- **仮説を持ちすぎる** → フィールドでは一旦仮説から離れ「観る」姿勢を優先
- **分析で止まる** → 観察データは必ず体系的にまとめ、関係者間で共有可能な形にする
- **人数が少ない** → 少人数でも振り返りを複数回挟むことで深度は確保できる

### 観察法

**目的**: 調査者が直接見聞きして、ユーザーの行動・ふるまい・行為環境を系統立てて記録する。

**適用段階**: ①利用文脈把握（エスノグラフィ内でも単体でも使用）

**実施ポイント**:
- **観察の状況で選ぶ**:
  - 自然観察：普段の行動をそのまま観察（干渉なし）
  - 実験的観察：調査者が状況を用意し、その中での行動を観察
- **観察方法で選ぶ**:
  - 交流観察（参加観察）：観察中にインタビューも行う。コンテクシュアル・インクワイアリーがこれにあたる
  - 非交流観察（フライ・オン・ザ・ウォール）：透明人間のように存在を消して観察
  - シャドーイング：行動する対象者の後をついて観察
  - 非参加観察法：ビデオ等で対象者を意識させずに記録
- **記録整理**: 観察後すぐに「フィールド/痕跡」「ユーザーの行動・発言」「考察」の3列でまとめる

**4つの心構え**:
1. 「問い」を立てて焦点を明確にし、観察を重ねるごとに焦点を具体化する
2. 「仮説・予見・思い込み」を持ってフィールドに臨まない
3. ユーザーを見る。ユーザーの行為を中心に見る
4. フィールドでは記録に徹する。解釈は後で行う

**よくある失敗と対策**:
- **「どこをみたらよいかわからない」** → 事前に「問い」を設定しておく
- **自分の仮説を検証する目で見てしまう** → 確認バイアスに自覚的になる。複数調査者で異なる視点を持ち寄る

### コンテクシュアル・インクワイアリー（文脈的調査）

**目的**: ユーザーの現場に出向き、行動を観察しながら文脈に応じてインタビューし、行動の背後にある考えや仕組みを明らかにする。

**適用段階**: ①利用文脈把握（作業プロセスが明確な対象に特に有効）

**4つの原則**:
1. **コンテクスト（現場での観察）**: ユーザーの行為が行われている現場で、普段通りの行動を観察する
2. **パートナーシップ（師匠と弟子モデル）**: 調査者はユーザー（師匠）に弟子入りし、作業を観察しながら根掘り葉掘り教えを乞う
3. **解釈**: 得られたデータはすべてユーザーにとっての意味を解釈し仮説を立てる
4. **焦点（調査対象者の立場）**: 調査者はユーザーの立場・視点から物事を見る

**実施ポイント**:
1. 調査フォーカスの設定（対象行為と周辺の活動も含める）
2. 対象者へのあいさつと仕事の概要確認（大まかなやり方をインタビュー）
3. 実際の仕事の観察・記録（写真等）
4. 弟子として作業に質問する（質問リストは作らず、対象者の行動や発言から都度質問を生成する）
   - 「たまに」→「具体的にどんなときか？」のように発言の言葉から掘り下げる
   - 要注意語：「検討する」「対応する」「管理する」「計画する」などは具体化が必要
5. 記録した作業内容の確認

**よくある失敗と対策**:
- **質問リストに縛られる** → 事前の質問リストは持ち込まない。対象者の行動・発言から質問する
- **現場に入れない場合** → 「人工物ウォークスルー」（実物や写真を持参してもらいインタビュー）や「ユーザビリティラウンドテーブル」（会議室に作業環境を再現）で代替

### 個人面接法（インタビュー）

**目的**: 対象者の経験・考え・意見・意識を直接聞き取る。UXデザインで最も汎用的なリサーチスキル。

**適用段階**: ①利用文脈把握（調査段階全般、評価段階でも活用）

**実施ポイント**:
- **半構造化インタビュー**が基本：大まかなトピックは用意するが、対話の流れに応じて柔軟に進める
- **過去のエピソードとして語ってもらう**: 直接「なぜ？」と聞くより「そのときどうしましたか？」の方が真の行動理由が出やすい
- **ユーザーの言葉から深掘り**: 「たまに」「よく」などの曖昧な表現は必ず具体化する

**補助手法**:

| 手法 | 用途 | タイミング |
|------|------|-----------|
| **脳内マップ** | 個人の価値観の重みと優先度を可視化。円を脳内に見立て、関心事の項目と大きさを書いてもらう | インタビューの最後 |
| **利用年表共作法** | 製品との過去の関わりを時系列で振り返る。「主な出来事」→「心理的事情」→「評価への影響」の順に記録 | 長期利用体験の調査時 |
| **UXカーブ** | 製品の使い始めから現在までの満足度変化を曲線で描いてもらう | 利用年表の最後 |

**よくある失敗と対策**:
- **インタビューが長いテーマ＝重要と思い込む** → 脳内マップで重要度を再確認する
- **記憶が曖昧な過去の体験** → 利用年表共作法で出来事を時系列に整理してから掘り下げる

### フォトエッセイ

**目的**: 調査対象者に写真とエッセイを作成してもらい、言語化しにくい価値観・内的動機・生活文脈を把握する。

**適用段階**: ①利用文脈把握（KA法の入力データとしても有効）

**実施ポイント**:
- **テーマ設定**: 広すぎず狭すぎない「やや広めのテーマ」にする
- **作成形式**: テーマについて写真2枚（自分目線のクローズアップ＋全体像）＋短い説明文＋タイトル
- **タイプA/B セットを使う**（推奨）:
  - タイプA：現在実際に行っていること
  - タイプB：現在やりたくてもできていないこと
  - 対比することでユーザーの本音（未充足ニーズ）が浮かび上がる
- インタビューと組み合わせると効果が高い（フォトエッセイを話題の起点にする）

**よくある失敗と対策**:
- **写真の編集スキル不足** → 事前にスマートフォンの操作方法を確認・サポートする
- **タイプAのみ収集** → タイプBを追加することで潜在ニーズが得られる

### エクスペリエンスフィードバック法（EFM法）

**目的**: タスク実施中の体験をビデオ録画し、ビデオを見ながら感情曲線を描いてもらうことで、エピソード単位のUX評価を行う。

**適用段階**: ①利用文脈把握（ユーザビリティテスト・実験的観察との組み合わせが基本）

**実施ポイント**:
1. **タスクの提示**（ビデオはタスク提示前から撮影し、期待度も記録）
2. **体験のビデオ記録**
3. **ビデオフィードバックによる感情評価**: タブレット等でビデオを再生しながら感情曲線を描く。変曲点に付箋でコメントを記録する
4. **総合評価の記入**: 総合評価と再利用意向を10段階で評価

**UXカーブとの使い分け**:
| | UXカーブ | EFM法 |
|---|---------|-------|
| 対象 | 長期利用体験の回顧 | 短時間エピソードの詳細評価 |
| 記録 | 対象者の記憶に依存 | ビデオで客観記録 |
| 精度 | 記憶の曖昧さあり | ビデオ確認で高精度 |

**よくある失敗と対策**:
- **変曲点の理由が不明** → 事前に「変化したときにコメントを書く」よう依頼する

## モデル化手法

### ペルソナ法

**目的**: ユーザー調査データから典型的なユーザーの行動パターンを導出し、デザインの基準となる仮想個人像を作成する。「みんなのため」ではなく「一人のため」にデザインする発想の転換。

**適用段階**: ②モデル化

**5ステップの作成手順**:

1. **行動変数の検討**: 調査結果から行動の違いに影響する変数を複数抽出する
   - 変数のタイプ：活動（何をしているか・頻度）／態度（技術への考え方）／適性（教育訓練・学習能力）／モチベーション（なぜ関わっているか）／技能（製品に関わる能力）

2. **調査対象者を行動変数に対応づける**: 各対象者のインタビューデータ・観察メモをもとに、各変数軸上での位置を示す。厳密さより調査者の印象を大切にする

3. **顕著な行動パターンの発見**: 複数の変数を通じて同じ対象者が集中している箇所を探す。これがペルソナの基礎（通常2〜3パターン）

4. **特徴とゴールの総合**: パターンごとにデータから詳細を集め、利用環境・現在の行動・不満・人間関係などを整理。ユーザーのモチベーションをゴールとして3〜5つ記述

5. **重複・完成度チェック**: 各ペルソナが少なくとも1つのふるまいで他と異なることを確認

**ペルソナのタイプ**:
| タイプ | 説明 |
|--------|------|
| 一般的ペルソナ | 典型的な利用シーンを持つユーザー像 |
| エクストリームペルソナ | 製品を熟知した先行ユーザー。潜在的ニーズが浮かびやすい |
| 主役（プライマリ） | デザインの主対象。このペルソナのニーズを最優先 |
| 脇役（セカンダリ） | 主役とは異なるニーズを持つ。部分的に考慮が必要 |

**ペルソナ作成方針の確認**（AskUserQuestion推奨）:
> データドリブン（行動変数ベース）/ 仮説ベースのプロトペルソナ / KA法からの体験価値ベース のどれか？

**よくある失敗と対策**:
- **ユーザー調査なしで作成** → 根拠のない「妄想の人」はバイアスを強化するだけ。プロトペルソナとして使う場合も「仮」と明記し早期に実調査で更新する
- **ペルソナを1体だけ作る** → 複数パターンを導出して優先順位づけすることがペルソナ法の本質

### ジャーニーマップ

**目的**: ユーザーと製品・サービスとの関わりを時間軸で視覚化し、複数のタッチポイントをまたいだ体験の全体像・問題点・改善機会を明らかにする。

**適用段階**: ②モデル化（ペルソナ作成後に作成）

**実施ポイント**:
- **AS-ISモデル**（現状モデル）: ペルソナの典型的な行為シナリオに基づき現状の体験を時系列で表現。問題点・課題・改善点の発見が目的
- **TO-BEモデル**（理想モデル）: アイデア実現後の理想的なUXを表現。各段階の課題と検討項目を整理するために使う

**マップの構成要素**:
| 要素 | 説明 |
|------|------|
| ステージ（時間軸） | 体験の前・中・後を段階に分割 |
| ユーザーの行動 | 各ステージでの行動を記述 |
| ユーザーの思考 | 各段階での認識・考えを記述 |
| 感情曲線 | ポジティブ/ネガティブ感情を曲線で表現。理由をコメントで補足 |
| タッチポイント | 製品・サービスとの接点 |
| 関連ユーザーの行動 | 脇役・端役ペルソナの行動も同時に表現 |
| 問題点・検討事項 | 各段階の課題や検討項目 |

**作成のポイント**:
- ワークショップ形式で実施すると効果的（マトリクス構造なので複数人での議論に向く）
- 行動シナリオとの違い：ジャーニーマップはマトリクス構造のため構造的・網羅的に理解しやすい

**よくある失敗と対策**:
- **デザインに時間をかけすぎる** → 目的に応じた適切な表現形式で十分。手書きでも機能する
- **ユーザー調査なしで作成** → AS-ISモデルは必ずユーザー調査結果に基づく

### KA法（価値マップ法）

**目的**: インタビューや観察で得られた定性情報から、人々の行為の背後にある体験価値・本質的ニーズを導出し、価値の構造を視覚化する。

**適用段階**: ②モデル化（フォトエッセイ・エスノグラフィの分析に特に有効）

**実施ポイント**:

**KAカードの3要素**:
```
[出来事]  状況/動機/行動/結果の2つ以上を30字程度で記述
[ユーザーの心の声]  出来事を読みユーザーになりきって内的感情を端的に表現
[体験価値]  心の声が出る理由を解釈し「〜する価値」「〜できる価値」の形で記述
```

**5ステップの分析手順**:
1. **事前準備**: インタビュー・観察結果をテキスト情報として整理
2. **KAカード作成（出来事）**: 特徴的なユーザー行動をピックアップし「状況+行動」「行動+結果」などの形でカードに記述
3. **心の声の解釈と体験価値の導出**: 出来事から「ユーザーの心の声」を書き、その理由を「体験価値」として導出。未充足の価値（やりたくてもできていないこと）は別色でマーク
4. **価値マップの作成**: 価値に着目してKAカードをKJ法でグルーピング→中分類ラベルを付ける→中分類間の因果・経時関係で構造化
5. **既存製品・技術のマップ展開**: 価値マップ上に競合製品・代替手段をマッピングし、機会領域を発見する

**体験価値の3タイプ**:
| タイプ | 例 |
|--------|-----|
| モノが生む価値 | 道具・製品を使うことで得られる |
| コトが生む価値 | ユーザーが行う行為そのものから得られる |
| 認識が生む価値 | 行為中の理解・安心感・自信から得られる |

**KJ法との違い**: KA法は出来事1つひとつを丁寧に解釈してから価値を導出する。KJ法はデータをグルーピングしてからまとめて解釈する。KA法の方が体験価値の詳細な分析に適している。

**よくある失敗と対策**:
- **分類名を先に作ってしまう** → 必ずボトムアップに。「○○系」「○○軸×□□軸」で先にフレームを作るのは禁止
- **抽象度が上がりすぎる** → 価値の表現は「動詞的表現＋価値」の形に限定する

### 上位・下位関係分析（ラダリング）

**目的**: インタビューで得られたニーズを階層化し、ユーザーの本質的ニーズ（Be ニーズ）を導出する。

**適用段階**: ②モデル化（グループインタビュー・個人インタビューの分析に使用）

**ニーズの3階層**:
| 層 | 名称 | 意味 | 例 |
|----|------|------|-----|
| 最上位 | **Be ニーズ** | ユーザーの本質的ニーズ（〜になりたい） | 「仲間とスポーツを楽しみたい」 |
| 中位 | **Do ニーズ** | Be実現のための行動目標（〜したい） | 「ちゃんとしたタイミングで応援したい」 |
| 下位 | **Have ニーズ** | Do実現のための事象（〜が欲しい） | 「応援グッズが欲しい」（インタビューから直接得られる） |

**実施ポイント**:
1. **ニーズのカード化**: インタビューデータから特徴的な行動・理由・要望を事象カードにする
2. **集約と上下関係の発見**: Have ニーズから「Have したい理由はどんなことをDoしたいからか？」と問いながら上位化。模造紙に3〜4段階の枠を作り、カードを移動させながら線で結ぶ
3. **上位ニーズの導出**: DoニーズのカードをさらにBeニーズへ上位化する。「Doしたい理由はどんなBeでありたいからか？」と問う
4. **レベルの調整**: 最上位が抽象度高すぎる場合は4段階に拡張（アイデア発想に使えるレベルに調整）

**KJ法との違い**: KJ法はグルーピングによる整理・命名。上位・下位関係分析は「なぜそう思うか？」という理由の上位化。グルーピングに似た作業にならないよう「上位に行く＝理由を考える」ことを意識する。

**よくある失敗と対策**:
- **最上位ニーズが抽象的すぎる** → 4段階に拡張し、アイデア発想できる粒度まで下げる
- **KJ法のグルーピングと混同** → 「なぜ？」を繰り返す上位化の作業であることを意識する

## 手法の組み合わせパターン

```
エスノグラフィ ─→ KA法 ─→ ペルソナ法 ─→ ジャーニーマップ
（利用文脈把握）  （体験価値分析）  （ユーザーモデル）  （体験可視化）

フォトエッセイ ─→ フォトKA法 ─→ 価値マップ ─→ ペルソナ法

コンテクシュアル・インクワイアリー ─→ 上位・下位関係分析
（作業プロセス把握）                    （本質的ニーズ導出）

EFM法 ─→ ジャーニーマップのTO-BEモデル
（感情評価）  （課題の特定と理想体験の設計）
```
