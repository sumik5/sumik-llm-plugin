# UX理論基盤

## UXの定義

UX（User Experience）とは、製品やシステム、サービスを利用した際、およびその利用を予想した際に生じる**人々の知覚と反応**の総体である（ISO 9241-210）。

ISO 9241-210の注釈が示す重要な特徴：

| 特徴 | 内容 |
|------|------|
| **時間軸の広さ** | 使用前・使用中・使用後に起こる感情・信念・嗜好・知覚・行動をすべて含む |
| **主観性** | UXはあくまでユーザー個人の内的なものであり、作り手が直接手を加えることはできない |
| **多因子性** | ブランドイメージ・見た目・機能・インタラクション・事前経験・利用状況などが複合的に作用する |

### UXをとらえる共通の視点

代表的な定義（ISO 9241-210、UXPA、Nielsen-Norman Group、Hassenzahl & Tractinsky）に共通する3つのポイント：

| 視点 | 内容 |
|------|------|
| **主観的評価** | UXの本質は、製品・サービスを使った体験を通じてユーザーが「どう感じたか」という感情的・認知的評価にある |
| **消費者とユーザーの連続性** | 購入前の「消費者」の体験と利用中の「ユーザー」の体験は、同じ人間の連続した体験として一体的に捉える |
| **時間的・長期的視点** | UXは利用の瞬間だけでなく、製品との関わり全体（使用前後を含む長期間）を対象とする |

UXはまた、Hassenzahl & Tractinsky（2006）の定義が示す通り、以下の3つの要因の掛け合わせによって規定される：

```
UX = f(ユーザーの内的状態 × システム特性 × 利用文脈)
```

- **ユーザーの内的状態**：体質的傾向・期待・ニーズ・動機付け・気分など
- **システム特性**：複雑さ・目的・ユーザビリティ・機能性など
- **利用文脈**：組織的・社会的条件・活動の重要さ・利用の自発性など

UXとUXデザインは明確に区別すること：

| 概念 | 内容 |
|------|------|
| **UX** | ユーザーの個人的な体験そのもの。ユーザーの内部に生じる主観的なもの |
| **UXデザイン** | どんな体験をしてもらうかを計画し、その体験が量産・再生産される製品・サービスと仕組みを作ること |

## UXの4つの時間軸分類

体験の「どの期間に注目するか」によってUXの評価メカニズムは異なる。

| 分類 | タイミング | 説明 | 評価メカニズム |
|------|-----------|------|--------------|
| **予期的UX**（anticipated UX） | 利用前 | 製品を使う体験を想像する段階。カタログ・レビュー・口コミなどから利用イメージを形成する | 過去の類似経験と期待に基づく評価。他のUXに影響を与える |
| **瞬間的UX**（momentary UX） | 利用中（瞬間） | 製品を使っている最中の体験。操作感・反応速度・驚きなどを直感的に感じる | 感情的・直感的反応に基づく評価 |
| **エピソード的UX**（episodic UX） | 利用後（1エピソード） | ある目的で使用した後、その体験を振り返るときのUX。成功・失敗体験として語れる期間が対象 | ユーザーの行動と結果に対する心理的判断に基づく評価 |
| **累積的UX**（cumulative UX） | 利用期間全体 | 製品との出会いから現在までのすべての関わりを振り返るUX。ブランドイメージや愛着も含む | 意味的・理念的な価値判断に基づく評価 |

### 時間軸を意識した設計の実践

- **予期的UX → 累積的UX** の連鎖を意識する：累積的UXは次回の予期的UXに影響する
- **エピソード的UXとの乖離**：同一製品でも注目する期間によって評価が反転することがある（例：お化け屋敷は瞬間的には恐怖→エピソード的には楽しい思い出）
- UXを議論する際は「どの期間に注目しているか」を明示することが重要

## 体験価値（Experience Value）

### ニーズの3階層モデル

ユーザーが製品・サービスに求めるものには階層構造がある。

| 層 | 名称 | 説明 | 例（デジカメの場合） |
|----|------|------|---------------------|
| **上位** | 体験価値 | 体験を通して得られる行為の価値。手段を特定しない抽象的な「うれしさの源」 | 「すぐにみんなと思い出を共有できる価値」 |
| **中位** | 本質的ニーズ | 得たい結果・効果・感情。特定の手段を含まない | 「すぐにみんなとの思い出を共有したい」 |
| **下位** | ニーズ | 特定の手段への欲求。経験的知識に基づく具体的な要望 | 「デジカメの写真をすぐに印刷できるプリンターが欲しい」 |

**UXデザインでは体験価値に着目する理由**：ニーズは手段を固定化してしまうため、より良い代替手段の発見を阻む。体験価値は手段を問わないため、イノベーションの余地が生まれる。

**設計上の実践**：UXデザインにおいては「ユーザーに体験させたい価値＝体験価値」を最初に定義し、その体験価値を実現する手段として製品・サービスのコンセプトを後から決める。製品の機能や仕様から出発する従来のアプローチと逆のプロセスをたどることが重要。

### 体験価値の形成プロセス

```
利用意欲（期待）
    ↓
製品の性質を知覚・理解（期待と結果のギャップを判断）
    ↓
自分の生活における意味の判断
    ↓
次の利用意欲の形成（体験価値によって調整される）
    ↓
利用エピソードの積み重ね
    ↓
体験価値の形成・更新
```

## ユーザーの分類とアクセシビリティ

### ユーザーの分類（ISO/IEC 25010）

UXデザインにおける「ユーザー」とは、製品・サービスと直接・間接的に関わるすべての人を指す。

| 分類 | 説明 | 例 |
|------|------|-----|
| **一次ユーザー（直接）** | 主に目標達成のためシステムと相互作用する人 | アプリを操作するエンドユーザー |
| **二次ユーザー（直接）** | システムへのサポートを提供する人 | ATMのメンテナンス担当者 |
| **間接ユーザー** | システムを直接操作しないが、その出力を受け取る人 | 図書館システムの処理結果を受け取る図書館利用者 |

UXデザインにおいては、誰をユーザーとして想定するかを最初に整理することが不可欠。特に複数の役割のユーザーが存在するサービスでは、それぞれの利用文脈と体験価値を別々に把握する必要がある。

### ユーザーの多様性

ユーザーの多様性に影響する要因は以下の3区分に整理できる。

| 多様性の区分 | 主な要因例 |
|-------------|-----------|
| **特性に関する多様性** | 年齢・性別・障害・識字率・身体特性・知識・技能 |
| **志向性に関する多様性** | 文化・宗教・社会的態度・嗜好・価値観 |
| **状況・環境に関する多様性** | 精神状態・一時的状態・経済状態・物理的環境・社会的環境 |

**設計上の留意**：障がい者・高齢者・外国語使用者など、すべてのユーザーグループを想定したアクセシビリティの確保は、どの製品・サービスにおいても検討が必要。ICT分野では JIS X 8341 シリーズのガイドラインを参照する。

## UXの基本フレームワーク：ハッセンツァールモデル

UXの生起プロセスを説明する代表的なフレームワーク。デザイナーの視点とユーザーの視点を明確に分けることで、設計者がどこまで関与できるかを示している。

### デザイナーの視点

```
製品の特徴（内容・表現・機能・インタラクション）
    ↓ デザイナーが設計
意図された製品の性質
  ├── 実用的属性：操作性（使いやすさ）
  └── 快楽的属性：刺激（受ける）・同定（自分らしさ）・喚起（気持ちの喚起）
    ↓（ここまでしか関与できない）
```

### ユーザーの視点

```
製品の特徴（同上）
    ↓ 利用文脈（状況）の影響を受けながら
表出した製品の性質（ユーザーが実際に知覚するもの）
    ↓ ユーザー自身が評価判断
結果：魅力・うれしさ・満足
```

**設計上の示唆**：デザイナーはUXそのものに直接アクセスできない。制御できるのは「製品の特徴」と「システムイメージ」だけであり、ユーザーが知覚する性質をいかに意図した通りにするかが設計の核心となる。

### 製品の性質の2種類

| 性質 | 説明 | デザインでの対象 |
|------|------|----------------|
| **実用的属性** | 操作性・使いやすさ。タスク達成を支援する側面 | ユーザビリティ設計 |
| **快楽的属性** | 刺激（新しい能力の習得）・同定（自分らしさの表現）・喚起（気持ちの喚起）。感情的側面 | 感情・価値観・アイデンティティへの訴求 |

## 利用文脈（Context of Use）

### ISO 9241-11による定義

利用文脈とは、「**ユーザー・タスク・設備（ハードウェア/ソフトウェア/資材）、および製品が利用されている物理的・社会的環境**」（ISO 9241-11）である。

同じ製品でも利用文脈が異なれば得られるUXは大きく異なるため、設計・評価において利用文脈を明確に特定することが不可欠。

### 把握すべき利用文脈の4要素（ISO 9241-210）

| 要素 | 内容 |
|------|------|
| **ユーザー** | 直接ユーザー・間接ユーザーの特性（知識・スキル・経験・身体的特性・習慣） |
| **タスク** | ユーザーの目標と仕事の内容。健康・安全への影響、誤操作リスクも含む |
| **設備** | ハードウェア・ソフトウェア・資材を含む技術的環境 |
| **環境** | 物理的環境（温度・照明・空間）、社会的・文化的環境（仕事の習慣・組織構造） |

### コンテクストの多重性

特にデジタルサービスにおいては、利用文脈は単層ではなく多重構造を持つ。

```
生活世界のコンテクスト（本来のユーザーの目標）
    ↓
手段選択のコンテクスト（Web・アプリを使うという判断）
    ↓
ネット世界のコンテクスト（サービス内・サービス間の利用目標）
```

**実践上の注意**：製品・サービスの利用中だけを利用文脈として捉えると、ユーザーの本質的なニーズを見落とす。利用前後の文脈まで広げて把握することが重要。

## ユーザビリティとUXの関係

### ISO 9241-11によるユーザビリティの定義

「ある製品が、**指定された利用者**によって、**指定された利用状況**下で、**指定された目標**を達成するために用いられる際の**有効さ・効率・満足度**の度合い」（ISO 9241-11）

| 指標 | 定義 | 設計への示唆 |
|------|------|-------------|
| **有効さ**（effectiveness） | 目標を達成する際の正確さと完全性 | 操作が完遂できるか |
| **効率**（efficiency） | 目標達成に費やした資源（時間・操作数など） | 無駄なく達成できるか |
| **満足度**（satisfaction） | 不快さのないことと、製品使用への肯定的な態度 | 使い続けたいと思えるか |

**ユーザビリティとUXの位置づけの違い**：

```
ユーザー体験（UX）← ユーザー側の主観的反応
         ↑
  利用文脈の影響
         ↑
ユーザビリティ ← 製品・サービス側の品質特性
```

ユーザビリティは製品・サービス側の概念であり、UXはユーザー側の概念。良いユーザビリティは良いUXの必要条件だが、十分条件ではない。

### 目標の階層性

ユーザーの目標には階層構造があり、上位ほど概念的（目的的）、下位ほど具体的（手段的）になる。

```
概念的
  ↑
「子どもと一緒に時間を楽しみたい」← Be Goal（存在ゴール）
  ↓ 手段として
「子どもと一緒に海までドライブしたい」← Do Goal（行動ゴール）
  ↓ 手段として
「目的地まで道案内できるようにしたい」← Do Goal
  ↓ 手段として
「カーナビに目的地を設定したい」← Motor Goal（操作ゴール）
  ↓
具体的
```

| 目標レベル | 説明 | UX設計上の意味 |
|-----------|------|--------------|
| **Be Goal**（存在ゴール） | ユーザーが根本的に「どうありたいか」の目標 | 体験価値・本質的ニーズに対応。製品の提供価値を定義する |
| **Do Goal**（行動ゴール） | ユーザーが「何をしたいか」の目標 | 機能要件・シナリオ設計に対応 |
| **Motor Goal**（操作ゴール） | 具体的な操作レベルの目標 | インタフェース設計・ユーザビリティ評価に対応 |

### ユーザーの指向性と満足感

目標の達成方法によって、ユーザーが重視する品質は異なる。

| 行動の指向性 | 説明 | 重視される評価軸 |
|-------------|------|-----------------|
| **目標指向的行動** | 目標を達成することが主眼 | 満足感（Satisfaction） |
| **プロセス指向的行動** | 目標達成の過程が主眼（ドライブを楽しむなど） | 楽しさ（Fun） |
| **状態指向的行動** | 特定の状態にあること自体が目的（お気に入りを眺めるなど） | 心地よさ（Comfort） |

## 利用意欲と利用態度：SEPIA法

インタラクティブ製品のUXに強い影響を与えるモチベーションとして、**製品利用の自己効力感（SE: Self-Efficacy）**と**製品関与（PI: Product Involvement）**がある。この2軸の組み合わせにより、ユーザーの利用態度を分類できる（SEPIA法）。

| | 製品関与（PI）高 | 製品関与（PI）低 |
|---|---|---|
| **自己効力感（SE）高** | **マニアユーザー**<br>積極的に使いこなし、愛着など好意的評価をする傾向 | **冷静・合理的ユーザー**<br>使えると思っているが製品への思い入れは薄く、合理的に判断 |
| **自己効力感（SE）低** | **期待先行ユーザー**<br>使いたい気持ちは高いが操作への自信がなく、満足・不満足が共存する傾向 | **ミニマム利用ユーザー**<br>使いこなしに消極的で最低限の利用に留まる傾向 |

**活用場面**：
- 調査対象者の選定（代表的なユーザータイプを網羅する）
- ペルソナの簡易フレームワーク
- UXの詳細な理解・ターゲット別設計方針の策定

## 認知工学の基礎

認知工学は、人間の記憶・学習・思考・判断・問題解決などの認知特性を理解し、それに適合した機器・システムを設計するための工学分野。**アフォーダンス・メンタルモデル・シグニファイア**などの概念はここから生まれた。

### メンタルモデルと概念モデル

| 概念 | 説明 | 設計への示唆 |
|------|------|-------------|
| **メンタルモデル（概念モデル）** | ユーザーが利用経験を通して構築した「システムがどう動くか」のイメージ | ユーザーの期待と製品の実際の動作が一致するよう設計する |
| **デザイナーの概念モデル** | 製品を作った側が持つ操作体系の概念 | デザイナーとユーザーの概念モデルを一致させることが設計の目標 |
| **システムイメージ** | マニュアル・UI・パッケージなど、製品周辺の情報群 | ユーザーが正しい概念モデルを形成できるよう、システムイメージを通じて伝える |

### アフォーダンスとシグニファイア

| 概念 | 説明 | 実践例 |
|------|------|--------|
| **アフォーダンス** | 環境や物理的対象物が人（動物）に提供する行為の可能性の情報。自然界に存在し、見るだけで知覚される | 40cmの木箱を見ると「座れる」「上に乗れる」と直感的に認識する |
| **シグニファイア** | ユーザーに適切な行為を促すためにデザインが提供する手がかり。良い・悪いがある | ボタンに影をつけて「押せる」と知覚させる。「押す」とラベルする |

アフォーダンスは自然界の性質であり、デザイナーが操作するのはシグニファイア。適切なシグニファイアを用いることで自然でわかりやすい製品が実現できる。

### 制約の4種類

ユーザーの行為を自然に正しい方向へ誘導する制約のタイプ：

| 制約の種類 | 説明 | 例 |
|-----------|------|-----|
| **物理的制約** | 物理的に誤った操作ができない | USBコネクタは逆向きに刺さらない |
| **意味的制約** | 状況の意味が特定の行為を示唆する | ヘルメットは頭部を守るものと理解しているため正しく被れる |
| **文化的制約** | 文化的慣習に基づく制約 | 赤は危険・停止、緑は安全・進行 |
| **論理的制約** | 論理的に考えて残りの選択肢が決まる | すべての部品を取り付けた後、残った1つの穴に対応するパーツが自明になる |

### 行為の7段階モデル

人が何かの行為を行う際の認知プロセスをモデル化したもの。**実行**と**評価**の2つの側面からなる。

**実行の側面**（頭の中から外界への流れ）：

```
ゴール（達成すべき状態）
    ↓
プラン（何をするかの意図）
    ↓
詳細化（具体的な行為への変換）
    ↓
実行（物理的な操作）→ 外界に効果を及ぼす
```

**評価の側面**（外界から頭の中への流れ）：

```
外界の変化
    ↓
知覚（外界を感知する）
    ↓
解釈（予期・期待に沿って意味を付与）
    ↓
比較（プランとゴールに照らし合わせる）
```

**設計への示唆**：各段階でユーザーがスムーズに進めるよう設計する。特に「詳細化→実行」段階の操作のわかりやすさと、「知覚→解釈」段階のフィードバックの明確さが重要。

### 認知工学に基づくデザインの7原則

1. **発見可能性**：行える操作と現在の状態が判断できる
2. **フィードバック**：行為の結果と現在の状態について完全かつ継続的な情報がある
3. **概念モデル**：システムの良い概念モデルを作るのに必要な情報を伝える
4. **アフォーダンス**：望ましい行為を可能にする適切なアフォーダンスがある
5. **シグニファイア**：効果的なシグニファイアにより発見可能性を確かにする
6. **対応づけ**：制御部と行為の関係が良い対応づけの原理に従う
7. **制約**：物理的・論理的・意味的・文化的な制約を与え、行為を導く

## ビジネスにおけるUXデザインの適用パターン

製品・サービスの実現手段と、提供する体験価値の組み合わせにより、3つの適用パターンが存在する。

| パターン | 実現手段 | 提供する体験価値 | ビジネス目標の例 |
|---------|---------|----------------|----------------|
| **A：新ビジネス・新製品開発** | 新規に製品・サービスを作る | 新しい・未充足の体験価値 | 顧客の獲得・事業の成功 |
| **B：既存ビジネスへの新価値追加** | 既存ビジネスに新機能・新サービスを追加 | 新しい・未充足の体験価値 | 満足度向上・新規顧客獲得 |
| **C：既存UXの品質向上** | 新規・既存いずれも | 既存の・充足されている体験価値 | 満足度向上・リピート率向上 |

### 各パターンの設計上の留意点

**パターンA（新ビジネス）**：ユーザーの本質的なニーズを深く掘り下げ、まだ市場にない体験価値を発見することが中心課題。リスクは高いがイノベーションの余地が大きい。

**パターンB（既存への新価値）**：既存ビジネスの強みや方向性を維持しつつ、ユーザーの未充足ニーズを発見する。既存ユーザーと潜在ユーザーの両方を対象に調査する。

**パターンC（品質向上）**：体験価値は既存だが、提供手段やプロセスを改善する。ユーザビリティ評価・エスノグラフィなど、現状把握の手法が中心的な役割を果たす。
