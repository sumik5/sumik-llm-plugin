## AI PM 3専門化 — 詳細比較

### AI-Experiences PM（体験設計スペシャリスト）

**コアミッション**: AIがユーザーと触れる全ての接点（UI/UX）を最適化する

#### 主要責任（Key Responsibilities）
```
1. AI機能のユーザー体験戦略の定義
2. AI出力の品質基準（Eval基準）の設定
3. ユーザーリサーチ・フィードバックの収集・分析
4. 説明可能性・透明性のUX設計
5. AI体験に関するA/Bテスト設計・分析
6. AI利用に関するユーザー期待値管理
```

#### 日常業務（Day-to-Day Activities）
```
- GenAI機能のEval（評価）セッションに参加
- ユーザーフィードバックを分析してMLチームにフィード
- プロンプトエンジニアリングチームと協働
- モデルアップデートのUXインパクト評価
- ユーザーテストセッションのファシリテーション
```

#### 必要スキルセット（Toolkit）
```
Core PM Craft:
  ✓ ユーザーリサーチ・ユーザビリティテスト
  ✓ プロダクトロードマップ管理
  ✓ データドリブンな意思決定

Engineering Foundations for PMs:
  ✓ API基礎知識（REST・GraphQL）
  ✓ プロンプトエンジニアリングの基礎
  ✓ AI/MLの出力形式と限界の理解

Leadership & Collaboration:
  ✓ UX/デザインチームとの協働
  ✓ ステークホルダーへのAI体験の説明・説得
  ✓ 倫理的AI実践の推進

AI Lifecycle Awareness:
  ✓ Evals（AI評価）の種類と実施方法
  ✓ Model Driftがユーザー体験に与える影響
  ✓ Human-in-the-Loop設計
```

### AI-Experiences PMの課題と報酬

**主要な課題**:
- **イノベーションとユーザビリティのバランス**: 最新AI機能を活用しつつシンプルなUXを保つことが最難関
- **ユーザー期待値管理**: AIは過剰に期待されがち。機能の能力と限界を誠実に伝える
- **不確実性・エラーへの対処**: モデルが誤動作した場合のUX設計（フォールバック・説明・フィードバック機能）
- **急速に変化する領域への追従**: 新モデル・手法・ツールが常に登場するため継続学習が必須
- **クロスファンクショナル調整**: PM・データサイエンティスト・ML Engineer・UXデザイナーの高度な連携が必要

**役割のやりがい**:
- ユーザーとAIの接点を設計し、未来のHuman-Computer Interactionを形成する
- ユーザーの実生活に直接インパクトを与える結果（エンゲージメント・満足度・タスク完了率）が見える
- 技術の最先端で常に学び・実験し続けられる

### AI-Experiences PMの「1日の流れ」（具体例）

```
08:30 – ユーザーフィードバック確認
  Smart Composeのアプリレビュー・サポートチケット・インアプリ評価をレビュー
  → AI提案の受容率（Acceptance Rate）がわずか低下していることを把握

09:00 – チームスタンドアップ
  データサイエンティスト・エンジニア・デザイナーとフィードバック傾向を共有
  → 言語モデルの最近の変更が原因かもしれないと議論

10:00 – 新AI機能のデザインレビュー
  「ミーティングサマライザー」機能のワイヤーフレームをレビュー
  → AI出力の表示方法・エラー処理・ユーザーフィードバック機能をデザイナーと検討

11:00 – プロンプトエンジニアリングワークショップ
  LLMの効果的なプロンプト戦略を学び、サマライザーモデルで実験

13:00 – A/Bテスト設計
  新バージョンのSmart Composeをテストするための仮説・対象指標・サンプルサイズを決定
  セグメント別のパフォーマンス差異も検討

14:00 – ロードマップ優先度会議
  ユーザーリサーチ・競合分析・ビジネスインパクトに基づきAI機能を提案

15:00 – GenAI評価指標のレビュー
  精度・一貫性・幻覚率（hallucination rate）を確認
  バイアス・公平性に関連する指標も専門に確認

16:00 – PRD草稿作成
  「スマートリプライ」機能のユーザー問題・解決策・機能要件・倫理的配慮を記述
```

### AI-Experiences PMとその他のロールとの関係

| 協働先 | 関係の性質 | PMの主な責任 |
|--------|-----------|------------|
| **Data Scientists** | 頻繁・緊密な協働 | ユーザーニーズ・成功指標・モデルフィードバックを提供 |
| **ML Engineers** | デプロイ・監視フェーズで連携 | パフォーマンス要件・ロールアウト戦略・モニタリング設計 |
| **UX Designers** | 極めて緊密な協働 | AI出力の表示方法・フィードバック機能・不確実性の表現設計 |
| **Marketing** | GTM戦略・メッセージング | AI機能の価値提案の正確な伝達（過大表現禁止） |
| **Legal/Compliance** | 規制遵守確認 | データプライバシー・差別禁止法への適合確認 |
| **AI-Builder PM** | 協働（能力・限界の共有） | 基盤モデルの能力を把握しUXに反映 |

### AI-Builder PM（インフラアーキテクト）

**コアミッション**: AIシステムの基盤・プラットフォームを構築し、エンジニアリングチームを率いる

#### 主要責任（Key Responsibilities）
```
1. MLプラットフォーム・インフラのPRD作成
2. データパイプラインの要件定義
3. APIインターフェースの設計とガバナンス
4. モデル訓練・評価・デプロイパイプラインの監督
5. Build vs Buy vs Open Source意思決定
6. AIインフラのコスト管理（AI is expensive）
```

#### 日常業務（Day-to-Day Activities）
```
- Feature Storeやモデルレジストリの要件化
- ML Engineerとのスプリントプランニング
- システムアーキテクチャレビューへの参加
- AI/ML技術的負債の管理・優先度付け
- クラウドプロバイダーとのコスト交渉・最適化
```

#### 必要スキルセット（Toolkit）
```
Core PM Craft:
  ✓ 技術的なPRD・API仕様書の作成
  ✓ テクニカルロードマップの管理
  ✓ エンジニアリングチームのスコープ調整

Engineering Foundations for PMs:
  ✓ MLOps基礎（CI/CD for ML）
  ✓ クラウドインフラ（AWS/GCP/Azure）
  ✓ データエンジニアリング概念
  ✓ APIデザインパターン

Leadership & Collaboration:
  ✓ ML Engineer・Data Scientist・Data Engineerとの橋渡し
  ✓ CTO/VPEngへのテクニカル戦略提言
  ✓ インフラ投資のROI説明

AI Lifecycle Awareness:
  ✓ ML訓練パイプラインの全段階
  ✓ Feature Engineering要件の定義
  ✓ モデル評価メトリクス（Precision/Recall/F1等）
```

### AI-Builder PMの「1日の流れ」（5大活動柱）

```
戦略的プランニング・ロードマップ管理:
  - 各プロダクトチームからの新AIケイパビリティリクエストを分析・評価
  - ビジネス目標と長期AIビジョンに基づき開発優先度を決定
  - AIプラットフォームロードマップを更新しステークホルダーに伝達

技術的監督・コラボレーション:
  - データサイエンティスト・ML Engineerとのスプリント計画・アーキテクチャレビュー
  - デプロイ済みモデルのAccuracy・レイテンシ・スループット・コストを監視
  - DevOps/MLOpsエンジニアとのモデルデプロイ自動化・監視ダッシュボード設計

クロスファンクショナルコミュニケーション:
  - AI-Experiences PMとの新モデルの能力・限界の共有
  - CTO/VPエンジニアリングへのAIインフラ投資ROI説明
  - 組織横断的なAI開発ベストプラクティスの共有・ワークショップ開催

GenAI評価・品質管理:
  - カスタマーサポートチャットボット等に使用するLLMの精度・一貫性・潜在的バイアスを定期評価
  - GenAI評価プロセスの基準とベンチマークを確立

パフォーマンス・最適化:
  - 本番AIシステムのパフォーマンス最適化戦略
  - クラウドプロバイダーとのコスト最適化交渉
  - AI技術的負債の可視化と優先度付け
```

### AI-Builder PMが構築する基盤モデルの例

各プロダクトチームが個別に構築するのではなく、組織共有APIとして提供:
- チャーン予測モデル（全プロダクトチームが利用可能なAPI）
- 顧客生涯価値予測
- 不正検知システム
- 商品カテゴリ自動分類

### AI-Enhanced PM（AIツール活用スペシャリスト）

**コアミッション**: PMの業務プロセス自体をAIツールで強化し、意思決定の質とスピードを上げる

#### 主要責任（Key Responsibilities）
```
1. PM業務のAI活用プロセス設計・標準化
2. AIツール（Copilot・Analytics・リサーチ支援）の評価・導入
3. データ収集・分析・インサイト抽出の自動化
4. AIが生成したプロダクト仮説の検証フロー設計
5. 組織全体へのAI活用文化の普及
```

#### 日常業務（Day-to-Day Activities）
```
- AIツールを使ったユーザーフィードバック分析
- 競合分析の自動化・整理
- ロードマップ生成・優先度付けのAI活用
- 会議・インタビューの自動文字起こし・要約
- データ分析ダッシュボードの整備
```

#### 必要スキルセット（Toolkit）
```
Core PM Craft:
  ✓ プロセス設計・最適化
  ✓ ツール評価・導入プロジェクト管理
  ✓ 変更管理（チームへのAI活用浸透）

Engineering Foundations for PMs:
  ✓ プロンプトエンジニアリング（実務レベル）
  ✓ APIインテグレーション基礎
  ✓ データ分析・BI ツール活用

Leadership & Collaboration:
  ✓ AI活用のROI測定・報告
  ✓ 組織横断的なAI活用ベストプラクティス共有
  ✓ AIリスク（誤情報・バイアス）の啓蒙

AI Lifecycle Awareness:
  ✓ 使用するAIツールの能力・限界を理解
  ✓ AI出力の批判的評価
```

### AI-Enhanced PMの「1日の流れ」（5コアエリア）

```
市場・競合分析（Market and Competitive Analysis）:
  - AI活用ツールで競合サイト・SNS言及・プレスリリースを自動追跡
  - 新機能ローンチ・マーケティングキャンペーン・顧客感情を自動集計
  - AIプラットフォームで市場調査レポート・業界ニュースを大規模分析し
    新興トレンド・潜在的破壊的変化を早期特定

ユーザーリサーチ・フィードバック分析（User Research & Feedback Analysis）:
  - NLPツールでユーザーレビュー・アンケート回答・サポートチケットを分析
    → 主要テーマ・ペインポイント・機能要求を抽出（手作業タグ付けを置換）
  - 教師なし学習（クラスタリング）でユーザーセグメントを発見的に特定
    → 事前定義なしで異なるニーズ・行動パターンを持つグループを識別

プロダクト戦略・プランニング（Product Strategy & Planning）:
  - AIツールでユーザーフィードバック・市場トレンド・技術的実現可能性・
    ビジネスゴールを統合分析し、ロードマップ優先度付けをデータドリブンに
  - LLMへのプロンプトでユーザーニーズ・市場機会に基づく機能アイデアをブレスト
  - AIアシスタントでPRD初稿を生成し、PM本来の判断業務に集中

コミュニケーション・コラボレーション（Communication & Collaboration）:
  - AI支援でデータサマリー・ビジュアル・プレゼンテーション草稿を生成
  - AI議事録ツールで会議を文字起こし・要約・アクションアイテム抽出を自動化
  - AI日程調整ツールで最適ミーティング時間の発見・フォローアップ自動送信

個人生産性・継続学習（Personal Productivity & Learning）:
  - メール分類・カレンダー管理・データ入力等のルーティンタスクを自動化
  - AIツールの最新動向をウェビナー・記事・実験で継続的にキャッチアップ
  - 「使っているだけのユーザー」から「プロセス変革者」へ: AI活用事例を
    チームに共有し、データドリブン文化の醸成をリード
```

## AI PM キャリアパス

### 4レベルキャリアロードマップ

```
Level 1: Foundation（基盤構築）
  期間: 0-2年
  フォーカス:
    - AI/MLの基礎理解（コードは不要、概念は必須）
    - 1つの専門化分野での経験積み上げ
    - メンターの確保・コミュニティ参加
  マイルストーン:
    - AI機能を含む製品の0→1リリース経験
    - 技術チームとの効果的な協働を確立
    - AI評価・Evals設計の経験

Level 2: Strategic Growth（戦略的成長）
  期間: 2-5年
  フォーカス:
    - 複数AIプロダクトの同時マネジメント
    - OKR設計・財務計画への参画
    - チームリーダーシップ経験
  マイルストーン:
    - AIプロダクトのGrowth・Scalingの経験
    - ROIの定量的証明
    - クロスファンクショナルチームのリード経験

Level 3: Specializing & Leading（専門化・リード）
  期間: 5-10年
  フォーカス:
    - 3専門化のうち1-2つでのエキスパート化
    - 組織AIロードマップへの戦略的関与
    - 後進の育成・メンタリング
  マイルストーン:
    - AI部門・チームのリード
    - 業界イベントでの発信・Thought Leadership
    - 組織横断的なAI文化の牽引

Level 4: Light for Others（灯台になる）
  期間: 10年+
  フォーカス:
    - 組織・業界全体へのAI戦略的インパクト
    - 次世代PM・リーダーの育成
    - AI倫理・政策への関与
  マイルストーン:
    - VP of Product AI / CPO / AI Strategy Leader
    - 書籍・論文・講演による知識共有
    - 組織のAI成熟度を複数段階引き上げた実績
```

## チームビルディング

### AI PMがチームに求めるもの

```
強いAIチームの構成原則:
1. T字型人材（広いAI理解 × 深い専門性）を中核に置く
2. 技術チームとPMの定期的な「AI知識共有」セッション
3. データサイエンティストとPMのペアリング（週1回の同期）
4. 倫理・法務・セキュリティの早期組み込み（Shift Left）
```

### AI時代のステークホルダーマネジメント

```
経営層への報告:
  - AI投資をビジネス成果（収益・コスト・ユーザー数）で語る
  - 技術的詳細より「AIが解決するビジネス問題」を前面に
  - リスクを隠さず、対策とともに提示

技術チームとの関係:
  - 「なぜ」（ビジネスゴール）を常に共有する
  - ML Engineer・Data Scientistの制約を尊重する
  - 技術的負債を可視化し、優先度付けに参加する

ユーザーとの関係:
  - AI使用を透明に開示する
  - フィードバックを積極的に収集しモデル改善に反映
  - AIミスへの誠実な対応・改善コミュニケーション
```
