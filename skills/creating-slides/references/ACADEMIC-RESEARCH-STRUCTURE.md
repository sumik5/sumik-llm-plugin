# 学術研究プレゼン構成ガイド

研究発表に特化した構成設計（Why→How→What→So What のアーク）、材料と方法・結果と考察・結論の各パートの実践手法、10通りのオープニングと5通りのクロージング、ストーリーアーク統合を体系化する。Wallwork『国際学会プレゼン戦略』Ch6, Ch8, Ch9, Ch10 を主出典とする。

---

## 1. 研究プレゼン特有のアーク

### 1.1 Why → How → What → So What

研究発表のすべての構成はこの4段階で捉えられる。

| 段階 | 役割 | 対応章 | 問いかけ |
|------|------|--------|---------|
| **Why** | 問いと動機 | オープニング | なぜこの研究が必要か？ |
| **How** | 方法と手順 | 材料と方法（Ch8） | どのように調べたか？ |
| **What** | 結果と考察 | 結果と考察（Ch9） | 何がわかったか？ |
| **So What** | 意義と展望 | 結論（Ch10） | それが世界にとって何を意味するか？ |

**重要な認識**: So What は「結論」ではなく「意義の宣言」。研究が与える影響を三層で伝える（→ §7）。

### 1.2 オーディエンスの集中力カーブとの対応

10分間のプレゼンでは集中力がこのように動く。

```
集中力
↑  ★高      ★高
|   /\         /\
|  /  \       /  \
| /    -------    
低←──────────────→ 時間
    0  2  4  6  8  10分
```

- **開始直後（0〜30秒）**: 集中力は低い。重要情報の提示を少し遅らせることも有効
- **2分前後**: 最初のピーク → 最重要ポイントを伝える
- **5〜8分**: 最も低い → 意識的にエンゲージメントを上げる施策が必要
- **終了1分前**: 2回目のピーク → 「終わりに近づいた」シグナルを明確に出す

## 2. 材料と方法

(Wallwork Ch8)

### 2.1 集中力を取り戻す（8.2）

方法の説明を始める時点で、オーディエンスの集中力はすでに低下している。以下で注意を引き戻す。

- 新しい視点を提示する一言を入れる
- 方法の「おもしろい部分」から入る
- 第12章的な手法（オーディエンスへの問いかけ・統計提示）を適宜挟む

### 2.2 例→技術説明の順序（8.4）

**❌ 逆順（理論先行）**:
> "The algorithm uses a non-parametric bootstrapping method..."

**✅ 例先行**:
> "Imagine you're trying to predict whether tomorrow will rain. We used a technique similar to asking 1000 different people and combining their guesses..."

直感を刺激してから技術的説明に入ることで、理解が定着しやすくなる。

### 2.3 必要な情報だけを話す（8.5 / 8.6 / 8.8）

| すること | しないこと |
|---------|-----------|
| 研究の理解に必要な情報だけ選ぶ | 全ステップを網羅しない |
| 重要な工程のみ強調 | 論文の図表をそのまま流用しない |
| 省略の理由を一言伝える | 詳細なアニメーション連続で時間を使わない |

全手順を省略する場合の表現:
```
We don't have time to look at the complete process, so I just wanted to show you this part.
If you are interested in the whole process then I can explain it at the bar
or you can look it up on my web page.
```

### 2.4 取扱説明書ではなくストーリーを語る（8.10）

- 選択過程をストーリーのように語る
- 受動態より能動態を使う（"I chose" not "X was chosen"）
- 人称代名詞を使う（I, we, you）
- 本質的でない情報は除外する

**スライドタイトルにステップを入れる（8.7）**: 手順説明が主体のプレゼンでは、各スライドのタイトルを工程名にすると流れが自然に伝わる。

### 2.5 方程式・公式の扱い（8.12）

方程式はできるだけ避ける。必要な場合は:
- 式の細部を解説するのではなく、**何を意味するか・なぜ重要か**だけを語る
- 詳細は配布資料またはウェブページへ誘導する

### 2.6 能動態と受動態の使い分け（8.13）

| 場面 | 推奨 | 理由 |
|------|------|------|
| 手順の説明 | 受動態も可 | 慣用的に許容される |
| データ選択・判断の説明 | 能動態 | プレゼンターが中心人物として見える |
| 研究の意図・動機 | 能動態 | 人間味が伝わる |

## 3. 結果と考察

(Wallwork Ch9)

### 3.1 重要な結果に絞る（9.2）

オーディエンスは4つの問いへの短い答えを期待している:

1. **何がわかったのか？**
2. **それは予想通りだったのか？**
3. **どのような意義があるのか？**
4. **注目すべき理由は？**

10分プレゼンなら結果スライドは2枚程度。「おもしろいが重要でない話題」は省く判断が重要。

### 3.2 結果の位置づけと価値を伝える（9.3）

プレゼンターには自明の位置づけも、オーディエンスには見えていないことがある。以下の表現で補う。

```
What this means is that ....           （この意味は〜）
The key benefit of this is ....        （こちらの主な長所は〜）
What I would like you to notice is ....（ここでお気づきいただきたいのですが〜）
Possible applications of this are .... （応用として可能性があるのは〜）
```

### 3.3 グラフ解説の原則（9.4）

**❌ やりがちな失敗**: 軸の説明をダラダラと行う
**✅ 正しいアプローチ**: グラフそのものの説明ではなく、**グラフが示唆する意味・知見**を語る

ポイント:
- 重要なポイントは2回以上繰り返す
- youを使ってオーディエンスに直接話しかける
- 3〜5秒のポーズを設け、情報を消化する時間を与える

### 3.4 ヘッジング（自信過剰表現の回避）（9.5）

断定的な表現は信頼性を損ない、批判にさらされるリスクを高める。

**避けるべき断定表現 → 推奨表現**:

| 強い断定 | ヘッジング表現 |
|---------|--------------|
| proves / demonstrates | would seem to / would appear to suggest |
| definitely / certainly | probably / possibly / likely |
| This is the first time | To the best of our knowledge |
| should be used | might / could be used |

助動詞（would, might, may, could）と譲歩表現（although, despite, nevertheless）を組み合わせることで、謙虚さと信頼性を同時に示せる。

### 3.5 否定的結果への正直な対応（9.8 / 9.9）

否定的な結果は隠蔽してはならない。以下の姿勢で臨む。

1. 「結果は自分が望んでいたものではない」とオーディエンスに認める
2. 問題解決のための次の計画を話す
3. 「このような経験をされた方はいませんか？」と問いかけ、共同研究の機会に変える

**共同研究の機会として提示する表現**:
```
Has anyone had similar experiences? How did you deal with it?
```

### 3.6 考察・討論を促す姿勢（9.10）

- 他の解釈を受け入れる寛大さを示す
- 研究の限界についての議論を積極的に歓迎する
- 講義ではなく「意見交換の場」として位置づける

## 4. 結論

(Wallwork Ch10)

### 4.1 事前準備と簡潔さ（10.2）

10分プレゼンの結論は約1分・3〜4文。事前に言葉を用意して、その通りに話す。

終わりの合図:
- 背筋を伸ばし、オーディエンスをまっすぐ見る
- "In conclusion..." または "To sum up..." で明確に区切る
- スライドやメモを見ずに話す（最後の60秒は暗記が理想）

### 4.2 最後のスライドに有用な情報を入れる（10.4 / 10.6）

| スライドの内容 | 評価 |
|--------------|------|
| Acknowledgments（謝辞）だけ | ❌ 印象に残らない |
| "Thank you" だけ | △ 使い古された表現 |
| 連絡先 + 招待表現 | ✅ 有効な終わり方 |

連絡先スライドを印象的に:
```
Please get in touch! [your email]
```

### 4.3 最後のスライドのコピーを用意する（10.7）

Q&A中はプレゼンの最後のスライドが表示されている。そのスライドが:
- 連絡先を含んでいる
- 研究の主要メッセージを示している

であれば、Q&A中も無言の「名刺」として機能し続ける。

### 4.4 5通りのクロージング（10.5）

(Wallwork Ch10.5)

| # | クロージングタイプ | 適用条件 |
|---|-----------------|---------|
| **[1] 画像を使用する** | 前半で使った重要画像に結論をスーパーインポーズ。オープニングで自国を紹介したなら類似写真で締める | 視覚的インパクトを求める場合 |
| **[2] 重要性を訴える** | 研究結果がオーディエンスの日常・社会に与える影響を直接語る | 応用研究・社会科学系 |
| **[3] 統計データを提示する** | 数値を会場の人数や身近なスケールに変換して提示 | 数値の力を最大化したい場合 |
| **[4] フィードバックを求める** | 3つの結論ポイントに対してオーディエンスに意見を求める | 対話的な締め、ワークショップ的場面 |
| **[5] 今後の研究について話す** | 研究の限界と次フェーズの計画を伝え、コラボへの招待を示す | 結果が不完全・否定的だった場合に特に有効 |

**クロージング共通表現**:
```
In conclusion, ...
To sum up, ...
What I'd like you to take away from this is ...
```

## 5. 10通りのオープニング

(Wallwork Ch6)

### 5.1 オープニングの重要性

プレゼンの成功の30%以上は最初の90秒で決まる。良いオープニングの条件:
- 自分が楽しんでいることが伝わる
- オーディエンスの好奇心を刺激する
- 研究テーマとの関連が明確

**悪いオープニング**（避ける）: 謝罪から始める・下を向いてためらいながら話す・独特の声色で話す

### 5.2 10通りのパターン一覧

| # | パターン | 難易度 | 概要 |
|---|---------|--------|------|
| **[1] 内容と理由の予告** | ★ 初級 | 仮説・方法・達成内容・影響を端的に伝える。最も標準的な始め方 |
| **[2] 出身地データ** | ★ 初級 | 自国についての新鮮な数値データで始める（30秒以内、研究との関連を明示） |
| **[3] 地図を見せる** | ★ 初級 | 研究場所や出身地を地図で示す。プレゼンターに心理的な強さをもたらす |
| **[4] 出身国の興味深いデータ** | ★ 初級 | 自国に関連した研究テーマのデータを提示。個人的エピソードと組み合わせても効果的 |
| **[5] 即理解できるデータ** | ★★ 中級 | 巨大な数字を「今この瞬間」「この会場の人数」など身近なスケールに変換して示す |
| **[6] 状況の想像** | ★★ 中級 | "Suppose..." で始め、研究テーマに関連した仮定の状況をオーディエンスに想像させる |
| **[7] オーディエンスへの質問** | ★★ 中級 | 問いを投げかけ2秒待ってから話す。答えさせると流れが崩れるため注意 |
| **[8] 個人的体験** | ★★★ 上級 | 研究テーマへの興味を持ったきっかけ・情熱の源泉を語る。情熱が声と表情に現れる |
| **[9] 時事ネタ** | ★★★ 上級 | 最近のニュースや学会関連の話題と研究テーマを結びつける |
| **[10] 逆説的事実** | ★★★ 上級 | 常識に反する研究結果や驚愕の事実を最初に提示する（宗教・倫理・政治は除く） |

### 5.3 各パターンの実践ポイント

**[5] データを身近なスケールに変換する例**:
```
例1: 1分間に14報の論文が発表されています。
例2: 皆さんにお話ししているこの10分間にも、世界中で140報の論文が発表されています。
例3（挙手テクニック）: この7日間に論文を書き上げた方は挙手を。
     世界では過去1週間に約14万報の論文が生み出されています。
```

**[7] 質問テクニック**:
- 答えを考えさせたら2秒後に自分で続ける
- "Do you know anyone who..."
- "Hands up if..." + 2秒待つ

**[8] 個人体験のポイント**:
- 話し言葉を使い、友人との会話のような雰囲気
- 他人の言葉を引用する
- 長文と短文を混ぜる
- 研究に対する情熱を隠さない

## 6. オープニング → 本論 → クロージングの統合

### 6.1 テーマの橋かけ原則

オープニングで使ったモチーフ・データ・問いかけを、クロージングで「回収」する。これによりプレゼン全体に統一感が生まれる。

| オープニング | クロージングでの回収例 |
|------------|-------------------|
| 自国の地図・データ [2][3][4] | 同じ国の写真でコラージュ。研究が貢献できることを伝える |
| 問いかけ [7] | 最後にその問いへの答えを示す |
| 個人的体験 [8] | その体験が研究で変化したことを語る |
| 驚愕事実 [10] | その事実に対する研究の貢献を語る |

### 6.2 本論との接続：トランジション

各セクション間には言語的な橋渡しが必要（詳細は Ch7 / `QUALITY-LOGICAL-STRUCTURE.md` 参照）。

**材料と方法に入る前に注意を引き戻す表現**:
```
Now let me explain how we went about investigating this ...
Let me walk you through our methodology ...
```

**結果セクションに入るシグナル**:
```
So what did we find? / So, moving on to our results ...
```

**結論へのシグナル**:
```
In conclusion ... / To sum up ... / I'd like to leave you with ...
```

## 7. So What? の三層化

研究発表の「So What?」（だから何なのか？）は、一層で語らず三層で構造化すると説得力が増す。

### 7.1 三層の構造

| 層 | 内容 | 問い |
|----|------|------|
| **学術的意義** | 既存研究との関係、知識体系への貢献 | この分野の理解がどう変わるか？ |
| **実用的意義** | 技術・産業・医療などへの応用可能性 | 誰が、どのように使えるか？ |
| **社会的意義** | 社会課題・政策・人々の生活への影響 | 社会全体にとって何が変わるか？ |

### 7.2 三層を使った表現例

```
学術: "To the best of our knowledge, this is the first demonstration that ..."
実用: "This could potentially be applied to [industry/medical field] to ..."
社会: "If widely adopted, this would mean that [social impact] ..."
```

### 7.3 スコープの選択

すべての層を語る必要はない。研究の性質と聴衆に応じて強調する層を選ぶ。

| 研究タイプ | 強調すべき層 |
|-----------|------------|
| 基礎研究 | 学術的意義（必須）+ 将来の実用的意義（任意） |
| 応用研究 | 実用的意義（必須）+ 社会的意義（推奨） |
| 社会科学 | 社会的意義（必須）+ 実用的意義（推奨） |

## Quick Reference: 研究発表構成チェックリスト

### オープニング（Ch6）
- [ ] 10通りのうち1つ以上のパターンを意識して設計した
- [ ] 最初の90秒でオーディエンスの好奇心を刺激できる
- [ ] 謝罪・自己卑下から始めていない
- [ ] クロージングで回収できる「仕込み」がある

### 材料と方法（Ch8）
- [ ] 例→技術説明の順序を意識した
- [ ] 不要なステップを省略し、その理由を一言伝えた
- [ ] 方程式・公式は最小限にした
- [ ] 能動態を基本として、ストーリー的に語った

### 結果と考察（Ch9）
- [ ] 重要な結果に絞り（10分なら2枚程度のスライド）
- [ ] グラフの意味・解釈を語った（軸の説明だけでない）
- [ ] ヘッジング表現を使って断定を避けた
- [ ] 否定的結果も正直に提示した

### 結論（Ch10）
- [ ] "In conclusion" など明確な終わりのシグナルを使った
- [ ] 5通りのクロージングから1つ以上選んだ
- [ ] 最後のスライドに連絡先 + メッセージを入れた
- [ ] So What? の三層（学術・実用・社会）を意識した
