# 提案書構成術ガイド

論理と共感を統合し、相手の意思決定を引き出す提案書設計の実践リファレンス。

---

## 1. パワポを開く前の10ステップ

> **いきなりパワポは禁物。**
> 構成が完成してから初めてスライドを作る。

### フェーズ1: 前提設計（Step 1-2）

```
Step 1: プレゼンのゴール明確化
  └── 「この資料を見た後、相手に何をしてほしいか」を1文で定義
      NG: 「提案を理解してほしい」（曖昧）
      OK: 「2週間以内に概算予算を承認してほしい」（行動レベル）
      ⚠️ 段階的ゴール: 最初から「成約」を狙わず
         「次回ミーティングの設定」など実現可能なゴールから始める

Step 2: 相手の分析
  └── 3側面を事前に把握する（次セクション詳述）
```

### フェーズ2: 課題解決設計（Step 3-6）

```
Step 3: 現状・あるべき姿・ギャップの設定
  └── 相手が「理想とする未来」との差分を明確化
      現状 ──→ [ギャップ] ──→ あるべき姿
      ※ギャップが小さければ提案の必要性がない

Step 4: ギャップの深掘り
  └── 「なぜ？」を3回問い、論点（課題の観点）を設定
      MECE原則で切り口を選ぶ（後述参照）

Step 5: 課題の特定
  └── ロジックツリーで初期仮説を検証
      根本原因（真因）を特定し、表面課題と区別する

Step 6: 解決策の導出
  └── 特定した課題に対応する具体的な解決策を導出
      「その課題にその解決策はなぜ効くのか」を論理で示す
```

### フェーズ3: 構成設計（Step 7-10）

```
Step 7: 構成の組み立て
  └── 課題→解決策→結論の基本フローを構築
      論理と共感の2軸で設計（後述「論理×共感構成」参照）

Step 8: メッセージ練成
  └── 提案の要点を1〜2文で要約
      「相手は何を得られるのか」が明示されているか確認

Step 9: 共感つかみの設計
  └── イントロダクション（冒頭）とエンディング（終盤）を設計
      感情的な接続点を作る（詳細は後述）

Step 10: ページ割りの決定
  └── 各セクションに必要なスライド枚数を配分
      本編は10枚前後が集中力の限界
      詳細・補足は付録に移動
```

## 2. 相手分析の3側面

プレゼン前に「1人の具体的な相手像」を設定し、この3軸で分析する。

### 2.1 知識レベル

```
高い（専門家）  → 専門用語可、詳細根拠を示す、前提説明を省略
中程度       → 用語の初出時に説明を付与、ケーススタディ多用
低い（初学者） → 平易な言葉、アナロジーや図解中心、「なぜ自分に関係するか」を明示
```

### 2.2 立ち位置

| 立ち位置 | 心理状態 | 対応戦略 |
|--------|---------|--------|
| **賛成** | すでに共感している | 背中を押す根拠と具体的なネクストアクションを提供 |
| **中立** | 興味はあるが決断していない | 課題認識を深め、リスクと効果を明確に比較 |
| **反対** | 懐疑的または利害が相反する | 反対意見を先回りして言語化し、正面から回答する |

### 2.3 関心

```
相手が最も気にしているKPI・課題・期待を特定する。
  例:
  - 経営層 → ROI・リスク・スピード
  - 現場担当者 → 工数・使いやすさ・既存業務への影響
  - 財務部門 → 予算・コスト削減・TCO

相手の関心に合わせてメッセージの切り口を変える。
```

## 3. ロジックツリーによる課題発掘

提案書の説得力は「課題の深さ」で決まる。表面課題に留まらず、真因まで掘り下げる。

```
問い: 「なぜ売上が伸びていないのか」
  │
  ├── 論点A: 新規顧客が取れていないのか？
  │     ├── 仮説A1: 認知度不足
  │     └── 仮説A2: 競合に価格負け
  │
  ├── 論点B: 既存顧客が離脱しているのか？
  │     ├── 仮説B1: 品質への不満
  │     └── 仮説B2: アフターサポート不足
  │
  └── 論点C: 客単価が下がっているのか？
        ├── 仮説C1: 安い商品への誘導
        └── 仮説C2: 値引き交渉に負けている
```

**検証プロセス**:
1. 初期仮説を立てる（ロジックツリーで分解）
2. データ・ヒアリングで仮説を検証
3. 真因に絞り込み、解決策を紐付ける

## 4. MECE原則のスライド適用

### 4.1 MECE の定義

> **M**utually **E**xclusive and **C**ollectively **E**xhaustive
> 「ダブリなく、モレなく」

提案書の課題・解決策・比較軸がMECEであると、聴衆に「漏れなく検討した」という信頼感が生まれる。

### 4.2 失敗3パターン

| パターン | 症状 | 修正方法 |
|--------|------|---------|
| **ダブり** | 「コスト削減」と「費用最適化」が別軸として並列 | 上位概念に統合するか、どちらかを削除 |
| **モレ** | 「人・モノ・金」で整理したが「情報」が抜けている | 切り口を見直し、網羅性を確認 |
| **粒度違い** | 「マーケティング戦略」と「SNS投稿の頻度」が同列 | 抽象度を揃えるか、階層構造に再整理 |

### 4.3 MECE の切り口4種

```
1. 層別: 対象を属性で区切る
   例: 顧客を「年齢層別」「地域別」「購買頻度別」に分ける

2. 因数分解: 全体を構成要素に分解する
   例: 売上 = 顧客数 × 購買頻度 × 客単価

3. 時系列: プロセスを時間軸で分割する
   例: 認知→検討→購買→継続の購買ファネル

4. フレームワーク活用: 既存のMECEな枠組みを借用する
   例: 3C（顧客・競合・自社）、SWOT、5Forces
   ⚠️ フレームワークに無理に当てはめず、必要な軸だけ使う
```

## 5. 「論理×共感」構成

論理だけでは納得しない。共感が論理を受け入れる「土台」を作る。

```
       ┌─────────────────────────────────────┐
       │         イントロダクション             │
       │     （相手の現状・課題への共感）         │
       └──────────────┬──────────────────────┘
                      │
       ┌──────────────▼──────────────────────┐
       │         論理の本体（本編）              │
       │  課題 → 解決策 → 根拠 → 結論           │
       └──────────────┬──────────────────────┘
                      │
       ┌──────────────▼──────────────────────┐
       │            エンディング                │
       │       （想い・未来像・行動促進）          │
       └─────────────────────────────────────┘
```

**ポイント**: 論理部分でどれだけ正確であっても、冒頭と終盤の「感情の接続」がなければ行動につながらない。

## 6. イントロダクションの作り方

**目的**: 提案を受け入れてもらうための「信頼の土台」を最初の30秒で構築する。

```
効果的なイントロの3要素:

1. 相手の現状を言語化する
   → 「現在、〇〇という状況ではないでしょうか」
   → 相手が「そうそう、その通り」と思える正確な現状描写

2. 課題への認識を示す
   → 「その結果、△△という問題が生じていると理解しています」
   → 「私たちはあなたの課題を理解している」という姿勢

3. 相手の立場への寄り添い
   → 「この課題を解決することで、□□を実現できると考えました」
   → ゴールのベクトルを相手と揃える
```

**NG例**: いきなり「本日は弊社のサービスについてご紹介します」
**OK例**: 「御社では現在〇〇というご状況と伺いました。このような課題を感じていらっしゃるのではと...」

## 7. エンディングの作り方

**目的**: 論理で積み上げた説得を「行動」に転換する最後のひと押し。

```
効果的なエンディングの3要素:

1. 想いを伝える
   → なぜこの提案をしたいのか、提案者としての動機・情熱を伝える
   → 論理の後の「人間的なつながり」が信頼感を高める

2. 未来像を描く
   → 「この提案が実現すると、〇〇というような未来が待っています」
   → 具体的で魅力的なビジョンを提示する

3. 行動を促す
   → 「本日お願いしたいのは〇〇のご承認です」と明確に
   → ネクストアクションを具体的に1つだけ提示する
```

## 8. 提案書の構成要素

提案規模に応じて必要なパートを選択する。

| # | セクション | 役割 | 含む内容 |
|---|----------|------|---------|
| 1 | **表紙** | 第一印象・提案タイトル | 提案名・提案先・日付 |
| 2 | **目次** | ストーリー全体の可視化 | セクション名・ページ番号 |
| 3 | **概要（エグゼクティブサマリー）** | 忙しい意思決定者向けの要約 | 課題・解決策・期待効果を1枚に集約 |
| 4 | **提案の背景** | 課題認識の共有 | 現状・あるべき姿・ギャップ |
| 5 | **課題の全体像** | 複数課題の俯瞰 | 課題マップ・優先度 |
| 6 | **各課題の詳細** | 根本原因の説明 | ロジックツリー・データ根拠 |
| 7 | **解決策の全体像** | 解決策の全体構造 | ソリューションマップ |
| 8 | **各解決策と施策** | 具体的な実行計画 | 何を・いつ・誰が・どのように |
| 9 | **費用・スケジュール・リスク** | 意思決定に必要な情報 | 概算費用・ROI・実施タイムライン |
| 10 | **まとめと想い** | 全体の総括＋感情的つながり | キーメッセージ・提案者の想い |
| 11 | **最後のひと押し** | 行動促進 | 具体的なネクストアクション依頼 |
| 12 | **参考資料** | 詳細データ・補足情報 | 本編で省略した詳細、導入事例 |

## 9. メッセージ定着化

聴衆がプレゼン後も提案を記憶し、行動するためのキーメッセージ戦略。

### 3回繰り返しの法則

```
1回目: 表紙・概要スライド
  → キーメッセージを最初に宣言する

2回目: 本編のまとめスライド
  → 本編を通じてキーメッセージを証明した後に再提示

3回目: エンディング・まとめ
  → 行動促進と一体化して最終強調
```

**重要**: 言い回しを変えない。「コスト削減」と「費用最適化」を混用すると記憶が分散する。同一表現で3回繰り返す。

### キーワードの一貫性

```
構成全体で「同じキーワード」を使い続けることで、
聴衆の記憶に「錨（アンカー）」を作る。

例: キーワードを「スピード」に設定したなら、
    表紙・各スライドタイトル・まとめの全てで「スピード」を使う
```

## 10. 段階的ゴール設定

一度の提案で「成約」を求めすぎると、相手の防衛本能が働く。

```
段階的なゴール設計の考え方:

Step A（初回接触）: 課題認識の共有
  → ゴール: 「課題を一緒に整理させてください」

Step B（ヒアリング後）: 解決策の方向性合意
  → ゴール: 「この方向性でよいか確認させてください」

Step C（提案書提出）: 概算承認
  → ゴール: 「詳細見積もりに進む承認をいただきたい」

Step D（正式提案）: 最終意思決定
  → ゴール: 「契約・発注の決定」
```

**なぜ段階的か**: 信頼は時間をかけて構築される。高すぎるゴールはNOを引き出すだけでなく、関係性自体を壊すリスクがある。

## チェックリスト

### 事前設計（パワポを開く前）
- [ ] 「相手に取ってほしい行動」が1文で定義されているか
- [ ] 相手の知識レベル・立ち位置・関心を分析したか
- [ ] 現状・あるべき姿・ギャップが明確か
- [ ] 「なぜ？」を3回問い、根本課題を特定したか
- [ ] 解決策と課題の論理的なつながりが説明できるか
- [ ] ゴールは「段階的に達成可能」な水準に設定されているか

### 構成・ロジック
- [ ] 課題→解決策→結論の論理フローが成立しているか
- [ ] 各セクションの並列要素はMECEか（ダブり・モレ・粒度違いがないか）
- [ ] 切り口（層別/因数分解/時系列/フレームワーク）は適切か
- [ ] キーメッセージは3回（表紙・本編まとめ・エンディング）繰り返されているか
- [ ] 同一キーワードで統一されているか（言い回しのブレがないか）

### 共感・感情設計
- [ ] イントロで相手の現状・課題への理解を示しているか
- [ ] エンディングに「想い・未来像・行動促進」の3要素があるか
- [ ] ネクストアクションは1つに絞って明確に伝えているか

### スライド構成
- [ ] 本編は10枚前後に収まっているか
- [ ] 詳細・補足は参考資料に移動したか
- [ ] 概要（エグゼクティブサマリー）が単独で全体を把握できるか
- [ ] タイトルを見るだけで各スライドの主張が分かるか
