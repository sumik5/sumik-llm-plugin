# スライド・資料デザイン原則リファレンス

資料の認知負荷を下げ、ミスを防ぎ、メッセージを正確に伝えるための設計原則。

---

## 1. 2つの基本原則

### 1.1 リスクフリー原則（ミスを防ぐ）

資料はプレゼンターの「第二の記憶」であり、誤解・ミスを防ぐ安全網でもある。

**リスクフリーの3要素**

| 要素 | 内容 | チェック |
|------|------|---------|
| **データの正確性** | 数値・固有名詞・日付の二重確認 | 別の人がレビューしたか |
| **曖昧性の排除** | 受け取り手によって解釈が変わらないか | 「なんとなく多い」ではなく「35%増」 |
| **一貫性の確保** | 用語・表記・フォーマットの統一 | 「売上」と「Revenue」が混在していないか |

**よくあるリスク箇所**

- 前スライドと矛盾するデータ
- 「昨年比増加」など比較基準が曖昧な表現
- 参照元が明記されていないグラフ・数値
- 複数の担当者が作成した資料の継ぎ目部分

### 1.2 ストレスフリー原則（読みやすさの最適化）

聴衆の認知資源は有限。資料を「読む」コストを下げることで、メッセージへの集中を促す。

**認知負荷を下げる5つの施策**

```
1. 情報密度の最適化
   → 1スライドに収める情報は「3点以内」を目安

2. 視覚的ノイズの除去
   → 不要な枠線・グラデーション・装飾を削除

3. 視線の流れの設計
   → 左上→右下のZ字型 or 上→下のF字型を基本に

4. コントラストの確保
   → テキストと背景の明暗差を十分に（目安: 4.5:1以上）

5. 余白の積極的活用
   → 詰め込みは読む意欲を削ぐ。余白は「贅沢」ではなく「機能」
```

## 2. 情報密度の最適化

### 1スライド1メッセージ

各スライドは「1つの主張」を持ち、タイトルにその主張を書く。

**悪い例**:
```
スライドタイトル: 「売上・コスト・利益の状況」
（3つの話題が混在）
```

**良い例**:
```
スライドタイトル: 「Q3売上は計画比115%—コスト削減が貢献」
（1つの主張 + 理由）
```

### 情報量の基準

| スライドタイプ | テキスト量の目安 | 図表の扱い |
|-------------|---------------|----------|
| タイトル / サマリー | 1文（キーメッセージのみ） | 不要 |
| データ提示 | 見出し + 3点以内の箇条書き | グラフ中心 |
| 比較 / 選択肢 | 評価軸を統一した表 | マトリクス可 |
| アクション促進 | 3ステップ以内 | 矢印・フロー |
| 詳細・補足 | 付録に移動 | 参照先を明示 |

## 3. 視覚的階層の設計

### 3層構造の基本

```
Layer 1: 見出し（最大の注意を引く）
  ├── フォントサイズ: 最大
  ├── 内容: キーメッセージ（動詞を含む1文）
  └── 役割: 「このスライドで何を言うか」を一瞬で伝える

Layer 2: 本文（情報を伝える）
  ├── フォントサイズ: 中
  ├── 内容: 根拠・データ・具体例（3点以内）
  └── 役割: 見出しの主張を支持する

Layer 3: 補足（詳しい人向け）
  ├── フォントサイズ: 最小
  ├── 内容: 出典・条件・注記
  └── 役割: 信頼性の補強（読まれなくても成立する）
```

### フォント選択の原則

- **プレゼン用**: サンセリフ系（読みやすさ優先）
- **印刷資料**: セリフ系も可（長文での読みやすさ）
- **フォントの種類**: スライド全体で2種類まで
- **サイズの最小値**: プロジェクター投影時は28pt以上を推奨

## 4. 余白とシンプル化の原則

### 「引き算」の思考

資料作成は「加える」より「削る」作業。

```
初期ドラフト → 「このスライドに本当に必要か？」でフィルタ
         ↓
不要な要素を削除（一度非表示にして影響を確認）
         ↓
「削除できるものが残っていない状態」が完成
```

### 削除候補の一覧

| 要素 | 判断基準 |
|------|---------|
| 会社ロゴ（全スライド掲載） | 表紙とラストのみに限定 |
| ページ番号 | 参照が必要な時のみ |
| 枠線・罫線 | 情報の区切りが自明なら不要 |
| 3色以上の色使い | 2色（メイン+アクセント）に絞る |
| 過去のバージョン情報 | 付録に移動 |
| 「〜について」というタイトル | キーメッセージを書く見出しに変換 |

### 余白の機能的活用

余白は「無駄なスペース」ではなく、次の機能を果たす：

1. **注目誘導**: 余白で囲まれた要素は強調される
2. **グルーピング**: 余白の差で関連性を表現（ゲシュタルト）
3. **呼吸感**: 聴衆が情報を処理する時間を提供

## 5. グラフ・図表の効果的活用

### グラフの種類と選択基準

| 見せたいこと | 推奨グラフ | 避けるべきグラフ |
|------------|----------|--------------|
| 時間経過による変化 | 折れ線グラフ | 円グラフ |
| カテゴリ間の比較 | 棒グラフ（縦/横） | 3D棒グラフ |
| 全体に対する割合 | 円グラフ / 積み上げ棒 | 複数の円グラフ |
| 2変数の相関 | 散布図 | 折れ線グラフ |
| プロセスの流れ | フロー図 | テキストのみ |
| 数値の大小比較 | 横棒グラフ | 縦棒グラフ（項目名が長い場合） |

### グラフデザインの5原則

1. **タイトルに解釈を書く**
   NG: 「売上推移（2020-2024）」
   OK: 「5年連続増収—特にデジタル領域が牽引」

2. **色は意味を持たせる**
   NG: 棒グラフに7色使用
   OK: 強調したいデータだけアクセントカラー、残りはグレー

3. **データラベルを直接記載**
   凡例と照合させるより、棒・線の近くに数値を配置

4. **グリッド線は最小限**
   主要な補助線のみ、色は薄いグレー

5. **縦軸は0から始める**
   0からでない棒グラフは変化を誇張して見せる（意図的な場合は注記）

## 6. 情報の視覚化テクニック

### テキストを図に変換する判断基準

| テキスト形式が向く場合 | 図形式が向く場合 |
|---------------------|----------------|
| 順序が重要でない3点以内の箇条書き | プロセス・手順（矢印でフロー化） |
| 強調すべき引用文 | 比較・選択肢（マトリクス化） |
| 定義・用語説明 | 階層関係（ツリー図） |
| 単一の重要数値 | 量の比較（グラフ化） |

### 「1枚100万円品質」の基準

資料の各ページが単独で意味を成すか確認する。

```
チェック: このスライドを切り取って誰かに送ったとき、
　　　　　説明なしでメッセージが伝わるか？

YES → 自立したスライド（高品質）
NO  → 追加情報が必要 → タイトル or 本文を改善
```

## 7. スライドタイプ別デザインガイド

### 表紙スライド

- 必須要素: タイトル、日付、発表者名
- 任意要素: サブタイトル（あれば）、ロゴ
- 避けるべき: 過剰な装飾、背景の複雑な画像

### サマリー / エグゼクティブサマリー

- 全体の結論を3点以内で箇条書き
- 忙しい意思決定者がここだけ読んでも全体を把握できる
- 各箇条書きに参照スライド番号を付記（例: → p.8）

### データスライド

- タイトル: データが示す「解釈」を書く
- 本文: グラフ + 最重要数値を強調テキストで添える
- 脚注: 出典・調査期間・n数

### アクションスライド（最終スライド）

- 「今日の依頼・次のステップ」を明示
- 期限・担当者・必要リソースを具体的に
- 「ご清聴ありがとうございました」のみのスライドは避ける

## 8. 数字の戦略的見せ方

数字は「読ませる」のではなく「見せる」。同じ数値でも提示方法によって伝わり方が大きく変わる。

### 4つの活用法

| 活用法 | 目的 | 実践のポイント |
|--------|------|--------------|
| **①比較** | 差を際立たせる | 単独の数値ではなく「対比」を見せる（「前年比3倍」「業界1位」） |
| **②傾向** | トレンドを示す | 時系列のデータで「動き」を見せる。折れ線・棒グラフで方向性を強調 |
| **③強調** | メッセージを際立たせる | キーナンバーをシンプルに大きく表示。5・6・7など少数の数字に絞る |
| **④ビジョン導出** | 未来を示す | 現在の傾向から「このまま進むとどうなるか」を推計・提示する |

**判断基準**: このスライドの数字は何を「言いたい」のか？→ 言いたいことに合った活用法を選ぶ。

## 9. ストーリーボード7種スライド

プレゼン資料を作る前に、7種類のスライドで**手書きストーリーボード**を作成する。構成を先に固めることで、実際の作成時間が大幅に短縮できる。

### 7種類の構成要素

| # | スライドタイプ | 役割 | 使い分け |
|---|-------------|------|---------|
| ① | **表紙** | タイトル・発表者・日付 | 必須。シンプルに |
| ② | **まとめ（冒頭）** | 全体のサマリー・目次 | 聴衆がゴールを把握できる |
| ③ | **ディバイダ** | セクション区切り | 本編が多い場合に挿入 |
| ④ | **本編** | 主張・根拠・データ | メインコンテンツ。10枚以内を目安 |
| ⑤ | **データ** | グラフ・数値の提示 | タイトルに解釈を書く |
| ⑥ | **図解** | 関係・プロセス・比較の可視化 | 4パターンから選択（並列/順序/整理/手順） |
| ⑦ | **まとめ（末尾）** | 結論・次のステップ・依頼事項 | アクションスライドとして機能させる |

**ポイント**: ①②はストーリーボード段階では省略可。③ディバイダも枚数次第で省略でよい。

### ストーリーボード活用の深化（菅野誠二）

7種類のスライドを単なる「種類の分類」ではなく、**構成設計ツール**として活用するための補強ポイント。

**本編10枚以内の徹底**

| 発表時間 | 本編スライド目安 | 1スライドあたりの時間 |
|---------|--------------|-------------------|
| 5分 | 3〜5枚 | 約1分 |
| 10分 | 5〜7枚 | 約1.5分 |
| 20分 | 8〜10枚 | 約2分 |
| 30分 | 10枚以内（厳守） | 約3分 |

**大ゴマ先行配置法との連携**

ストーリーボード作成時は「レンガ積み（順番に考える）」を禁止し、以下の手順で進める：

```
Step 1: ⑦まとめ（末尾）を先に決定 → What（何をすべきか）とCTA（行動促進）を明確化
Step 2: ④本編の核スライド（3〜4枚）を先に配置 → これが「大ゴマ」
Step 3: ②まとめ（冒頭）で全体サマリーを設計
Step 4: ③ディバイダと残りの④本編・⑤データ・⑥図解を間に挿入して完成
```

**7種の役割再定義**

| スライドタイプ | 聴衆から引き出したい反応 | 設計のポイント |
|-------------|-------------------|-------------|
| ①表紙 | 「今日は何を聞くのか」という期待感 | シンプルに。タイトルで主張を示す |
| ②まとめ（冒頭） | 「全体像が分かった」という安心感 | メンタルモデル先提示。3点以内に絞る |
| ③ディバイダ | 「次のセクションに進む」という区切り感 | セクション番号 + 見出しのみ |
| ④本編 | 「なるほど」「確かに」という納得感 | 1スライド1主張。核スライドを大ゴマに |
| ⑤データ | 「数字が示すことは明確だ」という確信 | タイトルに解釈（So What）を書く |
| ⑥図解 | 「関係性・プロセスが視覚的に理解できた」 | 並列/順序/整理/手順から選択 |
| ⑦まとめ（末尾） | 「次に何をすればよいか分かった」 | アクションスライドとして機能させる |

## 10. スライドに頼らないデザイン選択肢

### ホワイトボードディスカッション

スライドが固まっていない段階や、相手と共に考えたいプレゼンでは、ホワイトボードが強力な代替手段となる。

**効果的なシーン**:
- 提携・交渉のプレゼン（定型的でない、双方向が必要な場合）
- 相手の反応を見ながら柔軟に内容を変えたい場合
- 考えを「生成する」過程を相手に見せることで納得感を高めたい場合

**進め方**:
1. まず白紙の状態から「今日の目的」を口頭で宣言する
2. ホワイトボードに要点・関係性・数値を手書きで書きながら説明
3. 相手の反応に応じてリアルタイムで内容を変えていく
4. 最終的に書き留めた内容がプレゼンのアウトプットになる

### 小道具・動画の活用

| 手法 | 効果 | 活用タイミング |
|------|------|--------------|
| **実物（小道具）** | 抽象的な概念を具体化し、聴衆の記憶に刻む | 新商品・新機能の体感を伝えたい場面 |
| **動画** | コンセプトを感情面からも届ける | テキスト・グラフでは伝わらない世界観の提示 |

**注意**: 小道具・動画はメッセージを補強するもの。それ自体が主役にならないよう使用時間・タイミングを設計する。

## 11. 資料設計チェックリスト

### リスクフリー確認
- [ ] 全ての数値に出典または確認プロセスがある
- [ ] 用語・表記が全スライドで統一されている
- [ ] 誰が読んでも同じ解釈になる表現を使っている

### ストレスフリー確認
- [ ] 1スライド1メッセージが守られている
- [ ] 各スライドタイトルがキーメッセージ文になっている
- [ ] 不要な装飾・枠線・色を削除した
- [ ] フォントサイズの最小値が28pt以上（投影時）

### 視覚設計確認
- [ ] グラフの種類が伝えたいことと一致している
- [ ] グラフタイトルに解釈（So What）が書かれている
- [ ] 余白が十分に確保されている
- [ ] 色は3色以内（背景色を除く）

### 数字・ストーリーボード確認
- [ ] 数字は4つの活用法（比較/傾向/強調/ビジョン）のいずれかを意識して提示している
- [ ] ストーリーボードを先に作成してから資料を作り込んだ
- [ ] 本編スライドは10枚以内（30分相当）に絞っている

### 反応設計・演出確認
- [ ] 各スライドに「聴衆から引き出したい反応」を想定して作っている
- [ ] 数字は競合比較×時系列比較の「二重比較」またはスケール変換で直感的に伝えている
- [ ] Before/Afterや対比表現に60の対極軸（存在/変化/量/空間/道徳等）を活用した
- [ ] HTMLスライドでソニック・ブランディング（サウンドエフェクト等）を検討した

### 接続・ナビゲーション・リズム確認
- [ ] スライド間接続ブリッジ（前スライドの図やキーワードを縮小配置、`.bridge-ref`）を実装しているか
- [ ] 全体位置インジケータ（`.progress-indicator`）が固定位置に表示されているか
- [ ] 情報密度の高いスライド3枚ごとに呼吸スライド（`.slide-breath`）を挿入しているか
- [ ] もくじスライド（`.slide-toc`）を節目ごとに反復挿入しているか
- [ ] スライド番号は「7/16」形式（現在位置/総枚数）か

## 12. スライドは「反応を引き出す装置」

スライドの役割を「説明補助」から「聴衆から特定の反応を引き出す装置」に再定義する。

### 設計の思考プロセス

```
各スライドについて「聴衆はこれを見てどんな反応をするか」を想定してから作る

想定反応の例:
- 「あ、それ自分も悩んでた！」（共感）
- 「そんな数字なのか…」（驚き）
- 「なるほど、これは確かに」（納得）
- 「じゃあ自分はどうすれば？」（行動意欲）
```

**応用**:

| 要素 | 判断基準 |
|------|---------|
| 文字量 | 想定反応を得られる最小限の量か |
| 画像・図 | 反応を増幅させているか、単なる装飾か |
| フォントサイズ | 反応を生む「読みやすさ」を確保しているか |
| アジェンダスライド | 演出として機能しているか（不要なら省略） |

## 13. 数字の「二重比較」技法

同一スライドで「競合他社との比較」×「過去との時系列比較」を組み合わせ、自社の優位性を多角的に示す。

### 二重比較のパターン

| 比較軸 | 内容 | 効果 |
|-------|------|------|
| **横断比較** | 競合他社との現時点の数値差 | 「今、自社の方が優れている」を示す |
| **時系列比較** | 自社の過去→現在→予測 | 「さらに伸びている」を示す |
| **二重比較** | 競合は縮小均衡、自社は拡大均衡を同時提示 | 二軸で優位性を確定させる |

### 数字のスケール変換

大きすぎる数字は身近な単位に変換して直感的理解を促す。

**例**:
- 「3000兆個のトランジスタ」→「3万年分の映画を保存可能」
- 「1億人のユーザー」→「日本の人口と同じ規模」

**目安**: 75%のスライドに何らかの数字の裏付けを入れる（説得力の担保）。

## 14. ギャップ設計の体系化（60の対極軸）

Before/Afterや問題提起スライドのバリエーションを豊かにするための対極軸体系。

### 普遍的5軸

| 軸 | 対極の例 | 活用場面 |
|----|---------|---------|
| **存在軸** | 有 ↔ 無 | 「〜がない時代」vs「〜がある今」 |
| **変化軸** | 変化有 ↔ 変化なし | 「動いた企業」vs「止まった企業」 |
| **量軸** | 多 ↔ 少 | 「コストが高い状態」vs「削減後」 |
| **空間軸** | 広 ↔ 狭、近 ↔ 遠 | 「グローバル展開」vs「国内限定」 |
| **道徳軸** | 正しい ↔ 誤り、安全 ↔ 危険 | リスク訴求、コンプライアンス提案 |

### X→Y→Z公式

```
X（課題のある現状） → Y（好ましい未来） → Z（解決メッセージ）
```

この公式を軸選択と組み合わせることで、あらゆる業界・テーマのBefore/Afterスライドを体系的に設計できる。

## 15. ソニック・ブランディングのプレゼン応用

音は視覚より速く脳に届く。HTMLスライドでは積極的に活用できる。

### 脳の反応速度

| 感覚 | 処理速度 |
|------|---------|
| **聴覚** | 0.05秒（最速） |
| **触覚** | 0.1秒 |
| **視覚** | 0.2秒（最遅） |

聴覚は視覚の4倍速く処理されるため、短い音が強い印象を残す。

### 設計パターン

| タイミング | 効果 | 例 |
|-----------|------|-----|
| **オープニング** | ブランド・世界観の確立 | Netflix「タ・ダム」のような特徴的な音 |
| **重要スライド切り替え** | 注意の引き締め | コイン音・ベル音 |
| **クロージング** | 余韻・記憶定着 | フェードアウトする音楽 |

### HTMLスライドでの実装

```html
<!-- オープニング効果音の例 -->
<audio id="opening-sound" src="sounds/opening.mp3" preload="auto"></audio>
<script>
  // 最初のスライド表示時に再生
  document.addEventListener('DOMContentLoaded', () => {
    document.getElementById('opening-sound').play();
  });
</script>
```

**注意**: 音は「メッセージを補強する脇役」に徹すること。音が主役になるとプレゼンの集中を妨げる。

## 16. スライド間の接続設計

スライド間の「論理的なつながり」を視覚化することで、聴衆が話の流れを追いやすくなる。（出典: ロジック構築の技術）

### 16.1 スライド間接続ブリッジ

前スライドの図やキーワードを縮小して次スライドに配置し、因果関係・論理の継続性を視覚的に示す技法。

**設計の原理**

```
[スライドA] → 大きく表示: 「戦略X」の全体図
       ↓
[スライドB] → 左上に縮小配置: 「戦略X」の全体図（グレーアウト）
              + 右側に大きく: 「戦略Xの要素①：コスト削減」の詳細
```

**効果**: 聴衆が「今どこを説明しているのか」を見失わない。特に複雑な因果関係チェーンの説明に有効。

**使い分け**

| 場面 | 接続ブリッジの必要性 |
|------|------------------|
| 概念の詳細展開（Overview → Detail） | 必須 |
| データの文脈説明 | 推奨 |
| 独立したトピック間の遷移 | 不要（ディバイダで区切る） |
| 時系列の流れの説明 | 推奨 |

### 16.2 全体位置インジケータ

横並びロジック単位（例: 3要素の比較、4ステップのプロセス）を個別に説明する際、全体図を固定位置に表示し、現在説明中の要素をハイライトする。

**設計パターン**

```
┌─────────────────────────────────────────────┐
│  現在の説明: ステップ② の詳細               │
├──────────────────────────────────────────────┤
│                        ┌──────────────────┐  │
│   メインコンテンツ      │ ①→[②]→③→④   │  │  ← 全体位置インジケータ（固定）
│                        └──────────────────┘  │     ② をハイライト
└─────────────────────────────────────────────┘
```

**配置の推奨位置**: 右上または右下（スライドの主要コンテンツと重ならない場所）

### HTMLスライドでの実装

```html
<!-- 接続ブリッジ: 前スライドの図を縮小配置 -->
<div class="bridge-ref">
  <img src="prev-slide-diagram.svg" alt="前スライドの概念図">
  <span class="bridge-label">前スライドより</span>
</div>

<!-- 全体位置インジケータ: ステップ② をハイライト -->
<div class="progress-indicator">
  <span class="step">①</span>
  <span class="step active">②</span>
  <span class="step">③</span>
  <span class="step">④</span>
</div>
```

CSS クラス参照:
- `.bridge-ref` — 接続ブリッジのコンテナ（前スライドの図を縮小配置）
- `.progress-indicator` — 全体位置インジケータのコンテナ
- `.progress-indicator .active` — 現在説明中の要素のハイライト

## 17. 情報リズムの設計

プレゼンは「音楽」のようにリズムがある。高密度のスライドが続くと聴衆の集中は続かない。「間」と「強調」を意図的に設計する。（出典: プレゼンの極意はマンガに学べ）

### 17.1 省略のメディアとしての呼吸スライド

マンガにおける「コマとコマの間（ガター）」と同様に、情報の「空白」は聴衆が意味を処理するための不可欠な時間を提供する。

**ルール**: 情報密度の高いスライド3枚ごとに、1枚の「呼吸スライド」を挿入する。

**呼吸スライドの要件**

| 要素 | 内容 |
|------|------|
| テキスト量 | 1メッセージのみ（または画像のみ） |
| フォントサイズ | 大きく（画面の30〜50%を占める） |
| 装飾 | 最小限（余白を最大化） |
| 役割 | セクションの「まとめ」または「次への橋渡し」 |

**構成パターン例**

```
[データスライド1] → [データスライド2] → [比較スライド3]
                                            ↓
                              [呼吸スライド: 「つまり、Xが最重要」]
                                            ↓
                   [図解スライド4] → [本編スライド5] → [データスライド6]
```

### 17.2 決めゼリフ・クローズアップ・比喩の視覚化

マンガの「決めゼリフコマ」「クローズアップ」「比喩の絵化」をプレゼンに応用する3技法。

**技法①: 決めゼリフの配置**

各セクションに「核フレーズ（決めゼリフ）」を1つ配置する。このフレーズが聴衆の記憶に残るキーメッセージとなる。

```
例: 「私たちの競争優位性は、たった1つの数字に集約される。それが—」
    → 次スライドで「37%」を大きく表示（クローズアップ）
```

**技法②: クローズアップ（フォント極大化）**

重要ポイントで、数値やキーワードを画面いっぱいに大きく表示する。聴衆の注意を強制的に引き付ける。

| 要素 | 通常スライド | クローズアップスライド |
|------|------------|-------------------|
| フォントサイズ | 32〜48pt | 100〜200pt |
| テキスト量 | 見出し+本文 | 1語〜1文のみ |
| 目的 | 情報伝達 | インパクト・記憶定着 |

**技法③: 比喩の視覚化**

抽象的な比喩をアイコンやイラストで視覚化し、メッセージを直感的に伝える。

```
「私たちのサービスは、お客様の課題を解決する『羅針盤』です」
→ 羅針盤のアイコン + ブランドカラーで視覚化
```

### HTMLスライドでの実装

```html
<!-- 呼吸スライド -->
<section class="slide slide-breath">
  <p class="breath-message">つまり、コスト削減が鍵だ。</p>
</section>

<!-- クローズアップスライド（決めゼリフ + 大数値） -->
<section class="slide slide-breath">
  <div class="closeup-number">37%</div>
  <p class="closeup-label">コスト削減率（業界平均の3倍）</p>
</section>
```

CSS クラス参照:
- `.slide-breath` — 呼吸スライドのコンテナ（1メッセージ/画像のみ、余白最大化）

## 18. ナビゲーション設計

聴衆はプレゼン中に「自分が今どこにいるのか」を常に確認したい。ナビゲーション設計は、その「位置感覚」を提供し、情報の消化を助ける。（出典: 浅田すぐる）

### 18.1 もくじスライドの反復挿入

冒頭に一度だけ表示するのではなく、**節目ごとに繰り返し挿入**することで、聴衆が全体構造の中での位置を把握できるようにする。

**挿入タイミング**

| タイミング | 役割 |
|----------|------|
| **冒頭** | 全体像の提示（メンタルモデル形成） |
| **各セクション開始直前** | 「これからここを説明する」という予告 |
| **各セクション終了直後**（任意） | 「ここまで説明した」という確認 |
| **末尾** | 全体の振り返り |

**現在位置の示し方**

```
冒頭のもくじ:                    セクション2開始時のもくじ:
┌─────────────────┐              ┌─────────────────┐
│ 1. 現状分析     │ ← 通常       │ 1. 現状分析     │ ← グレーアウト（済）
│ 2. 課題の特定   │              │ 2. 課題の特定   │ ← ハイライト（今ここ）
│ 3. 解決策の提案  │              │ 3. 解決策の提案  │ ← 通常（これから）
│ 4. アクション   │              │ 4. アクション   │ ← 通常
└─────────────────┘              └─────────────────┘
```

### 18.2 スライド番号の形式

スライド番号は「現在ページ数のみ」ではなく、**「7/16」形式（現在位置/総枚数）** で表示する。

**効果**:
- 聴衆が「残りどれくらいか」を把握できる
- 発表者も時間管理がしやすい
- 「あと少し」という心理的安心感を与える

### HTMLスライドでの実装

```html
<!-- もくじスライド（セクション2開始時に挿入） -->
<section class="slide slide-toc">
  <h2 class="toc-title">目次</h2>
  <ol class="toc-list">
    <li class="toc-done">1. 現状分析</li>
    <li class="toc-current">2. 課題の特定</li>  <!-- 現在位置 -->
    <li>3. 解決策の提案</li>
    <li>4. アクション</li>
  </ol>
</section>

<!-- スライド番号: 7/16 形式 -->
<div class="slide-counter">
  <span class="current">7</span>/<span class="total">16</span>
</div>
```

CSS クラス参照:
- `.slide-toc` — もくじスライドのコンテナ
- `.slide-toc .toc-current` — 現在位置のハイライト
- `.slide-toc .toc-done` — 完了済みセクションのグレーアウト

## 19. 写真スライドの高度設計

インパクトのある写真スライドは「全面写真 + 透過オーバーレイ + テキスト」の3層構造で設計する。（出典: 浅田すぐる）

### 19.1 3層構成の原理

```
Layer 1（最背面）: 全面写真
  → 感情的インパクト。メッセージと強く関連する写真を選ぶ

Layer 2（中間）: 透過カラーオーバーレイ
  → 写真のコントラストを調整しテキストを読みやすくする
  → 透明度: 40〜60%が目安（写真が透けて見える程度）
  → 色: ブランドカラーまたはメッセージの感情に合った色

Layer 3（最前面）: テキストボックス
  → 核フレーズのみ（1〜2文）
  → 中央配置が基本
  → 白または高コントラスト色で読みやすく
```

### 19.2 写真選定の原則

| 基準 | 良い選択 | 悪い選択 |
|------|---------|---------|
| **メッセージとの一致** | 「成長」→ 木の年輪、建物の建設中 | 「成長」→ グラフのアイコン（抽象的すぎる） |
| **被写体の明確さ** | 主役が一目で分かる写真 | 複雑な構図で視線が散らばる写真 |
| **テキストの配置スペース** | 中央または下部に空白がある写真 | 細部まで描写が詰まった写真 |
| **感情的トーン** | スライドの目的（訴求/緊急/希望）に合致 | 雰囲気がちぐはぐな写真 |

### 19.3 オーバーレイカラーの戦略的活用

| 感情的目的 | 推奨オーバーレイカラー | 例 |
|----------|-------------------|-----|
| 緊急・警告 | 赤系（透明度50%） | 「このまま放置すると…」スライド |
| 信頼・安定 | 青系（透明度40%） | 「私たちの実績」スライド |
| 成長・希望 | 緑系（透明度45%） | 「これからの可能性」スライド |
| プレミアム・高級 | 黒系（透明度55%） | 「エグゼクティブ向け提案」スライド |
| ブランド統一 | ブランドカラー（透明度50%） | 全体を通じた一貫性 |

### HTMLスライドでの実装

```html
<!-- 写真+透過+メッセージの3層構成 -->
<section class="slide slide-photo-overlay"
         style="background-image: url('images/growth.jpg')">
  <!-- Layer 2: 透過カラーオーバーレイ -->
  <div class="overlay" style="background: rgba(0, 82, 204, 0.5);"></div>
  <!-- Layer 3: メッセージ -->
  <div class="message">
    <p class="photo-headline">成長の鍵は、たった1つの選択にある</p>
  </div>
</section>
```

CSS クラス参照:
- `.slide-photo-overlay` — 全面写真背景のコンテナ
- `.slide-photo-overlay .overlay` — 透過カラーオーバーレイ（`background: rgba(...)` で色と透明度を指定）
- `.slide-photo-overlay .message` — 前面テキストボックス（中央配置、高コントラスト）
