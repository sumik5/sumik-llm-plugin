# 国際学会ネットワーキング実践リファレンス

学会の懇親会・1on1面談・キーパーソン戦略。発表後の人脈構築を体系的に設計する。

---

## 1. 学会の本質：発表の場ではなく宣伝とネットワーキングの場

プレゼンやポスターセッションは、研究成果を伝えるだけでなく、自分自身と研究を参加者に知ってもらう機会だ。単に聴衆として多くのセッションに参加することより、空き時間をネットワーキングに投資することが長期的なキャリアにとって価値がある。

### 学会参加で得られるもの

| 目的 | 具体的な成果 |
|------|------------|
| 知識の最前線を把握する | 旬の研究テーマ・技術動向 |
| 自分の研究へのフィードバックを得る | 外部視点からの改善案 |
| 新しいアイデアの創出 | 他者との議論から生まれる発想 |
| 旧知との再会・新しい協力体制の構築 | 長期的な研究ネットワーク |

### マインドセットの転換

- 「プレゼンが終わればOK」ではなく、懇親会・コーヒーブレイクこそ本番
- 一流の論文は自宅で読める。生身の人との対話は学会でしかできない
- **印象の記憶はコンテンツの記憶より長く残る**。発表後10日経っても相手が覚えているのは「どんな人か」が多い

## 2. 懇親会：基本設計と会話の技術

### 2.1 自己紹介：フォーマル vs. インフォーマル

英語圏の典型的な自己紹介はシンプルだ。フォーマル度に応じて切り替える。

```
インフォーマル：Hi, I'm [ファーストネーム].
フォーマル    ：Hello, I'm [ファーストネーム] [ファミリーネーム].
初対面確認    ：I don't think we've been introduced — I'm [名前].
名前の交換    ：By the way, my name is [名前].
```

**名前を聞き取れなかったとき**: 遠慮せずに「Sorry, I didn't catch your name.」と尋ねる。相手の名前を覚えようとする姿勢は好意的に受け取られる。

### 2.2 肩書きの使い方

| 状況 | 推奨する呼称 |
|------|------------|
| 相手が教授 | Professor [ファミリーネーム] |
| 相手が博士 | Dr [ファミリーネーム] |
| 「ファーストネームで」と言われたら | その時点からファーストネームへ移行 |
| 研究職以外（男性） | Mr [ファミリーネーム] |
| 研究職以外（女性） | Ms [ファミリーネーム]（既婚未婚不問） |

英語では「エンジニア」「弁護士」などの職業肩書きを相手への呼びかけに使わない。Dr / Professor でカバーできる場合はそちらを優先する。

### 2.3 会話の輪への入り方

**早く到着する**が最大のコツ。話したい人が誰かと会話を始める前に接触できる。

会話中の人に加わるときはボディランゲージを観察する。
- 体を寄せ合ってアイコンタクトが強い → しばらく様子を見る
- 適度に距離を保っている → 参加を申し出るサイン

```
加わる：Do you mind if I join you?
        I don't really know anyone here — mind if I join?

聞き入れる：Sorry, I was listening from a distance and it sounds really interesting.
```

会話の輪に入ったら自己紹介し、自分の仕事を「私は博士課程の学生です」で終わらせず、研究内容を1文で加える。詳しいほど相手の反応を引き出せる。

### 2.4 会話終了の表現

現在完了を使うと「もうすぐ席を外す」ことが自然に伝わる。

```
It's been really interesting talking to you. I'll see you around.
I've really enjoyed talking to you. Enjoy the rest of the conference.
```

## 3. 会話の話題設計

### 3.1 よく使える話題リスト

懇親会では以下のトピックが汎用的に使える。事前に関連語彙を強化しておくと会話の主導権を保ちやすい。

- 開催地・学会の運営（地元情報・おすすめスポット）
- 天気・食文化
- 印象的だったプレゼン・セッション
- 最新テクノロジー（スマートフォン・アプリ）
- 余暇・休暇の過ごし方
- 国・地域ごとの文化的な違いや類似点

### 3.2 避けるべき話題（英米圏の基準）

英米人がインフォーマルな場で不適切と感じる質問の代表例:

| 避けるべき質問 | 理由 |
|--------------|------|
| 年齢・体重 | プライバシー侵害 |
| 給与・家の価値 | 金銭情報はデリケート |
| 宗教・政治（深入り） | 摩擦になりやすい |
| 結婚の予定・子どもの予定 | 個人の選択領域 |

一方、「どこで研究しているか」「何を専攻したか」「次はどの学会に参加するか」は英米人には普通の会話と受け取られる。

### 3.3 類似点を探す戦略

相違点より類似点に焦点を当てると、良好な雰囲気を生み出せる。倫理・政治的テーマでなくても、次のような「軽い文化比較」で十分に盛り上がる。

- 法定年齢（運転免許・選挙権など）の違い
- 方言・地域差
- 家族の役割・高齢者との生活スタイル
- 余暇・休日の過ごし方
- チップの習慣

### 3.4 エピソードストックを準備する

会話のネタを1〜2本事前に用意しておくと、話題に詰まったときに役立つ。

- 旅先でのトラブル（フライトに乗り遅れた、ひどいホテルに泊まった）
- 研究室での予期しない出来事
- これまで経験した最悪または最高の学会

ファクトイド（興味を引く統計データや驚きの事実）も有効。読書や日常で出会った面白い情報をメモしておき、「Did you know that ...?」形式で使う。

## 4. プレゼン後の質問への準備（Ch16.3）

プレゼン後は懇親会でも関連質問を受けることがある。準備できる想定問答を作っておく。

### 準備すべき質問の種類

1. **研究の動機・背景** — なぜこの問題を選んだか
2. **手法の選択理由** — なぜこの実験設計か
3. **結果の解釈** — 想定外の結果があったか
4. **今後の展開** — 次のステップは何か
5. **関連研究との比較** — 他の研究と何が違うか

懇親会中の会話でも、プレゼン発表者への質問として活用できる定番フレーズ:

```
Could you give me some more details about ...?
Where can I get more information about ...?
Have you uploaded your presentation? If so, where can I find it?
I'm not sure I understood your point about ... — could you clarify?
```

## 5. キーパーソンへのアプローチ（Ch17 要約）

### 5.1 事前準備：キーパーソンを特定する

- 学会プログラムからリストアップ
- 所属機関のウェブサイトで顔写真・研究内容を確認
- 論文を読み、「どの部分に共感しているか」を言語化しておく
- LinkedInや研究者プロフィールページで共通点を探す

### 5.2 事前メール：5つの構成要素

会う前にメールで連絡を取ると、実際の接触がずっとスムーズになる。

| 要素 | 内容 |
|------|------|
| 1. 相手についての情報 | 同じ学会に参加することを確認したことを述べる |
| 2. 自己紹介 | 自分の所属と専門を1〜2行で |
| 3. 共通点 | 相手の研究と自分の研究の接点を示す |
| 4. 所要時間 | 10〜15分など短時間であることを明記する |
| 5. 時間・場所の提案 | ディナー前15分など具体的に提案（柔軟性も添える） |

### 5.3 電話でのアプローチ

メールよりも電話のほうが最初の印象形成に有効な場合がある。電話前に話す内容を書き出し、英語表現を準備する。通話中はメモを取り、最後に合意内容を復唱して確認する。

### 5.4 喫茶コーナー・廊下での機会

一人でいるキーパーソンを見かけたら、コーヒーブレイクなどで話しかける絶好の機会。

```
Excuse me, I heard you speak in the round table — I thought it was really useful.
Hi, do you have a couple of minutes for some questions?
```

肯定的なコメントを添えてから、「少し座って話しませんか」と提案する。相手が急いでいる場合は後日のアポを取り付ける。

### 5.5 面談で使うテクニック（ダイアログ比較）

**NG パターン**: 自分の話を延々と続ける → 相手に「自分のことしか興味がない人」という印象を与える

**OK パターン（カルロスのアプローチ）**:

1. 相手に断る選択肢を与える（「今少しお時間ありますか?」）
2. 所要時間を最初に伝える（「2分で済みます」）
3. 相手のプレゼンへの具体的な感想を述べる
4. 相手が発言する余地を作る質問をする
5. 相手にとってのメリットを伝える
6. 会うための場として学会主催のイベントを活用する

> **会話は卓球**: 自分が数秒話したら相手にボールを返す。独り占めしない。

### 5.6 面談のフォローアップ

面談後は必ずフォローアップメールを送る。フォローなしでは人脈が育たない。

メールに含める内容:
- 面談への感謝
- 話し合った内容の要約と確認
- 次のアクション（CVの添付、論文の共有、指導教授への紹介など）

## 6. リスクの低い場での注目練習（Ch16.13）

懇親会で積極的に発言することが難しい場合、日常の低リスクな場で練習する。

- 友人との食事で自分から話題を提供する
- 研究室・学科内で講師役を自ら買って出る（教えることはプレゼンの訓練になる）
- 2分間スピーチを母語で、その後英語で練習する

**練習トピックの例**:
- 毎日の生活で楽しんでいること
- 好きな映画・本とその理由
- これまで経験した最悪・最高の旅
- 将来の目標

## 7. ネットワーキング即戦力表現集

### 懇親会での話しかけ・輪への参加

| 場面 | 表現 |
|------|------|
| 輪に加わる | Do you mind if I join you? |
| 輪に加わる（別表現） | Is it OK if I listen in? |
| プレゼンターへの接触 | Excuse me, do you have a minute? |
| 質問の前置き | Would you mind answering a few questions? |
| 詳細を求める | Could you give me some more details about ...? |
| 資料の場所を聞く | Have you uploaded your presentation? |

### 自己紹介・名前の確認

| 場面 | 表現 |
|------|------|
| 未紹介の確認 | I don't think we've been introduced — I'm ... |
| 名前の確認 | Sorry, I didn't catch your name. |
| スペルの確認 | Can you spell it for me? |
| 名前の発音確認 | How do you pronounce your name? |

### 会話の継続・終了

| 場面 | 表現 |
|------|------|
| 共感・体験の共有 | I know exactly what you mean. In fact, ... |
| 同意 | I completely agree with what you are saying. |
| 丁寧な異議 | I'm not sure I totally agree. In my country, ... |
| 会話の締め | It's been really interesting talking to you. |
| 締め（別表現） | I've really enjoyed talking to you. Enjoy the rest of the conference. |

## 8. ネットワーキング全体フロー

```
事前準備
├── キーパーソンリストアップ（学会プログラムから）
├── 顔写真・論文・プロフィールの確認
├── 事前メール送信（1〜2週間前）
└── 想定質問・トークネタの用意

学会中
├── 懇親会に早く到着
├── 会話の輪への積極的参加
├── 自分の発表へのキーパーソン招待
├── コーヒーブレイクの活用（1on1のチャンス）
└── 面談の設定と実施

学会後
├── 面談相手へのフォローアップメール
├── CV・論文の添付（合意した場合）
└── 次回学会・共同研究の可能性を温める
```

## 9. インフォーマル面談の進め方（17.11〜17.12）

### 9.1 面談の始め方

面談の成功は冒頭の数秒で決まる。相手が遅れて来ても問題ないことを伝え、ポジティブな雰囲気を作る。

- 来てもらったことへの感謝を最初に伝える
- 「事前に送った質問を読んでいただけましたか?」で会話を開始
- 学会・開催地・運営についてはポジティブなコメントのみ
  - ネガティブなコメントは共同研究への意欲を削ぐ
  - 「昨日の○○のプレゼンが素晴らしかった」など肯定的な話題を選ぶ

### 9.2 相手に話す機会を与える

会話を独り占めしない。**会話は卓球のラリー**として設計する。

| してはいけないこと | すべきこと |
|------------------|-----------|
| 自分の話を3分以上続ける | 30秒〜1分話したら相手に質問する |
| 相手の研究に「very good」とだけ評する（上から目線に聞こえる） | 「challenging だったと思います」など共感の言葉を使う |
| 共同研究の要求を最初に切り出す | メリットを示してから自然な流れで提案する |

### 9.3 面談の締め方

設定した時間枠を守り、気持ちよく終わらせる。

```
締める合図  ：OK, I don't want to keep you any longer.
後日連絡確認：Would it be OK if I email you if I need any further clarifications?
感謝       ：What you said has been really interesting and useful, thank you.
気遣い     ：I am sure there are other people you will be wanting to meet.
```

最後に面談内容を1〜2文で要約・確認することで、後のフォローアップメールと認識を合わせる。

## 10. キーパーソンに自分の発表を見に来てもらう（17.7）

会話の機会に、自分のプレゼンへの招待を加えると二重の接触が生まれる。名刺の裏に発表のタイトル・時間・場所を手書きメモして渡すと印象に残りやすい。

```
案内（基本）  ：I am doing a presentation on X tomorrow at 10:00 in Room 2.
              It would be great if you could come.

案内（共通点）：If you are interested in X, you might like to come to my presentation.
              It's at 11:00 in Room 13.
```

---

## 11. 面談準備の深化（17.10）

事前に質問リストを用意しておくと、真剣さと相手の時間への配慮が伝わる。

- 質問は3〜5点に絞ってA4 1枚にまとめる
- 面談前にメールで送付し、相手が考える時間を与えると答えの質が上がる
- メモを取りながら聞く（記憶の補助と敬意の表明を兼ねる）
- 重要なポイントはパラフレーズ（言い換え）して理解を確認する
- 面談翌日には内容を要約してフォローアップメールを送る（第一印象を強化する）
- 成果がなくても「お時間をいただきありがとうございました」の一言は忘れずに送る
