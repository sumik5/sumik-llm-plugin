# ビジュアルデザイン実践ガイド

視覚的に伝わるスライドを作るための図解技法・デザイン原則・グラフ改善の実践リファレンス。

---

## 1. 見える化の5ステップ

情報を図解する前に、思考の流れを整理する5段階プロセス。

```
Step 1: 全体像を捉える
  └── テーマ・スコープを定義する
      「どこからどこまでを1枚の図で説明するか」を決める

Step 2: 要素を抽出する
  └── 構成要素をすべてリストアップ（漏れなく）
      箇条書きで書き出し、後から取捨選択

Step 3: 関係性を整理する
  └── 要素間の「つながり方」を特定
      因果か？並列か？階層か？循環か？

Step 4: 視覚化する
  └── 下記「図解18の型」から最適な型を選択
      色・アイコン・配置で意味を補強

Step 5: 検証する
  └── 「この図だけで伝わるか？」を第三者視点で確認
      説明なしで理解できない場合は型の見直しが必要
```

## 2. 図解18の型

要素間の「関係性の種類」で分類した、図解の選択フレームワーク。

### 2.1 比較型（差を見せる）

| 型 | 使う場面 | HTML実装方針 |
|----|---------|-------------|
| **対立** | 善悪・正誤・有無・Before/Afterの二項対立 | 左右分割レイアウト、対比色（例: 青vs橙） |
| **段階** | レベル・ランク・フェーズの変化（成長曲線等） | 横並びの矢印付きボックス、色の濃淡で序列 |
| **構成要素比較** | 複数対象の内訳・シェアを横並びで比較 | 横積み上げバーまたはグループ棒グラフ |

### 2.2 系統型（つながりを見せる）

| 型 | 使う場面 | HTML実装方針 |
|----|---------|-------------|
| **流れ** | 時系列・プロセス・手順・変化の推移 | 横または縦の矢印フロー、ステップ番号付与 |
| **階層** | 上下関係・組織図・分類体系 | ツリー構造、インデントまたは線で親子関係を表現 |
| **循環** | サイクル・PDCAのような繰り返し・相互関係 | 円形矢印、各要素を等間隔に配置 |

### 2.3 分析型（全体を把握させる）

| 型 | 使う場面 | HTML実装方針 |
|----|---------|-------------|
| **マトリックス** | 2軸（重要度×緊急度等）で全体を4象限に分類 | CSS Grid 2×2、各象限に背景色 |
| **グラフ** | 量・割合・推移の数値変化を可視化 | Chart.jsまたはSVG直書き（後述「グラフ選択基準」参照） |
| **リスト** | 要素の列挙・優先順位付け | 箇条書き、番号付きリスト、アイコン付きカード |

### 2.4 発展型（変化のメカニズムを見せる）

| 型 | 使う場面 | HTML実装方針 |
|----|---------|-------------|
| **因果関係** | 原因→結果の一方向の影響関係 | 矢印でつないだボックス、原因と結果を異色で表現 |
| **相互関係** | 複数要素の双方向の影響・相乗効果 | 双方向矢印、Venn図、中間ゾーンで共通効果を表現 |
| **トリガー** | きっかけとなる事象→連鎖反応 | 起点を強調（色・大きさ）し、連鎖を矢印で展開 |

### 2.5 構造型（仕組みを分解して見せる）

| 型 | 使う場面 | HTML実装方針 |
|----|---------|-------------|
| **入出力型** | インプット→プロセス→アウトプットの変換機構 | 左→中央→右の3ゾーン分割 |
| **分解型** | 全体を機能別・役割別に分割（システム構成等） | 包含関係を示す入れ子ボックス |
| **組合せ** | 複数要素の掛け合わせによる相乗効果 | ×記号またはVenn図、結合ゾーンを強調 |

## 3. フォーカス＋ノイズカット

> **デザインの本質は「気遣い」**
> わかりやすさとは、相手の認知コストを下げる配慮の実現である。

### 3.1 フォーカス：重要な要素を目立たせる

```
1. 1スライドで「見てほしい箇所」は1か所だけ
2. その箇所に: 色（アクセントカラー）・サイズ（大きく）・位置（中央または左上）のいずれかを適用
3. フォーカス箇所以外は: グレーダウン・縮小・余白で後退させる
4. 矢印・吹き出し・下線は「補助」として使う—多用は逆効果
```

### 3.2 ノイズカット：不要な情報を削減する

| 削除候補 | 判断基準 | 代替手段 |
|---------|---------|---------|
| 不要なグリッド線・枠線 | 隣接配置で区切りが自明なら削除 | 余白でグルーピング |
| 過剰なグラデーション・シャドウ | 情報を読み取る妨げになるなら削除 | フラットな色面 |
| 装飾的なアニメーション | 意味を補強しない動きはすべて削除 | 静止スライドで完結 |
| テキストの重複 | 図解とキャプションで同じことを言っていないか | どちらか一方に集約 |
| ロゴ・日付の全スライド掲載 | 表紙・ラストスライドのみで十分 | マスタースライドで管理 |

## 4. 統一感の5要素

「バラバラに見える」資料の根本原因は、この5要素のいずれかの不統一にある。

```
1. デザインスタイル
   └── フラット / モダン / クラシック のいずれか1スタイルに統一

2. 配色
   └── メインカラー1色 + アクセントカラー1色 + グレー（補助）
       の3色体制を厳守（後述「配色実践ルール」参照）

3. フォント
   └── 見出し用・本文用の2種類のみ使用
       スライド全体で同一フォントファミリーを使用

4. 余白
   └── スライド端からの余白（マージン）・要素間の余白（ガター）を統一
       余白量はシステム的に決定（例: 8pxグリッド）

5. 基本図形
   └── 角丸ボックス・直線矢印など使用する図形の種類を限定
       同じ意味には必ず同じ形を使う（形の意味を一貫させる）
```

## 5. 配色実践ルール

### 5.1 基本3色体制

```
メインカラー  : テーマの核心を表す色（見出し・強調ボックス）
アクセントカラー: 行動・注目ポイントを示す色（CTA・重要数値）
グレー       : 補助情報・非アクティブ要素・背景（他の色を引き立てる）
```

### 5.2 同系色の濃淡による序列化

同一の意味カテゴリに属する複数要素は、**濃淡のみ**で優先度を表現する。

```
最重要: メインカラー 100% 濃度
重要 : メインカラー 60% 濃度
補足 : メインカラー 30% 濃度
参考 : グレー
```

### 5.3 言葉と色イメージの一致

| 伝えたい意味 | 適切な色のトーン |
|------------|----------------|
| 成長・ポジティブ・増加 | 暖色（青緑〜緑）または上昇矢印の色 |
| リスク・警告・課題 | 橙・赤（アクセントカラー） |
| 中立・現状・事実 | グレー・ブルーグレー |
| 解決策・未来・可能性 | 主テーマのメインカラー（明るめ） |

## 6. レイアウト3パターン

コンテンツの性質に応じた基本レイアウト選択。

### 6.1 全面レイアウト

```
┌─────────────────────────┐
│                         │
│   大きな図・グラフ・       │
│   インパクト画像          │
│   （キャプション小さく）   │
│                         │
└─────────────────────────┘
```
**使う場面**: インパクト重視のキーメッセージ、データの一目可視化

### 6.2 半面レイアウト

```
┌──────────┬──────────────┐
│          │              │
│  図・グラフ │ テキスト説明  │
│  (左50%)  │  (右50%)    │
│          │              │
└──────────┴──────────────┘
```
**使う場面**: 図解＋テキスト説明の組み合わせ、Before/After比較

### 6.3 3分の1レイアウト

```
┌───────┬───────┬───────┐
│       │       │       │
│ 項目1 │ 項目2 │ 項目3 │
│       │       │       │
└───────┴───────┴───────┘
```
**使う場面**: 3要素の並列比較、3ステップのフロー

## 7. グラフ実践改善

### 7.1 目的別グラフ選択

| 目的 | 推奨グラフ | 避けるべきグラフ |
|------|----------|----------------|
| 量・大小を比較する | 縦棒グラフ（正値のみ） | 3D棒グラフ |
| 時系列の変化を見せる | 折れ線グラフ | 面グラフ（複数系列は読みにくい） |
| 全体に対する割合を見せる | 円グラフ（4要素以下） | 3D円グラフ |
| 複数要素の構成変化を見せる | 積み上げ棒グラフ | 複数の円グラフを並べる |
| 長い項目名を比較する | 横棒グラフ | 縦棒グラフ（項目名が重なる） |
| 2変数の相関を見せる | 散布図 | 棒グラフで代替しない |

### 7.2 グラフ作成の8つの基本ルール

```
1. 数値の増減をアピールするときだけグラフを使う
   → 増減がない数値は表や箇条書きで十分

2. 余計な目盛り・数値・単位は極力カット
   → データバーの先端に値を直書きし、Y軸目盛りを非表示に

3. 本題を目立たせ、参考情報は控えめに
   → 強調したい系列のみメインカラー、他はグレー

4. 色と添え書きでメッセージを強調
   → グラフ上部に「5年間で売上高2倍」などのメッセージを記載

5. 矢印・凡例・吹き出しでポジティブ感を強調
   → 上昇矢印をグラフに重ねてトレンドを視覚化

6. データの取り方・見方を工夫する
   → 基準年・比較対象の選び方でメッセージが変わる

7. ネガティブな数値は「上にメッセージ、下にグラフ」で緩和
   → 課題提示スライドで感情的な受け取り方をコントロール

8. グラフはメッセージから逆算して設計する
   → 「何を言いたいか」が決まってからグラフ種類を選ぶ
```

### 7.3 グラフのメッセージを伝える4手法

| 手法 | 方法 | 効果 |
|------|------|------|
| **添え書き** | グラフ上部に「〇〇が△△%増加」と記載 | 数字の読み取り不要でメッセージが伝わる |
| **色による強調** | 重要データポイントのみメインカラー、他はグレー | 視線が自然にポイントに集まる |
| **矢印・吹き出し** | 注目すべき箇所に矢印や吹き出しを重ねる | 「ここが大事」という意図を明示 |
| **単位・凡例の分離** | グラフから独立させて余白に配置 | グラフ領域がすっきりし可読性が向上 |

## 8. Before/After改善メソッド

「なぜダメか」を理解してから直す、3パターンの変換技法。

### パターン1: 情報過多 → 分割

```
NG: 1スライドに課題・原因・解決策・スケジュールが混在
    → 読むべき情報が多く、何が重要か分からない

OK: スライドを3〜4枚に分割し、各スライドに1メッセージ
    → タイトルを見るだけで内容が把握できる
```

### パターン2: 色の乱用 → 統一

```
NG: 強調したい箇所を赤・黄・オレンジ・青と多色で塗り分け
    → どこが「最重要」か視覚的に不明

OK: アクセントカラー1色のみを強調に使用、他はグレーダウン
    → 色が「意味を持つ記号」として機能する
```

### パターン3: メッセージ不在 → 明確化

```
NG: グラフや図解だけがあり、タイトルは「売上推移」
    → 何を主張しているのか聴衆が判断しなければならない

OK: タイトルを「Q3売上は計画比115%—コスト削減が寄与」に変更
    → 資料を見ただけで結論が伝わる（1スライド1メッセージ）
```

## チェックリスト

### 図解・レイアウト
- [ ] 図解の型は「伝えたい関係性」に合致しているか（18の型で確認）
- [ ] 1スライドのフォーカスポイントは1か所か
- [ ] 削除できる装飾・枠線・重複テキストを取り除いたか
- [ ] レイアウトはコンテンツ性質に合ったパターン（全面/半面/3分の1）か

### 統一感
- [ ] デザインスタイルがスライド全体で一貫しているか
- [ ] 使用色はメイン・アクセント・グレーの3色体制に収まっているか
- [ ] フォントは2種類以内で統一されているか
- [ ] 余白量とマージンが統一されているか
- [ ] 同じ意味には同じ図形を使っているか

### グラフ
- [ ] グラフ種類は「伝えたい目的」に合っているか
- [ ] 強調したい系列・データポイントだけに色がついているか
- [ ] グラフ上部にメッセージ（添え書き）が記載されているか
- [ ] 目盛り・凡例・装飾は最小限になっているか

### Before/After確認
- [ ] 1スライドに複数メッセージが混在していないか
- [ ] 色は意味を持つ記号として使われているか
- [ ] タイトルにキーメッセージが書かれているか（「〜について」でないか）
