# 英語発音・デリバリーリファレンス

日本人研究者が国際学会で通用する発音・話し方を習得するための実践ガイド。完璧な発音を目指すのではなく、「伝わる発音」への最短経路を示す。

---

## 1. 発音への向き合い方：「ある程度うまく」で十分

### 1.1 目標設定の考え方

発音の目標は「ネイティブと区別がつかないレベル」ではない。典型的な10〜15分の学術プレゼンで登場する**個別単語数は 300〜450 語程度**で、そのうち発音を意識的に練習すべき単語は 10〜20 語に絞られる（学術プレゼン指導の経験則）。

- 代名詞・前置詞・接続詞・冠詞：すでに知っている単語がほとんど
- キーワード・専門用語：スクリプトを書き出せば「落とし穴」の場所が特定できる
- 残りの 10〜20 語：正しい発音を意識的に練習すれば十分

目指すのは「プレゼンで通用する発音」。母語訛りを消す必要はなく、**聴衆が内容を理解できる明瞭さ**があれば目標達成。

### 1.2 スクリプトを書き出すことの発音チェック効果

スクリプトを書き出す行為（→ QUALITY-PREPARATION-PROCESS 参照）は、発音の観点でも有効。書いた単語をテキスト読み上げソフトや音声辞書で確認することで、発音の落とし穴を事前に特定できる。

## 2. 日本人研究者がつまずきやすい発音パターン

### 2.1 綴りと発音のズレ

英語の綴りは発音を反映しないケースがある。代表例：

| 単語 | 注意点 |
|------|--------|
| Wednesday | "Wed-nes-day" ではなく "Wenz-day" に近い発音 |
| comfortable | "comf-ter-ble"（3音節）と読まれることが多い |
| doubt | b は黙字 |
| business | "biz-ness"（i は発音しない） |

プレゼン前にスクリプトを音声辞書で通しチェックし、黙字や不規則発音を把握しておく。

### 2.2 アクセント位置の誤り

複数音節の単語は、アクセント位置が日本語カタカナとずれることがある。

| カテゴリ | 原則 |
|--------|------|
| 2語複合語（smartphone / laptop）| 第1音節にアクセント |
| 後半が前置詞の句動詞（back up / log in）| 第1語にアクセント |
| 頭字語（PCなど）| 各文字に等しくアクセントを置く |

**innovativeの例**：英語では "in-NOV-a-tive"。日本語的な "IN-no-va-tive" は誤り。音声辞書で確認しておく。

### 2.3 母音の混同

英語の母音は種類が多く、日本語の5母音とは体系が異なる。

| よくある混同 | 例 |
|-----------|-----|
| "i" と "e" の混同 | ship / sheep |
| "a" と "æ" の混同 | cat / cut |
| "er" 音 | bird と word は同じ母音、beard とは異なる |

発音が難しい単語がキーワードでなければ、**発音しやすい同義語への置き換え**が現実的な解決策。

| 置き換え例 | 理由 |
|-----------|------|
| innovative → new, novel | 多音節語を単音節語へ |
| worldwide → globally | 発音しにくい母音を避ける |
| usually → often | 子音 "th" "s" の連続を避ける |

## 3. 発音確認のオンラインリソース

### 3.1 推奨ツール

| ツール | 用途 |
|--------|------|
| [howjsay.com](https://howjsay.com/) | 単語の正しいアクセント位置を音声で確認 |
| [Cambridge Dictionary](https://dictionary.cambridge.org/) | イギリス英語・アメリカ英語の両発音を確認 |
| [Oxford Learner's Dictionaries](https://www.oxfordlearnersdictionaries.com/) | 発音記号と音声を同時に確認 |
| Google翻訳（音声機能） | テキスト全体を流し聞きしてリズムを確認 |
| テキスト読み上げソフト（Odd Cast 系） | 地域差（英米印など）を選んで比較できる |

### 3.2 活用手順

1. スクリプトを書き出す（→ Ch.3 相当のスクリプト作成プロセス）
2. テキスト読み上げソフトでスクリプト全体を再生し、不自然な箇所をメモ
3. 特定したキーワードを音声辞書で個別確認し、アクセント位置を把握
4. 発音が難しく置き換えも難しい重要専門用語をリストアップ → 集中練習の対象に

## 4. モデリング学習法

### 4.1 プレゼンターを模倣して学ぶ

BBC News・TED・学会動画など、スクリプトまたは字幕付きの動画を使った模倣学習は発音向上に効果的。

**手順**:
1. 字幕付きのプレゼン動画を1本選ぶ（自分の研究分野に近いもの推奨）
2. 音声を止めてスクリプトを音読し、自分の発音を確認
3. 音声を再生してプレゼンターの発音と比較
4. ギャップがあった箇所を繰り返し練習

BBC のビジネス英語シリーズは、プレゼンで頻出する語句の発音練習に特化したコンテンツがある。

### 4.2 自己録音・分析

リハーサルを録音して批判的に聴く。チェックポイント：

- 間投詞（er, um, erm）が多発していないか
- 話すスピードが速すぎないか
- キーワードに十分な強調がついているか
- 文末の音量が下がりすぎていないか

## 5. スピード・声のトーン・強調

### 5.1 話すスピード

速すぎると「重要でない内容」という印象を与えるだけでなく、内容の理解も阻害する。適切なペースの目安：

- **通常のプレゼン**：1分あたり100〜130語程度
- **ノンネイティブ聴衆が多い場合**：やや遅めに。スライド間・センテンス間に1〜3秒のポーズを置く

ポーズは弱さではなく、聴衆が理解を整理するための設計的な「余白」。

### 5.2 声のトーン・変化

単調なイントネーションが続くと聴衆は内容を追えなくなる。変化させる4要素：

| 要素 | 活用場面 |
|------|--------|
| **スピード**（速 / 遅） | 重要な箇所ではゆっくり、遷移部分は少し速く |
| **ボリューム**（大 / 小） | 核心的な主張は少し大きく |
| **ピッチ**（高 / 低） | 新しいセクションの開始は少し高め |
| **トーン**（熱量 / 冷静） | 驚くべき発見は熱量を上げて伝える |

練習ヒント：目が不自由な聴衆に声だけで伝えると想像してみる。声が情熱と意味の唯一の伝達手段になったとき、自然と変化がつく。

### 5.3 キーワードの強調

英語は**強勢拍リズム**（stress-timed）の言語。重要な単語を強く・ゆっくり発音し、重要でない単語を弱く・速く発音することで、聴衆は「何が核心か」を音で判断できる。

**同じ文でも強調位置で意味が変わる例**:

```
Please PRESENT your paper next week. （readではなくpresentであることを強調）
Please present YOUR paper next week. （my ではなくyourを強調）
Please present your paper NEXT week. （this weekではないことを強調）
```

強調するときのテクニック：
- その単語だけ少しゆっくり発音する
- 音量をわずかに上げる
- 声のトーンを微妙に変える

## 6. 専門用語・頭字語の発音

### 6.1 借用語の落とし穴

多くの英単語が他言語に借用されており、カタカナ発音とは異なる場合がある。

**2語複合語**：アクセントは第1音節
```
HArdware（ハードウェア）
SUpermarket（スーパーマーケット）
MOBile phone（携帯電話）
```

**頭字語**：各文字に等しくアクセント
```
P-C、C-D、D-V-D
```

頭字語をプレゼンで使う場合は、スライド上にも表示する。母音が多い頭字語（IAEなど）は特に聞き取りにくいため、初出時にフルネームを述べる。

### 6.2 数字の明瞭化

似た発音の数字は文脈がなければ聞き分けにくい。

| 混同しやすい組 | 発音の差異 |
|-------------|-----------|
| 13 と 30 | thirTEEN（語尾を強調）vs THIRty（頭を強調） |
| 14 と 40 | fourTEEN vs FORty |

重要な数字はスライドにも表示する。数字の最後の文字（n）を意識的に発音する。

## 7. -ed 語尾の発音

過去形の -ed は原則として新たな音節を増やさない。

| 例 | 音節数 |
|----|--------|
| focus → focused | 変わらず2音節 |
| follow → followed | 変わらず2音節 |
| inform → informed | 変わらず2音節 |

**例外**：動詞の原形が -d または -t で終わる場合は音節が増える。
```
add → added（add-DID、2音節 → 2音節だが発音が変わる）
paint → painted（paint-TID、2音節 → 3音節）
```

## 8. 間投詞の回避と置き換え戦略

### 8.1 なぜ er/um が出てしまうのか

次の言葉を考えながら話しているとき、"er" が口をつく。特に `and, but, also, however, which, that` の後に続きやすい。

**悪い例**:
```
And er then er I analyzed er the data, but er this also meant that er...
```

### 8.2 回避戦略

| 方法 | 効果 |
|------|------|
| センテンスを短く切る | 次の言葉を考える時間を減らす |
| er の代わりにポーズを置く | 間ができても無音の方がプロフェッショナルに聞こえる |
| 接続詞を意識的に減らす | and / but を使う回数を減らせば er も減る |
| 繰り返し練習 | 言いたいことがすぐに出てくるようになると er は消える |

**改善例**:
```
I analyzed the data. This revealed an unexpected pattern. The pattern suggested...
```

### 8.3 自己点検

自分が間投詞を多発しているかどうか、気づいていないことが多い。リハーサルを**録音・録画**して客観的に確認する習慣が効果的。

## 9. ノンネイティブ聴衆への対応

### 9.1 ノンネイティブが好むプレゼンスタイル

国際学会では聴衆の多くがノンネイティブ。彼らが評価するプレゼンには共通した特徴がある：

- テキストを抑えた、視覚的に整理されたスライド
- 親しみやすいトーン（lecturingではなくconversing）
- 豊富な具体例
- 詳細に踏み込みすぎない（概念を押さえることを優先）
- 発表者の情熱が伝わる話し方

これらは発音の完璧さよりも高い評価につながる要素。

### 9.2 スライドを2タイプ用意する

スクリーン表示用と、ダウンロード用（追加情報入り）を分けて用意するとノンネイティブ聴衆の理解を助ける。

| タイプ | 内容 | 目的 |
|--------|------|------|
| スクリーン用 | 最小限のテキスト + ビジュアル | 発表中の視聴覚情報を整理 |
| ダウンロード用 | 詳細説明・数値・文献情報 | 発表後の復習・理解補完 |

プレゼン冒頭で「資料はダウンロードできます」と告知することで、聴衆はメモを取る必要がなくなり、内容に集中できる。

### 9.3 語彙の選択

直感に反するが、ノンネイティブ聴衆にはラテン語・フランス語由来のフォーマルな単語の方が通じやすいことがある。これは多くのノンネイティブがフォーマルな英語テキストで学習してきた背景による。

| 口語表現（通じにくいことがある） | フォーマル表現（広く通じる） |
|--------------------------------|---------------------------|
| get | obtain / acquire |
| use | utilize / employ |
| show | demonstrate / indicate |
| find out | discover / identify |

ただし、短く明瞭に発音した短いセンテンスはフォーマルな長文より理解しやすい。語彙の格調とセンテンスの簡潔さのバランスが重要。

### 9.4 聴衆の緊張をほぐす

ノンネイティブ聴衆は「聞き取れなかった」「理解できていない」と感じても表明しにくい。発表者としてできること：

- オープニングで「質問はゆっくり話してください」と一言添える
- 質問を受けたら全体に向けて繰り返し、全員が聞けるようにする
- アジェンダに戻って「今どこにいるか」を定期的に示す
- 要約スライドを挿入し「ここまでの主要ポイント」を確認する

## 10. ネイティブチェックを依頼する際の観点

### 10.1 何をチェックしてもらうか

ネイティブに発表を聴いてもらう機会があれば、以下の観点を事前に伝えておく：

| チェック観点 | 具体的な依頼 |
|------------|------------|
| キーワードの発音 | 「この10語の発音はネイティブに通じますか？」 |
| アクセント位置 | 「強調すべき箇所を強調できていますか？」 |
| 話すスピード | 「速すぎる / 遅すぎると感じる箇所はありますか？」 |
| 間投詞の頻度 | 「er / um を多発している箇所はどこですか？」 |
| 全体の明瞭さ | 「内容を理解するうえで聞き取りにくい部分はありましたか？」 |

### 10.2 フィードバックの活用

発音の誤りをリストアップしてもらい、正しい発音を音声で教えてもらう。テキストだけでなく、ネイティブの口の形・舌の位置を直接見るとより効果的に習得できる。
