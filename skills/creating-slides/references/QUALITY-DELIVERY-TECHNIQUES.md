# デリバリー・話し方・場づくりリファレンス

プレゼンテーションの実演技術：説得力のある話し方・ボディランゲージ・場のデザイン。

---

## 1. 説得の3要素：ロゴス・パトス・エトス

古代から実証されてきた説得の3本柱。プレゼンで高い説得力を生むためには3要素のバランスが不可欠。

### 3要素の定義

| 要素 | 意味 | 具体的手段 |
|------|------|-----------|
| **ロゴス**（理性・論理） | データと論理で「正しい」と納得させる | 統計・エビデンス・ロジカルな構成 |
| **パトス**（感情） | 感情に訴えて「共感・行動」を生む | ストーリー・具体的な人物像・情熱 |
| **エトス**（信頼・人柄） | 発表者への信頼が「信じる」を支える | 実績・専門性・誠実な振る舞い |

### バランスの調整

```
提案型（意思決定者向け）
→ ロゴス重視（データと論理）＋ エトス（信頼）

啓発型（一般聴衆向け）
→ パトス重視（共感・感情）＋ ロゴス（根拠）

ピッチ型（投資家・パートナー向け）
→ エトス重視（信頼性）＋ ロゴス（実現可能性）
```

### ロゴス強化の技術

- **数値の人間化**: 「100万人が影響を受ける」→「東京ドーム10杯分の人々」
- **比較の活用**: 単独の数値より「〇〇と比較して3倍」の方が伝わる
- **難しいことをやさしく**: 専門用語は聴衆が知っている概念で言い換える
- **具体→抽象→具体**: 事例から原則を導き、別の事例で確認させる

### パトス強化の技術

- 具体的な「1人の人物」のストーリーを使う（統計より個人の話が心を動かす）
- 聴衆が「自分ごと」として感じられる問いかけを冒頭に置く
- 感情の変化を意図的に設計する（問題提起→共感→希望→行動）

### エトス構築の技術

- 資格・実績を自慢せず「なぜこの問題に取り組んでいるか」を語る
- 弱点や限界を正直に認めると信頼が増す
- 聴衆の立場への深い理解を示す（「皆さんが〜で困っているのはよくわかります」）

### 論理が通用しない3条件

ロゴス（理性・論理）が高いプレゼンを設計しても、以下の3条件が成立すると論理は効果を発揮しない。事前に聴衆の状態を診断し、ロゴス以外の訴求を優先する設計が必要になる。

| 条件 | 説明 | 対策 |
|-----|------|------|
| **① 論理より習慣が優先** | 「ずっとこうやってきた」「変えるのは面倒」という行動パターンが固着している | 小さな変化から始める段階設計。「今日から1つだけ」の行動提案 |
| **② 論理で価値観は変えられない** | 深く根付いた信念・価値観・アイデンティティには論理が届かない | 価値観に共鳴するストーリー（パトス）を先に置く。「あなたが大切にしているXと、この提案は同じ方向を向いている」 |
| **③ 論理が通用するのは論理的な人だけ** | 感情的状態にある聴衆（怒り・不安・防衛心）は論理を受け付けない | まず感情を受け止める（エトス＋パトス）。聴衆が「聴く姿勢」になってから論理を展開する |

**実践の指針**: プレゼン設計の最初に「この聴衆に論理は通用するか？」を自問する。通用しない条件が当てはまる場合は、ロゴスを3要素の3番目に置き、エトス（信頼）→パトス（感情）→ロゴス（論理）の順序で構成する。

## 2. 会話のように話すテクニック

### 「会話」と「スピーチ」の違い

| 会話的話し方 | スピーチ的話し方 |
|------------|--------------|
| 相手の反応を見ながら調整 | 一方的に話す |
| 自然なポーズ・間がある | 途切れなく続く |
| 「えっと」「そうですね」が自然 | 完璧な文章を読む |
| 視線が合う | 原稿・スライドを見る |
| 感情が動いて見える | 感情が平坦 |

### 会話的話し方のトレーニング

**ステップ1: 原稿を捨てる**
完全な文章の原稿ではなく、キーワードのみのメモで練習する。

**ステップ2: 友人に説明する練習**
「友人に5分で説明するとしたら？」という設定で話す。専門用語が自然に消え、具体例が増える。

**ステップ3: 録画して観察**
自分のプレゼンを録画し、「自分が聴衆だったら退屈するか」を確認する。

**ステップ4: リアクションの取り込み**
発表中に聴衆の表情・姿勢を読み、理解度に応じてペースや説明を調整する。

## 3. 18分ルール：時間設計の科学

### なぜ18分が最適なのか

- 人間の注意集中の自然な限界が約15〜20分
- 情報が多すぎると「認知負荷の過負荷」が起き、記憶定着率が下がる
- 制約があることで発表者自身が「何が本当に重要か」を選択する

### 時間別の設計指針

| 持ち時間 | 構成の原則 |
|---------|-----------|
| 5分以内 | 結論→主な根拠1点→行動依頼 |
| 10分 | 問題提起→解決策（Why+What）→次のステップ |
| 18〜20分 | Why-What-How完全版 + 1〜2つの具体事例 |
| 30分以上 | 構造を分割。10〜15分ごとに「休憩ポイント」を設ける |
| 60分以上 | 対話型（Q&Aや演習）を組み込む。一方的な講義は避ける |

### 時間管理のテクニック

- **バッファを持つ**: 準備時間の75%で話し終わる練習をする（残り25%がQ&A・遅延のバッファ）
- **削減単位の設計**: 「ここ5分は削れる」と判断できるセクションを事前に決めておく
- **ステータスポイント**: 「ここで半分です」と自分に言えるポイントをプレゼン中間に設定

## 4. ボディランゲージの基本

### アイコンタクト

| 状況 | 推奨 |
|------|------|
| 聴衆が少人数（〜10人） | 全員に均等に視線を配る |
| 聴衆が多人数（10人以上） | 「セクション」を区切り、各エリアに向けて話す |
| キーポイントを伝える時 | 特定の1人と目を合わせながら話す（他の人も「自分に言われた」と感じる） |
| スライドを確認する時 | 正面を向いてから話す（スライドに背を向けて話さない） |

**避けるべきアイコンタクトパターン**:
- 特定の人ばかり見る
- 天井・床・スライドだけを見る
- 視線が落ち着かなく動き続ける

### ジェスチャー4ゾーン

ジェスチャーは行うゾーンによって伝わる意味と印象が変わる。

| ゾーン | 身体の位置 | 使い場面 |
|--------|----------|---------|
| **①ダイナミックゾーン** | 頭部より上 | 注目を集めたい時・遠くの相手への合図・注意喚起 |
| **②インパクトゾーン** | 両腕のつけ根から左右 | 強調・実物を見せる・方向を指し示す |
| **③セーフティーゾーン** | 胸からおへそ | 最も使われる安定ゾーン。程よい安心感で伝える |
| **④ミステリーゾーン** | おへそから下 | 後方から見えにくい→なるべく使わない |

**有効なジェスチャー**:
- 数を示す（1・2・3と指で示す）
- 大きさ・範囲を手で示す
- 重要なポイントで両手を広げる
- 動きや変化を手の動きで表現する

**避けるべきジェスチャー**:
- 無意識の自己接触（髪・顔・腕）
- ポケットに手を入れたまま
- 腕組み（防御的に見える）
- ペン・レーザーポインターを振り回す

**オンライン時の注意**: カメラ画角によってセーフティーゾーンがミステリーゾーンになる場合がある。事前にカメラ位置を確認する。オンラインでは横の動きより縦の動きが立体感を生む。

### 姿勢・ポジショニング

```
基本の立ち方
├── 肩幅程度に足を開く（安定感）
├── 体重を均等にかける（片足重心は不安定に見える）
├── 肩を引き、胸を開く（自信のシグナル）
└── ステージ上を適度に移動する（ただし歩き回りすぎない）

聴衆との距離
├── 近すぎ（1m以内）: 圧迫感、親密すぎる
├── 適切（2〜3m）: 自然な会話距離
└── 遠すぎ（5m以上）: 疎遠感、声が届きにくい
```

## 5. 声の使い方：ペース・間・トーン

### スピードの設計

| スピード | 使い場面 | 効果 |
|---------|---------|------|
| **ゆっくり（意図的に落とす）** | キーメッセージ、結論、重要な数値 | 強調・聴衆が追いつける |
| **通常（会話程度）** | 一般的な説明、情報伝達 | 自然さ |
| **やや速め** | リスト読み上げ、背景説明 | 時間節約、テンポ感 |

**「1分あたり125〜175語」が適切なプレゼンペースの目安**（日本語では毎分250〜300文字程度）。

### 「間」の活用：3種類の使い分け

> プレゼンターが感じる「沈黙の3秒」は、聴衆には「自然な考える時間」

**3つの間**:

| 間の種類 | タイミング | 効果 |
|---------|-----------|------|
| **理解の間** | 重要ポイントの「後」 | 聴衆が情報を消化する時間を与える |
| **強調の間** | キーメッセージの「前」 | 期待感を高め、次の言葉に注目を集める |
| **集中の間** | 注意が散漫になった時 | 意図的な沈黙でリセット・再集中させる |

**間を入れるべきタイミング**:
- 質問を投げかけた後
- スライドが切り替わった後
- セクションが変わるとき

### トーンの変化

単調なトーンは睡眠を誘う。意図的な変化が注意を維持する。

```
エネルギーを上げる場面: 発見・解決策の提示・行動促進
エネルギーを下げる場面: 共感を示す・深刻な問題提示・沈思を促す
平坦なトーン: データ列挙のみ（必要最低限）
```

## 6. ユーモアの適切な活用

### 「ジョークなし」のユーモア技法

ユーモアは笑いを取ることが目的ではない。**聴衆との距離を縮め、記憶に残るフックを作ること**が目的。

| 技法 | 説明 | 例 |
|-----|------|-----|
| **意外性** | 予想を裏切る展開 | 「最も効果的な解決策は意外にも〜でした」 |
| **自己開示** | 失敗談・恥ずかしい経験 | 「私も同じ間違いをしていました」 |
| **観察の共有** | 「あるある」の提示 | 「会議が終わって誰も何もしない、あの感覚はわかりますか」 |
| **比喩の意外な転換** | 既知の概念を別の文脈で使う | 「このシステムは、料理で言えば…」 |
| **軽いオーバーアクション** | 大げさな表現で要点を際立てる | 「2ページのメールが届いた時、私はPDFだと思いました」 |

### ユーモア使用のガイドライン

**使って良い**:
- 自分を笑い者にする（自己開示型）
- 状況・プロセスの矛盾を指摘する
- 広く共有されている「あるある」

**使ってはいけない**:
- 特定の人・グループを対象にした笑い
- 差別的・排除的なジョーク
- 業界内輪ネタ（全員が知らない）
- 無理に笑いを取ろうとする試み

## 7. リハーサル手法と準備の科学

### 練習の科学的根拠

プレゼンで最も大きな準備投資が必要なのは「最初の30秒」。冒頭が自然に出てくるまで練習することで、緊張感が緩む。

### リハーサルの段階設計

**Stage 1: 構造確認**
スライドを見ながら「何を言うか」を確認。原稿不要。

**Stage 2: 声出し練習**
実際に声に出して話す。自分の声を録音する。

**Stage 3: タイム計測**
実際の時間で通し練習。予定より10〜15%短く終わるペースを確認。

**Stage 4: 本番環境シミュレーション**
同等の空間で、立って、スライドを投影して練習。

**Stage 5: 観客付きリハーサル**
1〜2名の聴衆に見せ、反応を確認。「わかりにくい点」のフィードバックをもらう。

### 緊張への対処

```
生理的アプローチ
├── 発表前の深呼吸（4秒吸って、4秒止めて、4秒吐く）
├── ウォームアップ（身体を動かして筋肉の緊張を解く）
└── 水を飲む（口の渇きを防ぐ）

認知的アプローチ
├── 「緊張は興奮と同じ生理状態」と再解釈する
├── 「聴衆は発表者の失敗を願っていない」と認識する
└── 「完璧より伝わることが目的」と目標を修正する
```

### パワーポーズによる緊張コントロール

本番前の2分間、**ハイパワーポーズ**をとることで、ホルモン状態が変化し自信・安心感・熱意が高まる。

**ハイパワーポーズの例**:
- 両手を腰に当て、足を肩幅に開いて堂々と立つ（「ワンダーウーマン」ポーズ）
- 両手を頭上に大きく開き、顔を上に向けて背筋を伸ばす

**ローパワーポーズ（避けるべき）**:
- 背中を丸めて手元の資料を見る
- うつむいてスマートフォンを見る

**実践タイミング**: 本番直前に自分だけの空間で2分間実施する。「うまくできていないのにポーズだけ」という感覚があっても続けてよい。脳は現実と虚構を区別しにくいため、姿勢が思考・感情に影響する。

### 緊張トリセツ（自分専用の取扱説明書）

緊張は「排除」するのではなく「コントロール」する。**自分の緊張取扱説明書（緊張トリセツ）**を事前に作成しておくことで、本番で緊張しても「大丈夫、対策がある」と自信を持って臨める。

**作成手順**:

1. **症状の洗い出し**: 「発表前1週間から肩こり」「本番に声が震える」など自分固有の症状を書き出す
2. **対策の設計**: 症状ごとに効果的な対処法を書く（深呼吸・パワーポーズ・ウォームアップなど）
3. **実践と更新**: 本番後に「効いた/効かなかった」を記録し、より精度の高い対策にアップデートする

**マインドセット**: 緊張は「その機会を重要と捉え、能力を引き出そうとしている状態」。緊張しない人はほぼいない。コントロールする方法を持つことが目標。

### 3ステップビデオチェック法（菅野誠二）

リハーサル録画を3段階で自己チェックする手法。各ステップで異なる問題点が浮き彫りになる。

```
Step 1: 音声のみで確認
　→ 論理の流れ・言葉の選択・説明の明瞭さのみを評価
　→ 「スライドなしでも理解できるか？」

Step 2: 画面（映像）のみで確認
　→ ボディランゲージ・ジェスチャー・表情・視線を評価
　→ 「見ているだけで何を伝えようとしているかわかるか？」

Step 3: 音声 + 画像の両方で確認
　→ 言葉とボディランゲージが一致しているか確認
　→ 「声とビジュアルが相乗効果を生んでいるか？」
```

**活用のタイミング**: Stage 3（タイム計測）以降の録画を使う。1回の録画で3回チェックできるため、効率的な自己改善が可能。

### 繰り返しの劣化（藤沢晃治）

同じプレゼンを繰り返すうちに品質が自然に低下するメカニズム。

**劣化の原因**:
- 内容を熟知するほど「説明を省いても大丈夫」という錯覚が生まれる
- 「前も問題なかった」という成功体験がチェックを甘くする
- 発表者の「慣れ」が聴衆にとっての「初見」との乖離を生む

**対策**:
- 毎回「聴き手は今日初めてこの話を聞く」を発表前に意識する
- 「説明を省いた箇所」を発表後に記録し、次回の改善に活かす
- 定期的に Stage 5（観客付きリハーサル）を実施して第三者フィードバックを得る

> 「慣れているほど、聴衆視点を失いやすい」

## 8. 場づくりとファシリテーション

### 生産的な場の3条件

1. **心理的安全性**: 聴衆が質問・発言しやすい雰囲気
2. **双方向性**: 一方的ではなく対話が生まれる設計
3. **目的の共有**: この場で何を達成するかを全員が理解している

### 冒頭の場づくり

```
効果的な冒頭の順序
1. 聴衆の関心を引く問いかけ or 驚きの事実
2. 「今日の目的/アジェンダ」の明示
3. 聴衆への期待（「皆さんに持ち帰ってほしいこと」）
4. Q&Aやコメントをどこで受け付けるかの案内
```

### 双方向性を生む技法

| 技法 | 実施タイミング | 効果 |
|-----|-------------|------|
| **挙手確認** | 冒頭・知識確認時 | 全員参加感・情報収集 |
| **ペアトーク** | 情報提示後 | 理解の確認・対話の活性化 |
| **問いかけ** | セクション転換時 | 注意リセット・次への期待感 |
| **事例収集** | 概念説明後 | 聴衆の経験を引き出す |
| **Q&Aタイム** | セクション末 or 最後 | 理解確認・関係構築 |

### 質問への対処

**想定質問を事前に準備する**:
- 「最も難しい質問」トップ3を想定し、回答を準備
- データの引用元、前提条件、例外ケースを把握しておく

**Q&A基本フロー**:
```
感謝（受け止める）→ 回答 → 腹落ち確認 → まとめ（要約）
```

**Q&A 5タイプと対応法**:

| タイプ | 特徴 | 対応策 |
|-------|------|-------|
| ①試す質問 | 「やる意味あるの？」「価値を感じない」—発表者を試している | 怯まず余裕の笑みで回答。「試されている」と受け止める |
| ②不安・懸念 | 悪い経験や情報から心配して質問してくる | 背景を聞く：「どんな懸念がありますか？」 |
| ③悪意のある質問 | 「このデータはミスリードでは？」—やり込めようとしている | 抽象化して受け止め（「データについてのご質問ですね」）→全体的な説明にとどめる |
| ④詳細すぎ・脱線 | 個人的なケースや些細な点への深入り | 「個別にお話しましょう」と誘導 |
| ⑤回答できない | 準備していない領域の質問 | 「後日回答します」と正直に。推測で答えない |

**3大NGリアクション**: 「ムッとした表情」「反論して言い負かす」「沈黙」→ いずれも逆効果。

## 10. 質疑応答の反応パターンマネジメント

Q&Aはプレゼンの「最後のデリバリー」である。聴衆からの質問への対応は発表者の評価を大きく左右する。

### 7つの反応パターン

| パターン | 使いどころ | 例 |
|---------|-----------|-----|
| ①専門スタッフへ紹介 | 自分の専門外・より詳しい人がいる | 「〇〇部の△△が詳しいのでご紹介します」 |
| ②後日回答 | 準備が必要・データ確認が必要 | 「明日中にメールでお送りします」 |
| ③調べ方を教える | 聴衆が自己解決できる情報 | 「〇〇という資料を検索すると出てきます」 |
| ④個別対応 | ケース固有の詳細な話題 | 「終了後に個別にお話ししましょう」 |
| ⑤一緒に考える | その場で答えが出せる問い | 「いい質問ですね、一緒に考えてみましょう」 |
| ⑥聴衆に問い返す | 意見収集・場を活性化したい | 「皆さんはどう思いますか？」 |
| ⑦丁重に断る | テーマ外・時間的に無理な質問 | 「重要な視点ですが今日のスコープを外れるため…」 |

### 悪意・脱線質問への対処

**悪意のある質問**（やり込めようとするタイプ）:
1. 感情的にならず「ご指摘ありがとうございます」と受け止める
2. 抽象化して捉え直す（「データの妥当性へのご質問として理解しました」）
3. 全体への影響に対する説明にとどめ、個別論争に引き込まれない

**テーマ外の質問**:
- 毅然と切る: 「今日のテーマとは少し異なりますが、個別にお時間をいただけますか」
- 一人の質問に引きずられて全体の時間を侵食しないことがマネジメントの核心

> 「プレゼンの最後の1秒まで、場をマネジメントするのが発表者の責任」

## 11. 手元資料の非配布戦略と動画配信の活用

### なぜ事前配布が逆効果なのか

人間の脳は3つの情報源（話し言葉・スクリーン・手元資料）を**同時処理できない**。手元資料があると聴衆は先読みし、発表者の話を聞かなくなる。

| 情報源 | 配布タイミング | 効果 |
|-------|-------------|------|
| 手元資料 | **事前配布** | 聴衆が先読み → 発表者の話が「二度目」になる |
| 手元資料 | **終了後配布** | 発表内容の復習・参照用として機能する |
| 動画 | 後日配信 | 欠席者・復習者へのフォローアップ |

### 「後からダウンロード可能」宣言

プレゼン冒頭で以下を宣言することで、聴衆の「資料を取らなければ」という不安を解消する:

```
「今日のスライドと動画は後ほどXXXからダウンロードできます。
 ですので、手元にメモを取る必要はありません。
 今日は画面と私の話だけに集中してください。」
```

**HTMLスライドとの親和性**: HTMLスライド形式は本質的に「配信可能なフォーマット」であり、URLを共有するだけで資料提供が完結する。後日配信の運用設計と相性が良い。

## 12. スライドは「脚本」＋脳内リハーサル同時作成法

### スライド作成と話し言葉を同時に設計する

最大の失敗パターンは「スライドを作ってから話し方を考える」こと。スライドは**脚本**であり、作成時点で「このスライドのとき何を話すか」を脳内シミュレーションしながら進める。

**実践手順**:

```
Step 1: 白紙スライドを数十枚用意し、先に「言葉（メッセージ）」だけを入れる
　→ デザインは後回し。「このスライドで何を言うか」を優先する

Step 2: サムネイル画面で全体を俯瞰し、流れを確認する
　→ スライドを随時入れ替えてナラティブの流れを調整する

Step 3: 作成中、「このスライドのとき自分はステージでどんな話し方をしているか」をシミュレーションする
　→ 「つなぎの言葉」がイメージできないスライドは削除か統合のサイン
```

### 脳内リハーサルのチェックポイント

- [ ] 各スライドで「最初に何を言うか」がイメージできているか（つなぎの言葉）
- [ ] スライドの切り替わりが「息継ぎ」のタイミングと合っているか
- [ ] 「沈黙を使う」スライドと「話し続ける」スライドが区別されているか

## 13. 接続詞の戦略的活用（10種分類）

接続詞は「次の話の方向」を聴衆に予告するナビゲーターである。戦略的に使うことで、聴衆の注意を操作できる。

### 10種の接続詞と効果

| 種類 | 代表語 | 効果 | 注意喚起力 |
|-----|-------|------|-----------|
| **順接** | だから・したがって | 論理の流れを示す | 低 |
| **逆接** | しかし・ところが | 方向転換・意外性 | **最高** |
| **並列** | また・さらに | 情報の追加 | 低 |
| **対比** | 一方・それに対して | 比較を促す | 中 |
| **列挙** | まず・次に・最後に | 構造の提示 | 中 |
| **換言** | つまり・言い換えると | 理解の確認・強調 | 中 |
| **例示** | たとえば・具体的には | 具体化 | 低 |
| **補足** | なお・ちなみに | 追加情報 | 低 |
| **転換** | さて・ところで | 話題の切り替え | **高** |
| **結論** | 以上から・結論として | まとめ・到達感 | 高 |

### 活用のコツ

- **逆接「しかし」と転換「さて」** が最も注意を引く ── これらを言う瞬間にアイコンタクトを合わせると効果が増幅する
- 接続詞をストーリーラインのテンプレートとして使う: 「まず（問題提示）→しかし（転換）→だから（提案）→つまり（核）」
- 口頭でも書面でも、接続詞を抜くと「話の方向が読めない」と感じさせてしまう

## 14. 3M説明フレームワーク（Meaning・Mechanism・Message）

各スライドやセクションの冒頭に適用する**説明の導入構造**。「なぜこれを話すのか」から始めることで、聴衆がその後の情報を正しいコンテキストで受け取れるようになる。

### 3Mの定義

| 要素 | 英語 | 役割 | 問いかけ例 |
|-----|------|------|-----------|
| **M1** | Meaning | これから話すことの**意味・目的**を示す | 「なぜ今これが重要なのか？」 |
| **M2** | Mechanism | 話す内容の**構成・仕組みの概要**を提示する | 「どんな流れで説明するのか？」 |
| **M3** | Message | 相手にとっての**意義・価値**を先に伝える | 「つまりこれはあなたにとって何を意味するのか？」 |

### マジックワード: 「つまりこれはあなたにとって」

M3で最も効果的な導入フレーズが「つまりこれはあなたにとって〜ということです」。
この一言で聴衆は「自分の話だ」と瞬時に受け取り、集中度が上がる。

**適用例（セクション導入）**:
```
M1 Meaning  : 「次に、コスト削減の仕組みを説明します。
               これが今日最も重要なパートです。」
M2 Mechanism: 「3つのステップで構成しています。まず現状分析、
               次に改善案、最後にROI試算です。」
M3 Message  : 「つまりこれはあなたにとって、来期の予算申請に
               そのまま使えるデータが手に入るということです。」
```

## 15. 3訴求バランス評価（菅野誠二）

プレゼンの説得力を「合理訴求」「信頼訴求」「感情訴求」の3軸で評価し、バランスの偏りを発見・修正するための評価シート。

### 3訴求の定義

| 訴求軸 | 意味 | 問いかけ |
|-------|------|---------|
| **合理訴求** | 論旨の論理性・データの妥当性・提案の合理性 | 「この主張は筋が通っているか？数値・根拠は正確か？」 |
| **信頼訴求** | 課題の共有・聴衆への理解・発表者の誠実さ | 「聴衆の課題を正確に言語化できているか？発表者は信頼できるか？」 |
| **感情訴求** | 自信・熱意・ビジョンへの共感・感情移入 | 「発表者の熱意が伝わるか？聴衆の感情が動くポイントがあるか？」 |

### 5点満点評価シート

自己評価または第三者評価で使用する。

| 評価軸 | 1（非常に弱い） | 3（普通） | 5（非常に強い） | 自己評価 |
|-------|-------------|---------|------------|---------|
| **合理訴求** | 論理の飛躍・データなし | 論理は通るがデータが薄い | 完璧な論理とデータ | ／5点 |
| **信頼訴求** | 聴衆の課題を理解していない | 部分的に課題を共有している | 聴衆の言葉で課題を語れる | ／5点 |
| **感情訴求** | 熱意が感じられない | 時々熱意が伝わる | 終始情熱が伝わり共感を生む | ／5点 |

### バランス診断と改善方針

| パターン | 症状 | 改善方針 |
|---------|------|---------|
| **合理高・感情低** | 「正しいけど退屈」「頭では理解したが心は動かない」 | オープニングにストーリーを追加。具体的な人物エピソードを入れる |
| **感情高・合理低** | 「熱意はわかるが信頼できない」「感動したが何を決めれば？」 | データ・根拠を補強。ロジックの橋渡しを明示する |
| **信頼低・両他高** | 「話はうまいが、この人に任せていいのか」 | 「なぜ自分がこの問題に取り組むのか」のエピソードを加える。限界・弱点を正直に認める |
| **3軸ともバランス良** | 聴衆が動く | 維持しつつ、最も弱い軸を重点的に磨く |

**活用場面**: プレゼン準備の最終確認時。特に「なぜか刺さらない」と感じる場合に診断として使用する。

## 16. 実践デリバリーTips集（倉島保美『論理が伝わる 世界標準のプレゼン術』）

現場で即効性の高い実践Tips。理論より経験則の凝縮。

### アイコンタクト

| Tips | 要点 |
|------|------|
| 口を見て話す | 目を見るのが苦手なら聴衆の口を見る——聴衆は「自分の目を見ている」と思い込む |
| スクリーンを見ない | スクリーンを見るのはポインティング時のみ。スライドはキーワードだけにして、話す内容はその場で考える |
| 最前列左右端を意識 | 無意識に視線を振ると最前列の左右端とアイコンタクトが取れない——意識して視線を振る |

### 声と話し方

| Tips | 要点 |
|------|------|
| 大きな声で説得力3割増し | 大きな声はプレゼンターの自信・熱意を感じさせ、内容の理解にも役立つ |
| 最初の一声を意識する | 第一印象はとても大事。最初の一声だけでも意識して大きく出す——その後小さくなっても印象は残る |

### 身体の使い方

| Tips | 要点 |
|------|------|
| 手は自然に下ろして話す | 両手を下ろすと手が居心地の悪い場所から出ようとし、自然なボディランゲージが生まれる |
| 手を居心地よい場所に置かない | 体の前後で組む・ポケットに入れる・腕組みはNG——自然な動きが消える |
| 利き腕側に立つ | 右利きはスクリーン向かって右に立つ——ポインティングしやすく、聴衆に背を向けずに済む |
| スクリーンから離れすぎない | スクリーンとプレゼンターが同時に視野に入る位置で話す |
| スクリーンを遮らない | スクリーンの脇でも最前列端の聴衆を遮る場合がある——最前列端を空席にする配慮も必要 |

### ツール使用

| Tips | 要点 |
|------|------|
| ポインターは強調時のみ | 話している箇所をすべてポインティングしない——うるさいだけ。アニメーション使用時はほぼ不要 |
| アニメーションは必須（必要最小限） | 余計な情報を隠せる・ポイントを強調できる・流れを表現できる。ただし小賢しいエフェクトは不要 |
| 発表者ツールを活用 | スクリーンにスライドショー、手元に原稿・次スライドを表示——聴衆に悟られずメモを確認できる |

### 心理マネジメント

| Tips | 要点 |
|------|------|
| 自信があがりを防ぐ | 重要なプレゼンでは10〜20回のリハーサル。自信は繰り返しから生まれる |
| 熱意があがりを吹き飛ばす | 「伝えたい」気持ちが前に出ると声が大きく、アクションも大きくなる。仕方なしにやるとあがる |
| 伝わらないのはプレゼンターの責任 | 一方通行コミュニケーションの責任は発信者側。伝わらなかったら「なぜ伝わらなかったか」を考える |

### 聴衆マネジメント

| Tips | 要点 |
|------|------|
| 笑顔で挨拶する | 笑顔は聴衆を味方にする最大の武器——せめて最初だけでも心がける |
| 身だしなみを整える | 着こなし・ネクタイ・前ボタンを事前チェック——聴衆の好感は細部から生まれる |
| 質疑応答の共有判断 | 全員に有益な質問はその場で回答。重箱の隅・特殊ケースは終了後に個別対応 |
| 複数質問はメモを取る | 1人から複数の質問が出たら必ずメモ——「2つ目は何でしたっけ？」は時間の無駄で失礼 |

### スライド作成の失敗回避

| Tips | 要点 |
|------|------|
| 情報の詰め込みすぎを避ける | 最も多い失敗。漏らさず伝えようとするあまり、大事なことが埋没する |
| 1スライド1トピック | スペースが空いても2〜3トピックを詰め込まない。空きスペースには図・イラストを入れる |
| キーワードのみ記載（24pt以上） | 文にしてはいけない。キーワードだけにしてフォントを大きく（24pt以上）する |
