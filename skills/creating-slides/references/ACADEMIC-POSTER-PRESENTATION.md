# 学術ポスター発表設計リファレンス

国際学会ポスターセッションの構成・各セクションの書き方・オーディエンスへの声かけ戦略。

---

## 1. ポスター発表の目的と口頭プレゼンとの違い

### 1.1 ポスターの目的

ポスター発表の主目的は口頭プレゼンと共通している。

- 研究成果を伝える
- 自分への注目を集める
- 将来の共同研究や資金獲得につながる接点を作る

### 1.2 ポスターの方がプレゼンより優れている点

| 観点 | 詳細 |
|------|------|
| 1on1の対話 | 相手と直接・深く話せる |
| 緊張の少なさ | 大勢の前での発表より心理的ハードルが低い |
| 時間の柔軟性 | 口頭発表（10〜15分）より長く説明できる |
| 意見交換の豊富さ | 双方向の対話がしやすい |
| 常時展示 | 学会期間中ずっと閲覧可能 |

初めて学会発表する場合、指導教授からポスター発表を勧められることも多い。

### 1.3 ポスター発表に向いている研究

- 研究方法・結果が複雑で、図表や対話で補足が必要なもの
- 高度に専門的で小規模な聴衆向けの研究（大会場の口頭発表では埋没してしまう）

---

## 2. コンセプトを絞る：1ポスター1メッセージの原則

口頭プレゼンではキーポイントを3つ選んでよい。しかしポスターでは **できるだけ1つの重要コンセプトに絞る**。

### 絞ることのメリット

- ポスターを文字と図表で埋め尽くさずに済む
- 1つのストーリーに集中できて構成が明確になる
- 説明がしやすくなり、聴衆との対話が深まる

コンセプトのシンプルさは評価につながり、問い合わせや共同研究の連絡が増える。

## 3. ポスターの構成セクション設計

標準的な学術ポスターに含まれるセクションと推奨フォントサイズの目安:

| セクション | 役割 | フォントサイズの目安 |
|-----------|------|-------------------|
| **タイトル** | 第一印象・引きつけ | 80pt以上（3m先から読める） |
| **サマリー** | 通りがかりに読む概要 | 大きめ |
| **イントロダクション** | 研究背景・ギャップの説明 | 標準 |
| **材料と方法** | 実験設計の概要 | 標準（詳細は省く） |
| **結果** | 最も広いスペースを確保 | 図表中心 |
| **結論** | 1〜2文で意義を伝える | やや大きめ |
| **連絡先** | メール・ウェブサイト | 大きめ・目立つ位置 |
| **付加情報** | 参考文献・謝辞・利益相反 | 小さめ |

## 4. 各セクションの書き方原則

### 4.1 タイトルの書き方（18.3.1）

- 1〜2行で研究テーマを簡潔にまとめる
- できるだけ多くの分野の人を引きつける表現を使う
- **すべて大文字は避ける**
- 太字・イタリック体・下線のうち1つだけ使用（3つ同時使用禁止）
- 問いかけ形式のタイトルは人を立ち止まらせる効果がある

### 4.2 サマリーの書き方（18.3.2）

通りがかりの人がポスターを詳しく読みたいと思わせることが目的。

含める要素:
1. **Aim（目的）** — 何を調べたか
2. **Example findings（主な発見）** — 具体的な数値・事例
3. **Conclusion（結論）** — 端的な答え

写真・スクリーンショット・グラフなどのビジュアルを添えると魅力が増す。

### 4.3 イントロダクションの書き方（18.3.3）

- 最小限の研究背景と必要な用語解説のみ
- 専門外の読者が多いことを想定して書く
- 研究の「ギャップ」を示し、自分がそれを埋めようとしていることを示す
- **長いセンテンスは短い複数のセンテンスに分割する**
- 参考文献は短いセンテンスのほうが挿入しやすい

```
❌ 悪い例（長い1文）:
   Googleが検索結果を...（続く複雑な文）...を調査することにした。

✅ 良い例（短文に分割）:
   Governments virtually control information via search engines. [Ref1]
   Returns are based on previous searches. [Ref2]
   We investigated what Google returns to websurfers in Italy vs. the US.
```

### 4.4 材料と方法の書き方（18.3.4）

- 装置や統計の詳細は最小限に抑える
- **電文スタイル（短く・受動態）** で書く
- センテンスごとに改行して速読を助ける
- 詳細な情報は図表・フローチャートに置き換える

### 4.5 結果の書き方（18.3.5）

最も広いスペースを確保する。確認すべき問い:

- 研究手順と方法は機能したか？
- 期待した結果が得られたか？仮説どおりか？
- 主な結果の意義は何か？

**図表・グラフを最大限活用する。** 文章より視覚情報が伝わりやすい。明解な説明文とともに必要情報だけを載せる。

### 4.6 結論の書き方（18.3.6）

- 1〜2文で研究結果の意義を述べる
- フォーマルなトーンでも、わずかなユーモアやウィットを添えると記憶に残る
- 「今後の展開」を1文添えると連絡を呼び込みやすい

### 4.7 連絡先の書き方（18.3.7）

大きなフォントではっきりと記載する必要のある情報:

- **メールアドレス（1つで十分）** — 電話番号・Skype番号は不要
- **ウェブサイト1つ** — 研究室のウェブページ・LinkedInページ・ResearchGateページなど
- **ポスター・論文ダウンロードリンク** — 上記とは別のURL

### 4.8 付加情報（18.3.8）

小さめのフォントで掲載:

- 本文・図表で言及した参考文献一覧
- 学会独自の文献引用ルールがある場合はそれに従う
- 協力者・装置提供者への謝辞
- 利益相反の有無の開示

## 5. 箇条書きの活用原則

ポスターでは箇条書きが散漫な文章より効果的。特に以下のセクションに向いている。

- 研究目的・仮説
- 研究方法
- 研究結果
- 研究意義
- 提案・推奨事項

**ただし全セクションを箇条書きで埋めてはならない。** 変化に富んだレイアウトが読者を引きつける。

重要な箇条書きのフォントを大きく・太字にすると、研究の意図・内容・結果・意義が数秒で伝わる。

## 6. 品質チェックリスト

### 6.1 下書きチェック（18.2.6）

- [ ] センテンスが不必要に長くないか
- [ ] 冗長な説明がないか（「単語数=費用」と想像して削る）
- [ ] 曖昧な表現がないか
- [ ] 結果の説明が明確か

### 6.2 クオリティチェック（18.2.7）

- [ ] ポスターのサイズが学会規定に適合しているか（標準: A0縦, 841×1189mm）
- [ ] 友人・家族に見せて誤字脱字・わかりやすさ・引きつける力を確認したか
- [ ] キーメッセージが最も説得力を持って目に入ってくるか
- [ ] フォントの色がどのような照明条件でも正しく発色するか（昼光・蛍光灯で確認）
- [ ] タイトル・サマリーは3m先から読めるか（80pt以上を目安）
- [ ] キーメッセージは5〜10秒で読めるか

## 7. オーディエンスへの声かけ戦略（18.2.8）

ポスターの前で立ち止まる人がいても、声をかけなければ目的は達成されない。

### 7.1 声かけの準備

1. **説明スクリプトを練習する** — 学会中に何度も繰り返すため、明瞭簡潔に説明できるまで練習
2. **想定Q&Aを準備する** — 口頭発表のQ&Aと同様に想定問答を用意
3. **手法・統計の根拠を再確認** — なぜその実験設計・統計検定を選んだか
4. **専門外の訪問客向け説明** — 「家族にも伝わる」レベルの平易な言葉で準備
5. **重要人物を認識する準備** — 会いたいキーパーソンの名前・顔を事前に確認

### 7.2 配付資料と名刺

- 1ページのサマリー（配付資料）を用意する
- 配付資料に含める情報:
  - 発表タイトル
  - 学会名・開催地・日程
  - 発表者名・共同研究者名
  - 所属機関名
  - メールアドレス

## 8. ポスター × QR コード × 連絡先導線設計（現代的補強）

Wallwork本執筆時（2016年）から技術環境が変化した。QRコードを活用した現代的な導線設計を加える。

### 8.1 QRコードで拡張できる情報

| QRコード先 | 用途 |
|-----------|------|
| 論文・プレプリント（arXiv等）のURL | ポスターの詳細版を提供 |
| 研究室のウェブページ | 研究グループ全体の紹介 |
| 発表資料のPDFダウンロード | 手元に残せる形で提供 |
| 連絡先（vCard形式） | スマートフォンで即座に保存可能 |

### 8.2 QRコードの配置原則

- **目立つ位置**（右下か左下）に配置
- QRコードの横に「このポスターをダウンロード」など用途を1行で説明
- 印刷後に必ず実際のスマートフォンで読み取りテストをする

### 8.3 SNS・研究者プラットフォームの活用

- **ResearchGate** や **Academia.edu** にポスターをアップロードしておく
- **Twitter/X の学会ハッシュタグ**でポスター発表を事前告知する（ポスター番号・時間帯を添える）
- **LinkedIn** に学会後の振り返りとともにポスターを投稿する

### 8.4 学会後の連絡先管理

- 名刺交換した相手は当日中にメモ（「ポスターの前でAIの話をした」など文脈を残す）
- 翌日以内にフォローアップメールを送ると印象が残りやすい
- CRM的な発想でコンタクトリストを管理し、次の学会前に再確認する

## 9. ポスター発表の英語表現集

### 通りがかりの人への声かけ

| 場面 | 表現 |
|------|------|
| 基本の声かけ | Hi, I'd be happy to tell you about my poster. |
| 内容を紹介する | This poster presents our research on ... |
| 要約を促す | Let me give you a quick overview of what we found. |
| 質問を促す | Do you have any questions about the methodology? |
| 名刺を渡す | Here's my card — feel free to email me with any follow-up questions. |

### 説明中のつなぎ表現

| 流れ | 表現 |
|------|------|
| 背景を説明する | The background to this research is ... |
| 問題提起 | The key question we addressed was ... |
| 手法に移る | To investigate this, we ... |
| 結果を示す | What we found was ... |
| 意義を述べる | The implication of this is ... |
| 次の研究 | Future work will focus on ... |

## 10. ポスター発表の当日タイムライン

```
前日
├── ポスター印刷・サイズ確認
├── 配付資料の印刷（想定人数＋余裕分）
└── 声かけスクリプトの最終練習

セッション開始前
├── ポスター掲示（会場の規定どおりに）
├── QRコードの読み取りテスト
└── 名刺・配付資料の準備

セッション中
├── 積極的に通りがかりに声をかける
├── 説明後に名刺・配付資料を渡す
└── キーパーソンの名札を確認

セッション後
├── 当日中に交換した名刺にメモ
└── 翌日以内にフォローアップメール送信
```
