# デッキ戦略リファレンス

シリーズ書籍のデッキ階層設計・年版差分の取り込み方針・複数版の併存命名・欠落差分判定の詳細ガイド。INSTRUCTIONS.md の Step 5a（デッキの選択）から参照される。

---

## シリーズ書籍のデッキ階層

同一資格・同一級でも複数の問題集シリーズを扱う可能性がある場合、`{資格名}::{級}::{シリーズ識別子}` のように3階層以上のサブデッキ命名を推奨する。

- 級レベルにフラットに保存すると、別シリーズが増えたとき再編成（多数カードの `changeDeck`）が必要になる
- 最初から1階層深く切るとAnkiの階層タグ機能と併用してフィルタリングしやすい
- シリーズ識別子には書籍タイトルの汎用識別子や刊行年など、内容を一意に識別できる文字列を使用する

---

## 年度版投入時のユーザー意図確認 — 差分投入 vs 独立リソース投入

同一資格試験対策書籍の新年度版を投入する際、「前年版との差分のみ追加したい」と「新年度版を独立した学習リソースとして丸ごと投入したい」では方針が真逆になる。

差分判定で既存扱いされた件数が大幅に多い場合（例: 新規扱いが全体の10%以下）、ユーザーが独立リソース投入を期待しているにもかかわらず差分モードで動いている可能性が高い。

**判断フロー:**
1. 年度版の投入セッション開始時に AskUserQuestion で「差分追加か独立投入か」を確認する
2. 独立投入を選択した場合は差分判定をスキップして全件投入し、専用サブデッキ（例: `資格名::2026年版`）に配置する

## 複数版書籍の併存サフィックス命名戦略

同じ試験回（例: 第41-44回）を **複数の出版社・複数の版**から取り込む場合、既存デッキを上書きすると元の出版社版が失われる。

**対策**: 既存デッキと並存させる場合は `過去問題第N回学科(YYYY-YYYY版)` のように **版年度サフィックス**を付けて別デッキ化する。

サフィックス例:
- `(2023-2024版)`
- `(出版社略称)`
- `(電子書籍版)`

同名サブデッキは Anki UI で並列表示されるため、ユーザーは両者を見比べながら学習できる。

**判断フロー:**
1. `findNotes deck:"<予定デッキ名>"` で件数を確認
2. (a) 0件 → そのままの命名
3. (b) ある程度埋まっている → サフィックス付与で併存
4. (c) 1件だけのプレースホルダー → ユーザーに「上書き/併存」を AskUserQuestion で確認

フォーマット差（別出版社の電子書籍由来で TOC ノイズ含む等）が大きい場合は併存が安全。

## 既存デッキの問番号取得による欠落差分判定

既存デッキに「30問あるべきところが28問」のような欠落がある場合、新規パース結果と差分を取れば不足分だけ追加投入できる。

**手順:**
1. `findNotes deck:"<既存デッキ>"` で既存ノートIDを取得
2. `notesInfo` で Question/Answer フィールドから問番号を抽出（regex例: `[フッター内の]問(\d+)` または `<b>問(\d+)\.`）
3. 期待番号 set（1-30 等）との差で欠落番号を確定
4. 新規パース結果から欠落番号のカードのみフィルタして投入

**注意事項:**
- 既存デッキが別出版社・別フォーマットの場合、問番号埋め込み位置が異なり regex 抽出に失敗することが多い（例: TOCカード1枚 + 本体27枚で28件、内訳が不規則）
- その場合は問番号比較ではなく **Front テキスト類似度比較**（`difflib.SequenceMatcher` 0.75以上で同一視）にフォールバックする
- 実観測: 28件のデッキで問番号 regex が `{1, 3, 29}` の3件しか取れなかったケースあり
- 問番号抽出失敗時は**複数版書籍の併存サフィックス命名戦略**（上記）に切り替えるのが安全

## 外部無料問題集サイト（kentei-lab 等）からの収集データの命名

URL収集ワークフロー（kentei-lab 等、詳細は [URL-COLLECTION.md](URL-COLLECTION.md)）から収集したデータは、コンテンツ種別ルート `検定試験::` 配下に
**`検定試験::<検定名>::<級>::kentei-lab`**（級を検出できない試験は `検定試験::<検定名>::kentei-lab`）
の階層で命名する。

**階層の意味:**

- ルート `検定試験`: Anki の他コンテンツ（自作教材・語学カード等）と資格試験デッキをカテゴリ単位で分離する。
- 検定名: `exam_title` から級を除いた本体（`kentei_lab_import.py` の `split_exam_title` が抽出）。
- 級: `split_exam_title` が `exam_title` の末尾から抽出するサブレベル。検出できなければこの階層は省略され、
  3 階層 `検定試験::<検定名>::kentei-lab` になる。
- 出典 `kentei-lab`: 葉に置き、データの出所を記録する。
- `kentei_lab_import.py` は `--deck` 省略時にこのデッキ名を `default_deck_name(exam_title)` で自動算出する。
  実際のデッキ確定は Step 5a の AskUserQuestion に委ねる（既定を「推奨」の先頭に提示する）。

**転換理由（旧・2 階層 `kentei-lab::<試験名>` 方式からの変更点）:**

- 出典をプレフィックス（先頭）からサフィックス（末尾）へ移したことで、同一の検定名・級を複数ソース
  （書籍由来 + kentei-lab 由来）から取り込んでも、同じ `検定試験::<検定名>::<級>` の親配下に並べられる。
  学習時に出典をまたいで見比べやすく、再編成コストも下がる。出典（`kentei-lab`）は葉に残るため出所管理も
  維持される。
- ルートを `検定試験::` に統一することで、Anki の他コンテンツと資格試験デッキをカテゴリ単位で明確に分離できる。
- 級を独立した階層レベルに切り出すことで、Anki の階層フィルタ機能で級ごとの絞り込み・出題が自然に行える
  （2 階層フラットだと級混在の再編成に多数の `changeDeck` が必要になる）。

**旧デッキからの移行について**: 旧 `kentei-lab::<試験名>` 形式で既に投入済みのユーザーは、Anki 側でデッキを
リネーム（または `changeDeck`）して新命名へ寄せる手動移行を行う。ブリッジは投入先デッキ内でのみ重複判定する
ため、移行しないまま同じ JSON を再投入すると旧・新デッキに同一内容が並存する（自動移行・旧デッキ横断の重複
判定は本変更のスコープ外）。
