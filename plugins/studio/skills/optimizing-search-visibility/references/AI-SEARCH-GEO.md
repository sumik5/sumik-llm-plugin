## AI検索の現状

### Google AI Overviews / AI Mode

**AI Overviews（複数形）**とは、Googleが検索結果のオーガニックリスト上部に表示するAI生成の要約枠。

| 特徴 | 内容 |
|------|------|
| 表示位置 | ブルーリンクより上部。第1オーガニック結果が約1,674px下（ファーストビュー外）に押し下げられることが多い |
| 平均構成 | 約169語・展開時に約7本の参照リンクを含む |
| 到達規模 | 月間約15億ユーザー・200か国超（2025〜2026年時点） |
| 引用特性 | 引用の約92.36%が既にオーガニック上位10位以内のドメイン由来 |

**AI Mode**は従来の「10本の青リンク」を置き換える対話型インターフェース。マルチターン質問に応え、参照リンクを回答内に埋め込む。AI Overviewsとは引用プールが異なり、AI OverviewsはURLが約615ドメイン、AI Modeは約3,621ドメインを引用（URL重複はわずか約13.7%）。AI Modeはより広い範囲、UGC（ユーザー生成コンテンツ）も多く引用する傾向がある。

**引用されるコンテンツの特徴**
- オーガニック順位との相関が最も強い（前述の92.36%）
- 明確な見出し階層・短い段落・統計・Q&A形式が有利
- AI Overviewsへの引用は**ボラティリティが高く**、引用されるページの約70%が2〜3か月で入れ替わる（従来のオーガニック順位変動とは無関係）

### ChatGPT search / Copilot 等

**ChatGPT search / OpenAI**
- AIリファラルの約78%（2025年6月時点）を占める最大の流入源
- リアルタイムクエリではBing検索を経由して上位を取得しインライン引用で合成する
- 2025年6月から参照リンクに `utm_source=chatgpt.com` を付与開始（それ以前は流入計測が困難だった）

**Perplexity**
- 独自クローラ（PerplexityBot）と各種検索APIを併用
- 1クエリで20〜40 URLを精査してから選定する
- 一次ソースへのバイアスが強く、原論文・公式ドキュメント・一次データは二次まとめより約3.1倍引用される傾向（業界報告値、方向性参考）

**Claude / Anthropic**
- 広いソースプールから参照（Perplexityに類似）
- ブランド言及は多いがURLリンクは少ない傾向（リンク獲得より言及獲得が重要）

**Copilot / Microsoft**
- Bingをバックエンドとして利用
- GA4で `copilot.microsoft.com` を計測対象に追加して監視する

### ゼロクリック化とトラフィック影響

AI Overviewsをはじめとするゼロクリック要因がオーガニックトラフィックに与える影響は、ベンダー・メディア・研究機関の報告で幅がある。以下は各ソースの報告値をそのまま示す（比較参考用）。

| 指標 | 報告値 | 出典種別 |
|------|--------|---------|
| AIサマリ提示時のクリック率 | サマリありで8% vs なしで15%（約46.7%減） | Pew Research（一次調査） |
| AI Overview併存時のオーガニックCTR | 61%減（1.76%→0.61%） | 業界調査会社（2025年9月） |
| 有料CTR | 68%減（19.7%→6.34%） | 同上 |
| Position 1 CTR | 34.5%減 | 業界ツールベンダー調査 |
| AI Overview併存時の平均CTR | 58%低下 | 業界ツールベンダー調査（2025年12月） |
| ゼロクリック検索率 | 56%→69%（2024年5月→2025年5月） | Similarweb調査 |
| 最悪ケース | 25.23%→2.79%（約89%減） | 特定メディアの観測値 |

**重要な補足（解釈の注意点）**
- AI Overviewsが**表示されないクエリでもCTRが41%低下**したとの報告がある。これはAI Overviews単独の効果でなく検索行動全体のシフトである可能性があり、因果関係の切り分けは困難（「議論あり」）
- ブランド名がAI Overviewsに含まれると、指名クエリのCTRが18%上昇するとの報告あり（業界ベンダー値、方向性参考）

**ゼロクリック時代のパブリッシャー対策チェックリスト**

- [ ] 指名検索を防御する: ブランド認知投資を継続する
- [ ] 直接オーディエンスを構築する: メルマガ・アプリ・SNS等の所有チャネルへ分散
- [ ] クエリ種別でコンテンツ戦略を分割: 汎用情報クエリへの投資比率を下げ、ブランド/比較/高購買意図クエリへ集中
- [ ] AI Overviewsへの引用を順位とは別に監視する（約70%が短期で入れ替わるため定期確認が必要）
- [ ] AI Modeの会話型フォーマットを今のうちにテストする
- [ ] コンテンツライセンス契約（AI企業との直接提携）を選択肢として検討する

## GEO（Generative Engine Optimization）

### 定義

**GEO（Generative Engine Optimization）**は、AIアンサーエンジン（ChatGPT・Perplexity・Google AI Overviews/Gemini・Claude等）が回答生成時に自社コンテンツを**引用・推奨・言及**するよう、コンテンツ構造とオンラインプレゼンスを最適化する実践。

- 用語の起源: 2023年11月のarXivプレプリント（Princeton主導）で正式提唱、KDD 2024で発表
- 古典SEO（オーガニック順位最適化）と並行して取り組むもので、代替ではない

### 研究で効果が示された手法

GEO論文は **GEO-bench**（9ドメイン・約1万クエリのベンチマーク）を構築し、**Position-Adjusted Word Count（PAWC）** と主観的印象スコアを引用品質指標として定義。9種の最適化手法を統計的に検証した。

**研究で可視性向上が示された手法（GEO論文の結果）**

| 手法 | 可視性向上（目安） | 備考 |
|------|-----------------|------|
| 引用文（Quotations）を追加 | 最大約+41% | 研究値。効果はドメイン依存 |
| 統計（Statistics）を追加 | 約+30〜40% | 特に「Law & Government」「Opinion」型のクエリで高い |
| 出典・引用（Cite Sources）を追加 | 約+30〜40% | 研究値 |
| 流暢性（Fluency）最適化 | 約+28% | 研究値 |
| 専門家の引用文（Expert quotations） | 約+35%前後 | 業界記事値（方向性参考） |
| 回答先出し（answer-first、冒頭200語以内） | 引用率に大幅向上傾向 | 業界記事値（方向性参考、約+67%との報告あり） |

**重要な注釈**
- これらの数値は**目安**であり、クエリ・ドメイン・エンジンにより効果は異なる
- 「業界記事値」はベンダー・コンサルタントブログ由来であり、GEO論文（査読研究）と同等の証拠水準ではない
- **反証された手法**: 従来SEOのキーワード詰め込みは、生成エンジンの文脈では低効果または逆効果

**LLMの引用メカニズム**
LLMは小さなテキストチャンクを抽出・利用するため、**ページ単位でなくパッセージ（一節）単位**で最適化することが重要。ページ全体が優れていても、個々のパッセージが抽出しにくい構造だと引用率が下がる。

### 引用されやすいコンテンツ構造チェックリスト

- [ ] 検証可能な**統計・数値**を本文に埋め込む（一次データが理想。出典を明記する）
- [ ] **権威ある一次ソースへの出典リンク**を付す（二次まとめより原論文・公式ドキュメント・一次データ）
- [ ] **専門家の引用文**を肩書き・資格付きで掲載（E-E-A-Tの権威性・信頼性を強化）
- [ ] **回答先出し（answer-first）**: 冒頭〜200語以内に結論・直接回答を配置する
- [ ] **見出し階層（H1/H2/H3）を明確に**・短い段落・箇条書き・表・明示的なQ&Aで構造化する
- [ ] バイライン・著者の経歴・資格を可視化する（権威性シグナル）
- [ ] パッセージ単位で独立して回答が成立するように各節を設計する

## AEO / LLMO

### 用語の整理（定義に議論あり）

AIアンサーエンジン最適化には複数の用語が混在しており、業界内で明確な標準定義はない。以下は現状の整理であり、定義は流動的（「議論あり」）。

| 用語 | 主な意味（現状の解釈） | 特徴 |
|------|---------------------|------|
| **GEO**（Generative Engine Optimization） | 生成エンジン全般での引用・推奨最適化（最広義） | 研究論文から生まれた用語 |
| **AEO**（Answer Engine Optimization） | 質問に直接答える構造化最適化（FAQスキーマ・Q&A形式重視） | フィーチャードスニペット最適化のAI版に近い |
| **LLMO** | 技術レイヤ（robots.txt・スキーマ・クローラ可達性）の最適化 | AIがコンテンツにアクセス・処理できる状態を担保 |

3者は重複が大きく、単一の包括的な概念としてGEOが使われる場面が増えつつある。ただし業界標準定義は未確立であり、ベンダーによって異なる意味で使用されていることに注意。

### 各AIエンジンの参照ソース傾向と施策

**ChatGPT / OpenAI**
- **索引**: 主にBingを利用（リアルタイムクエリでBing検索→上位取得→インライン引用で合成）
- **流入シェア**: 全AIリファラルの約78%（2025年6月時点）で最大
- **許可するクローラ**: GPTBot, OAI-SearchBot, ChatGPT-User
- **施策**: 統計入りのファクト密度が高いコンテンツ・一次研究へのリンク・購買リサーチのプロンプトを軸にした構成

**Perplexity**
- **索引**: PerplexityBot＋各種検索API。1クエリで20〜40 URLを精査してから選定
- **一次ソースへのバイアス**: 二次まとめより原論文・公式ドキュメント・一次データを約3.1倍引用する傾向（業界報告値、方向性参考）
- **施策**: モジュール化したセクション構造・関連質問への展開（query fan-out）・測定可能な成果を含むケーススタディ

**Claude / Anthropic**
- **参照範囲**: 広いソースプールから参照
- **リンクより言及**: ブランド言及率は97.3%とされるがURL直接引用は少ない傾向（業界ツール調査値、方向性参考）
- **許可するクローラ**: ClaudeBot, Claude-SearchBot, Claude-User
- **施策**: リンク配置よりブランド言及を優先・著者バイライン＋資格・一次体験によるE-E-A-T強化

**Google（Gemini / AI Overviews / AI Mode）**
- **前提**: オーガニック順位との相関が強い（引用の約92.36%が上位10位以内）
- **許可するクローラ**: Googlebot, Google-Extended
- **施策**: まずオーガニック順位を確保・FAQPageスキーマの実装・AI Modeの幅広い引用プールを狙った可視性向上（UGCも対象になりやすい）

**Copilot / Microsoft**
- Bingをバックエンドとして利用。汎用施策を踏襲し、GA4で `copilot.microsoft.com` を計測対象に追加して監視する

### 共通AEO/LLMO技術チェックリスト

- [ ] robots.txt で主要AIクローラを許可する（GPTBot, OAI-SearchBot, ChatGPT-User, PerplexityBot, ClaudeBot, Claude-SearchBot, Claude-User, Googlebot, Google-Extended 等）
- [ ] FAQPage スキーマ（JSON-LD）を実装する（リッチリザルト目的ではなく構造化データ・AI検索基盤として）
- [ ] XMLサイトマップの `lastmod` を正確に維持する
- [ ] Organization / Article / Product 等のスキーマで主要エンティティを明示する
- [ ] 著者・組織を `sameAs` で外部権威（Wikipedia/Wikidata/LinkedIn等）に接続する

**30日実装の型**
- W1: 技術基盤（robots.txtクローラ許可・スキーマ・サイトマップ更新）
- W2: 上位20ページを回答先出し＋統計/出典付きに書き換え
- W3: 購買プロンプト狙いの新規記事3〜5本＋専門家引用を追加
- W4: GA4 AIチャネル設定＋プラットフォーム別の手動引用テストを実施

## llms.txt

### 仕様・設置

`llms.txt` はサイトルート（`/llms.txt`）に置くMarkdownファイル。LLM（大規模言語モデル）向けにサイトの主要コンテンツとリンクを整理して提示する提案標準として2024〜2025年に注目された。`llms-full.txt`（全文版）という派生仕様もある。

**設置方法の基本**
- Markdownで主要ページへのリンクリストと概要を記載する
- ファイルサイズを小さく保ち、最重要コンテンツを優先順に並べる
- `llms-full.txt` はより詳細なコンテンツを含む大きなファイルで、設置コストが高い

### 採用状況・有効性（2026年6月時点）

**現状の客観的評価: 効果未実証・「議論あり」・現状ネガティブ寄り**

`llms.txt` の有効性については、2026年6月時点で強い議論があり、現状ではネガティブな評価が優勢。

**主要LLMプロバイダーの公式対応**
- **Google**: John Mueller（2025年6月）「どのAIシステムも現状 `llms.txt` を使っていない。サーバーログを見れば一目瞭然」。Gary Illyes（2025年7月）「Googleは未対応で対応予定もない」。Muellerは廃れた `keywords` メタタグになぞらえた
- **Google**: 2025年12月に開発者ドキュメントへ一時掲載されたものの即日削除。公式サポートなしとMuellerが明言
- その他の主要LLMプロバイダーも公式採用を発表していない

**定量的な調査結果**
- SE Rankingによる30万ドメイン分析: 採用率は約10.13%
- AI引用データで学習したXGBoostモデルでは、**`llms.txt` 変数を除いた方が予測精度が向上**（予測シグナルでなくノイズの可能性）
- AIクローラの関心も僅少: 90日間で5億超のAIボット訪問のうち `llms.txt` を直接狙ったのは408件のみ

**公平な評価**
> 設置コストは低いが、2026年6月時点で実効性の客観的証拠はほぼない。「やっても害は小さいが効果も実証されていない」が現状最も公平な評価。将来的にLLMプロバイダーが対応する可能性はゼロではないため、コストが低ければ設置しておくことは妨げないが、SEO施策の優先度は低い。

## 構造化データのAI引用効果

### 採用状況

- schema.org 構造化データの導入は4,500万ドメイン超（全登録ドメインの約12.4%）
- 2025年の業界的傾向として、スキーマは「SEO戦術」から「AI検索の基盤インフラ」へ位置づけが変化している（業界コンセンサス）

### AI引用への効果（肯定的主張と反証を峻別）

**肯定的な見解**
- MicrosoftはスキーマがLLMのコンテンツ理解を助けると公式に認めた
- ナレッジグラフで接地（grounding）したLLMは非構造データのみより**精度が300%高い**との主張（ベンダー由来、方向性参考）
- AIはエンティティグラフで文脈理解・情報検証・引用信頼度判定を行うとの見方がある
- FAQPage スキーマを持つページはAI Overviewsに約3.2倍引用されやすいとの報告（ベンダー記事値、独立検証なし）

**反証・懐疑的な見解**
- Search/Atlas（2024年12月）の調査: スキーマ網羅率と引用率に**相関なし**と報告。包括的スキーマのサイトが最小/無スキーマのサイトを一貫して上回るわけではない
- Evertune調査: 「スキーマ有無で決定的な差は出ない」傾向を示唆
- Search Engine Land: 「誇張に注意」と題した解説記事がスキーマのAI引用効果の過大評価を指摘

**公平な評価**
> スキーマは「機械可読性・エンティティ明確化の土台」として推奨されるが、**単独でAI引用を保証するものではない**。コンテンツ品質（統計・出典・専門家引用・回答先出し）との組み合わせが本質的に重要。スキーマのみへの過剰投資は費用対効果が低い可能性がある。

### 実務チェックリスト

- [ ] JSON-LDで Organization / Article / FAQPage / Product 等の主要タイプを実装してエンティティを明示する
- [ ] 著者・組織を `sameAs` で外部権威（Wikipedia/Wikidata/LinkedIn等）に接続しエンティティを強化する
- [ ] サイト内でエンティティ関係を一貫させ Content Knowledge Graph を形成する
- [ ] スキーマは「土台」と位置づけ、本文最適化（統計・引用・出典・回答先出し）と必ず併用する
- [ ] FAQPage スキーマをリッチリザルト目的でなく構造化・AI検索基盤として設置する（リッチリザルト目的のFAQPageは廃止済み）
- [ ] Googleリッチリザルトテスト・Schema.org バリデーターで実装を検証する

## クエリインテントとビジネスモデルへの影響

### クエリ種別によるリスク格差

AI Overviewsやゼロクリック化が与える影響はクエリの性質によって大きく異なる。

| クエリ種別 | ゼロクリック化リスク | 対策の方向性 |
|----------|-----------------|-------------|
| 汎用情報・How-to | 高（AIが直接回答しやすい） | 投資比率の見直し |
| ブランド指名 | 低〜中（AI引用でCTR上昇ケースあり） | ブランド認知投資・エンティティ強化 |
| 比較・検討 | 中 | 詳細な一次データ・比較表・一次体験を含める |
| 高購買意図（transactional） | 低（商業意図クエリはAIが慎重） | SEO＋コンバージョン最適化に注力 |
| ローカル | 中〜高（Ask Maps等の影響拡大） | GBP最適化・ローカルエンティティ強化 |

### ビジネスモデルへの中長期的影響

**トラフィック収益モデルへの影響**
- 広告収益がページビューに依存するパブリッシャーは、AI Overviewsによる大幅CTR低下（前述の数値参照）が直撃する可能性がある
- ゼロクリック傾向は「汎用情報提供」から「一次情報・体験・ツール提供」へのモデル転換を促す

**コンテンツライセンスの台頭**
- 大手メディア・パブリッシャーはAI企業との直接コンテンツライセンス契約（AP通信・News Corp等の事例）を選択する動きがある
- 一方でライセンスを拒否し訴訟に踏み切るメディアもあり、業界内での方針は分かれている

**所有チャネルへの移行加速**
- SEOトラフィックへの過度な依存を避け、メール・アプリ・コミュニティ等の直接オーディエンスへの移行が加速
- ブランド認知投資（指名検索の防御）がSEO戦略と並立して重要性を増している

**AIリファラルの質**
- AI経由の訪問はGoogle Organic比でコンバージョン率が高い傾向がある（あるB2B事例: ChatGPT流入のコンバージョン率15.9%、Perplexity 10.5%、Google Organic 1.76%）。ただしこれは単一事例であり、業種・サイト・計測設定によって大きく異なる
- 「量より質」を前提にKPIを設計し、AI流入のコンバージョン寄与を追跡することを推奨

## AI検索最適化 全体チェックリスト

### 基盤（既存SEOとの連動）

- [ ] オーガニック順位の土台を維持する（Google AI Overviews引用の約92%は上位10位以内が前提）
- [ ] E-E-A-T（Experience, Expertise, Authoritativeness, Trust）を全ページで強化する

### コンテンツ構造（GEO最適化）

- [ ] 回答先出し（answer-first）: 冒頭〜200語以内に結論を配置する
- [ ] 検証可能な統計・数値に出典リンクを付ける
- [ ] 専門家の引用文を肩書き・資格付きで掲載する
- [ ] パッセージ（一節）単位で独立した回答が成立するよう設計する
- [ ] 見出し階層を明確にし、短い段落・箇条書き・表を活用する

### 技術的アクセス可能性（LLMO）

- [ ] 主要AIクローラを robots.txt で許可する
- [ ] FAQPage・Article・Organization 等のスキーマを JSON-LD で実装する
- [ ] `sameAs` でエンティティを外部権威に接続する
- [ ] XMLサイトマップの `lastmod` を正確に維持する

### llms.txt

- [ ] 設置コストが低い場合は設置を検討する（優先度は低い・効果未実証）
- [ ] 効果に過度な期待をせず、他の施策を優先する

### 計測

- [ ] GA4でAIリファラル専用のカスタムチャネルグループを設定する（`chatgpt|openai|perplexity|gemini|copilot|claude|edgeservices` 等の正規表現）
- [ ] GSCの「AI Mode」アピアランスフィルタでGoogle AI面の露出を監視する
- [ ] プラットフォーム別に手動引用テストを定期実施する（ChatGPT/Perplexity/Geminiへ直接質問し自社引用を確認）
- [ ] 「引用シェア」「ブランド言及率」「AIリファラルのコンバージョン率」を従来CTRに加えてKPIとして追跡する
