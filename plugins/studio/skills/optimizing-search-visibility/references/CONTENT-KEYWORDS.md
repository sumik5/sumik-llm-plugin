## キーワードリサーチ

キーワードリサーチは「ユーザーがどんな言葉で検索しているか」を把握し、コンテンツ戦略の優先順位を決める起点となる作業だ。ボリュームや難易度の数値を見るだけでなく、そのKWを検索したユーザーの意図を読み解くことが本質。

### シードKW発見とロングテール展開

**シードKW**とは、事業・商品・サービスの中核を表す1〜2語の起点キーワード。これを軸にロングテールへ広げることで、現実的に上位表示できる候補群を整備する。

**ロングテールKWの特性**:
- 検索ボリュームは低いが、意図が明確で競合が少ない
- コンバージョン率が高い傾向がある（4語以上の具体的なフレーズ）
- 1つの上位ページが数十の関連ロングテールで上位に入ることもある = トピック網羅度の指標

**シードKW発見・ロングテール展開チェックリスト**:
- [ ] Googleオートコンプリート「アルファベット・スープ展開」（シード＋a〜z／あ〜ん）で候補を機械的に収集
- [ ] SERPの「他の人はこちらも検索（People Also Ask）」「関連キーワード」を抽出
- [ ] GSC（Google Search Console）のパフォーマンスレポートで**平均掲載順位8〜30位（2〜3ページ目）のクエリ**を抽出 — 既に関連性はあるが未最適化のロングテール候補
- [ ] GSCで「インプレッション > 10 かつ 4語以上」のクエリをフィルタ — 自然発生したロングテールを拾う
- [ ] 競合上位サイトのキーワードギャップ分析（自社未獲得・競合獲得中のKW）
- [ ] Q&Aサイト・コミュニティ・レビューから実際のユーザー言語を収集

### ボリューム・難易度・CPCの読み方

| 指標 | 意味 | 読み方の勘所 |
|------|------|------------|
| **検索ボリューム（Volume）** | 月間検索回数の推定値 | ツール値は推定であり絶対視しない。高ボリューム＝流入大だが競合大 |
| **キーワード難易度（KD）** | 0〜100（%）被リンク・権威ベースの上位獲得難度 | 0-30: 易（新規サイト可）／31-50: 中（一定の権威が必要）／51-70: 難（大きな労力）／71-100: 非常に難（高い権威が必要） |
| **CPC** | Google広告での1クリック単価 | 高CPC＝商業性・競合性が高い。低ボリュームでも商業意図が強いKWは高CPCになる（1CVの価値が高いため）→ 収益化狙いの優先候補に |

**実務上の優先順位付け**:
- [ ] 新規・低権威サイトは「KD低 × 意図明確 × 適度なボリューム」のロングテールから着手
- [ ] 商業価値の判断にCPCを補助指標として使う（高CPC＝売上に近いKW）
- [ ] ボリュームだけでなく「自サイトの権威で現実的に狙えるか」をKDで照合

### 検索意図グルーピング・意味的関連グルーピング

検索意図（Search Intent）は4種類に分類される。同じトピックでも意図が異なればコンテンツ形式を変える必要がある。

**4つの検索意図**:

| 意図 | 特徴 | 最適なコンテンツ形式 |
|------|------|-------------------|
| **Informational（情報収集型）** | 「〜とは」「やり方」「原因」 | ブログ記事・ガイド・FAQ |
| **Navigational（指名型）** | 特定サイト・ツール・ブランド名 | ブランドページ・TOPページ |
| **Commercial（商業調査型）** | 「おすすめ」「比較」「レビュー」 | 比較記事・ランキング |
| **Transactional（取引型）** | 「購入」「申込」「料金」 | 商品ページ・LP・カテゴリページ |

**グルーピング（クラスタリング）チェックリスト**:
- [ ] **同一SERPに出るKWは同一ページで狙う**（SERP上位の重複度でクラスタを判定するのが現在の主流）
- [ ] 意味的に類似するKW（同トピック・言い換え・同義語）をまとめて束ねる
- [ ] 検索意図（ユーザーゴール）で束ねる → 1意図1ページにマッピング
- [ ] クラスタごとに「ピラーページ＋クラスター記事」の内部リンク構造を設計し、トピック権威を積み上げる（詳細は `FOUNDATIONS-RANKING-EEAT.md`）
- [ ] 各クラスタを意図に応じたコンテンツ形式（ガイド・比較・商品ページ等）へ割り当て

### ツールの使い方

| ツール | 用途 | 費用 |
|--------|------|------|
| **GSC** | 自サイトの実クエリ・順位・CTR・インプレッションの一次データ源。新規KW発掘と既存ページ改善の起点 | 無料 |
| **Google Keyword Planner** | ボリューム帯・CPCの目安取得 | 無料（広告アカウント必要） |
| **Ahrefs / Semrush / Moz** | KD・競合ギャップ・SERP分析・クラスタリング | 有料 |
| **Ubersuggest 等** | 低コストでのKW調査補助 | 一部無料 |

**推奨フロー**: まずGSCで自サイトの実態を把握 → 有料ツールで競合・市場全体を補完する順で運用する。

## オンページ最適化

オンページ最適化は、個々のページがターゲットKWで上位表示されるよう、HTMLの各要素を整える作業。Google公式のSEOスターターガイドと業界標準を統合したベストプラクティスを示す。

### タイトルタグ

タイトルタグはSERPに表示されるページの「見出し」であり、ランキングシグナルとしても機能する最重要要素の一つ。

**タイトルタグ最適化チェックリスト**:
- [ ] **50〜60文字**を目安（超過すると検索結果で切り詰められる）
- [ ] ターゲットKWを含め、ページ内容を正確・簡潔・一意に表現する
- [ ] 全ページでユニークにする（重複させない）
- [ ] H1のテーマと整合させる（完全一致は不要）
- [ ] キーワードの詰め込み（stuffing）は厳禁

**注記**: Googleは独自にタイトルを書き換えることがある（H1や本文の内容を参照）。タイトルタグはあくまで「提案」として機能する側面もある。

### メタディスクリプション

メタディスクリプションはランキングへの直接影響はないが、CTR（クリック率）を通じて間接的にSEOに貢献する。

**メタディスクリプション最適化チェックリスト**:
- [ ] ページごとにユニークな1〜2文の要約を書く
- [ ] クリックを促す訴求を入れる（ベネフィット・独自性・行動喚起）
- [ ] **おおむね105〜160字程度**（表示は文脈・デバイスで変動）
- [ ] ターゲットKWを自然に含める（太字表示される場合あり）

### 見出し階層（H1〜H6）

見出し階層はコンテンツの構造をユーザーと検索エンジンの両方に伝える要素。

**見出し最適化チェックリスト**:
- [ ] **H1はページに1つ**のみ（ページのメインテーマを表す）
- [ ] H2でセクションを分け、H3で小節を設ける（トピック→サブトピックの階層）
- [ ] 見出しレベルを飛ばさない（H2→H4はNG。H4→H2の戻りは可）
- [ ] フォントサイズ調整やナビゲーション目的で見出しタグを使わない
- [ ] 長文は段落・セクションに分割して可読性を確保する

### URL設計

URLはSERPにも表示されるため、ユーザーにとっての分かりやすさが重要。

**URL最適化チェックリスト**:
- [ ] ユーザーに有用な語を含める（ランダムなID・数字の羅列を避ける）
- [ ] ディレクトリでトピック近接ページをグルーピング（例: `/blog/seo/`）
- [ ] 簡潔で説明的な構造（パンくずリストにも反映される）
- [ ] URLの変更は極力避ける（変更時は301リダイレクトを必ず設定）

### 画像alt属性

alt属性は画像の内容を検索エンジンとスクリーンリーダーに伝える属性。

**画像alt最適化チェックリスト**:
- [ ] 短く説明的なalt属性を全画像に設定する
- [ ] 画像ファイル名も内容を表す命名にする（`image001.jpg` ではなく `seo-checklist.jpg`）
- [ ] 装飾目的の画像は`alt=""`（空文字）を指定しスクリーンリーダーをスキップさせる
- [ ] キーワードの詰め込みは禁止

### 内部リンク

内部リンクはサイト内の「道路網」として、リンクエクイティの分配とユーザーの回遊性に寄与する。

**内部リンク最適化チェックリスト**:
- [ ] 関連トピックで読者に有益なページへリンクを設置する
- [ ] 新規コンテンツ → 既存関連コンテンツへリンクし発見性を高める
- [ ] 高権威ページから重要ページへリンクし順位向上を後押しする
- [ ] 過剰リンクを避ける（1ページに数十本は多過ぎ・スパム的に映る）
- [ ] **説明的なアンカーテキスト**を使う（「こちら」「詳細はこちら」等の汎用語を避ける）
- [ ] リンク先のページが解決する疑問をアンカーテキストで示すと、ユーザーとGoogleの双方に価値を伝えやすい

## コンテンツ網羅性と品質

### 網羅性とコンテンツギャップ

「網羅性（トピッククバレッジ）」とは、あるトピックに関連するサブトピック・疑問・ロングテールを包括的にカバーすることで、検索エンジンにとっての「トピック権威」を確立する考え方。

**コンテンツ網羅性チェックリスト**:
- [ ] 検索意図を満たすことが核心 — ユーザーが本当に探している答えを明確に提示する
- [ ] 関連するサブトピックとロングテールKWを自然に取り込む（トピック権威の醸成）
- [ ] 独自性・有用性・E-E-A-T（Experience・Expertise・Authoritativeness・Trust）を担保する
- [ ] 競合上位コンテンツが扱っているが自社が扱えていないトピック（コンテンツギャップ）を特定し埋める

**コンテンツギャップ分析チェックリスト**:
- [ ] 競合ページのH2/H3を抽出し自社との差分を可視化する
- [ ] 有料ツール（Ahrefs・Semrushのコンテンツギャップ機能）を活用
- [ ] GSCで検索インプレッションはあるが順位が低いKWを特定 → 既存コンテンツの充実か新規ページ作成を判断

### Googleが避けるべきとする事項

Googleのスパムポリシー・コアアップデートで繰り返し標的とされるパターンを排除する。

**禁止・回避事項チェックリスト**:
- [ ] キーワード詰め込み（キーワードスタッフィング）— スパムポリシー違反
- [ ] `<meta name="keywords">` タグ — Googleは使用しない（無効）
- [ ] KW狙いのみのドメイン名取得（パンくず表示以外ほぼ効果なし）
- [ ] 重複コンテンツ・スクレイピングコンテンツ
- [ ] 邪魔な広告・インタースティシャル（特にモバイル）
- [ ] UGC（コメント・フォーラム）へのリンクに`nofollow`付与漏れ
- [ ] AIを使って大量に薄いコンテンツを生産する Scaled content abuse — 2025-11アップデートで強化された取り締まり対象（詳細は `RISK-GOVERNANCE.md`）

## リンクビルディング

被リンク（バックリンク）は依然として重要なランキングシグナルだが、質・関連性が量を大きく上回る。2025年11月のGoogleコアアップデートではリンクスパムへの取り締まりが強化された。

### 現行ホワイトハット手法

**ホワイトハットリンクビルディングの考え方**: 無理やりリンクを「作る」のではなく、引用されるに値するコンテンツを「作り」プロモーションを通じて自然リンクを「誘発する」。

| 手法 | 概要 |
|------|------|
| **リンク獲得型アセット（Linkable Assets）** | 無料ツール・インフォグラフィック・独自データ・調査レポート・徹底ガイドなど、引用されやすい資産を作る |
| **デジタルPR** | メディア露出経由で被リンク獲得（データインサイト・プレスリリース・ジャーナリスト連携・情報提供）|
| **ブロークンリンクビルディング** | 他サイトのリンク切れを発見し、自社コンテンツを代替提案する（双方にメリット）|
| **質の高いゲスト投稿** | 関連性・権威性のあるサイトへの寄稿（低品質サイトへの量産は逆効果）|

**ホワイトハット実行チェックリスト**:
- [ ] まず「引用したくなるアセット」を作り、プロモーションで自然リンクを誘発する
- [ ] 関連性・権威性のあるドメインからのリンクを優先する（量より質）
- [ ] アンカーテキストは自然な分散を維持する（完全一致KWの過多は危険）
- [ ] デジタルPRでデータ・独自調査を起点にメディア露出を狙う

### 避けるべき手法（2025-11アップデートの標的）

2025年11月のGoogleアップデートではリンクスパム・人工リンク全般への検出精度が大幅に向上した。以下のパターンはリスクが極めて高い。

**禁止・高リスク手法チェックリスト**:
- [ ] **PBN（プライベートブログネットワーク）** — 検出精度が向上し壊滅的なペナルティリスク
- [ ] 低品質サイトへの大量ゲスト投稿（量産＝グレー〜ブラック）
- [ ] リンク目的で低権威サイトに薄い記事を量産するパターン
- [ ] 有料リンク（rel指定なし）・リンクファーム・過剰な相互リンク
- [ ] アンカーテキスト完全一致の不自然な集中

## ローカルSEO

ローカルSEOは、特定の地域でビジネスを展開する事業者が、ローカル検索（「近くの〇〇」「地域名＋業種」等）での可視性を高めるための施策群。GBP（Googleビジネスプロフィール）の最適化が中心となる。

### GBP（Googleビジネスプロフィール）最適化

GBPはローカル可視性の中心ハブ。リアルタイム更新・AI提案投稿・サービスメニュー・予測プロンプトに対応し、ローカルパック（地図上の3件表示）への掲載を左右する最重要要素。

**GBP最適化チェックリスト**:
- [ ] プロフィールを完全に埋める（カテゴリ・営業時間・サービス・属性・写真・説明文）
- [ ] 投稿・サービスメニューを定期的に更新する
- [ ] 商品・サービスごとの情報を充実させる
- [ ] 多拠点の場合はNAP（後述）・営業時間を拠点別にユニーク化し、対応するローカルページにリンクする
- [ ] 写真・動画を定期追加する（新鮮さのシグナルになる）

### ローカルパック（3-Pack）ランキング要因

Googleのローカル検索結果上部に表示される「地図＋3件」の表示枠がローカルパック（3-Pack）。以下の5要因が主要なランキングシグナル。

| 要因 | 内容 |
|------|------|
| **距離（Distance）** | 検索地点とビジネス所在地の物理的距離 |
| **関連性（Relevance）** | 検索クエリとGBPカテゴリ・説明の一致度 |
| **知名度（Prominence）** | 被リンク数・レビュー数・オンライン言及量 |
| **レビュー** | 評価数・星評価・新しさ・返信の有無 |
| **エンゲージメント** | クリック・電話・経路検索等のユーザー操作数 |

**ローカルパック最適化チェックリスト**:
- [ ] GBPの完全性を確保する（未記入フィールドをゼロに）
- [ ] レビュー獲得の継続サイクルを設計する（数・新しさ・評価すべてが重要）
- [ ] NAP一貫性を確保する（後述）
- [ ] ウェブサイト側のローカルコンテンツを充実させる（店舗・サービスエリアページ）

### NAP一貫性

NAP（Name・Address・Phone）の表記がオンライン上で一致していないと、検索エンジンが「同一事業者」と認識できず、ローカルランキングを押し下げる要因となる。

**NAP一貫性チェックリスト**:
- [ ] **Name / Address / Phone をオンライン全所在地で完全一致させる**
- [ ] 「Street」vs「St.」のような表記揺れも排除する（検索エンジンの混乱要因）
- [ ] GBP・ウェブサイトフッター・各ディレクトリ・引用元（citation）を横断的に確認・統一する
- [ ] NAP情報を構造化データ（`LocalBusiness` スキーマ）でマークアップする

### レビュー管理

レビューは「知名度」シグナルとしてGoogleが明示的に言及している要素。継続的なレビュー獲得が不可欠。

**レビュー管理チェックリスト**:
- [ ] レビュー数・新しさ・星評価が競合より劣る状況を改善するサイクルを設計する
- [ ] 購入・来店・サービス提供後にレビューを依頼するフロー（メール・SMS）を整備する
- [ ] 全レビューに返信してエンゲージメントを示す（特にネガティブレビューは丁寧に対応）
- [ ] 偽レビューの依頼・購入は厳禁（Googleのポリシー違反）

## EC商品ページ最適化

ECサイト固有のSEO課題（ファセットナビゲーションによるURL爆発・在庫切れ対応）は `TECHNICAL-SEO.md` を参照。ここでは商品ページのコンテンツ品質に焦点を当てる。

### 商品ページコンテンツ最適化

**商品ページ最適化チェックリスト**:
- [ ] **独自の商品説明を書く**（メーカー提供のコピーをそのまま流用すると重複コンテンツになる）
- [ ] 商品スキーマ（`Product` 構造化データ）で価格・在庫状況・レビューをマークアップする
- [ ] 高品質画像＋全画像に説明的なalt属性を設定する
- [ ] 顧客レビュー・UGC（ユーザー生成コンテンツ）を積極的に取り込む
- [ ] 関連商品・よく一緒に購入される商品へのリンクを設置する
- [ ] 検索意図に合致したフィルタ済みページをインデックスし、ロングテール（例: 「メンズ ランニングシューズ 5000円以下」）での上位表示を狙う

**コンテンツの差別化観点**:
- 使用方法・活用シーン・対象ユーザーを詳しく記述する
- FAQ（よくある質問）を商品ページに組み込み検索意図を多面的にカバーする
- スペック表は構造化して視認性を高める（テーブル形式）
