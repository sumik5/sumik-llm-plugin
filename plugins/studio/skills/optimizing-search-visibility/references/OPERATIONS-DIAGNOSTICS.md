## 既存サイト資産の監査と意思決定

新規記事を積み増す前に、既にあるページ群を**3区分**に仕分けて優先順位を決める。仕分けをせず一律に「増やす」方向に走ると、質の低いページが積み上がり品質比率（quality ratio）が悪化する。

### 3区分監査フレーム

| 区分 | 対象 | 対応方針 |
|---|---|---|
| **維持** | 上位表示・成果を出している記事 | 定期モニタリングのみ。手を加えて壊さない |
| **改善（統合を含む）** | 中位表示・伸びしろがある記事群 | リライト・重複記事の統合・内部リンク強化 |
| **手を付けない** | 成果が乏しく改善余地も薄い記事 | 剪定判断（次節）へ回すか、明確に放置と決める |

- [ ] 全記事をGSC掲載順位・表示回数・クリック数でエクスポートし、この3区分で棚卸しする
- [ ] 「改善」区分は次節の剪定判断（統合/リライト対象）と重ねて確認する
- [ ] 棚卸しは一度きりで終わらせず、後述の運用ケイデンスに組み込む

### 改善の優先順位序列

やることを全部同時に始めない。効果の波及範囲が広い施策から着手する。

```
1. 信用（トラスト系ページ・発信主体の明示）の土台固め
   └─ サイト全体のE-E-A-T評価に波及するため最優先
2. 主力記事の鮮度・経験（Experience）更新
   └─ 既に評価されている資産の劣化を防ぐ
3. 薄い記事の整理（次節のcontent pruning）
   └─ 品質比率を悪化させている要因を除去
4. 導線（内部リンク）の再設計
   └─ 整理後のページ構成に合わせてリンクを再配線
5. 技術施策（クロール・構造化データ等の細部）
   └─ コンテンツ・信用の土台が固まってから着手すると投資対効果が高い
```

> 技術施策を後段に置くのは「軽視してよい」という意味ではない。クロールエラー等の致命的な技術問題は別途最優先で是正する（`TECHNICAL-SEO.md`）。ここでの序列は「同程度の緊急度なら何から手を付けるか」の判断軸である。

### 既存記事を「育てる」戦略

新規記事を増やすより、**既存記事を厚くする**方が投資対効果が高い局面が多い。特に検索結果2ページ目付近（10〜20位）にいる記事は、後一押しで1ページ目に届く可能性がある「伸びしろの塊」である。

- [ ] GSCで**10〜20位**のクエリ・ページを抽出する（クイックウィン候補）
- [ ] 該当ページの検索意図とのズレ・網羅性ギャップを確認する（`CONTENT-KEYWORDS.md` § コンテンツ網羅性と品質）
- [ ] 「新規記事を1本作る工数」と「既存10〜20位記事を1本改善する工数」を比較し、後者を優先する運用を基本線にする

## コンテンツ剪定（content pruning）

低品質・低成果のページを放置すると、サイト全体の品質評価に悪影響を及ぼしうる。ただし機械的な削除は逆効果になりうるため、判断木に沿って進める。

### 低品質記事の発見（GSCでの洗い出し基準）

- [ ] 表示回数はあるがクリックがほぼゼロのページ
- [ ] 掲載順位50位以下で長期間（半年以上）停滞しているページ
- [ ] 同一トピックで複数記事が競合し合っている（カニバリゼーション）ページ群
- [ ] 公開から1年以上更新がなく、トラフィックが継続的に逓減しているページ

### 削除/統合/リライト/noindex の4択判断木

```
Q1. このページにしか無いユニークな価値があるか？
    No  → Q2へ
    Yes → 「維持」または「リライト」で価値を強化

Q2. 他の自社記事と内容が重複しているか？
    Yes → 「統合」（正規記事へ301リダイレクト）
    No  → Q3へ

Q3. 情報が古いだけで、構成・検索意図の合致は妥当か？
    Yes → 「リライト」
    No  → Q4へ

Q4. 検索意図には応えているが、検索結果に出す必要がないページか？
    （社内向け・重複性の低い薄いユーティリティページ等）
    Yes → 「noindex」
    No  → 「削除」（外部被リンクの有無を必ず確認してから実行）
```

### やってはいけないこと

- 🔴 **一括大量削除**: 同時多発的なURL削除・noindex化はクロールバジェットを圧迫し、サイト全体の信頼シグナルを悪化させるリスクがある。段階的に実施する
- 🔴 **被リンクのあるページをリダイレクトなしで削除**: リンクエクイティ（被リンクが持つ評価の蓄積）を無駄に失う。削除前に外部被リンクの有無を確認し、ある場合は関連性の高いページへ301リダイレクトする
- **トラフィックの少なさだけで機械的に判断しない**: ロングテールで着実に成果を出しているページを誤って削除するリスクがある。上記4択判断木を必ず経由する

## 診断ワークフロー

### アクセス減の原因切り分け6パターンと手順

**🔴 原因を特定しないまま新しい施策を追加しない。** 原因ごとに打つべき手が異なり、誤った施策は工数を浪費するだけでなく状況を悪化させることもある。

| # | パターン | 見分け方の手がかり |
|---|---|---|
| 1 | アルゴリズムアップデートによる順位変動 | 変動タイミングとGoogleのアップデート実施期間が一致するか（`FOUNDATIONS-RANKING-EEAT.md` § アップデート年表と突合） |
| 2 | 季節性・トレンド変化 | 検索需要自体（インプレッション総量）が減っているか、過去の同時期と比較する |
| 3 | 競合の新規参入・強化 | 対象クエリのSERPで自社より上位に新しい競合が出現していないか |
| 4 | 技術的問題 | インデックスカバレッジレポートで除外理由が増えていないか、直近のサイト変更・移行の副作用がないか |
| 5 | AI Overviews等によるゼロクリック化 | 表示回数（インプレッション）は据え置きでクリックのみ減少しているか（`MEASUREMENT.md` § ゼロクリック計測の限界） |
| 6 | コンテンツの陳腐化 | 内容の更新日・検索意図との現在のズレを確認する |

**診断の手順（この順で切り分ける）**:

1. GSCでインプレッションとクリックの推移を**分離**して見る（両方減 vs クリックのみ減で原因群が変わる）
2. 順位変動の時期とGoogleのアップデート実施履歴を突合する
3. 対象ページの最終更新日・内容の鮮度・検索意図との現在のズレを確認する
4. インデックスカバレッジ・クロールエラーの有無を技術面から確認する
5. 上記1〜4で説明がつかない場合のみ、競合の新規参入・強化を疑う

### 生成AIを診断コンサルとして使う

自社の現状認識の言語化・競合との比較・文章上の弱点抽出を、無料の壁打ち相手として生成AIに行わせる運用パターンがある。本ファイルでは「診断ワークフローのどの工程でAIを使うか」の位置づけのみを扱い、具体的なプロンプト設計・出力の扱い方の注意点は `GENAI-WORKFLOWS.md` を参照する。

> 生成AIによる診断はあくまで補助であり、上記6パターンの切り分け手順を代替しない。AIの回答を一次データ（GSC・GA4の実測値）の裏取りなしに鵜呑みにしないこと。

## 高難度キーワード攻略の時系列運用モデル

弱小ドメインから競合の強いキーワードで上位表示を狙う場合、施策を並行させるのではなく**時系列で投下順序を変える**運用モデルが有効とされる。

> 🔴 以下の各フェーズ・期間はあくまで**事例依存の目安**であり、ドメインの強さ・競合密度・業種によって大きく変動する。断定的な保証ではなく、計画立案のための出発点として扱うこと。

### 量→質→権威の3フェーズ

| フェーズ | 主眼 | 典型的な内容 |
|---|---|---|
| **1. 量** | インデックス面積の確保 | 対象トピッククラスターの記事群を一定量揃え、サイトが当該領域をカバーしていることをシグナルとして示す |
| **2. 質** | 個々のページの評価向上 | 量が揃った記事群をリライトし、E-E-A-T（特に経験・専門性）を強化。上位表示済みページの厚みを増す |
| **3. 権威** | 外部権威の獲得 | 質が伴った段階で被リンク獲得・言及獲得（後述の多角チャネル展開）へ投資の重心を移す |

- 質が伴わない段階で権威獲得（被リンク営業等）に先行投資しても、リンク先のコンテンツが弱く効果が薄い
- フェーズは厳密に順送りである必要はないが、「量が全く無い段階で権威投資を最優先する」逆転は避ける

### 競合分析の定量チェック項目（上位10サイト構造分解）

狙うキーワードのSERP上位10サイトを構造的に分解し、参入難易度と打ち手を定量的に把握する。

- [ ] 文字数・コンテンツ量の分布（上位10サイトの中央値・レンジ）
- [ ] 見出し構成（H2/H3の数・扱っているサブトピックの網羅範囲）
- [ ] ドメインの年齢・第三者ドメイン指標（DR/UR。Google公式指標でない旨に留意。`MEASUREMENT.md` §第三者ドメイン指標）
- [ ] 被リンク数・被リンク元の質
- [ ] コンテンツの更新頻度
- [ ] 著者情報・監修者情報の明記有無（E-E-A-T実装粒度）
- [ ] 構造化データの実装有無
- [ ] 内部リンクの本数・アンカーテキストの傾向

### 1位獲得後の維持・拡大運用

1位到達はゴールではなく次の運用フェーズの開始点。

- [ ] モニタリング頻度を上げる（順位低下の早期検知）
- [ ] CRO（コンバージョン率最適化）へ投資の重心を移す（順位向上の余地が縮小した分、流入の質転換に投資する）
- [ ] マルチチャネル展開（SNS・メルマガ・広告等）で総合的な露出を維持し、単一チャネル依存を避ける
- [ ] 競合の追随を前提に、コンテンツ更新・拡張を継続する（1位を「維持」区分に固定して放置しない）

## 実行ロードマップと運用サイクル

### 90/180/365日ロードマップ

> 🔴 以下の期間・区切りは**事例依存の目安のテンプレート**であり、必達スケジュールではない。実際のドメイン状況に応じて期間配分を調整すること。

| 期間 | 主な焦点 |
|---|---|
| **〜90日** | 基盤構築。技術監査・E-E-A-T土台整備・優先キーワード群の記事着手 |
| **〜180日** | 拡張。コンテンツ拡充・内部リンク強化・初期成果の観測とリライト着手 |
| **〜365日** | 定着・拡大。高難度キーワードへの到達・権威獲得施策・運用体制の定常化 |

### リライトループ

既存記事の改善は一度きりでなく、観測を挟んだループとして運用する。

```
公開・改訂 → 30日観測 → 補筆・改善 → 再観測 → （必要なら再度補筆）
```

- 「30日」は検索エンジンが新規・更新コンテンツを評価するまでに要する典型的な期間の目安であり、絶対的な固定値ではない
- 観測せずに次の補筆を重ねると、どの変更が効いたか分からなくなる。1サイクルごとに変更点と結果を記録する

### 継続運用ケイデンス

| 頻度 | 実施内容 |
|---|---|
| **月次** | 順位・トラフィックのレビュー、クイックウィン候補（10〜20位ページ）の抽出 |
| **四半期** | コンテンツ監査（3区分棚卸しの再実施）・優先順位の見直し |
| **半期** | 競合分析の更新（構造分解の再実施） |
| **年次** | 戦略全体の見直し（アルゴリズム動向・事業目標との再整合） |

## コンテンツ・オペレーション体制

大量の記事を継続的に運用する体制では、個々の施策より**運用の仕組み**が成果を左右する。

### 記事管理シート/重複回避/執筆校正フロー/カレンダー

- [ ] 記事管理シート（企画/執筆中/校正中/公開済み等のステータス管理）を用意する
- [ ] 新規記事の企画前に既存記事との重複（カニバリゼーション）チェックを行う
- [ ] 執筆→校正→公開の役割分担フローを明文化する（執筆者と校正者を分離する）
- [ ] 編集カレンダーで公開ペースを管理し、量産の波を平準化する

### 被リンクを生む仕組み化（多角チャネル）

被リンク獲得を単発の施策ではなく、継続する仕組みとして設計する。

- [ ] SNSでの露出を継続的な導線として設計する
- [ ] ウェビナー・オンラインイベントを情報発信とリンク獲得の機会にする
- [ ] プレスリリースを配信する（設計指針は `AI-SEARCH-GEO.md` § AI引用源の獲得も参照）
- [ ] 業界イベント登壇・専門家コミュニティでの発信を継続的なチャネルとして育てる

## 「追わないキーワード」の割り切り

すべてのキーワードを追う必要はない。以下の基準に該当するキーワードは、意図的に「追わない」と割り切ることが有効な資源配分につながる。

- 商業的意図（コンバージョンへのつながりやすさ）が事業と噛み合わない
- 競合の参入障壁が極端に高く、投下リソースに対して見合う成果が見込めない
- 自社のトピック権威（トピカルオーソリティ）と関連性が薄く、獲得しても事業へのシナジーが乏しい

キーワード選定そのものの基準・リサーチ手法は `CONTENT-KEYWORDS.md` § キーワードリサーチを参照。本節はその選定結果を「運用上どう割り切るか」の判断のみを扱う。

## ツール投資の見極め

LLMO専用ツール・AI可視性計測ツール・高額な外部コンサルへの投資は、**基本的なSEO（E-E-A-T・テクニカルの健全性・検索意図充足）が固まってから**検討する。

- 基礎が伴わない状態でツール投資を先行させても、測定対象・改善対象となる土台が無いため効果が出にくい
- ツールは診断・計測を効率化する手段であり、施策そのものを代替しない
- 投資判断の前に、本ファイル §1「既存サイト資産の監査と意思決定」で棚卸しを済ませ、土台の欠落がないか確認する
