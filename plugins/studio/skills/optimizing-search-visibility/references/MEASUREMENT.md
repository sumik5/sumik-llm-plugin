## SEO KPI設計

KPI（Key Performance Indicator）は「何のために計測するか」という目的から逆算して設計する。「順位が上がった」は観察であり成果ではない。ビジネス成果（収益・CV・ROI）に直結した指標を中心に据えることが重要。

### KPIの階層（順位・流入・CV・ROI）

SEO KPIは4階層に整理できる。下位は観察指標、上位はビジネス成果指標として使い分ける。

| 階層 | 指標 | 役割 |
|------|------|------|
| **ランキング指標** | キーワード順位・SERP可視性スコア | 施策の方向性確認。ビジネス成果との直結は弱い |
| **流入指標** | オーガニックセッション数・新規ユーザー数・インプレッション・CTR | 検索結果からの集客量。SEOの「入り口」指標 |
| **エンゲージメント・CV指標** | コンバージョン率・キーイベント数・エンゲージメント率 | SEO流入の「質」。ビジネス成果への橋渡し |
| **ROI・収益指標** | オーガニック収益・LTV:CAC比・SEO ROI | 投資対効果。経営層への報告指標 |

**コアKPI一覧（推奨）**:
- オーガニックセッション数
- キーワード順位（主要クラスタ）
- CTR（クリック率）
- オーガニックコンバージョン率 ← **ビジネス価値証明の最重要KPI**
- オーガニック収益・ゴール完了数
- 被リンク成長率

**ステージ別KPI（事業フェーズで変える）**:

| フェーズ | 重点KPI |
|---------|---------|
| 立上期（シード） | 流入量・メール登録数 |
| 成長期（アーリー〜Series A） | デモ申込数・パイプライン寄与 |
| 拡大期（グロース〜） | チャネルROI・オーガニックのLTV:CAC（有料チャネルとの比較） |

**二軸計測（2026年〜の標準）**:
- [ ] 従来SEO指標（クリックベースSERPでの成果）
- [ ] **AI可視性KPI（AI回答内の引用・ブランド言及＝AEO/GEO）** — 後述「AI検索流入の計測」「AI可視性計測の独自軸」参照（Mentions/Citationsの分離計測を含む）

> KPI設計の優先順位付けや事業目標との接続方法については、`product:practicing-product-management` が詳しい手順を提供している。

### ROI算出

**基本定義**: SEO ROI ＝（SEO施策が生む収益 − SEO投資）÷ SEO投資

**ROI算出チェックリスト**:
- [ ] 直接収益指標（オーガニック収益・CVR・AOV・CPA）＋補助指標を統合した一枚のフレームに整理する
- [ ] **マルチタッチアトリビューション**で各接点に按分する（単一チャネル帰属は過大/過小評価を生む）
- [ ] 単一期間でなく**トレンド**で評価する（購買サイクルが長いほどロングビューが重要）

**ベンチマーク目安**（あくまで参考値・業種・競合状況で大きく変動）:
- ROI 300%以上が良好の目安
- LTV:CAC = 3:1以上が持続可能な基準

#### マルチタッチアトリビューションの配分テンプレート

大半の見込み客は購買行動の前に複数経路で複数回サイトを訪れる。last-clickだけで評価すると「最初にオーガニック検索で発見→後日リターゲティング広告で再来訪してCV」のようなケースで、オーガニックに一切クレジットが付かない。これを避けるための**first/last重視型の配分テンプレート（一例）**:

| 接点 | 配分 | 意図 |
|------|------|------|
| **First（初回接点）** | 50% | 認知のきっかけを作ったチャネルを重く評価 |
| **Second（2番目）** | 10% | 検討開始への寄与 |
| **Third〜中間** | 合計10% | 中間接点で均等割り（接点数で按分） |
| **Final（直前接点）** | 30% | 最終CVの引き金となったチャネル |

- このモデルは不正確で合計が±100%前後にぶれることもあるが、**全チャネルの重要性を経営層に伝える**ことが主目的（厳密な配分の正確さより、単一チャネル帰属の弊害を補正することに価値がある）。
- GA4のようなlast-click環境では「先にorganic来訪→後にsocial再マーケ」でもorganicにクレジットが付かない。多くの組織はプラットフォームが対応済でもマルチタッチモデルを適用していない。
- **assisted conversions（間接コンバージョン）**の概念を関係者に伝えるだけでも、直接CVに現れないチャネル（コンテンツSEO・SNS等）への投資を正当化しやすくなる。

> マルチチャネルのROI比較・投資優先順位付けには `product:practicing-product-management` のフレームワークが活用できる。

## GA4（Google Analytics 4）

GA4はWebサイトのユーザー行動を計測する標準ツール。2024〜2025年に用語・機能が変更されたため、旧Universal Analyticsの知識をそのまま適用すると誤計測が生じる。

### 基本指標と設定

**2024〜2025年の重要な用語変更**:
- 「コンバージョンイベント」 → **「キーイベント（Key Event）」** に名称変更（GA4上でスターマーク★でキーイベント化し、コンバージョン指標として集計される）

**主要指標の定義**:

| 指標 | 定義 |
|------|------|
| **キーイベント（Key Events）** | ビジネス価値の高いアクション（例: `generate_lead`）。カスタマイズ・インポート・優先付けが柔軟にできる |
| **イベント数** | クリック・動画再生・フォーム送信等の総アクション数 |
| **エンゲージメント率** | 10秒超継続 / コンバージョン発生 / 複数ページ閲覧のいずれかを満たすセッション割合 |
| **直帰率（GA4版）** | 10秒以内かつ無操作・単一ページで離脱したセッション割合（UA版とは定義が異なる） |
| **新規ユーザー** | 流入の増減監視に有効 |

**SEO計測のためのGA4設定チェックリスト**:
- [ ] イベント命名は一貫した小文字スネークケース（例: `generate_lead`、`file_download`）
- [ ] ビジネス成果と相関する少数のイベントを選びキーイベント化する（価値主導アプローチ）
- [ ] **オーガニックトラフィックのセグメント・比較**を作成し、検索流入のコンバージョンを分離する
- [ ] マイクロCV（メール登録・PDF DL）とマクロCV（購入・デモ申込）の双方を計測する
- [ ] オーガニックコンバージョンレポートでSEO施策の収益寄与を可視化する

### GA4の限界

GA4はSEO計測の主力ツールだが、以下の構造的な限界がある。

**GA4の構造的限界**:
- GA4には専用の「AI」チャネルがなく、**生成AI（ChatGPT・Perplexity等）からの流入はreferralに分類**される（オーガニック検索ではない）。カスタムチャネルグループの設定が必須
- ChatGPTが `utm_source=chatgpt.com` を渡し始めたのは2025年6月以降 → それ以前の流入・モバイルアプリ経由・ゼロクリック（AIが答えてクリックしない）は捕捉できない
- **実際のAI影響は計測値の2〜3倍と推定される**（ある業界調査による）

### 「direct」トラフィックの正体（計測の落とし穴）

GA4のようなJavaScriptベースの計測で「direct」は「URL直接入力」ではなく、実態として**「発生源が特定できなかった」を意味する分析用語**である。真のdirect（ブラウザにURLを手入力）は極めて稀で、directが大きい場合はその裏に取りこぼした流入経路が隠れている。AI流入計測（後述）はチャネルを正確に切り分ける前提に立つため、directの正体を理解し最小化しておくことが土台になる。

**directに紛れ込む典型原因**:
- cookie / JavaScript が無効、またはトラッキングコードが未読込
- UTMパラメータを付けていないSNS投稿・メール・メッセージアプリ経由のリンク
- ブックマーク・UTM無しのQRコードからのアクセス
- リダイレクトがURLからUTMパラメータを剥離してしまう
- リダイレクトの発生源を計測できない・タグ設定の不備

**directを最小化するチェックリスト**:
- [ ] 自社が配信するSNS・メール・広告の全リンクにUTMパラメータを付与する
- [ ] QRコード・短縮URLにもUTMを埋め込む（リダイレクトでの剥離有無を実機確認）
- [ ] リダイレクト経路でUTMが保持されるか検証し、剥離するものは修正する
- [ ] directセッションの着地ページ・地域分布を点検し、特定経路の取りこぼしを推定する

## Google Search Console（GSC）

GSCは自サイトのSearch API経由の一次データ源（クエリ・インプレッション・クリック・順位）。GA4と組み合わせてSEO分析の基盤を構成する。

### 主要レポートと活用

**2025年の主要アップデート**:
- **AIパフォーマンスレポート設定（2025年12月）**: 自然言語プロンプトでフィルタ・比較・指標を自動生成
- **インデックス/カバレッジ詳細化（2025年11月）**: 「除外理由」がより詳細化（`Crawled–Not Indexed` に「低品質シグナル」「独自性不足」等のサブ理由を追加）、クロール頻度トレンドを可視化
- **Search Console Insightsのソーシャル統合**: 対象プロパティで検出したソーシャル指標を同一ダッシュボードで閲覧

**GSC活用チェックリスト**:
- [ ] パフォーマンスレポートを**週次**で確認し、可視性の低下を早期検知する
- [ ] 新規ページ公開後はインデックスレポートを確認する（意図しないnoindex・技術エラーの検出）
- [ ] サイトマップ送信ページにフィルタし、インデックス優先度を管理する
- [ ] **クイックウィン**: 高インプレッション・掲載順位4〜10位のページを特定し上位3位以内を狙う（既にある程度評価されているページへの改善投資）
- [ ] 主要クエリの低CTRを分析し、タイトル・メタディスクリプション・編集角度を改善する
- [ ] リンクレポートは**月次**で確認する

### GSCの限界

**GSCの構造的限界**:
- AI Overviewsのクリックは通常オーガニックと**同一カテゴリに混在**し、長らく明確な分離フィルタが存在しなかった
- 2026年初頭にGSCへ「AI Mode」検索アピアランスフィルタが追加され、Google AIサービス由来のインプレッション・クリックを分離可能になった — ただし**Google自身のAI機能のみ対象**であり、ChatGPT・Perplexity・Claude等は追跡対象外
- GSCのデータはサンプリングされており、特にロングテールKWは表示されないクエリが多い

## ランクトラッキング

順位は「入り口指標」であり成果そのものではないが、施策の方向性確認・競合比較に不可欠。単発の数値ではなくトレンドで判断する。

**ランクトラッキングチェックリスト**:
- [ ] 主要KWクラスタの順位を継続的にトラッキングする（週次または月次）
- [ ] GSCの平均掲載順位と専用ランクトラッカーを**併用**する（GSCは平均値のため単体では解像度が低い）
- [ ] デバイス別（PC・モバイル）・地域別の順位差を把握する
- [ ] 競合上位サイトの順位動向を比較トラッキングする

**推奨アプローチ**:
- クラスタ単位で「クラスタ内上位KWの平均順位」を追跡すると、ページ単位よりトレンドが安定する
- 大きな順位変動は「アルゴリズムアップデート」「コンテンツ変更」「被リンク増減」と紐付けてログを残す

## AI検索流入の計測

生成AIを使った検索（ChatGPT・Perplexity・Google AI Overviews等）が普及するにつれ、従来のGA4・GSCのみでは全貌を捉えられなくなっている。AI検索の「引用」はゼロクリック（クリックされずに回答として消費される）が多く、計測の構造的限界が存在する。

### ゼロクリック計測の限界

AI Overviewsをはじめとする生成AI回答では、**ユーザーが答えを得てもリンクをクリックしない**ケースが多い。このゼロクリック問題は以下の課題をもたらす。

**ゼロクリック計測の主な限界**:

| 限界 | 内容 |
|------|------|
| **GA4でのAI流入捕捉漏れ** | 生成AI流入はreferralに分類。ゼロクリックはサイトに来ないため計測不能 |
| **GSCのゼロクリック非計測** | クリックが発生しないとGSCにはインプレッションのみ記録。AI引用 ≠ クリック |
| **ChatGPT等の外部AI計測不全** | GSCのAI Modeフィルタは「Google AIのみ」が対象。ChatGPT・Perplexity等は対象外 |
| **推定値の不確実性** | 実際のAI影響は計測値の2〜3倍との推定もあるが、正確な把握は困難 |

**参考：AI Overviewsによるトラフィック影響の規模感（各調査報告より）**:
- AI Overviewsが表示されるクエリでオーガニックCTRが大幅低下するとの複数報告がある
- ただし影響は業種・クエリタイプ・ブランド強度によって大きく異なる
- 「AI Overviewに引用されたブランドはオーガニッククリックが増加した」との報告も存在する（ある業界調査による）

### 計測の実装チェックリスト

現時点で実施可能な計測アプローチを組み合わせる。計測できるものと計測できないものを明確に区別し、「計測値の範囲外で起きていること」を補助的手段で把握する構造にする。

**GA4でのAI流入セグメント設定**:
- [ ] GA4でセッションソースの正規表現セグメント「AI Traffic」を作成する（例: `chatgpt|openai|perplexity|gemini|copilot|claude|edgeservices` 等）
- [ ] referralを上書きするカスタムチャネルグループを定義する（手動チャネルグループ機能）
- [ ] コンバージョン質に注目する（参考事例：ChatGPT流入の高いCVR報告あり — GA4の流入量より質で評価）

**GSCでのAI面モニタリング**:
- [ ] GSCの「AI Mode」検索アピアランスフィルタ（2026年初頭追加）でGoogle AI面の露出を監視する
- [ ] 「AI Mode」フィルタはGoogle AIのみ対象であり、外部生成AIは別途手動で確認が必要と認識しておく

**補助的な計測手段（定量化が難しい部分を補う）**:
- [ ] プラットフォーム別に**手動の引用テスト**を定期実施する（ChatGPT・Perplexity・Geminiへ実際に質問し自社が引用されるか確認）
- [ ] 「引用シェア（share of citations）」「ブランド言及率」「AIリファラルのコンバージョン率」を従来CTRに加える新指標として整備する
- [ ] ブランド指名検索（GSC）の推移をGEO施策の間接的指標として使う

**計測設計の考え方**:
- 「クリックベースのSERP指標」と「AI引用・ブランド言及指標」の二軸で評価する
- 量（PV・セッション）より質（CVR・AIリファラルのエンゲージメント）で判断する

> AI検索最適化（GEO・AEO）の詳細については `AI-SEARCH-GEO.md` を参照。AI引用に特化したコンテンツ設計の手法をカバーしている。

## AI可視性計測の独自軸

前節の「AI検索流入の計測」が従来のGA4/GSCをAI時代に拡張する話だったのに対し、本節は**AI回答そのものへの可視性**（AIが自社をどう扱い、どう露出させるか）を測るための、従来ツールには無い計測軸を束ねる。

> 🔴 **前提（全体に効く注意）**: AI可視性の計測はいずれも**不正確で、トレンド把握用の補助指標**である。値の絶対水準を信じて投資判断の一次根拠にしてはならない。「先月より自社が出始めたか」「競合と比べてどの面で言及されるか」といった**方向性・相対変化の把握**に用いる。

### Mentions（言及）と Citations（引用）の分離トラッキング

GenAIツールでの可視性は2つの要素を**別個に**計測する。混同すると「言及されているのに気づかない」死角が生まれる。

| 指標 | 定義 | 追跡可否 |
|------|------|---------|
| **Mentions（言及）** | AI応答の**本文中**にブランド名・製品名が登場するか | 自社サイトへ来ないため**直接の被リファラルでは追えない** |
| **Citations（引用）** | 自社サイトが「回答の出典」として提示され**クリック可能なリンク**になっているか | クリック発生で被リファラルとして追跡しやすい |

- Citationはクリックという行動が起きるため、後述のチャネル定義で被リファラルとして捕捉できる。
- 一方 **Mentionのみで、ユーザーがそこから次のアクションを取らない**場合、ブランドは自分が言及されたことすら気づけない。
- この死角を埋めるのが**プロンプトトラッキング**である。各AIツールに固定プロンプトを投げ、応答にMentions / Citationsが含まれるかを継続観測する。

### プロンプトトラッカー vs ランクトラッカー（機能対応表）

AI可視性計測ツール（プロンプトトラッカー）は、従来のランクトラッカーと**できること・できないこと**が異なる。導入前に役割を切り分ける。

| 計測したいこと | ランクトラッカー | プロンプトトラッカー |
|---------------|:---:|:---:|
| 特定プロンプト/KWの追跡 | Yes | Yes |
| 関連プロンプトの特定 | Yes | Sometimes |
| トピックを広く追跡 | No | Sometimes |
| 競合の特定 | Yes | Yes |
| コンテンツギャップの特定 | Yes | Sometimes |
| 検索ボリュームの提供 | Yes | No |
| センチメント分析 | No | Yes |

**運用上の課金注意**:
- プロンプトトラッカーは**追跡したいLLMごとに課金**されることが多い。同一プロンプトをChatGPT / Perplexity / Claude の3つで追えば実質3プロンプト分のコストになる。
- 追跡するドメイン・言語が増えると課金は指数的に膨らむ。**重要プロンプトに絞って観測する**運用が現実的。

### AI被リファラルの実務トラッキング3手段

GenAIツールからの流入は既定で「Referral」バケツに混ざる。独立したチャネルとして切り出し、かつ「学習クロール」を「実顧客の訪問」と誤認しないことが要点。

1. **User agent による分離**: 各AIツールは固有のuser agentを使う。これを調べてチャネル定義に適用し、AI流入を独立計測する。
2. **サーバーログ（ログファイル）調査**: 訪問のたびに生成される網羅データ。検索スパイダー / スパマー / 顧客 / AIツールの訪問を区別でき、**どのAIがどのページを何回訪れたか**まで把握できる。
3. **Training-crawl（学習クロール）の識別**: ChatGPT・Claude・Perplexity等は検索エンジン同様にクロールして学習データを更新する。**連続した複数ページ訪問を「実顧客の被リファラル」と誤計上しない**こと。大半はトレーニング収集で実トラフィックではなく、多くのGenAIは学習クロールに通常と微妙に異なるuser agentを使う。

> 学習クロールと実流入は**隔離・ラベル付け**し、他のエンゲージメント指標を汚染させないこと。これは前節の「direct」最小化（チャネルを正確に切り分ける土台）と地続きの作業である。

### Gemini流入のGSC側の扱い（3手順）

Geminiの流入は独立チャネルとして切り出せず、通常のオーガニック検索に混ざる。そのためorganicチャネル側で次の補正を行う。

1. **GSCベンチマークのリセット**: GSCのインプレッション/CTRのベンチマークを**2025年4月（Gemini 2.0がAI Overviewsに投入された時点）にリセット**する。この時期にインプレッションが明確にスパイクするため、それ以前との前月比/前年比比較は無効になる。
2. **10語以上のクエリを別扱い**: **10語以上のクエリは大半がAI Overviews / AI Modeで、通常検索ではない**というヒューリスティック。存在をトラッキングし、必要なら正規表現＋GSC APIでCTR評価から除外する（通常検索のCTR分析が歪むため）。
3. **Google Local / Mapsを別計測**: GBP（Googleビジネスプロフィール）のローカルリスティングにUTMを付与しないと、**Maps/GBP由来の流入は既定で全て「direct」**に落ちる。GBPのリンクにUTMを付けて分離する。

### alligator mouth（ワニの口）パターンの解釈

AI Overviews拡大後、GSCで頻出するのが「**インプレッション上昇線**」と「**CTR下落線**」が乖離してワニが口を開けたように見える形である。

- 直感的には「AIOがクリックを奪っている」悪兆候に見える。しかし**一部の観測例では**、収益・予約数・フォーム完了といったビジネス成果がほぼ横ばい（むしろ微増）で、**残ったクリックの質が高い**（行動意図のあるユーザーが残る）傾向が報告されている。実店舗の歩道セールに例えると「browseは増えるが購入者数はほぼ同じ」という構図に近い。
- ⚠️ これは**因果として断定できる現象ではない**。インプレッションはGoogleが重くサンプリングし、検索機能ごとに計測法も異なる「クリックの代理指標」に過ぎず、トレンド把握にのみ使う。
- **例外（必ず併記）**: 一般知識の提供が中核ビジネスのサイトは、ゼロクリック化で**本当に打撃を受ける**。どこでも得られる情報を提供するモデルは元来持続が難しく、AIOがそれを顕在化させる。自社がどちらのタイプかで解釈を変えること。

### AI可視性のproxy（代理）指標

AIへの露出はクリックに現れにくいため、**間接的な代理指標**で補完する。

- [ ] **intake-form設問**: 問い合わせ・申込フォームに「ChatGPT/Claude/Perplexity等のAIで見つけましたか?」という設問を加え、AI由来リードを定量化する。
- [ ] **ブランド指名検索リフトをproxy KPIに**: AI推薦で社名を知ったユーザーは、後で**指名検索**する傾向がある。GSCのブランド指名検索の推移をAI Overviews露出のproxy KPIとして使う。
- [ ] **自前アシスタントの会話ログ**: 自社サイトに設置したAIアシスタントの会話ログを、頻出質問・離脱点を含むKPIソースとして活用する。
- [ ] **固定テストプロンプトの月次パネル**: 自社・競合に関する固定プロンプト群を毎月同条件で再実行し、「自社が出始めたか」を追跡。同時に**競合がどの面（Reddit/レビュー/YouTube等）で被引用されているか**を分析する。

> GEO（Generative Engine Optimization）の計測軸は、クリック/訪問ではなく**AI回答への露出（impression / 参照採用）**で評価する点が従来SEOと根本的に異なる。施策側の詳細は `AI-SEARCH-GEO.md` を参照。

## GenAI効率・生産性メトリクス

ここまでは「検索/AI経由でどれだけ可視化・集客できたか」の計測だった。本節は視点を変え、**GenAIツールを業務に導入したこと自体のROI・生産性**を経営層へ報告するためのメトリクス群を扱う。コンテンツ制作の効率化を投資判断に乗せる際のKPIテンプレートとなる。

> 補足: **組織の20%未満しかGenAIツールのKPIを追跡していない**（ある業界調査による）。メトリクスを定義して継続計測すること自体が差別化になる。

### 8つの効率メトリクス（定義と式）

各メトリクスの**定義（計算式）**を恒久的な軸として用いる。下表の数値は理解を助けるための例示であり、**公開ベンチマークの一例**として扱うこと（自社の絶対目標として鵜呑みにしない）。

| メトリクス | 単位 | 定義（式） | 公開ベンチマークの一例 |
|-----------|------|-----------|----------------------|
| **Tool ROI** | % | （年間の財務的リターン − 投資額）÷ 投資額 | 投資2.5万ドル→リターン22.5万ドルで約800%という例が示されることがある |
| **Process improvement** | % | （導入前の作業時間 − 導入後の作業時間）÷ 導入前の作業時間 | 週次タグ付け20時間→8時間で約60%短縮という例 |
| **Turnaround time reduction** | % | （導入前の依頼→公開日数 − 導入後の日数）÷ 導入前の日数 | 6日→3日で50%短縮という例 |
| **Content acceleration factor（CAF）** | 比率 | 導入前の1アセット作成時間 ÷ 導入後の作成時間 | ブログ1本6時間→1.5時間で4倍という例 |
| **Cost per asset** | 金額 | 制作総コスト ÷ 生成アセット数 | 2,000ドルで100投稿＝1投稿20ドルという例 |
| **Output usability rate** | % | 最小限の編集でそのまま使える成果物数 ÷ 全生成数 | 大幅編集が必要な割合が20%未満＝可用性80%という例 |
| **Error reduction rate** | % | （導入前のエラー件数 − 導入後の件数）÷ 導入前の件数 | typo 10件→4件で60%削減という例 |
| **Compliance improvement** | % | （前年のコンプラエラー件数 − 当年の件数）÷ 前年の件数 | 年10件→7件で30%改善という例 |

**運用上の注意**:
- 数値（ROI・コスト削減率・CAF等）は環境・タスク・モデルで大きく変動する。**式を固定し、自社の実測値で埋めて時系列で追う**のが正しい使い方。
- 効率メトリクスは「速く・安く作れたか」を示すが、**品質・ブランド適合・可視性の指標とは別軸**である。効率だけを最大化すると低品質量産に傾く（量産ペースの罠は `RISK-GOVERNANCE.md` を参照）。
