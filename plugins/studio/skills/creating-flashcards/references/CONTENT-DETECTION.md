# 書籍構造判別マニュアル

EPUB/PDF/Markdown ソースを初見で読み解き、**どの構造パターンに該当するか**を判別するための診断ガイド。判別後の具体的なパース戦略は [CONTENT-BY-TYPE.md](CONTENT-BY-TYPE.md) を、全タイプ共通の前処理・後処理は [CONTENT-COMMON.md](CONTENT-COMMON.md) を参照。

> 🔴 ここで判別したパターンに基づく `parse()` は `scripts/parser_scaffold.py` をコピーして**毎回手書き**する（過去のパーサーを再利用しない＝ジェネリック化禁止）。投入・HTML整形・冪等性は `scripts/anki_toolkit.py` を再利用する（手書きしない）。

**使用フロー**:
1. ソースを全文俯瞰し、本ファイルの「検出すべきマーカー」セクションでマーカー種別を特定
2. 「構造パターン判別マトリクス」で7パターンのいずれに該当するかを決定
3. [CONTENT-BY-TYPE.md](CONTENT-BY-TYPE.md) の該当判定セクションへ進み、パース戦略を実装

## 検出すべきマーカー

### 問題マーカー

| マーカー種別 | 例 | 検出パターン |
|------------|-----|------------|
| 問題番号（日本語） | 問1、問題1、第1問、第一問 | `問\d+`、`問題\d+`、`第\d+問` |
| 問題番号（英語） | Q1、Question 1 | `Q\d+`、`Question\s+\d+` |
| 括弧番号 | (1)、【1】、[1] | `\(\d+\)`、`【\d+】` |
| 見出し付き番号 | `## 問題 1`、`# Q1` | 見出しレベル + 問題キーワード |
| 見出し埋め込み型 | `##### 問題 1. [問題文全体]` | 問題番号と問題文が同一見出し内 |
| インラインテキスト型 | `[Question 1]{.class}: question text` | 見出しではなく本文中に埋め込み |
| Bold装飾型（日本語） | `**問題1 ★★**`、`**問題 5** ★★★`、`**問題1** ★ **重要!**` | `\*\*問題\s*\d+\*\*` |
| Bullet prefixed 番号型 | `- **095** 問題文…` | `^(?:-\s+)?\*{0,2}\d{1,3}\*{0,2}\s+` |
| ○×問題本文（選択肢なし） | 問題番号の直後に平叙文1つのみが続く | 問題ブロック内に `A.` `B.` 等の選択肢マーカーが存在しない |

> ⚠️ **問題マーカー3形式の優先順位付き統一処理**: 同一EPUBで問題マーカーが `**問N**`（Bold装飾型）/ `### 第N問`（見出し型）/ `問N`（プレーン型）の3形式で混在する場合、それぞれ単独で regex を適用すると二重検出が発生する。**対策**: 抽出前に3段階の正規化を適用する: (1) 見出し記号付き形式 `^#{1,6}\s+(?=問\s*\d+|第\s*\d+\s*問)` → 見出し記号除去、(2) Bold装飾形式 `^\*\*(問\s*\d+|第\s*\d+\s*問)\*\*\s*$` → Bold除去、(3) 残りをプレーン形式として処理。正規化後は単一の `^(?:問\s*\d+|第\s*\d+\s*問)` パターンで全形式を網羅できる。章ごとにマーカー形式が変わる書籍では必須。

> ⚠️ **`〔問N〕` 形式のヘッダ有無混在 (同シリーズの別冊間)**: 同じ「過去問一問一答」シリーズの異なる科目本でも、問題マーカーが (a) `# 〔問 N〕[出典コード] {.heading_sW}`(Markdownヘッダ + 出典コード + クラス属性)と (b) `〔問 N〕[A-Eア-オｱ-ｵ]`(`#` なしのプレーン行 + 小問識別子)で**ヘッダの有無が混在**するケースがある(同じシリーズの科目本同士で観測)。単一の `^#\s+〔問` パターンで両方を検出しようとすると一方で取得 0 件の致命的失敗。**対策**: 検出 regex は `^#?\s*〔問\s*([０-９0-9]+)〕\s*(.*?)\s*$` の `#` オプショナル化で両形式に対応するか、ファイル先頭の構造プロービング段階で (a)/(b) どちらかを判定して該当パーサに分岐する。

> ⚠️ **「第N問」マーカーの bold 位置揺れ 4 形式が同一書籍内に混在**: 章末問題集型・テーマ分離型などで、同一書籍・同一章内でも問題マーカーが (a) `第 1 問`（無装飾＋スペース区切り）/ (b) `**第 2 問** &nbsp;&nbsp;...`（マーカー全体 bold ＋ `&nbsp;` トレイル）/ (c) `第 **4** 問`（**数字のみ** bold）/ (d) `## 第12問`（markdown ヘッダー形式）の **4 形式** で章ごとにランダムに変異する。既存の「3形式の優先順位付き統一処理」だけでは (c) の数字のみ bold が抜け落ちる。単一の `^\*\*第\s*\d+\s*問\*\*` regex だと検出件数が章単位で 0 になる罠。**対策**: `^(?:#{1,4}\s+)?\**第\s*\**(\d+)\**\s*問\**\s*` のように **(i) markdown ヘッダープレフィックス optional、(ii) 「第」「数字」「問」それぞれの直前直後で `\**` を許容、(iii) 数字部だけ bold のケース `第 \*\*4\*\* 問` も `\**` の入れ子で吸収** する。前段に `&nbsp;` → 半角空白の正規化処理を必ず置くこと（[CONTENT-COMMON.md](CONTENT-COMMON.md) の `&nbsp;` 行内フィラー混入 警告参照）。検出の兆候: 章ごとの問題抽出数が「期待値の N 倍 / 1/N 倍」のように倍数で揃ってブレる（特定 bold パターンの章だけ全滅）。

> ⚠️ **スキャンOCRで `［N］` 問題番号マーカーが多様に崩壊する → 単調増加割り当てで頑健化**: 画像スキャン本を OCR すると `［N］` マーカーが次のように壊れる: (a) 開き括弧 `［` が `1`＋空白に化け `1 27」`（=27）/ (b) 開き括弧が `1` 化し丸囲み選択肢と融合 `16①`（=6・①が直結）/ (c) さらに3桁化 `140①`（=壊れ括弧1＋実番号40＋①）・`126］`（=26）/ (d) 閉じ括弧が `」` に化ける `［ 10」` / (e) 丸囲み選択肢 `③` が ASCII `3 ` に退行（行頭が「数字＋空白＋本文」）。**単一 regex で全形を拾うと誤検出が増え、「次は必ず番号 N」とする逐次（期待値）マッチングは OCR 欠落1つで以降が連鎖崩壊する**（実観測: 1問の誤検出で後続28問が全滅）。**対策**: 各行から候補番号の集合を生成し（曖昧な `16①` は先頭 `1` 剥がし候補も入れて `{16, 6}` の両方）、設問が `1..N` 順に並ぶ事実を使って「**直前に割り当てた番号より大きい最小の候補**」を採用する**単調増加割り当て**にする。欠落があってもギャップを飛ばすだけで破綻せず、`16①` のような曖昧候補も直前番号で一意化できる（直前=5 なら 6、直前=15 なら 16）。上限 N は「解答キーを真実源にする」戦略（[CONTENT-BY-TYPE.md](CONTENT-BY-TYPE.md)）で得た各回／各章の問題数を使うと安定する。なお選択肢テキスト末尾に紛れ込むマーカー断片（`スロープ126］` の `126］`、`スコータイ朝［ 33` の `［ 33`）は開き括弧の有無の両方を末尾除去する regex（`\s*\d{1,3}\s*[\]］」]\s*$` と `[\[［]\s*\d{1,2}\s*$`）で掃除する。マーカー・正解レター判定を NFKC ではなく生テキストで行うことも必須（[CONTENT-COMMON.md](CONTENT-COMMON.md) の NFKC 丸囲み罠を参照）。

> ⚠️ **章見出しが「単独行 `第 1 章`」と「markdown ヘッダー `# 第8章`」の 2 形式で同一書籍内混在**: pandoc 変換時、原典紙面のフォントサイズ・装飾の違いから、同じ書籍内でも序盤の章は無装飾の単独行 `第 1 章 / 章名`、後半の章は markdown ヘッダー `# 第8章 章名` のように **章境界マーカーが章ごとに変異** することがある。既存の `^#\s*第\d+章` 単一形式 regex だと、無装飾側の章境界をすべて取りこぼし「章メタデータが空のカードが大量発生」する罠。**対策**: 章境界検出 regex を `^(?:#{1,4}\s+)?第\s*[一二三四五六七八九十\d]+\s*章\s*[\S].*$` のようにヘッダー有無の両形式に対応させる。さらに、章番号と章タイトルが**2 行に分かれる**ケース（`第 1 章\n章タイトル`）も観測されるため、章番号行検出後の直後行を章タイトルとして連結する fallback も用意する。実観測: 一書籍内で第1〜7章は単独行、第8章以降は markdown ヘッダーという形式変異を確認。

> ⚠️ **`第N問` マーカーの装飾内部混入と空白フィラー**: pandoc変換時に同一書籍内で次の5変種が混在するケースがある:
>
> | 変種 | 例 |
> |------|-----|
> | 標準 | `第3問` |
> | 内部空白 | `第 3 問` |
> | 全体bold | `**第 3 問**` |
> | 数字のみbold | `第 **3** 問` |
> | 見出し化 | `### 第13問` / `#### 第38問` |
>
> 単一の `第\d+問` パターンでは検出漏れが多発する。検出用正規表現は `第\s*\*{0,2}\s*(\d+)\s*\*{0,2}\s*問` のように **`*` の0〜2個許容 + 数字前後の空白許容** を必須とする。マッチ前に `text = text.replace(' ', ' ')`（NBSP→半角空白）の正規化を実施すること。

> ⚠️ **小問プレフィックス `ア・イ・ウ・エ・オ` の bullet + bold 混在**: 同一書籍内で小問のリードイン記法が次の変種で混在することがある: `ア．`（プレーン）/ `**ア**．`（boldのみ）/ `*   **ア**．`（箇条書きマーカー + bold + 全角ピリオド）/ `- **ア**．`（dash + bold）/ `ア)` 等。**対策**: 小問抽出 regex は `^(?:[\*\-]\s+)?\*{0,2}\s*([アイウエオ])\*{0,2}[．\.\)）]` のように、箇条書きマーカーとboldを両方許容する。

> ⚠️ **選択肢マーカー「▶」セパレーター付き4形式混在と組み合わせ問題インライン形式**: 資格試験対策テキストでは、選択肢マーカーが章ごとに次の4形式で混在することがある: (a) `**ア** 選択肢本文`（標準 bold + 空白区切り）/ (b) `ア▶選択肢本文`（プレーン + ▶ U+25B6 セパレーター）/ (c) `*   ア▶選択肢本文`（箇条書きマーカー + ▶ セパレーター）/ (d) インライン形式 `ア▶1と2 イ▶2と3 ウ▶3と4 エ▶1と4`（組み合わせ問題で典型的、1行内に4選択肢が▶区切りで横並び）。**対策**: 選択肢抽出 regex は `^(?:[\*\-]\s+)?\*{0,2}\s*([アイウエオ])\*{0,2}\s*[▶\.．\)）　 ]+\s*(.+?)$` のように `▶` を区切り文字クラスに含める。(d) のインライン形式は専用パーサーで分解する: `re.findall(r"([アイウエオ])▶([^アイウエオ\n]+?)(?=\s+[アイウエオ]▶|\s*$)", line)`。判定8（サブアイテム + 選択肢の二層構造）の組み合わせ問題と併用される確率が高く、両者を同一書籍内で扱う場合は CONTENT-BY-TYPE.md の判定8 パーサーに分岐する。

> ⚠️ **同一書籍内の小問識別子レター表記揺れ (ア〜オ ⇔ Ａ〜Ｅ)**: 国家資格本試験形式の過去問解説書では、問1〜問N まではア・イ・ウ・エ・オで小問記述が並ぶのに、後半の問では Ａ〜Ｅ（全角英字）に表記が切り替わる書籍がある(pandoc変換時の原書EPUB内部スタイル差に由来)。同一書籍内で「問題編はア〜オ表記、解答編は Ａ〜Ｅ 表記」のパターンも観測される。**対策**: 解答セクションの小問マーカー検出 regex は両方を許容する `^([アイウエオ]|[ＡＢＣＤＥ])\s*$` を使用し、`statement_letters` リストで実際に検出された記号を保存して Front の HTML 生成時にも同じ記号を使う(混同したまま投入すると Anki Back で「ア の解答」がない状態で「Ａ の解答」が並ぶ違和感が出る)。

> ⚠️ **全角数字選択肢ラベル `１．`/`２、` と半角正解の照合ズレ**: 選択肢ラベルが**全角数字＋全角区切り**（`１．` `２．` `３．`、または `１、`）の書籍があり、半角 `1)` 形式と同一シリーズ内・同一書籍内で混在する。一方 `正解：1` は半角数字で記載されることが多く、**ラベル全角・正解半角**だと `_choices_by_letter`（[CONTENT-COMMON.md](CONTENT-COMMON.md) の正解レター照合）が外れ、正解本文がマップされない。**対策**: 選択肢開始検出を `^\s*[1-9１-９][)）.．、]` で**両幅・複数区切り**に対応させ、ラベルは `str.maketrans("１２３４５６７８９", "123456789")` で**半角に正規化**してから `N. text` を組む（correct と突き合うように）。実観測: 実技問題集の単問グループが全角 `１．` 選択肢で抽出0件 → 半角正規化で全問成立。

> ⚠️ **インライン番号並びの「連番性チェック」による誤分割回避**: 1行に複数の番号付きサブ問題が並ぶインライン形式（例: `1 著作権法 2 意匠法 3 商標法 4 個人情報保護法 5 種苗法`）を分解する際、単純な `(\d+)\s+([^\d]+?)` パターンでは**解説文中の参照番号**（例: `特許法2条`、`民法90条`）まで誤検出してしまう。**対策**: 検出した番号列が `[1, 2, 3, 4, 5, ...]` のような**連続昇順**になっているかを後処理でチェックし、連番性が満たされない場合はインライン分解を諦めて行頭マーカー分解にフォールバックする。

> ⚠️ **設問語幹「ア〜ウを比較して」を選択肢「ア」と誤検出する罠**: 「次のア〜ウを比較して…」のような**設問語幹**は行頭が `ア` で始まるため、選択肢マーカー検出 regex `^[アイウエオ]` がこれを選択肢「ア」と誤検出し、選択肢リストが1個ズレる/設問文が選択肢へ混入する。**対策**: 選択肢は**レター昇順連続（ア,イ,ウ,…）の塊**として検出されたもののみ採用し、(1) `ア〜[イ-カ]`（範囲ダッシュを伴う行）、(2) `を比較して`・`のうち` 等の設問語を含む行を選択肢候補から除外する。上記「インライン番号並びの連番性チェック」のレター版に相当する。

> ⚠️ **書籍末尾セクションのみ bullet list 形式に変わる罠**: ○×型一問一答セクション（例: 100問中の最終10問）では、章末・最終ページのレイアウト都合により bold装飾型 `**095** 問題文` から **bullet 形式** `- **095** 問題文` に切り替わるケースがある。**問題番号 regex の先頭に `(?:-\s+)?` を必ず許容**すること。

> ⚠️ **出典行が bold / 箇条書きマーカーで囲まれる変種**: 過去問題集の出典行は `(47回 実技 問17)`（プレーン）だけでなく `**(47回 実技 問17)**`（bold囲み）や `*   (49回 実技 問5～問8)`（箇条書きマーカー + 範囲表記）の変種で混在する。プレーン形式のみを想定した出典 regex では bold/bullet 版を取りこぼす。**対策**: 出典検出 regex の前後に `(?:[\*\-]\s+)?` の箇条書きマーカーと `\*{0,2}` の bold 装飾を許容し、`問M` 単独だけでなく `問M～問N`（範囲表記）もキャプチャする。範囲表記は組合せ大問の親出典であることが多い（[CONTENT-BY-TYPE.md](CONTENT-BY-TYPE.md) の「組合せ大問の小問は出典(src)を持たず範囲親から継承」参照）。

> ⚠️ **事前ガードでの「問題見出し vs 解答見出し」混同回避**: 解答見出し検出 regex は `^#\s+問[０-９0-9]+　解答　[アイウエオ]` のように「解答 X」必須にし、「問題見出し」と「解答見出し」を正規表現レベルで区別する。

> ⚠️ **過去問択一の「各選択肢に付く ○/× は選択肢の真偽であり、正解選択ではない」**: 本試験形式の択一過去問解説では、各選択肢(ア〜オ)ごとに `ア ○` / `イ ×` のような ○/× が付くが、これは**その記述(選択肢)が正しいか誤りかの真偽判定**であって「正解の選択肢」そのものではない。「正しいものを選べ」型なら ○ の選択肢が正解だが、「誤っているものを選べ」型では × の選択肢が正解であり、○/× をそのまま正解フラグに使うと**全選択肢を正解扱いする/正解が反転する**誤りが起きる。**対策**: 正解の選択は必ず明示の解答マーカー `[問N　解答　X]` / `# 問N　解答　X`(上記「問題見出し vs 解答見出し」参照)から取得し、各選択肢の ○/× は Back の選択肢別解説(真偽＋理由)としてのみ使う。両者は別フィールドで保持する。明示マーカーが無く真偽数から最終正解を導出する形式は [CONTENT-BY-TYPE.md](CONTENT-BY-TYPE.md) の「正しいものはいくつあるか」型を参照(本ケースはマーカーが有る側の対処で補完関係)。

> ⚠️ **資格試験対策書の「もう一度check! ▶ 問題 N」参照行の exclusion**: 資格試験対策書では各テーマ内の関連問題への相互参照として `もう一度check!▶ 問題 3` / `もう一度check! ▶ 問題 5` のような行が頻繁に出現する。問題マーカー検出 regex `^問題\s*\d+` をそのまま適用すると、これら**参照リンク行**を問題ブロック開始として誤検出する。**対策**: 問題マーカー検出時に exclusion regex `check|もう一度|▶|→` を併用し、これらの語を含む行は問題マーカー候補から除外する。

> ⚠️ **書籍冒頭セクションのみ問題見出しが異質**: 書籍冒頭の最初の問題（Q1）だけ問題見出しが `# 問1 {.heading_s51}`（半角数字 `1` + 別 heading クラス `s51`）で、Q2 以降は `# 問２ {.heading_s4V-0}`（全角数字 + クラス `s4V-0`）の形式になる Kindle EPUB が存在する。**対策**: 問題見出しの正規表現は `^#\s+問\s*([0-9０-９]+)\s*\{[^}]*\.heading_s[A-Za-z0-9_-]+\}` のように「全角・半角両対応」かつ「`heading_s` 配下のクラス名差異を許容」する。

> ⚠️ **同一シリーズ EPUB の見出しレベル H1/H2 混在**: 同じ出版社・同じシリーズの過去問題集 EPUB でも、ファイル（=実施回）ごとに問題見出しが **H1 (`# 問N`) と H2 (`## 問N`) で混在**することがある。pandoc 変換時のヘッダーレベル推定が EPUB 内部 CSS スタイル設定差で変動するため。単一の `^#\s+問N` パターンで全ファイルを処理しようとすると、H2 のファイルだけ**小問総数ゼロ**の致命的失敗が発生する。**対策**: 過去問題集の汎用パーサーでは問題見出し検出を **`^#{1,2}\s+問\s*([０-９0-9]+)`** （H1/H2両対応）にし、大問見出し検出も両対応にする。解答編開始マーカー `# 解答・解説` も同様に `^#{1,2}\s+解答[・･]?\s*解説` で両対応必須。

> ⚠️ **総仕上げ・模擬試験型セクションの「解答一覧テーブル + 詳細解説」2層構造**: 一問一答セクションが「問題ページ→解答ページ」の単純2分割であるのに対し、総仕上げ・模擬試験型は (A) 全問題、(B) **短い解答一覧テーブル** `| 第N問 | 解答 |`、(C) 各問題ごとの **詳細解説ブロック** `第N問\n解答 X\n解説\n…` の3層構造で記載される。`<table>` から正解だけ拾うと解説が欠落、`第N問` の出現位置を全件マッチすると (B) と (C) が重複検出される。

> ⚠️ **3桁問題番号付き書籍特有の「装飾セクションヘッダー」誤検出**: 「最重要項目 100」「重要 100 問」等を冠する書籍では、pandoc EPUB 変換で**装飾用の section header「100」が単独行として本文に残る**ことがある。問題番号 `001`〜`100` を `\d{3}` でマッチすると、この装飾「100」を**問題番号として誤検出**する罠がある。**対策**: `^\s*100\s*$` の単独行は、直前行に「最重要項目」「最重要 100」「100 問」等の語が含まれていれば decorative とみなしてスキップする。

> ⚠️ **章セクション見出しの装飾抜け落ち**: pandoc が `## 5 章 債権の管理と回収` を `5 章 債権の管理と回収` のように **`#` を完全に失った形**で本文に残すことがある。直前ブロックの末尾に付着し、最終問題の解説に章タイトルが混入する。**対策**: クリーニング regex に `^\s*\d{1,2}\s*章\s*[ぁ-んァ-ヶ一-龥]+\s*$`（`#` 有無を問わず行頭の章ラベルを除去）を追加する。

> ⚠️ **年度別章見出しの複数クラス併用 (Kindle EPUB特有)**: 過去問題集を年度別に区切る章見出しは、同一書籍内で**最新年と過去年で異なる pandoc heading クラス**を使うケースがある(実観測: 7年分過去問で `::: heading_sY` が最新年、`::: heading_sKT` が過去6年分)。最新年セクションが strong emphasis 付きで原書テンプレート上別 CSS class になっているため。単一の `heading_sY` クラスだけで章見出しを検出すると最新年以外がすべて「不明」になり、全カードが同年度扱いされる。**対策**: 章見出し検出 regex は `:::\s*heading_s(?:Y|KT|[A-Za-z0-9_-]+)\s*\n([^\n]+)\n:::` のように複数クラスを許容するか、ファイル全体の `heading_s\w+` クラス分布を初期スキャンで集計して該当クラス群を動的決定する。

> ⚠️ **同一過去問シリーズで「回(ファイル)ごとに問題・解答マーカーの pandoc クラスが変異」→ クラス名非依存の語彙ベース検出**: 同じ過去問題集シリーズの複数回分を別ファイルとして処理する際、問題マーカー・解答マーカーに付く pandoc クラス名が**回(ファイル)ごとに変異**することがある(実観測: ある回は問題ブロック `s8MW`/解答ブロック `s8MY`、別の回は `s8N1`/`s8MZ`)。原書 EPUB のテンプレート CSS が回ごとに別採番されるため。上記「年度別章見出しの複数クラス併用」は1冊内の章見出しクラス変異だが、本ケースは**回をまたいだ問題・解答マーカー両方のクラス変異**で別軸。特定クラス名 (`class_s8MW` 等) を決め打ちした検出 regex は、クラスが変異した回で**取得 0 件**の致命的失敗を起こす。**対策**: クラス名に依存せず `問\s*[0-9０-９]+` / `解答` / `正解` のような**語彙(マーカー文字列)ベースで検出**する。`{.class_s\w+}` は [CONTENT-COMMON.md](CONTENT-COMMON.md) のクリーニングで一律除去した後、残った語彙でブロック境界を判定すれば回ごとのクラス差を吸収できる。

> ⚠️ **章タイトル無装飾フッター汚染の多パターン対応**: 章タイトルがフッターとして本文中に混入する際の表記は複数パターンが混在する。**対策**: `^\s*第?\s*[一二三四五六七八九十\d]{1,2}\s*章\s*` のように数字表記・漢数字・`第` 有無を許容し、続行テキストの文字種を限定しない regex に拡張する。さらに2行分割パターンは「1〜2桁数字単独行 + 直後の章タイトル行」を連続して除去するループを実装する。

> ⚠️ **Bold装飾カテゴリ見出しの構成要素独立欠落と検出のための2行結合**: 問題群を区切る複合カテゴリ見出し（`**N. [カテゴリ名]問題大特訓(10問)**` のような「番号＋カテゴリ名＋件数」の合成）は、OCRで構成要素が**独立にバラバラ欠落・変形**する: カテゴリ名のみ欠落（`**問題大特訓(10問)**`）/ 件数 `(N問)` のみ欠落（`**7. 特殊構文問題大特訓**`）/ Bold見出しが2行に分割（`**3. 呼応...` と続きが別行）。上記の章フッター除去とは目的が逆で、**こちらは構造マーカーとして"取りこぼさず検出・復元"する**のが狙い。**対策**: (1) カテゴリ名が欠落しても**見出し番号→カテゴリ名の辞書補完**で復元する（後述「Part」セクションの `PART_TITLES` 辞書と同方式。番号キーで目次由来のカテゴリ名を引く）、(2) 件数 `(N問)` は **optional 扱い**にして count 検証の補助に留める（欠落しても見出しとして成立させる）、(3) 検出 regex を当てる前に**隣接行を結合**して2行分割を吸収する。`^\s*\*{0,2}\s*(\d{1,2})[.．]?\s*(.*?)(?:[(（]\s*(\d+)\s*問\s*[)）])?\s*\*{0,2}\s*$` のように番号・カテゴリ名・件数を別キャプチャにし、欠落分を辞書で補う。

> ⚠️ **同一書籍内の page footer 表記揺れと累積適用**: 同一書籍内で page footer/header が `2 最重要項目100の一問一答` / `最重要項目 ●100` / `最重要項目●100` / `# 最重要項目` / `# 100` 等、5〜6 変種で混在することがある。**対策**: クリーニング処理を**複数 regex の累積適用ループ**にし、各変種を独立パターンで除去する（順序依存に注意：長いパターンを先に適用）。

> ⚠️ **「Part」セクションヘッダーの3形式混在と PART_TITLES 辞書補完戦略**: 資格試験対策書のシリーズ書籍では、章単位ではなく「Part」単位でセクション分けされるパターンが頻出する。同一EPUB内でも Part マーカーが次の3形式で混在する: (a) Markdown見出し型 `## Part5 不正競争防止法・独占禁止法・その他` / (b) Bold段落型 `**Part 1 特許法・実用新案法**` / (c) ページヘッダー型 `Part 1`（本文中にページ帯としてプレーンテキスト挿入、タイトルなし）。**対策**: Part検出 regex は `^(?:#{1,4}\s*)?\**\s*Part\s*(\d+)\b[\s　]*(.*?)\**\s*$` のように見出し記号・Bold装飾・空白を全て許容する。タイトルが取得できない (c) 形式の場合は、目次から抽出した `PART_TITLES = {"1": "特許法・実用新案法", "2": "意匠法", ...}` 辞書で補完する。

> ⚠️ **「Part」セクション帯のページヘッダーノイズ除去**: 上記の Part セクション3形式混在ケースで、本文中に挿入される「Part 1」「特許法・実用新案法」「意匠法」「商標法」「著作権法」のような**Part帯タイトル**（紙面のページ帯デザインに由来）が、解説テキストや問題本文に混入する。**対策**: 既存の「章タイトル無装飾フッター汚染」と同じく、目次から抽出した Part タイトル群を `PART_NOISE_PATTERNS` として明示リスト化し、ファイル全体をスイープして除去する。**位置依存ではなく語彙依存**にすることで、ページ内位置に関わらず確実にスイープできる。

### 解答マーカー

| マーカー種別 | 例 |
|------------|-----|
| セクション見出し | `## 解答`、`# 答え`、`## Answers` |
| インライン見出し | `**解答**`、`**正解**`、`**Answer**` |
| ラベル付き | `解答:`、`答え:`、`Answer:` |
| 括弧付き | `【解答】`、`【答え】`、`【正解】` |
| 解説付き | `解説:`、`Explanation:` |
| 解答例見出し | `##### 解答例` |
| 数字レスポンス型 | `Correct Response: 4` |
| Bold装飾型（番号付き○×） | `**解答1 ○**`、`**解答2 ✕**`、`**解答 3 ×**` |
| ○×判定マーカー | `○` (U+25CB), `✕` (U+2715), `×` (U+00D7), `〇` (U+3007) |

> ⚠️ **判定マーカーの文字クラス明示（`×` vs `✕`）**: 正規表現で判定マーカーを一括マッチする場合、`[○×✕〇◯❌☓]` の文字クラスは少なくとも以下の7コードポイントを含める: `○` (U+25CB LARGE CIRCLE) / `×` (U+00D7 MULTIPLICATION SIGN) / `✕` (U+2715 MULTIPLICATION X) / `〇` (U+3007 IDEOGRAPHIC NUMBER ZERO) / `◯` (U+25EF BIG CIRCLE) / `❌` (U+274C CROSS MARK) / **`☓` (U+2613 SALTIRE)**。フォントによって字形がほぼ同一に見えるため混同しやすいが、コードポイントが異なると正規表現マッチが外れ判定マーカーを読み飛ばす。**`☓` (U+2613 SALTIRE)** は Kindle EPUB 過去問題集の「解答一覧」セクションで `問１　　☓` のような形式で頻出する変種。正規化マップに `☓ → ✕` を追加すること。

> ⚠️ **ローマ数字ページ番号と○×判定マーカーの衝突**: pandoc変換後のMarkdownにはローマ数字のページ番号（`ii`, `iii`, `iv` 等）が残ることがある。これらを検出する正規表現で `[ivxIVX]{1,5}` のように1文字以上を許容すると、○×型問題の判定マーカー `X` (U+0058) や `V` を「ローマ数字ページ番号」として誤検出して食い潰してしまう。**対策: ローマ数字ページ番号の検出は2文字以上を必須とすること**（`[ivxIVX]{2,5}`）。

> ⚠️ **CJK統合漢字とカタカナの形状混同 OCR誤認**: `recognize-image-to-markdown` コマンドはこの字形混同をプロンプトで予防するが、すり抜けた混同は消費側（本スキル）でここに記す機械置換で対処する（二層防衛の最終段）。画像OCRでカタカナ `エ` (U+30A8) が **CJK統合漢字「工」(U+5DE5)** と、`オ` (U+30AA) が **「才」(U+624D)** と systematic に混同される現象がある。両ペアは活字フォントで字形がほぼ同一のため。**対策**: (1) 抽出 regex を `[アイウエオ工才]` に拡張、(2) 抽出後に `.replace("工", "エ").replace("才", "オ")` で正規化、(3) **位置ベース fallback**（小問ブロック内の出現順で ア→イ→ウ→エ→オ を割り当てる）を最終手段として用意する。🔴 この機械置換は誤爆リスクがあるため `parser_scaffold.py` の `strip_thinking_logs()` 等の汎用ヘルパには入れず、ソース固有の `parse()` 内で抽出文脈を限定して適用すること。

> ⚠️ **「語群」セクション境界の括弧表記揺れ**: 穴埋め問題で選択肢候補リストを区切る「語群」マーカーは、同一書籍内でも5変種が混在する: `[語群]`（半角角括弧）/ `［語群］`（全角角括弧）/ `**[語群]**`（bold + 半角）/ `【語群】`（全角隅付き括弧）/ `（語群）`（全角丸括弧）。さらに `[語 群]`（内部空白）も観測される。**対策**: 語群検出 regex は `\*{0,2}\s*[\[［【（]\s*語\s*群\s*[\]］】）]\s*\*{0,2}` のように全括弧種別・空白・bold を許容する。

> ⚠️ **半角コロン `:` と全角コロン `：` の混在**: 同一書籍内でも `正解：` と `正解:` が混在するケースがある。すべての解答マーカー検出正規表現で `[：:]` 文字クラスを使用し、半角・全角どちらも許容すること。例: `^\*\*問\s*\d+\s+正解\s*[：:]\s*` のように文字クラスで統一対応する。

> ⚠️ **解答マーカー4種表記揺れの正規化順序**: 同一EPUB内でも章ごとに `**問N 正解：X**` / `**問N** 正解：X` / `# 問N 正解：X` / `問N 正解：X` の **4形式が混在**することがある。
>
> **3段階の事前正規化（順序厳守）**:
>
> ```python
> # 段階1: 行頭の見出し記号 # / ## / ### を除去
> text = re.sub(r'^#{1,6}\s+(?=問\s*\d+\s+正解\s*[：:])', '', text, flags=re.MULTILINE)
> # 段階2: 単独装飾 `**問N**` を除去
> text = re.sub(r'^\*\*\s*問\s*(\d+)\s*\*\*\s*$', r'問\1', text, flags=re.MULTILINE)
> # 段階3: `**問N 正解：X**` の外側 `**` を除去
> text = re.sub(r'^\*\*(問\s*\d+\s+正解\s*[：:][^*]+?)\*\*', r'\1', text, flags=re.MULTILINE)
> ```
>
> **段階1→段階2→段階3 の順序が必須**。正規化後は単一の `^問\s*\d+\s+正解\s*[：:]\s*(\S+)` パターンで全形式を検出できる。

> ⚠️ **解答見出しの3形式（A:見出し型 / B:インラインspan型 / C:完全欠落型）の混在**: 同一EPUB内で解答見出し形式が次の3形式で混在することがある: **A型** `# 問N　解答　X {.heading_sPD}` または `{.heading_s4V-1}`（見出し型、大多数）/ **B型** `[問N　解答　X]{.class_s1GV}`（`#` がない、インラインspan形式、1〜数件）/ **C型** 解答見出し完全欠落。**対策**: 解答ブロック抽出を3段階で実装する: (1) A型 regex マッチを優先 / (2) B型 regex 追加（**ただし「解答編開始位置」より前のマッチは目次として除外**）/ (3) C型救出: `[問Nに戻る]` 終端マーカーから逆引きし、解説範囲を切り出す。

> ⚠️ **B型インラインspan解答見出しの目次側マッチとの衝突回避**: `[問N　解答　X]{.class_s1H3}` 形式は EPUB の**目次と解答編本文の両方**に出現する。**対策**: A型（`# 問N　解答` の見出し型）の最初のマッチ位置を `answer_section_start` とし、それ以前の B型マッチおよび `[問Nに戻る]` マッチをすべて目次として除外する。

> ⚠️ **B型インラインspan解答見出しの「閉じ括弧前の全角スペース」検出漏れ**: B型 `[問N　解答　X]` マーカーは、閉じ括弧 `]` の直前に**全角スペースがトレイル**するケースがある(実観測: `[問３３　解答　イ　]` のようにレター `イ` と `]` の間に全角空白)。レター直後に `]` を要求する regex (`\[問\s*[0-9０-９]+\s*解答\s*([アイウエオ])\]`) はこの全角スペースで**マッチ脱落**し、当該問の解答が欠落する。**対策**: 閉じ括弧前に空白を許容する `\[問\s*[0-9０-９]+\s*解答\s*([アイウエオ])[　 \s]*\]` を使う。A型見出し (`# 問N　解答　X`) 側も同様に末尾空白を許容しておくと安全。

> ⚠️ **C型救出（解答見出し欠落）の block_start アンカー戦略**: 「**直前解答見出し本文中の `[問Mに戻る]` 終端位置の直後**」を block_start とする:
>
> ```python
> prev_block = md[prev_header_end:prev_next_start]
> prev_back_m = re.search(rf"\[問\s*[0-9０-９]+\s*に戻る\][^\n]*", prev_block)
> block_start = prev_header_end + prev_back_m.end() if prev_back_m else prev_next_start
> ```

> ⚠️ **「答え合わせ」見出しを解答集セクション開始マーカーとして追加**: 既存の解答マーカーキーワード（解答/答え/Answer/正解/解説）に加え、`答え合わせ` を登録する。

> ⚠️ **解答マーカーの複合前置記法**: `**解答N**` 形式の前に複数のMarkdown装飾・記号が付くケースがある（箇条書きプレフィックス `- `、verdict プレフィックス `○ ` / `✕ ` / `× `、三重アスタリスク装飾 `***`、見出し記号 `### ` 等）。これら複合前置を許容するパターン例:
>
> ```
> ^(?:-\s+)?(?:[○〇◯×✕XxOo]\s+)?(?:\*\*\s*|###\s+)?解\s*答
> ```

> ⚠️ **過去問題集の角括弧解答マーカーと全角解答行**: 過去問題集の一部書籍では選択式問題の解答セクションを `[ 選択肢 ]` / `[ 解答 ]` / `[ 解説 ]` の **半角角括弧 + 半角空白囲み** で記述する (pandoc 変換後は `\[ 解答 \]` → `[ 解答 ]`)。解答行も `【 A 】（4）8割` ではなく `Ａ：（3） ドイツ` のように **全角英文字 + 全角コロン + 全角番号** 形式が混在する。**対策**: 解答 section 検出 regex は `(?:【解答[・、]?(?:解説)?】|【正解】|\[\s*解答\s*\])` で全形式 OR にする。解答行 regex は `(?:【([A-Z])】|([ＡＢＣＤＥＡＢＣＤＥ])[：:])\s*[\(（]?\s*\d*\s*[\)）]?\s*([^\n【Ａ-Ｚ]+)` のように **両形式を OR キャプチャ** して全角letter は `normalize_choice_letter` で半角化する。実観測: 平成28年一般常識/社会保険過去問で 選択式の解答section検出に失敗 (0件)。`【解答】` のみを想定した regex が `[ 解答 ]` を取りこぼし、選択式カードがすべて欠落した。

> ⚠️ **「正答」と「正解」の表記ゆれ（同一書籍内混在）**: 試験対策書では解答セクション内で `【正答 ア】` と `【正解 ア】`、`正答：イ` と `正解：イ` のように **同一書籍内・章ごとに表記が揺れる** ことがある。原因は (1) 著者・校閲者間の用語統一漏れ、(2) 出版社のテンプレート混在、(3) 模擬問題編と章末演習で別人が執筆。**対策**: 解答マーカー regex は `【(正答|正解)\s*([アイウエ])】` のように両表記を OR でキャプチャする。実観測: 同一書籍40問の上級模擬問題で 38問が `【正答】`、2問だけ `【正解】` だった。`正答` のみマッチすると 2問が欠落し、上級扱いの試験準備で穴ができる。

> ⚠️ **PDF→Markdown 変換の OCR 無限ループ崩壊**: 画像由来のページ（スキャン画像混入PDF、画像ベース電子書籍のJPG埋め込みページ等）を AI OCR ツールで Markdown 変換すると、**同一テキストが数十〜100行以上繰り返される無限ループ的崩壊**が発生することがある（例: `*   画像のテキスト：「ア 特許権の侵害者に対し…」` が 80行近く重複し本来の問題文・選択肢が消失）。LLM の自己ループの結果で、**該当問題はソースから復元不可能**。
>
> 🔵 **コマンドが反復圧縮＋自動再OCRで一次対処済み**（`recognize-image-to-markdown` 経由なら検出時に `--max-repeat` 閾値で畳み、必要なら自動再OCRする）。本注記はコマンド未経由ソース／残存崩壊ページ向けの**残存フォールバック**。
>
> **検出手順（残存フォールバックとして残す）**: (1) 消費側は `parser_scaffold.py` の `collapse_repeated_lines(text, max_repeat=3)`（消費側閾値3。コマンド側=4 と非対称）で「同一行が3回以上連続する」崩壊を1回に畳み、戻り値の `collapsed` フラグで崩壊検出をログ記録して該当問題番号をユーザーへ報告、(2) 復元不能ならスキップして他の問題に進む、(3) 一括処理完了後に「欠落問題リスト」を提示し、ユーザーが手動で原典から補完する経路を残す。実観測: 4回分の過去問のうち 1回で問19-20 が、別の回で問7-8 が崩壊。同一書籍内でも崩壊箇所はランダム。

> ⚠️ **「問題不成立」による解答欠落**: 試験施行後に「誤記により問題不成立と判断したため、無解答を含め、全ての解答を正解として採点します。」という注記で解答が空欄になる特殊問題が存在する。解答ヘッダー `**問N解答・解説**` の直後に `※検証の結果、誤記により〜` のような注釈文だけがあり、正解レターが存在しない。**対策**: 解答パース時に `※検証の結果` `問題不成立` `全ての解答を正解` 等のキーワードを検出したら、(1) その問題を「全解答正解」フラグ付きカードとして登録する（オプション1）、または (2) スキップして欠落報告に含める（オプション2）。ユーザーへの最終報告で「OCR崩壊」と「問題不成立」を**別カテゴリで提示**すべき（前者は手動補完可能、後者はそもそも解答が存在しない）。

> ⚠️ **2級・上級資格試験の Markdown リスト形式解説**: 一部の上位級試験書籍では、選択肢解説が `*   **ア ×**\n    [解説text]` のような **markdown 箇条書き + bold ヘッダー + 4スペースインデント本文** の形式で記述される（3級書籍の `ア ×\n[解説]` プレーン形式とは異なる）。pandoc 変換後そのまま残る。**対策**: 解説パース前に preprocessing で (1) `^\s*\*\s+\*\*([アイウエ])\s+([○×〇Ｘ])\*\*\s*$` → `\1 \2` に変換、(2) `^\s{4,}(.+)$`（4スペース以上のインデント）→ `\1` に変換、の2段階で正規化してから本処理に進める。実観測: 同シリーズの3級と2級で全く異なる解説形式が使われており、3級用パーサーは2級で48問中5問の解答ブロックが取得できない（解説抽出失敗）。

> ⚠️ **章末問題「別ページ参照」型のリンクパターン**: テキスト+問題集形式の書籍では、章末練習問題の解答が**章ごとには併記されず、巻末に集約**されるパターンがある。問題ブロックの末尾に `正答・解説は211ページ参照` や `[解答](#section-link)` のようなページ参照リンクが置かれ、実際の解答は **別セクション（巻末）に章単位でまとめて配置**される。問題と解答が物理的に離れているため、`---` ページ区切りベースの単純な交互配置パーサーでは解答とのマッチングに失敗する。**対策**: (1) 問題ブロックに章番号/単元名のメタデータを付与（直近の `# 第N章` ヘッダーから推定）、(2) 巻末の解答セクション（マーカー例: `# 演習問題の正答・解説`、`演習問題の正答・解説` 単独行、`# 解答編`）を一括取得、(3) 章番号 + 問題内番号で問題と解答をマッチングする2パスパースを採用。実観測: 各章末の練習問題が `正答・解説は211ページ参照` で巻末参照され、巻末の `# 演習問題の正答・解説` セクションに章単位で集約されている書籍がある。判定6（テーマ単位ページ分離型）の変種として扱える。

> ⚠️ **章末演習問題型「問題」マーカーの4変種混在と後方探索アンカー戦略**: 章末1問・巻末解答集約型の書籍では、各章末の「問題」マーカーが同一書籍内でも次の4変種で混在する: (a) `**問題** [本文]`（bold装飾＋同一行本文）/ (b) `**問題**\n[本文]`（bold装飾＋改行分離本文）/ (c) `問題  次の記述のうち〜`（プレーン＋複数空白フィラー）/ (d) `問題 [本文]`（プレーン＋単一空白）。単一の `^\*\*問題\*\*` regex では (c)(d) を取り逃し、`^問題\s+` regex では (a)(b) を取り逃す。**対策**: (1) 検出 regex は `^(?:\*\*)?問題(?:\*\*)?\s*[\s\n]+(.+?)$` のように bold optional + 空白/改行両対応にする。(2) より安定する戦略として、**`正答・解説は\s*\d+\s*ページ参照` をアンカーに後方探索**する。各章末で1回しか出現しないこのリンクテキストを起点に、直前段落（最大20行）を問題本文とみなして抽出する。前方検索（章見出しから前進方向）は問題マーカーの変種に弱く、ミスマッチしやすい。

### 境界マーカー

| マーカー種別 | 例 | 意味 |
|------------|-----|------|
| 水平線 | `---`、`***` | ページ区切りの可能性 |
| ページ番号 | `- 1 -`、`Page 1` | 元のページ区切り |
| 大きな空行 | 3行以上の空行 | セクション区切りの可能性 |

> ⚠️ **3桁ゼロパディング問題番号とページマーカーの混同**: 「短い数字 + 短文」パターンでページマーカー（例: `10 最重要項目チェック`）を検出するヒューリスティクスを実装する場合、**ゼロパディングされた問題番号**（例: `029 使用者責任の成立要件として誤っているものは…`）を誤ってページマーカーとして除去してしまう罠がある。
>
> **対策**:
> - 許容する数字桁数を **1〜2桁のみ** に限定する（`^\d{1,2}\s+\S`）
> - ゼロ始まり数字は問題番号と推定して除去対象から除外（`^0\d+` を許容しない）
> - 数字直後の文字列長で識別する（短い見出し語のみ許容、長文が続けば本文と判断）

> ⚠️ **同一書籍内の級別/部別連続配置（初級+上級の問題番号ゼロリセット）**: 「初級・上級」「基礎・応用」のような複数レベル試験を一冊にまとめた書籍では、同一書籍内に **問題1-30（初級セット）→ 問題1-40（上級セット）** のように問題番号が**ゼロリセットして再カウント**される連続配置が観測される。同じ「問題1」というマーカーが書籍内に2回以上現れるため、単純な regex マッチでは初級と上級が混在してしまう。**検出マーカー**: (1) `模擬問題N {検定名} 初級` / `模擬問題N {検定名} 上級` のようなセクション見出し、(2) 「問題1 から問題 30 まで〜選びなさい」「問題１から問題40まで〜選びなさい」のような**問題数告知文**（30問/40問の数字が級判別の鍵）、(3) 解答セクションも同様に分離される（初級用集計テーブル → 上級用集計テーブル → 初級用詳細解説 → 上級用詳細解説）。**対策**: セクション境界を **(問題数告知文の行番号, 問題数, 級名)** の三つ組として事前抽出し、各セクションは独立した `parse_section(start, end, num_q)` を呼ぶ設計にする。デッキも `検定名::模擬問題1初級` と `検定名::模擬問題1上級` に分離する。

> ⚠️ **科目区分ごとに問題番号が振り直される単一デッキ○×集（級別ゼロリセットの同一デッキ版）**: 上記「級別/部別ゼロリセット」は級ごとに別デッキへ分離する対処だが、**1冊が複数の科目区分（例: 物理・化学／性質・消火／法令）に分かれ各区分で 問題1 から振り直す**○×問題集では、同じ「問題1」が科目数だけ出現するものの、**全問を1つのデッキにまとめ科目はタグで分類する**のが自然なケースがある。**対策**: (1) 識別キーを問題番号単独でなく **`(科目, 番号)` のタプル**にして問題↔解答をペアリングする（番号衝突での上書きを防ぐ）、(2) 科目見出しページ（例: `# <科目名>` + `問題は次のページから`）で current_section を更新し各カードに `分野:<科目>` タグを付与する、(3) Anki 側の重複判定は Front（問題文）が科目間で異なるため `duplicateScope:"deck"` で問題なく機能する。デッキ分離が過剰な小規模○×集（数十問）で有効。

> ⚠️ **セクション見出しの2形式混在（markdown ヘッダー + プレーンテキスト `=== ##問題##`）**: 同一書籍内で `## 問題 {.heading_sXX}`（markdown見出し）と `=== ##問題##` `=== ##解答と解説##`（`=== ` 前置・`##` 囲みのプレーンテキスト段落）が混在することがある。前者だけを `^##\s*問題` で検出すると、後者の節で mode（問題/解答）が切り替わらず、問題群が解答セクションに（逆も）紛れ込んで番号の欠落・重複を起こす。**対策**: 見出し判定を両形式吸収の関数に集約し、`^=*\s*#*\s*問題\s*#*\s*$` のように行末アンカー付き regex でプレーン形式も拾う（行末アンカーで本文中の「問題/解答」誤検出を防ぐ）。事前に全見出しを両形式で grep カウントし、問題セクション数と解答セクション数が一致するか検算する。

> ⚠️ **章・難易度の境界見出しがプレーンテキストに退化する**: `# 難易度1`〜`# 難易度4` が markdown 見出しなのに、最後の `難易度5（最上級）` だけ `#` の無いプレーンテキスト行になる等、境界マーカーの一部だけ書式が退化することがある。`^#\s*難易度` だけで章分割すると最終章がまるごと抽出漏れする。**対策**: 章境界 regex は `^#*\s*難易度\s*([1-5])` のように `#` を optional にして見出し形式・プレーン形式の両方を拾う。冒頭で章数を本文宣言（「5段階」等）と grep カウントで突き合わせ、検出章数が宣言と一致するか検算する。

## 構造パターン判別マトリクス

ソースを全文読み込んだ後、**以下の順序で判定する**。先に該当したパターンを採用し、[CONTENT-BY-TYPE.md](CONTENT-BY-TYPE.md) の該当セクションへ進む。

| 判定 | パターン名 | 判定条件 | パース戦略の所在 |
|------|----------|---------|-----------------|
| **判定1** | 前半後半分離型 | ファイルの中盤〜後半に「解答」「答え」等の大きなセクション見出しが1つだけ存在し、前半に問題、後半に解答が並ぶ | [CONTENT-BY-TYPE.md#判定1](CONTENT-BY-TYPE.md#判定1-前半後半分離型) |
| **判定2** | 交互配置型 | 問題マーカーと解答マーカーが交互に出現する | [CONTENT-BY-TYPE.md#判定2](CONTENT-BY-TYPE.md#判定2-交互配置型) |
| **判定3** | 同一セクション型 | 各セクション内に問題マーカーと解答マーカーの両方が含まれる | [CONTENT-BY-TYPE.md#判定3](CONTENT-BY-TYPE.md#判定3-同一セクション型) |
| **判定4** | 章末解答型 | 複数の章に分かれ、各章の末尾に解答セクションがある | [CONTENT-BY-TYPE.md#判定4](CONTENT-BY-TYPE.md#判定4-章末解答型) |
| **判定5** | その他（フォールバック） | 上記に該当しない場合 | [CONTENT-BY-TYPE.md#判定5](CONTENT-BY-TYPE.md#判定5-その他フォールバック) |
| **判定6** | テーマ単位ページ分離型 | テーマ毎に「問題N群ページ」→「解答N群ページ」が `---` 区切りで対を成し、繰り返される | [CONTENT-BY-TYPE.md#判定6](CONTENT-BY-TYPE.md#判定6-テーマ単位ページ分離型) |
| **判定7** | 混在型 | 1つのEPUB内に複数の構造パターンが共存 | [CONTENT-BY-TYPE.md#判定7](CONTENT-BY-TYPE.md#判定7-混在型) |
| **判定8** | パッセージ共有型 | 1つの長文（Eメール・記事）に複数の空欄/設問がぶら下がり、`No.N`（パッセージ番号）→本文→設問群→`No.N 解答と解説` のセグメント構造をなす（長文穴埋め・長文読解） | [CONTENT-BY-TYPE.md#判定8](CONTENT-BY-TYPE.md#判定8-パッセージ共有型) |

> ⚠️ **パッセージ共有型の判別シグナル**: (1) `No.N` 単独行（パッセージ番号・全体連番で再利用や欠番がありうる）、(2) その後に長文プロース、(3) 設問が `1.`/`No. N`/空欄番号 `(n)` + 選択肢（`(A)`/`A)`/`A.` 揺れ）、(4) `No.N 解答と解説` 見出し配下に `(n) 正解: X`。各空欄/設問を1カードにし、パッセージ全文を各カードの Front に文脈として共有する。設問番号はパッセージ内でリセットするため、解答との対応は番号でなく出現順で取る。

### 判別フローチャート

```
ステップ1: ファイル全体を俯瞰し、解答セクションの位置を確認
   ├─ 解答が単一の大セクション（後半に集約） → 判定1（前半後半分離型）へ
   ├─ 解答が問題と交互に出現 → 判定2（交互配置型）へ
   ├─ 解答が各章末に分散 → 判定4（章末解答型）へ
   └─ それ以外 → ステップ2

ステップ2: テーマ毎のページ区切り `---` の有無を確認
   ├─ テーマごとに `---` 区切り + 「問題ページ → 解答ページ」ペアが繰り返される
   │     → 判定6（テーマ単位ページ分離型）へ
   └─ それ以外 → ステップ3

ステップ3: 1セクション内に問題と解答の両方が含まれるか
   ├─ Yes → 判定3（同一セクション型）へ
   └─ No → ステップ4

ステップ4: 複数の異なるパターンのセクションが共存するか
   ├─ Yes → 判定7（混在型）へ
   └─ No → 判定5（その他/フォールバック）へ
```

## 判定時の注意事項

- **選択肢との混同回避**: 連番（1., 2.）は問題番号ではなく選択肢の可能性がある。周辺コンテキストで区別
- **解説の扱い**: 「解説」は解答の一部として扱う
- **図表の参照**: テキストはそのまま保持（画像自体は抽出不可）
- **参考資料/URL**: Backフィールドには含めない（除去する）
