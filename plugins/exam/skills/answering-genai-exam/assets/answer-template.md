# 出力サンプル（1大問分・穴埋めテンプレート）

このファイルは、ある1大問（小問3つ・提出物が `.md` / `.png` / `.jpg`）を解いたときの **出力ファイル群のサンプル** を1ファイルに連結したもの。実際の出力では `answers/<問番号>/` 配下に下記の各ブロックを**別々のファイル**として保存する（区切り `=== ファイル: … ===` は説明用で、実ファイルには含めない）。

`<>` で囲んだ箇所が穴埋め対象。下のサンプル本文は「抽象的な構造原理を別領域へ転用する」型の問題を想定した記入例。

---

## 出力構成（この大問の場合）

```
answers/問<番号>/
├── _summary.md
├── q<N>_1.md               # 小問(1) 提出物 = markdown 回答本文
├── q<N>_1_対話の要点.md
├── q<N>_2_画像生成指示.md   # 小問(2) 提出物 = q<N>_2.png（比較図）
├── q<N>_2_対話の要点.md
├── q<N>_3_画像生成指示.md   # 小問(3) 提出物 = q<N>_3.jpg（抽象図）
└── q<N>_3_対話の要点.md
```

```markdown

- 小問(2) <タイトル> → q<N>_2_画像生成指示.md（提出物 q<N>_2.png）/ q<N>_2_対話の要点.md

- 確定方針: <Step 2 で確定した試験分野・レベル・解答方針>
```

```markdown
対象提出物: q<N>_2.png（この指示を画像生成AIに渡して画像化する）
図の種類: 比較図
目的: 「同型パターン（同じ構造原理で動く事例）」と「反例パターン（似て非なる事例）」の構造の違いを一目で示す。

## 描画要素
- 左カラム見出し「同型: <事例名>」、右カラム見出し「反例: <事例名>」。
- 各カラムに3つの箱「入力」「臨界点」「結果」を縦に配置し、ラベルを記入。

## 配置
- 画面を左右2カラムに分割。左=同型、右=反例。上部に図タイトル「<構造原理>の成立条件比較」。

## 矢印・線
- 左カラム: 「入力」→「臨界点」→「結果」を赤い実線矢印（単方向）でつなぎ、矢印脇に「非線形悪化」。
- 右カラム: 「入力」→「結果」を灰色破線矢印で直結し、「臨界点」の箱には×印を重ねて「臨界点なし＝線形」。

## ラベル
- 凡例: 赤実線=「構造原理が成立」、灰破線=「成立しない」。

## スタイル
- フラットで角丸の箱。同型=暖色系、反例=寒色系で役割を色分け。手描き風は不要。

## 制約反映
- <問題が課す制約（例: 図中の文言は短い単語に限る 等）をここに明記し逸脱しない>。
```
