## Bedrockコンソール画面構成

### 概要（Overview）

AWSコンソールで "bedrock" を検索してBedrockコンソールへ移動。初回アクセス時に「Amazon Bedrockへようこそ！」ダイアログが表示される。

| セクション | 説明 |
|-----------|------|
| **開始方法 > 概要** | Bedrockのホーム。基盤モデル一覧・プレイグラウンドへのリンク |
| **開始方法 > 例** | プロンプトの利用例サンプル集（33種類以上） |
| **開始方法 > プロバイダー** | モデルプロバイダーとモデルの説明 |
| **基盤モデル（ベースモデル）** | 利用可能なモデルの一覧・検索・フィルタ |
| **カスタムモデル** | ファインチューニング済みモデルの管理 |
| **モデルアクセス** | 利用するモデルのリクエスト・許可状態確認 |
| **プレイグラウンド > テキスト** | テキスト生成AIとの単発やり取り |
| **プレイグラウンド > チャット** | 会話形式のやり取り |
| **プレイグラウンド > イメージ** | 画像生成 |

### リージョン設定

```
推奨リージョン: us-east-1（バージニア北部）
理由: すべてのAIモデル・機能が最初に実装されるリージョン
     他のリージョンでは一部モデルや機能が使えない場合がある
```

## 基盤モデルの有効化手順

### モデルアクセスのリクエスト

左メニュー「モデルアクセス」→「モデルアクセスを管理」ボタンをクリック。

| アクセスステータス | 意味 |
|-----------------|------|
| **リクエスト可能** | チェックをONにして即リクエスト送信可能 |
| **ユースケースの詳細は必須です** | フォーム入力後に審査あり（Anthropicのみ） |
| **アクセスが付与されました** | 利用可能な状態 |

### すぐに使えるモデル（チェックONで即利用可能）

| プロバイダー | モデル | モダリティ |
|-------------|--------|-----------|
| **AI21 Labs** | Jurassic-2 Ultra / Mid | テキスト |
| **Amazon** | Titan Text / Embeddings / Image Generator | テキスト・埋め込み・画像 |
| **Meta** | Llama 2 (13B / 70B) | テキスト |
| **Stability AI** | SDXL 0.8 / 1.0 | 画像 |
| **Cohere** | Command / Command Light / Embed | テキスト・埋め込み |

### Anthropicのみ審査あり

「ユースケースの詳細を送信」ボタンから利用目的フォームを送信。審査に通過するとClaudeが使用可能になる。

> **⚠️ 注意**: Claudeは日本語に正式対応している唯一のモデル（他モデルは日本語未対応または部分対応）。本格的な日本語用途はClaudeを推奨。

## テキストプレイグラウンド

### 基本画面構成

左メニュー「プレイグラウンド > テキスト」から開く。

| UI要素 | 説明 |
|--------|------|
| **モデルを選択** | プロバイダーとモデルを選択するパネルを呼び出す |
| **テキストエリア** | プロンプト入力と応答表示が1つのエリアに混在 |
| **実行ボタン** | プロンプトを送信してAIから応答を受け取る |
| **設定（右側）** | パラメータ設定エリア（「×」で折りたたみ可） |
| **ゴミ箱/コピーアイコン** | テキストエリアの内容を削除/コピー |

### モデル選択手順

1. 「モデルを選択」ボタンをクリック
2. カテゴリ（プロバイダー）を選択（例: AI21 Labs）
3. モデルを選択（例: Jurassic-2 Mid）
4. スループット: 「オンデマンド」のまま（従量制）
5. 「適用」ボタンで確定

> **スループット補足**: プロビジョンドスループットは商業サービス向けの定額契約。個人学習ではオンデマンドで十分。

### AIの動作原理を理解する

テキストプレイグラウンドでAIは「送られたテキストの続きを生成する」動作をする。

```
# プロンプト例: 文の続きを補完させる
生成AIについて説明します。生成AIとは、
→ AIがこの文の続きを作成して返す
```

これがテキスト方式の基本動作。チャット形式と異なる点でもある。

### 日本語対応状況

| モデル | 日本語対応 |
|--------|-----------|
| **Claude (Anthropic)** | ✅ 正式対応 |
| **Jurassic-2 (AI21 Labs)** | △ 部分対応（簡単な会話程度） |
| **Llama 2 (Meta)** | ❌ 未対応 |
| **Titan Text (Amazon)** | ❌ 未対応（2024年初時点） |

## チャットプレイグラウンド

### テキストとチャットの違い

| 比較軸 | テキスト | チャット |
|--------|---------|---------|
| **やり取り形式** | 1回限り（プロンプト→応答） | 会話形式（複数回の連続やり取り） |
| **表示方法** | 1つのテキストエリアに混在 | ユーザー/AI別のメッセージとして積み上げ |
| **会話履歴** | なし | あり（蓄積していく） |
| **用途** | プロンプト設計・ユーティリティ | 会話・サポートBot向け |
| **Few-shot設定** | プロンプト内に直接記述可能 | コンソール上では事前設定が難しい |

### 使い分けの指針

```
テキスト方式が向く場面:
- 複雑なプロンプトの設計・研究
- 「選択テキストを翻訳する」等のユーティリティ
- Few-shot学習を試したい場合

チャット方式が向く場面:
- 会話を続けながら情報を深掘りする
- コンテキストを引き継いだ連続質問
- ChatGPTのような対話型アシスタント
```

### チャット操作の注意

- **会話クリア**: モデル名右側のゴミ箱アイコン
- **コピー**: コピーアイコンで会話全体をコピー
- **モデル変更**: 「変更」リンクで別モデルに切り替え

## プロンプトデザイン

### プロンプトの基本構造

プロンプトの基本は「**指示 + 対象**」の組み合わせ。

```
# パターン1: プレフィックス（指示→対象）
以下のテキストを英訳してください。

今日いい天気で気分がいい。

# パターン2: サフィックス（対象→指示）
地球温暖化

上記について説明しなさい。
```

### プレフィックス vs サフィックス チューニング

| 方式 | 特徴 | 向いている用途 |
|------|------|--------------|
| **プレフィックス（指示→対象）** | タスクが先に固定されリソース効率が良い | 翻訳・要約など明確なタスクの繰り返し |
| **サフィックス（対象→指示）** | 文章全体の文脈を処理するため柔軟な応答 | 複雑な文脈を考慮した応答が必要な場合 |

> **実用的なTips**: 高性能モデル（Claude等）では両者の差はほぼなくなる。低性能モデルほど差が顕著に現れる。

### Few-shot プロンプティング

サンプル（例）を与えてAIに応答フォーマットを学習させる手法。

ワンショット学習（1例を与える）テンプレート:

    System: Humanからの質問に対し、指定された形式で答えなさい。

    Human: フランス革命

    Assistant:
    概要：封建制度と絶対主義に反対し、平等と自由を求める社会政治的変革をもたらした。
    年月日：1789年7月14日 - 1799年11月9日
    主要人物：マリー・アントワネット、ジャン＝ジャック・ルーソー、ナポレオン・ボナパルト

    Human: 明治維新

    Assistant:

| 手法 | 例の数 | 用途 |
|------|--------|------|
| **ゼロショット** | 0 | シンプルな指示のみ |
| **ワンショット** | 1 | フォーマットを1つ例示 |
| **Few-shot（少数ショット）** | 複数 | より精確なフォーマット学習 |

### 指示補足の定型句

| 補足 | 効果 |
|------|------|
| `小学生でもわかるように` | 易しい説明 |
| `高校生向けに` | 専門的な説明 |
| `100文字以内で` | 簡潔にまとめる |
| `500文字以上で詳しく` | 詳細な説明 |
| `専門用語を使わずに` | わかりやすい説明 |
| `具体的なデータや数字を用いて` | 根拠を示す |
| `わかりやすい例をあげて` | 具体例つき |

> **英語の活用**: 日本語未対応モデルを使う場合、プロンプトを英語にすると格段に精度が上がる。翻訳ツールで英訳してから送信するのが実用的。

## パラメータ調整

### パラメータの3グループ

| グループ | 含まれるパラメータ | 目的 |
|---------|-----------------|------|
| **ランダム性と多様性** | 温度・トップP・トップK | 応答のランダム性・創造性を制御 |
| **長さ** | 最大長・停止シーケンス | 応答の長さを制御 |
| **繰り返し（Repetitions）** | プレゼンスペナルティ・カウントペナルティ・頻度ペナルティ | トークンの繰り返し抑制 |

### Temperature（温度）

| 値の範囲 | 効果 | 推奨用途 |
|---------|------|---------|
| 0.0〜0.3 | 決定論的・堅実な応答 | FAQ回答・翻訳・要約 |
| 0.4〜0.6 | バランス型 | 汎用テキスト生成 |
| 0.7〜1.0 | 創造的・多様な応答 | 物語・アイデア出し |

モデルデフォルト値: Jurassic-2=0.7 / Llama 2=0.5 / Claude=1.0

### Top-P（トップP）

- 確率上位P%のトークンからサンプリング
- 0〜1の実数で指定（1.0=全候補からサンプリング）
- 値を絞るほど確率の高いトークンが選ばれやすくなる
- GPT系モデルでよく採用される手法

### Top-K（トップK）

- 確率上位K個のトークンからサンプリング
- 0〜500の整数で指定（例: 250=上位250個から選択）
- トップPと同様の役割だが「割合」ではなく「個数」で絞る

### Top-PとTop-K両方あるモデルの動作

1. まずTop-Kサンプリングで上位K個を選択
2. 選ばれたK個の確率合計がTop-Pのしきい値を超えるか確認
3. 超えない場合は次に高い確率のトークンを追加調整
→ Top-Pの柔軟性とTop-Kの確定性を組み合わせた手法

### 最大長（Max Tokens）

| モデル | 最大値 | 説明 |
|--------|--------|------|
| Jurassic-2 | 8191 | 最大トークン数 |
| Claude | 2048 | 最大トークン数 |

> **コスト注意**: 最大長が大きいほど課金額も大きくなる。必要最小限に設定すること。

### 停止シーケンス（Stop Sequences）

応答中に指定テキストが現れると生成を強制終了させる。複数登録可能。

    例: "\n\n" を設定すると改行2回で生成終了

### Repetitions（繰り返し制御）

Jurassic-2のみで利用可能（Claude/Llama 2/Cohereにはない）。

| パラメータ | 範囲 | 効果 |
|-----------|------|------|
| **プレゼンスペナルティ** | 0〜5 | 1回でも出現したトークンの再出現確率を下げる |
| **カウントペナルティ** | 0〜1 | 出現回数に比例して再出現確率を下げる |
| **頻度ペナルティ** | 0〜500 | 出現頻度に比例して再出現確率を下げる |

### パラメータ調整の実践指針

1. **まずデフォルト値のまま使う**（モデルごとに最適なデフォルトが設定済み）
2. **最大長から調整**（応答が途切れる場合は増やす）
3. **次に温度を調整**（堅実な回答=低く、創造的=高く）
4. **その他は慣れてから**（基本は最大長と温度だけで十分）

## イメージプレイグラウンド

### 概要

左メニュー「プレイグラウンド > イメージ」から開く。

| UI要素 | 説明 |
|--------|------|
| **モデルを選択** | イメージ生成モデルを選択 |
| **中央エリア** | 生成されたイメージを表示 |
| **プロンプト入力フィールド（下部）** | プロンプトを記述して実行 |
| **設定（右側）** | イメージ生成用パラメータ |

### 利用可能なイメージ生成モデル

| モデル | 特徴 | デフォルト解像度 | 生成枚数 |
|--------|------|--------------|---------|
| **SDXL 0.8** | シンプルな構成。編集機能なし | 512x512 | 1枚 |
| **SDXL 1.0** | 編集機能あり。推論イメージ対応 | 1024x1024 | 1枚 |
| **Titan Image Generator G1** | 高機能。3枚同時生成 | 1024x1024 | 3枚 |

### イメージ生成のプロンプト記述パターン

プロンプトは**英語**で記述（日本語は未対応または精度が低い）。

    基本構造: [何が] + [どこで] + [何をしているか] + [どんなスタイルか]

    例:
    A cat walking on the main street.
    A cat sleeping in the park, Watercolor painting.
    A cat in the park, Japanese anime style.

#### 場所の指定

    in the living room / on London Bridge / in Paris
    → "in/on/at 〜" で場所を指定

#### 状況の指定

    walking / sleeping / running / dancing / eating
    → 動詞で対象の状態を表現

#### スタイル（描画方法）

| 指定ワード | スタイル |
|-----------|---------|
| `Oil painting` | 油絵 |
| `Watercolor painting` | 水彩画 |
| `Pencil drawing` | 鉛筆画 |
| `Colored pencil drawing` | 色鉛筆 |
| `Japanese anime style` | 日本のアニメ風 |
| `Van Gogh style` | ゴッホ風 |
| `3D model style` | 3Dモデル風 |
| `Abstract painting style` | 抽象画風 |
| `Photo of 〜` | 写真風 |

→ プロンプトの末尾にカンマをつけて追記。

## SDXL 0.8 のパラメータ

| パラメータ | 範囲 | デフォルト | 説明 |
|-----------|------|-----------|------|
| **プロンプト強度** | 0〜30 | 10 | 高い=プロンプト忠実。低い=自由な創造性 |
| **生成ステップ** | 10〜150 | 50 | 高い=詳細で正確。低い=高速・低コスト |
| **シード** | 0〜4294967295 | 0 | 同じシード+同じプロンプト=毎回同じイメージ |

### SDXL 0.8 パラメータ調整指針

- **プロンプト強度**: 詳細なプロンプトがある→低め、アイデア出し→高め
- **生成ステップ**: まずデフォルト(10)のまま使う。不満があれば10単位で増やす
- **シード**: 「悪くないが少し違う」→シードを変えてバリエーションを探す

## SDXL 1.0 / Titan Image Generator G1 の高度な機能

### Titan Image Generator G1 の追加パラメータ

| パラメータ | 説明 |
|-----------|------|
| **モード（Generate/Edit）** | Generateは新規生成、Editは既存イメージを編集 |
| **負のプロンプト** | 「描かせたくない内容」を指定 |
| **推論イメージ** | バリエーション生成の参照イメージをアップロード |
| **質（標準/プレミアム）** | 画質の選択 |
| **オリエンテーション** | 横長（ランドスケープ）/ 縦長（ポートレート） |
| **サイズ** | 生成イメージのサイズ（デフォルト1024x1024） |
| **画像数** | 1〜5枚（デフォルト3枚） |

#### 負のプロンプトの活用例

    プロンプト: "People in the park."
    負のプロンプト: "child"
    → 子供が含まれない公園の人々が描かれる

#### バリエーション生成

1. 気に入ったイメージをクリック → 応答パネルが開く
2. 「バリエーションを生成」ボタンをクリック
3. 同じテイストで少しずつ異なるバリエーションが生成される

または推論イメージに参照ファイルをアップロード → 適当なプロンプトで実行

### コスト注意事項

    イメージ生成はテキスト生成より遥かにコストが高い

    試行段階での推奨設定:
    - サイズ: 小さめ（512x512程度）
    - 画像数: 1枚
    - 生成ステップ: デフォルト値

    本番・高品質生成時:
    - サイズ: 1024x1024以上
    - 画像数: 必要な分だけ
    - 生成ステップ: 高め（50以上）

## イメージ編集（Editモード）

### Editモードとは

既存イメージを読み込み、一部を変更・追加する機能。SDXL 1.0とTitan Image Generator G1で利用可能。

設定の「モード」から「Edit」を選択して有効化。

### SDXL 1.0 のEditモード

1. 設定「モード」→「Edit」に切り替え
2. 「推論イメージ」→「画像をアップロード」でイメージを読み込む
3. プレビュー内の**選択エリア枠**をドラッグで位置・サイズを調整
4. 枠の内部が再描画対象エリア
5. プロンプトに追加・変更したい内容を記述して実行

    例: 枠を画像の空き部分に配置して「A red balloon」と実行
    → 枠内に赤い風船が追加される

### Titan Image Generator G1 のマスクプロンプト

「残したい要素を指定して、それ以外を描き直す」方式。

1. 設定「モード」→「Edit」に切り替え
2. 「推論イメージ」から元イメージをアップロード
3. 「マスクプロンプト」に**残したい要素**を記述

       例: "woman"（女性を残す）

4. 通常のプロンプトに**変更後の内容**を記述

       例: "Penguin"（ペンギンに変える）

5. 実行 → 女性の部分がペンギンに変わる（背景はそのまま）

| 方式 | 得意なこと |
|------|----------|
| **SDXL（枠選択）** | 特定エリアに要素を追加 |
| **Titanマスクプロンプト** | 特定の人・物体だけ別のものに置き換え |

> **注意**: 編集は万能ではない。思ったように編集できない場合は枠位置・プロンプト・パラメータを調整して試行錯誤が必要。

## まとめ: ハンズオンの進め方

1. **リージョン確認**: us-east-1 を選択
2. **モデルアクセス**: 使いたいモデルのチェックをON → リクエスト
3. **テキストプレイグラウンド**: Jurassic-2またはClaudeで基本動作確認
4. **プロンプトデザイン**: 指示補足・Few-shot・プレフィックス/サフィックスを試す
5. **パラメータ調整**: まず最大長・温度のみ調整
6. **イメージプレイグラウンド**: SDXL 0.8から始めてTitan G1へ
7. **Editモード**: 既存イメージへの追加・変更を体験
