# モーションデザイン理論 詳細リファレンス

## なぜUIにモーションが必要か

モーションは単なる「見た目の装飾」ではない。鹿野護・森田考陽の論考が明らかにした3つの根拠:

1. **空間的メタファー**: 画面は物理的な2Dだが、ユーザーは脳内で「レイヤー構造・奥行き・画面の外側」を認識する。モーションがその空間的関係を体験で伝える
2. **感情の非言語表現**: 言語が比喩を使うように（「怒りがこみ上げる」「勇気がわく」）、UIも動きで感情・状態を伝えられる
3. **心理学的時間操作**: 待ち時間の「感じ方」はモーションによって変化する

---

## モーション4分類（鹿野護）

### 分類1: 身体延長（Body Extension）

タッチへの直接的な物理フィードバック。「自分の指がUIと繋がっている」感覚を生む。

**特性**:
- リアルタイム・1:1対応: 指の動きに完全同期
- 物理法則に従う: 慣性・弾性・摩擦
- 自己帰属感の基盤

**代表的なパターン**:
| パターン | 体験 |
|---------|------|
| 慣性スクロール | 指を離した後も速度に応じて続く |
| プルトゥリフレッシュ | 引っ張ってから離すと更新 |
| バウンスバック | 端まで来ると弾力で戻る |
| ドラッグ＆ドロップ | 要素が「つかめる」感覚 |
| 引き戻し（ゴム紐） | 限界を超えると抵抗が増す |

### 分類2: 構造認知（Structure Cognition）

画面空間の構造・階層関係をモーションで伝える。「今どこにいるか」「どこから来たか」を動きで示す。

**特性**:
- 方向性が重要: スライドの方向が「前/後」「上位/下位」を示す
- 遷移速度が重要: 早い遷移は「近い」、遅い遷移は「遠い」を示す

**代表的なパターン**:
| パターン | 構造の表現 |
|---------|----------|
| 右スライドイン | 新しい画面へ進む |
| 左スライドアウト | 前の画面へ戻る |
| ズームイン（拡大） | リスト→詳細への「掘り下げ」 |
| ズームアウト（縮小） | 詳細→リストへの「引き上げ」 |
| Hero Animation | 同じ要素が「成長した」ことを示す |
| フェードイン/アウト | レイヤーの前後関係（奥から手前へ） |

### 分類3: メッセージ（Message）

感情・状態・結果を非言語で伝えるモーション。テキストを使わなくても意味が伝わる。

**哲学的根拠**: 日本語の比喩表現「怒りがこみ上げる・勢いがある・重くなる」は動きを感情に対応させている。UIモーションでも同じ対応が可能。

**代表的なパターン**:
| 感情・状態 | モーションパターン | 身体的メタファー |
|-----------|-----------------|---------------|
| エラー・拒否 | 左右に揺れる（首振り） | 「ダメ」のジェスチャー |
| 成功・完了 | 弾む・拡大してから縮小 | 「やった！」の身体反応 |
| 警告・注意 | 小刻みに振動 | 「気を付けて」の身振り |
| 削除・消去 | 縮小してフェード | 消えていく感覚 |
| 新着・出現 | ポップアップ・バウンス | 「来た！」の出現感 |

### 分類4: 演出（Direction）

プロダクト固有の世界観・ブランドを表現するモーション。機能とは独立した「個性」。

**特性**:
- OSガイドライン準拠では実現できない領域
- プロダクトの感情的価値を高める
- 過度な演出は機能性を損なうため慎重に

**設計アプローチ**:
1. プロダクトの「人格・世界観」を言語化する
2. その人格に合うモーションのリズム・スピード・形状を定義する
3. コンテンツと統合されたモーション（コンテンツが変わると動きも変わる）を目指す

## 心理学的時間とイージング

### なぜ等速（リニア）はダメか

人間の自然な動きは等速ではない。腕を振る、歩く、車が発進する─全て加速・減速がある。等速のアニメーションは「機械的で人工的」に感じられる。

### イージングの種類と心理的効果

| イージング | 形状 | 心理的印象 |
|-----------|------|----------|
| ease-in（加速） | ゆっくり→速く | 重力に従う感じ。終了が速いので「去っていく」感覚 |
| ease-out（減速） | 速く→ゆっくり | 軽快に始まりゆっくり止まる。自然な着地感 |
| ease-in-out（加減速） | ゆっくり→速く→ゆっくり | 最もなめらかで自然。UIの標準 |
| spring（弾性） | オーバーシュート→バウンス | 物理的・有機的。自己帰属感を高める |
| custom bezier | 任意 | ブランド固有のリズムを表現する時に |

### 待ち時間の心理学

**フロント・ローディング（前半加速型）の効果**:
```
プログレスバーの速度パターン:
通常:  ░░░░░░░░░░░░░░░░░░░░  (等速)
推奨:  ████████░░░░░░░░░░░░  (前半が速い)
```
前半に素早く進む（例: 0→70%を速く、70→100%をゆっくり）と、体感待ち時間が短くなる。

**理由**: 「もうすぐ終わる」という期待感が「まだかかる」という不安より待ち時間を短く感じさせる。

## 自律的UIとプロシージャルモーション

### 次世代のモーションデザイン

従来: デザイナーが全モーションを事前に設計・固定
次世代: **システムがユーザーの心理・状況に応じて動的にモーションを生成**

**プロシージャルモーション**:
- ユーザーのスクロール速度に応じてアニメーションが変化
- デバイスの傾きにUIが反応する
- AIがユーザーの感情状態を推定してモーションを調整

**自律的UI**: UIが単なる「ツール」ではなく、自律的な表現主体として振る舞う。コンテンツの文脈・ユーザーの状態・環境に応じてモーションを選択する。

## 映像的ストーリーテリングとUIモーション

UIモーションは映画の編集技法と共通する概念を持つ。

| 映画技法 | UIでの対応 |
|---------|---------|
| カット（直接切り替え） | フェードなしの瞬間切り替え |
| ディゾルブ（重ね合わせ） | クロスフェードトランジション |
| ワイプ | スライドトランジション |
| ズームイン | 要素の拡大トランジション |
| 空間的連続性の維持 | Hero Animation（共有要素） |
| ナレーション（時間経過） | 進捗インジケーター |

**実践原則**: UIアニメーションを設計する前に「この遷移を映画で表現するとどうなるか」と考えると、適切なモーションが見えてくる。

## モーション設計の実践フロー

### Step 1: 分類の決定

```
このモーションは何のためか？
├─ タッチへの物理フィードバック → 身体延長
├─ 空間・階層の説明 → 構造認知
├─ 感情・状態の伝達 → メッセージ
└─ 世界観・ブランドの表現 → 演出
```

### Step 2: イージングの選択

```
動きの印象は？
├─ 自然・なめらか → ease-in-out
├─ 軽快に現れる → ease-out
├─ 勢いよく消える → ease-in
└─ 物理的・有機的 → spring
```

### Step 3: 持続時間の設定

| 操作の距離・複雑さ | 目安の持続時間 |
|-----------------|------------|
| 小さな状態変化（色変化等） | 100-200ms |
| モーダルの表示/非表示 | 200-300ms |
| 画面遷移 | 300-400ms |
| 大きな変形・演出 | 400-600ms |

**注意**: 長すぎるアニメーションは「機械が遅い」と感じさせる。短すぎると変化が伝わらない。

### Step 4: 世界観との統合チェック

- OSデフォルトのモーションと比較して違和感はないか？
- プロダクトの「人格」と一致しているか？
- コンテンツが変わった時、モーションも自然に変化するか？
