# 表層段階：感覚デザインとビジュアルデザイン

> 表層段階は5段階モデルの最上位。コンテンツ・機能性・美しさが一体となって、下位4段階のゴールをすべて達成しながら、感覚的に楽しめるデザインを完成させる段階。

骨格段階が「配置（要素をどこに置くか）」を扱うのに対し、表層段階は**感覚デザイン**と「その配置をどのように視覚的に表現するか」を扱う。

---

## 感覚の意味を理解する

私たちが体験するすべては感覚を通じてもたらされる。表層段階はデザインがすべての人の感覚にどのように現れるかを決定する最終段階。

### 五感とUXデザインの関係

| 感覚 | UXデザインとの関連 |
|------|------------------|
| 嗅覚・味覚 | 食品・香り系製品を除き、ほとんど関与しない |
| 触覚 | 物理製品 → インダストリアルデザインの領域。バイブレーション → 画面ベース製品にも関与 |
| 聴覚 | 操作フィードバック音・ブランド個性表現に使える |
| 視覚 | UXデザイナーが最も得意とする分野。ほとんどの製品で重要な役割を果たす |

**触覚の注記**: 携帯電話・ゲームコントローラのバイブレーションはユーザーとのコミュニケーション手段として機能する。形状・テクスチャ・素材の選択はインダストリアルデザイナーの判断。

---

## ビジュアルデザインの評価基準

```
If ビジュアルデザインを評価する
Then 「見た目の美しさ」ではなく「いかに機能するか」で評価する
（下位の各段階で定義された目標をどれだけ効果的にサポートしているか）
```

**確認すべき問い**:
- 製品の外観がアーキテクチャのセクション間の区別を曖昧にしていないか？
- ユーザーの選択肢を明確にし、構造を強化しているか？
- ビジネス目標（ブランドアイデンティティ等）を伝えているか？

ビジュアルデザインで伝えたいアイデンティティが「技術的で権威ある」なら、マンガフォントや明るいパステルカラーは適切ではない。これは美しさの問題ではなく戦略の問題。

## 視線の動きに従う

### 視線分析の基本手法

| 手法 | 精度 | 適した用途 |
|------|------|-----------|
| アイトラッキング機器 | 高精度・微細なニュアンスも把握 | 詳細な研究・大規模プロジェクト |
| 目を細めてページを見る | 低精度 | 簡易チェック（目立つ要素を見つける） |
- 視点をぼかす / 部屋の反対側から見る | 低精度 | 主要要素の確認 |
| ユーザーに声に出してもらう | 中程度 | 注意が向いた順序の把握 |

```
If デザインを評価したい
Then まず「最初に目が行くのはどこか？」と問う
→ それはユーザーのゴールに必要な要素か？
→ それはビジネスの戦略目標の重要な部分か？
```

### 優れた視線フローの2条件

1. **スムーズな流れ**: 「ごちゃごちゃ」「乱雑」と言われるのは、ページ内がスムーズに誘導されていないから。主張する要素が多すぎて視線が行き来してしまう。

2. **ガイドツアー型の誘導**: ユーザーを圧倒せずに選択肢を提示する。ただし、ゴール到達に必要な情報や機能を妨げないことが最重要。

## コントラストと均一性

### コントラスト

コントラストのないデザインは「特徴のない灰色の塊」。ユーザーの視線はどこにも定まらない。

```
If 重要な要素にユーザーの注意を引きたい
Then 十分なコントラストをつける（色・サイズ・スタイル）
ただし コントラストを使いすぎると乱雑になる
```

**コントラスト活用の事例**:
- エラーメッセージを赤テキストまたは大胆なグラフィックで強調
- 重要度の高い要素を他より目立たせる

**注意点**: 2つの要素が微妙に違うだけの場合、ユーザーは「どうして違うのか？同じものか？間違いか？」と混乱する。コントラストは、それが意図的なものであることをユーザーが納得できるほど明確でなければならない。

### 均一性

デザインの一貫性を保ちながら効果的にコミュニケーションするために重要。

**要素サイズの均一化**: ナビゲーション画像ボタンをすべて同じ高さにすることで、自由に組み合わせ可能になりレイアウトが乱れない。

**グリッドベース・レイアウト**: 印刷デザインからウェブに効果的に引き継がれたテクニック。マスターレイアウトをテンプレートとして均一性を確保。

```
If グリッドシステムを採用している
Then そのグリッドが明らかに機能していないなら見直すタイミングを見極める
（グリッドに固執することはニーズに合わないデザイン標準に束縛されるのと同様）
```

画面ベースのデザインにグリッドを適用するのは、デバイス・画面サイズ・解像度の多様性から、印刷デザインより困難。

## 一貫性：内部と外部

### 2種類の一貫性の問題

| 問題の種類 | 内容 |
|-----------|------|
| 内部の一貫性 | 製品の部分によって違うデザインアプローチが反映されてしまっている |
| 外部の一貫性 | 同じ組織の他製品に用いられているデザインアプローチが反映されていない |

### 内部一貫性の解決策

**アプローチ**: 製品のさまざまなコンテクストで繰り返し出てくるデザイン要素を特定し、各要素をコンテクストから分離してデザインする。

```
If 同じ要素が複数コンテクストで使われる
Then 一度デザインした要素を製品全体で使用する
（ただし各コンテクストで動作確認し、必要に応じてやり直す）
```

**ポイント**: 優れたデザインは「小さくうまくデザインされたオブジェクトの集合体」ではなく、「それらのオブジェクトが一貫したシステムとして機能するもの」。

### 外部一貫性の価値

- **ブランドアイデンティティの統一**: 顧客・見込み客・従業員など全ステークホルダーへの印象を統一
- **組織全体の信頼感**: ビジュアルスタイルに一貫性がないと「自分が何者かわかっていない会社」という印象を残す

ブランドアイデンティティは、主要ナビゲーション要素から一度しか表示されない地味なボタンまで、あらゆるレベルで一貫している必要がある。

## カラーパレットとタイポグラフィー

### カラーパレットの設計原則

カラーパレット = 「すべての色を使う」ではなく「ビジネス全体に使える幅広い色を戦略的に選ぶ」。

| 用途 | 色の特性 |
|------|---------|
| 前景・注意を引く要素 | 明るく大胆な色 |
| 背景・目立たせない要素 | 控えめな色 |
| ブランドカラー | 核となる1色（＋補完色でパレット構成） |

```
If 似た色を複数使いたい
Then ユーザーが十分見分けられる違いを持たせ、
     それぞれの色を一貫した方法で使う
（微妙に違うだけの2色はカラーパレットの効果を損ねる）
```

### タイポグラフィーの使い分け

| テキストの種類 | 推奨スタイル | 理由 |
|-------------|------------|------|
| 長文・本文 | シンプルなフォント（Helvetica等） | 読み疲れを防ぐ |
| 大きなテキスト・短いラベル | 個性のある書体 | スタイルを表現できる |

**共通ルール**:
- 使用するフォントの種類を無駄に増やさない
- 微妙に違うスタイルの使用を避ける
- 情報の違いを示す目的でのみ異なるスタイルを使う
- ほんの一握りのフォントですべてのコミュニケーションニーズを満たす

コントラストと均一性はカラーパレット・タイポグラフィーにも同じく適用される。

## デザインカンプとスタイルガイド

### デザインカンプ（Design Comp）

「カンプ（comp）」= composite（構成物）の略。選択されたコンポーネントで構成された完成品のビジュアル化。

**ワイヤーフレームとの関係**:
- カンプはワイヤーフレームを忠実に再現しなくてよい（ビジュアルデザイン問題を考慮するため）
- ただしワイヤーフレームの核となる情報デザインの意図（要素の相対的重要度・グループ化）は明確に反映する

```
If ワイヤーフレーム作成後にデザインカンプを作る
Then 骨格の問題を先に検討し、
     その後ビジュアルデザインが骨格の決定にどう影響するかを確認できる
```

### スタイルガイド

判断を下してきたデザイン決定事項を文書化したもの。

**スタイルガイドが必要な理由**:
1. 時間が経てば判断の理由が記憶から消える
2. その場しのぎの判断と将来の基盤を作る判断が混同される
3. 人は組織を去り、知識が失われる（組織の「記憶喪失」）

**スタイルガイドの構成**:

| レベル | 内容 |
|--------|------|
| グローバル基準 | デザイングリッド・カラーパレット・タイポグラフィー基準・ロゴガイドライン |
| セクション/機能特化 | 特定のセクションや機能に特化した基準 |
| コンポーネントレベル | 個別のインターフェース・ナビゲーション要素の基準 |

**目標**: 将来の人々が賢明な判断を下せるよう十分な詳細を提供すること。スタイルガイドを作ることで、必要な検討の大半は済んでいる。

**組織規模での価値**: 分散した組織（世界中に散らばるオフィス）でデザインに一貫性を持たせる最も効果的な手法。スタイルガイドの実施判断は、予想以上に組織上層部で行われることが多い（全体的なまとまりは経営判断）。
