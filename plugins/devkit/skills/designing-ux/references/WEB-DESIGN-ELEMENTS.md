## 1. コンテナとデザイン要素の構造

### UI構造におけるデザインの位置づけ

「デザインする」とは、**機能性と情緒性をコンテナを通じてマテリアル（コンテンツ）に設定する行為**と定義できる。コンテナとはデザイン要素の集合体であり、8つの要素に分解される。

```
コンテンツ（マテリアル）
        ↑
  コンテナ（デザイン要素の集合体）
  ├── 配色
  ├── タイポグラフィ
  ├── イメージ（写真・イラスト）
  ├── アイコン
  ├── シェイプ
  ├── デコレーション
  ├── インタラクション・アニメーション
  └── レイアウト
        ↑                 ↑
  機能性7軸           情緒性6軸
```

### デザイン要素 vs UIコンポーネント

デザイン要素は「原材料」であり、UIコンポーネントはそれらを組み合わせた「部品」である。

| 概念 | 定義 | 例 |
|------|------|-----|
| **デザイン要素** | 機能性・情緒性が設定される原材料 | 配色、タイポグラフィ、シェイプ |
| **UIコンポーネント** | デザイン要素から構成される具体的な部品 | ボタン（シェイプ＋配色＋アイコン＋インタラクション）|
| | | 見出し（タイポグラフィ＋配色＋レイアウト） |
| | | ナビゲーション（レイアウト＋タイポグラフィ＋インタラクション） |

**「デザインする」の具体的な定義**: 8つのデザイン要素の属性を変化させ、7つの機能性と6つの情緒性を設定する行為。

## 2. 8つのデザイン要素の定義と属性

### ① 配色

| 項目 | 内容 |
|------|------|
| **定義** | 色の種類・量・配置の設計。どのような色をどれくらいの分量でどこに配置するか |
| **属性: 色相** | 色味・色合い |
| **属性: 色数** | 使う色の数 |
| **属性: 彩度** | 使う色の鮮やかさ |
| **属性: 明度** | 使う色の明るさ |
| **属性: コントラスト** | 使用する色相・彩度・明度の差異 |
| **実装詳細** | → `WEB-COLOR.md`（カラーシステム・配色手法・アクセシビリティ） |

### ② タイポグラフィ

| 項目 | 内容 |
|------|------|
| **定義** | 文字・文字情報の処理方法 |
| **属性: 出力方法** | デバイスフォント・Webフォント・画像フォント |
| **属性: サイズ** | 文字のサイズ |
| **属性: 行間** | 複数行テキストの行と行の間隔 |
| **属性: 文字間** | 文字同士の間隔・スペース |
| **属性: ウェイト** | 文字の太さ |
| **属性: 書体の種類** | 明朝体・ゴシック体・丸文字・手書き風 等の文字セット |

### ③ イメージ（写真・イラスト）

| 項目 | 内容 |
|------|------|
| **定義** | 画像の選定・加工・編集（イメージ自体はマテリアルだが選定はデザイン要素） |
| **属性: 静止画・動画** | 写真・映像の別 |
| **属性: 写真・イラスト** | 被写体撮影 vs モチーフ描写の別 |
| **属性: サイズ** | 画面・ページ内のイメージの大きさ |
| **属性: 形式** | jpg・gif・png 等の表示形式 |
| **属性: 比率** | イメージの縦横比率 |
| **属性: 被写体** | 表示・表現される物体・人物・風景 |
| **属性: トリミング** | イメージをエリア内に収める際の切り取り方 |
| **属性: アングル** | イメージをエリア内に収める際の角度 |

### ④ アイコン

| 項目 | 内容 |
|------|------|
| **定義** | 情報を絵柄で記号化して表現したもの |
| **属性: 出力方法** | 画像 vs Webフォント（Font Awesomeなど） |
| **属性: モチーフ** | アイコンの着想元 |
| **属性: デフォルメ** | モチーフの忠実度・再現具合（強〜弱） |
| **属性: 配置箇所** | 配置場所のルール |

### ⑤ シェイプ

| 項目 | 内容 |
|------|------|
| **定義** | ボタン・エリア・フォームなどのエレメント・コンポーネントの矩形やその形状 |
| **属性: 出力方法** | 画像 vs CSS・HTML |
| **属性: サイズ** | シェイプの大きさ |
| **属性: 形状** | 四角・円・星型・波型など |
| **属性: ボーダー** | 枠線の太さ・色・位置 |
| **属性: 角丸の大きさ** | 長方形・正方形の角の丸みの半径（アール） |

### ⑥ デコレーション

| 項目 | 内容 |
|------|------|
| **定義** | あしらい・意匠・飾り |
| **属性: シャドウ** | 影 |
| **属性: テクスチャ** | 質感・手触り |
| **属性: 光彩** | 光 |
| **属性: ベベル・エンボス** | 立体（面取り）加工・浮き彫り加工 |
| **属性: グラデーション** | 色や濃淡の階調・連続的変化 |

### ⑦ インタラクション・アニメーション

| 項目 | 内容 |
|------|------|
| **定義** | ユーザのアクションに対するシステムのリアクション |
| **アクション（PC）** | ホバー・クリック・ドラッグ・ドロップ・インプット・スクロール |
| **アクション（タッチ）** | タップ/ロングタップ/ダブルタップ・スワイプ/フリック・ピンチイン/アウト |
| **リアクション（種類）** | 決定/送信・選択・同画面内移動・別画面遷移・出現/消失・開閉・変形・拡大縮小・点滅/透過・変色・回転・浮遊・吸着・分離・振動 |
| **リアクション（属性）** | 速度・イージング（加速/減速）・軌道・起点/終点・方向・順番 |
| **実装詳細** | → `WEB-MOTION.md`（モーション種類・持続時間・イージング設定） |

### ⑧ レイアウト

| 項目 | 内容 |
|------|------|
| **定義** | 要素の配置・割り付け |
| **属性: 種類** | 固定（pixel指定） vs リキッド（可変幅） |
| **属性: サイズ** | 要素の大きさ |
| **属性: 余白・距離** | 要素間の距離 |
| **属性: 反復** | 要素の繰り返し具合 |
| **属性: 整列** | 要素の並び方の整い具合 |
| **属性: 位置** | 要素の位置 |
| **属性: アングル** | 要素を置く角度 |
| **実装詳細** | → `WEB-LAYOUT.md`（レイアウトパターン・レスポンシブ設計・原則） |

## 4. 要素×情緒性マトリクス

情緒性6軸の詳細定義・トーン設定方法は `WEB-EMOTIONALITY.md` を参照。

**記号凡例**（各軸の方向性）:
- **装飾性**: `L`=シンプル ← → `R`=デコラティブ
- **成熟度**: `L`=爽やか・若々しい ← → `R`=伝統的・円熟
- **態度**: `L`=フォーマル・真面目 ← → `R`=ポップ・カジュアル
- **愛嬌**: `L`=スタイリッシュ・スマート ← → `R`=キュート・プリティ
- **品性**: `L`=静かで落ち着いた ← → `R`=アクティブ・ダイナミック
- **温度感**: `L`=冷たく無機質 ← → `R`=暖かく開放的

| デザイン要素 | 装飾性 | 成熟度 | 態度 | 愛嬌 | 品性 | 温度感 |
|------------|-------|-------|-----|-----|-----|-------|
| **配色** | 色数少→L / 色数多→R | 高明度・高彩度→L / ゴールド等重色→R | ネイビー→L / カラフル多色→R | 色数絞る→L / 原色近似→R | 暖色系→R | 茶・緑・ベージュ→R |
| **タイポグラフィ** | 装飾少書体→L / 装飾多書体→R | ゴシック体→L / 明朝体→R | フォントサイズ大→R | 英文・細字→L / 丸文字→R | ウェイト太→R / 細→L | フォントサイズ小→L / 手書き風→R |
| **イメージ** | — | 光・空・若者→L / 伝統建築・ダーク→R | 抽象化イラスト増→R | 削ぎ落としダーク→L / 小動物・幼児→R | 動画・奥行きアングル→R | 画像量減→L / 自然素材→R |
| **アイコン** | デフォルメ強→L / リアル忠実→R | — | アイコン多用→R | 枠線構成→L | — | — |
| **シェイプ** | — | — | — | 角丸小→L / 角丸大→R | 枠線太→R / 細→L | サイズ小→L |
| **デコレーション** | 排除→L / シャドウ・エンボス多用→R | フラット→L / アンティーク多用→R | ストライプ・ドット→R | 円・曲線意匠→R | — | 自然現象意匠・木目テクスチャ→R |
| **インタラクション・アニメーション** | — | 展開速度速い→L / 輝き・点滅→R | 躍動的アニメーション多い→R | 滑らか・イージング控えめ→L / 跳ね・揺れ→R | 大コントラスト・高速→R / 緩やか→L | 浮遊・吸着・自然現象の動き→R |
| **レイアウト** | 1カラムに収める→L | — | 整列線厳密→L / 整列線から外す→R | リキッド・大余白→L / 詰め込み→R | 余白コントラスト大・斜め配置→R | 余白狭い→L / 余白広い→R |

## 5. 要素別インパクトサマリー

各デザイン要素が「何に最も強く影響するか」の早見表。デザイン方針立案時に優先的に操作すべき要素を特定するために使用する。

| デザイン要素 | 機能性への主影響 | 情緒性への主影響 | 判断の起点となる属性 |
|------------|:------------:|:------------:|---------------|
| **配色** | 誘目性・アイデンティティ | **全6軸**（最も横断的） | 色数・コントラスト・色相の選択 |
| **タイポグラフィ** | 視認性・情報理解 | 成熟度・愛嬌・品性 | 書体種類・サイズ・ウェイト |
| **イメージ** | 誘目性・アイデンティティ | 成熟度・態度・愛嬌 | 被写体・写真 vs イラスト・アングル |
| **アイコン** | 操作性・情報理解 | 装飾性・態度 | デフォルメ度・配置ルール統一 |
| **シェイプ** | 操作性（第一） | 愛嬌・品性 | 角丸・枠線・サイズ |
| **デコレーション** | 誘目性（存在感） | 装飾性・成熟度・温度感 | 追加 vs 排除の方向性 |
| **インタラクション・アニメーション** | 誘目性・操作性 | 成熟度・態度・愛嬌・品性 | 速度・イージング・リアクション種類 |
| **レイアウト** | 操作性・習熟度・可変性（最も幅広い機能性影響） | 装飾性・態度・愛嬌・温度感 | 余白量・整列・カラム数 |

### 機能性向上を優先する場合の要素選択

| 課題 | 優先操作すべきデザイン要素 |
|------|------------------------|
| ユーザが目的の要素を見つけられない | 配色（コントラスト）・レイアウト（位置・サイズ）・アイコン |
| 情報が理解されにくい | タイポグラフィ（構造化）・レイアウト（グルーピング）・イメージ（図解化） |
| 操作ミスが多い・使いにくい | シェイプ（サイズ・角丸）・インタラクション・レイアウト（マージン） |
| ブランドが伝わらない | 配色（コーポレートカラー）・イメージ（キービジュアル）・タイポグラフィ（ブランドフォント） |

### 情緒性設定を優先する場合の要素選択

| 目標トーン | 特に影響が大きいデザイン要素 |
|----------|--------------------------|
| シンプル・ミニマル（装飾性L） | デコレーション排除・配色の色数削減・デフォルメの強いアイコン |
| 伝統的・高級感（成熟度R） | 配色（重色・ゴールド）・タイポグラフィ（明朝体）・デコレーション（アンティーク） |
| フォーマル・信頼感（態度L） | レイアウト（整列厳密）・配色（ネイビー）・タイポグラフィ（標準サイズ） |
| 活発・スポーティ（品性R） | インタラクション（高速・大コントラスト）・レイアウト（斜め配置・余白コントラスト大） |
| 親しみ・温かさ（温度感R） | 配色（茶・緑・ベージュ）・デコレーション（木目テクスチャ）・タイポグラフィ（手書き風） |

## 6. 要素選択の判断フロー

```
デザイン課題を受ける
       ↓
機能性と情緒性のどちらが優先か？
       ↓
┌─────────────────────────────────────────┐
│ 機能性優先                情緒性優先      │
│ ↓                           ↓          │
│ 課題の種類を特定         目標トーンを特定  │
│（視認性・操作性・理解促進等）（6軸のどこか）│
│ ↓                           ↓          │
│ 上記「機能性課題別」      上記「目標トーン別」│
│ テーブルで要素を特定      テーブルで要素を特定│
└─────────────────────────────────────────┘
       ↓
要素の属性変化を具体的に決定
（例: 配色→コントラスト↑、レイアウト→余白量増加）
       ↓
機能性と情緒性が矛盾していないか確認
（例: 高密度な情報量[機能性] vs 余白重視[情緒性]はトレードオフ）
       ↓
WEB-CONCEPT-STRATEGY.md の
デザイン詳細方針立案プロセスで統合
```
