# デザイン理論基盤（Design Theory Foundation）

エンジニアのためのビジュアルデザイン理論。
「なぜこのデザインが機能するのか」を理論的に理解し、根拠ある判断を下すための知識体系。

---

## 1. デザインコンセプトの構築

デザインは感性ではなく**思考プロセス**。目的から逆算して構成要素を選定する。

```
目的（Why）→ ターゲット（Who）→ メッセージ（What）→ エレメント選定（How）
```

| 構成要素 | 定義 | 例 |
|---------|------|-----|
| 目的 | このデザインで達成したいこと | 新規ユーザーの登録率向上 |
| ターゲット | 誰に届けるか | 20代・スマートフォンユーザー |
| メッセージ | 伝えたい核心 | 「3分で始められる」 |
| エレメント選定 | 目的・ターゲット・メッセージに合致する視覚要素 | シンプルな色・大きなCTA・短いコピー |

**原則**: エレメント（色・フォント・レイアウト）はすべてコンセプトから導出する。装飾のための装飾は行わない。

## 2. タイポグラフィ理論

### フォント分類

| 分類 | 特徴 | 用途 |
|------|------|------|
| セリフ（明朝体） | 文字端に飾り（ひげ）あり | 長文・格式・伝統的印象 |
| サンセリフ（ゴシック体） | 飾りなし・均一な線幅 | UI・デジタル・現代的印象 |
| モノスペース | 等幅・機械的 | コード・ターミナル表示 |

### フォント選定基準

1. **可読性**: 小サイズで認識できるか
2. **印象**: ブランドの世界観と一致するか
3. **ブランド適合性**: 競合との差別化ができるか
4. **技術的制約**: Webフォントのロード時間、ライセンス

### 和文フォントの使い分け

| 書体 | 印象 | 適切な用途 |
|------|------|-----------|
| 明朝体 | 格式・伝統・品格 | 新聞・書籍・高級ブランド |
| ゴシック体 | 現代・清潔・親しみやすい | Web・UI・アプリ |
| 丸ゴシック | 柔らかい・フレンドリー | 子供向け・ヘルスケア・SNS |

### フォントサイズスケール設計

**Modular Scale（比率ベース）を使用する。**
推奨比率: `1.25`（Major Third）または `1.333`（Perfect Fourth）

| 用途 | 比率×基準サイズ（16px） |
|------|------------------------|
| xs（補足テキスト） | 12.8px |
| sm（ラベル） | 14px |
| base（本文） | 16px |
| lg（小見出し） | 18px |
| xl（見出し） | 20px |
| 2xl（大見出し） | 24px |
| 3xl（ヒーロー） | 30px |

**法則**: 同一ページ内で使うフォントサイズは3〜5種類に絞る。

## 3. 情報整理の原則

### CRAP原則（4大原則）

情報の関係性を視覚化する基本原則。

| 原則 | 英語 | 概要 | 実践 |
|------|------|------|------|
| **近接** | Proximity | 関連する要素は近づける | ラベルと入力欄を近づける |
| **整列** | Alignment | 要素を見えない線に揃える | グリッドシステムの使用 |
| **反復** | Repetition | パターンを繰り返し統一感を出す | 同じスタイルを一貫して使う |
| **コントラスト** | Contrast | 異なる要素は明確に差別化する | 大きさ・色・太さで強弱をつける |

**応用**: この4原則のどれかが崩れているとき、デザインが「なんとなく違和感がある」状態になる。

### ゲシュタルト法則（知覚の原則）

人間の脳が視覚情報をどのようにまとめて認識するかの法則群。

| 法則 | 概要 | UI応用例 |
|------|------|---------|
| **閉合** | 不完全な形でも閉じた形として認識する | 進捗バー・アイコンの簡略化 |
| **近接** | 近くにある要素を同じグループとして認識する | カード内の情報グルーピング |
| **類同** | 見た目が似た要素を同じグループとして認識する | テーブルの交互背景色 |
| **連続** | 連続した点・線を一つの方向として知覚する | ステッパー・ブレッドクラム |
| **共通運命** | 同じ方向に動く要素を同グループと認識する | アニメーションでの関連表現 |
| **プレグナンツ（単純化）** | 複雑な形を最もシンプルな形として認識する | ロゴ・アイコンのシンプル設計 |

## 4. レイアウトとスペーシング

### 均等スペーシングの原則

- **8px基準グリッド**: すべての余白・パディングを8の倍数で設計（4px単位も可）
- **要素間の関係**: 関連する要素ほど間隔を狭く、独立した要素ほど広く

```
関連グループ内: 4〜8px
グループ間: 16〜24px
セクション間: 32〜64px
ページ余白: 64〜128px
```

### 黄金比（Golden Ratio）

**比率: 1 : 1.618**（φ = 約1.618）

| 活用場面 | 例 |
|---------|-----|
| レイアウト分割 | メインコンテンツ:サイドバー = 61.8% : 38.2% |
| 見出しと本文の比 | 見出し24px : 本文14.8px ≒ 1.618 |
| 余白設計 | コンテナ幅1000px → 余白を618px:382pxに |
| カードの縦横比 | 幅:高さ = 1.618:1（横長）または 1:1.618（縦長） |

**実践**: `calc()` や CSS変数で比率を固定すると一貫性が保ちやすい。

### 三分割法（Rule of Thirds）

**画面を縦横3等分し、交点（アクアポイント）に重要要素を配置する。**

```
┌─────┬─────┬─────┐
│     │  ★  │     │  ← 上1/3のライン
├─────┼─────┼─────┤
│  ★  │     │  ★  │  ← 交点4か所が視線を引くポイント
├─────┼─────┼─────┤
│     │  ★  │     │  ← 下1/3のライン
└─────┴─────┴─────┘
```

- ヒーローセクションのCTAボタンは右下の交点付近に配置
- ランディングページのメインビジュアルは左上の交点付近に要素を配置
- 写真のトリミング時に被写体を交点に合わせると安定感が生まれる

## 5. 色彩理論

### 色の三属性

| 属性 | 定義 | 範囲 |
|------|------|------|
| **色相（Hue）** | 色の種類（赤・青・黄など） | 0〜360° |
| **明度（Value/Lightness）** | 明るさ | 0%（黒）〜100%（白） |
| **彩度（Saturation）** | 鮮やかさ | 0%（無彩色）〜100%（純色） |

### トーンと印象

| トーン | 彩度 | 明度 | 印象 |
|--------|------|------|------|
| ビビッド | 高 | 中 | 活発・鮮やか・目立つ |
| ブライト | 中〜高 | 高 | 明るい・元気・ポップ |
| ライト | 低〜中 | 高 | 柔らか・爽やか・清潔 |
| ペール | 低 | 高 | 淡い・上品・繊細 |
| ダル | 中 | 中 | 落ち着き・成熟・安定 |
| ダーク | 低〜中 | 低 | 重厚・高級・格式 |

### 配色手法

| 手法 | 定義 | 特徴 | 用途 |
|------|------|------|------|
| **類似色** | 色相環で隣接する色 | 統一感・穏やか | コーポレート・ナチュラル系 |
| **補色** | 色相環で正反対の色 | 強いコントラスト・目立つ | CTA・エラー表示 |
| **トライアド** | 色相環を3等分した3色 | バランス・活気 | エンタメ・子供向け |
| **スプリットコンプリメンタリー** | 補色の隣2色を使用 | コントラストを抑えた調和 | 汎用性が高い配色 |

### 色の使用比率（70:25:5ルール）

```
ベースカラー  : 70% ── 背景・余白（白・グレーなど）
メインカラー  : 25% ── ヘッダー・ナビ・主要ブロック（ブランドカラー）
アクセントカラー: 5%  ── CTA・強調・通知（補色や高彩度色）
```

**理由**: 視覚的な重みを分散させることで、ユーザーの視線を自然にアクセントカラーへ誘導できる。

### 色彩の役割（4つの視覚特性）

| 特性 | 定義 | UI活用例 |
|------|------|---------|
| **誘目性** | 無意識に目が引き寄せられる強さ | エラーメッセージの赤・成功の緑 |
| **視認性** | 背景との判別しやすさ | コントラスト比（WCAG 4.5:1以上） |
| **可読性** | テキストとして読みやすいか | 本文は低彩度・高明度の組み合わせを避ける |
| **識別性** | 複数の要素を区別できるか | グラフの系列色・ステータスバッジの色分け |

## 6. 色彩調和論と対比

### ジャッドの色彩調和論（4原則）

| 原則 | 内容 | 実践 |
|------|------|------|
| **秩序の原則** | 規則的な関係にある色は調和する | トーンを統一した配色（同トーン配色） |
| **馴染みの原則** | 自然界に見られる配色は調和する | 空・大地・森をモチーフにした配色 |
| **類似の原則** | 共通要素（色相・明度・彩度）を持つ色は調和する | 同系色グラデーション |
| **明瞭性の原則** | 曖昧でなく明確な関係性を持つ色は調和する | 補色対比・強いコントラスト |

### 色彩対比の種類

| 対比 | 概要 | UI上の注意点 |
|------|------|------------|
| **同時対比** | 隣接する色が互いに影響し合う現象 | 背景色が変わるとテキスト色も違って見える |
| **色相対比** | 同じ色でも周囲の色相で違って見える | アクセントカラーの選定時に確認必須 |
| **明度対比** | 同じ色でも周囲の明度で違って見える | 暗背景上では色が明るく見える |
| **彩度対比** | 周囲が無彩色だと中央の色が鮮やかに見える | 白背景上では色が強く出る |
| **補色対比** | 補色同士が隣接すると両方が鮮やかに見える | 赤と緑は可読性が下がりアクセシビリティに問題 |

### 色彩同化現象

**対比の逆**: 周囲の色に引き寄せられて同じように見える現象。

- 細いストライプや小さいパターンで発生しやすい
- デジタル画面でのアイコン・テクスチャ設計時に考慮する

## 7. ビジュアル選定とトリミング

### ビジュアル素材の選定基準

| 基準 | 内容 |
|------|------|
| **メッセージ整合性** | コンセプトのメッセージを強化するか |
| **ターゲット共感性** | ターゲットユーザーが共感・親近感を感じるか |
| **品質** | 解像度・照明・構図が十分か |
| **一貫性** | 他のビジュアルとトーンが統一されているか |
| **ライセンス** | 商用利用可能か（著作権・商標の確認） |

### トリミング手法

| 手法 | 目的 | 活用場面 |
|------|------|---------|
| **三分割法に合わせる** | 被写体を交点に配置し安定感を出す | ヒーロー画像・OGP |
| **余白の活用** | テキストを重ねるスペースを確保する | コピー付きバナー |
| **注目点の誘導** | 視線を特定のUI要素に向ける | 人物の視線をCTAへ向ける |
| **フォーカルポイント** | 最も重要な部分に視線が行くよう切り抜く | サムネイル・カードビジュアル |

## 8. デザインスキル向上の実践法

### 基本ループ

```
観察（Observe）→ 分析（Analyze）→ 模倣（Imitate）→ フィードバック（Feedback）→ 再観察
```

| ステップ | 具体的な行動 |
|---------|------------|
| **観察** | 日常のUI・広告・パッケージを意識的に見る。「なぜこの色か」「なぜこのフォントか」を問う |
| **分析** | 優れたデザインを分解する（色・フォント・余白・グリッドを測定） |
| **模倣** | 良いデザインを意識的に再現する（トレースから始める） |
| **フィードバック** | 他者の意見・A/Bテスト結果から客観的に評価する |

### 著作権・商標への配慮（デザインと法律）

| 項目 | 注意点 |
|------|--------|
| **フォントライセンス** | 商用利用・Webフォント埋め込み可否を確認 |
| **画像・素材** | ロイヤリティフリーと著作権フリーの違いを理解する |
| **色と商標** | 特定ブランドカラーが商標登録されている場合がある（ティファニーブルー等） |
| **UI模倣** | Trade dressとして訴訟リスクになる場合がある |
