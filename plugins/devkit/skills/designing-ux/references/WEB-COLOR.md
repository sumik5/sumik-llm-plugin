## 1. Webでの色の4つの役割

Webサイトにおける色はビジュアルの第一印象を瞬時に決める。以下の4つの役割を意識して設計する。

| 役割 | 目的 | 実装のポイント |
|------|------|--------------|
| **ブランド訴求** | ひと目でブランドを想起させる | ブランドカラーを配色の軸に据える。VI（ビジュアル・アイデンティティ）準拠 |
| **行動促進** | CTAボタン等でユーザーの行動を誘導する | 背景・隣接要素とのカラーバランス・レイアウトとの相乗効果を考慮 |
| **トーン設定** | サイトのパーソナリティを一貫して伝える | 単調すぎず・散漫にならないトーン管理。スタイリッシュ→単色、楽しい→カラフル |
| **可能性拡張** | 認知拡大・付加価値創出・競合との差別化 | ベースが白黒でも色の扱いで印象が大きく変わる。情報設計にカラー心理を組み込む |

## 2. カラーシステム: 4つの色の機能

Webデザインでは使用する色に明確な機能を持たせる。

### 各カラーの定義と使い分け

| カラー | 役割 | 使用面積 | 選び方のポイント |
|--------|------|---------|----------------|
| **メインカラー** | 第一印象を形成する配色の軸 | 中程度 | ブランドカラーまたはターゲットの嗜好に合った色。高彩度すぎると煩わしくなるので注意 |
| **ベースカラー** | 最大面積を占める背景・余白の色 | 60〜80% | メインカラーを引き立てる色。補色・類似色・無彩色が定番。メインと衝突しないようトーン差をつける |
| **サブカラー** | メインカラーを補い華やかさを加える | 小〜中 | メインカラーが強すぎる場合に調整役。使わない選択もあり（写真がカラーを担う設計） |
| **アクセントカラー** | 注目させたい要素を際立たせる | 最小（点的） | CTAボタン等に使用。赤・オレンジなど暖色系が多い。大面積に使うと印象を崩すので注意 |

### カラー設計の原則

```
メインカラー（軸）
  ↓ 引き立てる
ベースカラー（60〜80%）
  ↓ 補完・アクセント
サブカラー ＋ アクセントカラー（点的）
```

- 彩りをビジュアル（写真・イラスト）に委ねる場合: ベースカラー＋メインカラーのみのシンプル設計が多い
- コンバージョンエリアには役割の明確なカラーを使い、ユーザーを明確に誘導する

## 3. ブランドカラーの活用

### ブランドカラーとVI（ビジュアル・アイデンティティ）

ロゴ・名刺・カタログ・Webサイトなど、ユーザーとの接点ごとにビジュアルが変わるとブランドイメージが定着しない。
**VIを設けてビジュアルに一貫性をもたせる**ことで、感覚的認知を確立する。

| ブランドカラーが果たす効果 | 内容 |
|--------------------------|------|
| **競合との差別化** | 独自性ある色で記憶に残る |
| **情緒的訴求** | 理念・ビジョン・パーソナリティを色で体現 |
| **感覚的認知の獲得** | 繰り返しの露出で「〇色のブランド」として定着 |
| **社内共有** | ブランドの方向性を従業員と共有し、行動・アイデアの指針に |

### カラーレギュレーション（デザインガイドライン）

ブランディングを重視する企業ではカラーを厳格にルール化している。
Webサイト制作時は**必ず確認**する。

| レギュレーション項目 | 内容例 |
|--------------------|--------|
| ロゴカラー値 | 印刷用 CMYK、DIC番号 / Web用 RGB・HEX |
| テキストカラー | 本文・見出し・リンクの指定 |
| テーマカラー | 全体配色の規定（ロゴ・テキスト・テーマ別のHEX値） |
| フォント指定 | 日本語フォント・英字フォントのセット |

## 4. 配色の基本手法

| 手法 | 説明 | 効果・注意点 |
|------|------|------------|
| **同系色（ドミナント配色）** | 同じ色相でトーンを変えて組み合わせる | 統一感・ブランドイメージが強まる。単調に見えやすい → 隣接色相で幅を出す |
| **異なる色相＋トーン統一** | 色相は異なるが明度・彩度を揃える | トーンのイメージ（明るい／暗い）が前面に出る。1色だけ彩度を上げると強調効果 |
| **補色の活用** | 色相環の反対側の色を組み合わせる | 互いを引き立て、アクセントカラーに補色を使うと印象が強まる。ハレーション注意 |
| **トライアド配色** | 色相環上で等距離3色を使う | カテゴリ分け・色差の確保に有効。色の偏りが出にくい |
| **セパレートカラー** | 境界に別の色（主に無彩色）を挟む | 補色同士のハレーション回避。類似色の境界を明確化（白フチ・シャドウも同様） |
| **グラデーション** | 色の境界をなじませて複数の色を表現 | 補色間のグラデーションは中間がグレーに濁る → 中間色または白を挟む |

## 5. イメージ別配色判断テーブル

Webサイトのターゲット・目的から配色の方向性を決定する。

| イメージ | トーンの方向性 | カラー傾向 | ベースカラー | アクセントカラー |
|---------|-------------|-----------|------------|----------------|
| **キュート・ポップ** | 高明度・高彩度 | パステル〜ビビッド。ピンク・水色・黄 | 白または淡色 | 暖色系の彩度高め |
| **エレガント・高級** | 低明度・低彩度 | モノクロ基調。ダークブラウン・ゴールド差し色 | 白または黒 | 明度高めの無彩色またはゴールド系 |
| **ビジネス・信頼** | 中〜低彩度、安定感 | 青・グレー系。コーポレートカラー軸 | 白・薄グレー | コーポレートカラー（青系多い） |
| **オーガニック・ナチュラル** | 中明度・低〜中彩度、柔らかい | グリーン・ベージュ・ブラウン | 白・クリーム | テラコッタ・オリーブ系 |
| **クール・先端** | 低彩度または大胆なビビッド | 無彩色＋1〜2色の強いアクセント。または多色を大胆に | 黒または白 | 電気系ブルー・ビビッドカラー |
| **和** | 低彩度・品のある淡色 | 藍・朱・深緑・金など日本の伝統色。彩度低め | 白・薄灰 | 日本の伝統色（藍・朱） |

### 判断フロー

```
ターゲット・コンセプトの確認
  ↓
上表でイメージカテゴリを選択
  ↓
メインカラー（ブランドカラーがあれば優先）を決定
  ↓
ベースカラー（メインカラーを引き立てる色）を設定
  ↓
コンバージョンエリア用のアクセントカラーを設定
  ↓
（必要に応じて）サブカラーを追加
```

### 各イメージの設計ポイント

**キュート・ポップ**
- 明度と彩度の組み合わせで落ち着き度を調整可能
- コンバージョンエリアには無彩色（黒系）を使い、明暗でメリハリを出す

**エレガント・高級**
- 明度の低い色を基調とし、無彩色の白をアクセントに使う「落ち着き＋上質さ」が定番
- ビジュアル（写真）の力強さと配色のバランスをとる

**ビジネス・信頼**
- コーポレートカラーをリンク・ボタン・アイコンなど機能的な要素に使用
- ホワイトスペースを広くとることで誠実さと可読性を両立

**オーガニック・ナチュラル**
- 緑・ベージュのビジュアルとホワイトスペースの組み合わせが定番
- セクションごとにベースカラーとメインカラーを使い分けてコンテンツを明確化

**クール・先端**
- 無彩色のみで力強さと落ち着きを両立できる
- 先端性を強調する場合は複数の強い色を大胆に組み合わせる（大胆配色 × まとまりのある画面設計）

**和**
- フォントカラーを完全な黒ではなく背景に馴染む落ち着いたカラーにすると空気感が生まれる
- ページ全体でセクションごとに明度・彩度を徐々に変化させるストーリー設計も有効

## 6. Webアクセシビリティと配色

あらゆる人（高齢者・障がい者・異なる色覚特性をもつ方）がWebサイトを利用できるよう、配色段階から配慮する。

### コントラスト比の原則

| 対象 | 推奨 |
|------|------|
| 通常テキスト | コントラスト比 4.5:1 以上（WCAG 2.1 AA）|
| 大きなテキスト（18pt以上・14pt以上の太字） | コントラスト比 3:1 以上 |
| UIコンポーネント・グラフィック | コントラスト比 3:1 以上 |

**ポイント**: コントラストは**明度差**で決まる。色相の違いだけでは不十分。

### 色覚特性への対応（カラーユニバーサルデザイン）

| 問題 | 対策 |
|------|------|
| 色相だけで情報を区別している | テキスト・アイコン・パターンも併用する |
| 赤・緑の組み合わせ | 明度差を大きくする、または別の色相の組み合わせに変更 |
| グラフの色分け | 隣接するカテゴリに色相環で等距離の色を選ぶ。ラベルも必ず表示 |

### 確認方法

- Figma / Photoshopの色覚シミュレーション機能（P型・D型）で確認
- ブラウザの DevTools のアクセシビリティ機能でコントラスト比を確認
- 自治体のカラーユニバーサルデザインガイドライン（東京都等）を参照

### 実装チェックリスト

- [ ] 背景と文字のコントラスト比が基準を満たしているか
- [ ] 色だけで情報を伝えていない（テキスト・形・パターンの補助あり）
- [ ] P型・D型色覚シミュレーションで問題がないか
- [ ] 色覚特性を考慮した配色ツール（Figma プラグイン等）で検証済みか
