## 1. ユーザーリサーチの手法

定性・定量の両アプローチを組み合わせることが基本。

| 手法 | 種類 | 得られる知見 | 主なツール |
|------|------|------------|-----------|
| インタビュー | 定性 | 行動の背後にある動機・理由 | Zoom, Dovetail |
| サーベイ | 定量 | 満足度・利用パターン・デモグラフィック | Typeform, Google Forms |
| ユーザビリティテスト | 定性 | 操作上の摩擦・直感性の評価 | Maze, UserTesting |
| データ分析 | 定量 | 利用率・コンバージョン・ドロップオフ地点 | GA4, Mixpanel, FullStory |

> **インタビューの原則**: オープンエンドな質問で深掘り → フォローアップで文脈を補完

## 2. リサーチの成功要因

| 成功要因 | 説明 |
|---------|------|
| 明確なリサーチ目標 | 「何を明らかにしたいか」を事前に定義。データ収集の焦点を維持する |
| ターゲットオーディエンスの定義 | ユーザーセグメントを特定し、そのセグメント特有の行動・ニーズに合わせたリサーチ設計 |
| 多様なサンプル | デモグラフィック・役割・利用形態が偏らないよう参加者を選定。代表性を確保する |
| プライバシーの尊重 | 参加者データの機密性を明示。信頼関係の構築がフィードバックの質を高める |

## 3. リサーチの課題と対策

### 課題マトリクス

| 課題 | リスク | 対策 |
|------|--------|------|
| **参加者リクルート** | ターゲットを代表しないサンプルによる偏り | 多様なチャネル（SNS・フォーラム・専門エージェンシー）を活用。謝礼でモチベーション向上 |
| **時間制約** | 深度不足・品質劣化 | アジャイル方式（スプリント単位）で実施。優先順位を明確化 |
| **データ分析の複雑さ** | 誤解釈・見落とし | データアナリストとの協業、BIツール活用、定性・定量の反復的な検証 |
| **前提への盲点** | プロダクトに近すぎて仮定を疑えない | 外部視点（UXリサーチャー、ユーザーアドバイザリーボード）を取り込む |

> **実践知**: カスタマー/パートナー・アドバイザリーボードは、ペインを継続的に収集する「常時オン」の仕組みとして機能し、製品ロードマップへの影響力をメンバーに与えることでリテンションにも寄与する。

## 4. B2C vs B2B ペルソナの違い

| 比較軸 | B2C バイヤーペルソナ | B2B バイヤーペルソナ |
|--------|--------------------|--------------------|
| **意思決定者** | 個人（単独） | 組織内の複数ステークホルダー（役割別に定義が必要） |
| **動機** | 個人の欲求・感情・痛点 | 組織目標・課題・期待するアウトカム |
| **購買複雑性** | シンプル・トランザクション型 | 多段階（リサーチ・評価・交渉・承認） |
| **意思決定基準** | 価格・利便性・ブランド知覚 | BANT / MEDDPICC・ROI・ベンダー信頼性 |
| **関係性** | 短期的インタラクション | 長期的パートナーシップ（信頼・実績重視） |
| **コミュニケーション** | SNS・ダイレクトマーケティング等広域 | 組織内フォーマルチャネル |
| **カスタマイズ要求** | 低〜中 | 高（ソリューションフィット必須） |
| **業界知識の必要性** | 低 | 高（市場動向・課題の深い理解が必要） |

### B2B意思決定フレームワーク

| フレームワーク | 構成要素 | 適用場面 |
|-------------|---------|---------|
| **BANT** | Budget（予算）/ Authority（権限）/ Need（必要性）/ Timeline（時期） | 初期リード資格確認 |
| **MEDDPICC** | Metrics / Economic Buyer / Decision Criteria / Decision Process / Paper Process / Identify Pain / Champion / Competition | 複雑なエンタープライズ営業・ペルソナ深掘り |

## 5. バイヤーペルソナ開発プロセス

### B2B ペルソナ開発（10ステップ）

| ステップ | 内容 |
|---------|------|
| 1. 主要ステークホルダー特定 | 意思決定者・影響者・ユーザー等の役割を洗い出す |
| 2. 社内インタビュー | 営業・CS・PMから既存顧客の知見を収集 |
| 3. 外部データ収集 | サーベイ・インタビュー・市場調査でターゲットの課題・目標・嗜好を把握 |
| 4. 顧客データ分析 | 既存データのパターン・トレンドを分析 |
| 5. デモ/ファームグラフィック取得 | 業種・規模・役職等の属性データ収集 |
| 6. 目標とペインポイントの定義 | ターゲットの達成したいことと障壁を明確化 |
| 7. 意思決定要因の特定 | 購買を左右するファクターを抽出 |
| 8. ペルソナナラティブ作成 | 名前・職種・責務・目標・課題を含む人物像を叙述 |
| 9. ペルソナの検証 | 内部ステークホルダーとのフィードバックループで精度を確認 |
| 10. 意思決定への活用 | マーケティング・製品開発・CS・営業戦略の参照モデルとして運用 |

### B2C ペルソナ開発（10ステップ）

| ステップ | 内容 |
|---------|------|
| 1. ターゲット市場の定義 | 年齢・性別・地域・興味・収入等の軸でセグメント設定 |
| 2. 市場調査の実施 | サーベイ・SNS分析・インタビューで消費者ニーズを把握 |
| 3. 顧客データ分析 | 購買履歴・Webアナリティクス・サポート履歴からパターン抽出 |
| 4. デモ/サイコグラフィック情報収集 | 価値観・ライフスタイル・態度まで含む人物像の輪郭を描く |
| 5. 目標とペインポイント把握 | 解決したい課題・達成したい目標を理解 |
| 6. 意思決定要因の特定 | 価格感度・ブランド知覚・レビュー・利便性等のファクター |
| 7. ペルソナナラティブ作成 | 架空人物として名前・属性・目標・課題・行動を叙述 |
| 8. ペルソナの検証 | 市場調査データと顧客接点スタッフのフィードバックで精度確認 |
| 9. マーケティング戦略への適用 | コンテンツ・メッセージング・広告をペルソナ基準で設計 |
| 10. 継続的精緻化 | 定期的に見直し、消費者行動の変化に合わせてアップデート |

> **実例（IKEA）**: 都心店 → 小スペース居住の学生（予算制約重視）、郊外店 → 子育て世代・シニアカップル。Mosaicの消費者プロファイリングと組み合わせ、立地ごとのターゲットを精緻化。

## 6. ペインポイント分析

### 特定の4手法

| 手法 | 方法 | 強み |
|------|------|------|
| **サーベイ・インタビュー** | オープンエンド質問で不満・課題を直接収集。アドバイザリーボードで継続的ダイアログを構築 | 直接的フィードバック |
| **ソーシャルリスニング** | SNS・業界フォーラム・レビューサイトを監視し、繰り返し登場するテーマを抽出 | 無意識の本音が現れやすい |
| **カスタマーサポート分析** | サポートチケット・メール・チャット履歴からパターンと再発痛点を特定 | 実際の摩擦点を網羅 |
| **顧客データ分析** | 利用パターン・行動アナリティクス・購買履歴のドロップオフ地点を分析 | 大規模・定量的 |

### ペインポイント特定の考慮事項

| 考慮事項 | 説明 |
|---------|------|
| 定量＋定性の組み合わせ | 統計的洞察と「なぜそうなるか」のコンテキストを両立させる |
| セグメント内の可変性 | 同一ターゲット内でも役割・業種・デモグラフィックでペインは異なる |
| 潜在的ペインの発掘 | 明示されないペインを探るために観察・プロービング質問・録画ユーザーテストを活用 |
| 優先順位付け | 顧客満足への影響度・発生頻度でスコアリング。高インパクト・高頻度を優先対処 |

## 7. RADモデル：獲得・リテンション・開発戦略

**RAD = Retention（リテンション）/ Acquisition（獲得）/ Development（開発）**

ペルソナ知識を活用した統合的顧客戦略フレームワーク。

### リテンション（既存顧客の継続・満足）

| 戦略 | 内容 |
|------|------|
| パーソナライズドコミュニケーション | ペルソナ情報を元にメール・オファー・コンテンツをカスタマイズ |
| テーラードプロダクトアップデート | ペルソナの痛点・フィードバックを製品改善サイクルに直結させる |
| カスタマーサクセスプログラム | ペルソナごとの目標達成を支援するリソース・トレーニング・サポートを設計 |
| プロアクティブエンゲージメント | フィードバック収集・満足度調査を能動的に実施。チャーンリスクを早期検知 |

### 獲得（新規顧客の取り込み）

| 戦略 | 内容 |
|------|------|
| ターゲットマーケティング | ペルソナの痛点・動機に響くメッセージ・コンテンツ・広告を開発 |
| チャネル選択 | ペルソナが最も活動するプラットフォームに集中投資 |
| プロダクトポジショニング | ペルソナ特有のニーズに合った価値提案を前面に出す |
| リードナーチャリング | バイヤーズジャーニーの各フェーズに合わせた情報・体験を提供し、信頼を構築 |

### 開発（既存顧客の価値最大化）

| 戦略 | 内容 |
|------|------|
| カスタマージャーニー強化 | ペルソナ別に既存顧客のタッチポイント・コンテンツを継続改善 |
| アップセル/クロスセル | ペルソナの進化するニーズに合ったオファーで CLV（顧客生涯価値）を拡大 |
| パーソナライズドエンゲージメント | 個別成功にコミットするコンテンツ・インタラクションで深い関係性を構築 |
| フィードバック起動の製品開発 | ペルソナ別のフィードバックを継続収集し、製品の価値持続性を保つ |

> **実例（Flow State）**: Intercomによる常時オンフィードバック＋デモ時の市場調査＋競合モニタリング＋A/Bテストを組み合わせ、新規高価値セグメント2つを発見。機能を「Exciter（獲得用）」と「Essential（リテンション用）」に分類し、投資配分を最適化。

## 8. PLGビジネスにおけるペルソナ活用

PLG（Product-Led Growth）では、プロダクト自体が獲得・リテンションの主ドライバーになる。ペルソナ知識はこの文脈で特に重要。

| 活用領域 | ペルソナ知識の役割 |
|---------|-----------------|
| **製品開発方向性** | ペインポイント・期待機能・ユーザー期待を特定し、プロダクトの優先事項を決定 |
| **UX設計** | 目標・嗜好・行動パターンからUIの直感性を最適化。エンゲージメント・定着率を向上 |
| **サポートシステム設計** | ペルソナのよくある問題を予測したドキュメント・FAQ・リソースを整備 |
| **市場戦略** | ペルソナ特有のニーズに合わせたメッセージング・チャネル選択でコンバージョン率向上 |
| **チャーン予防** | 潜在的なペルソナニーズの変化を予測し、プロアクティブにチャーンリスクを低減 |

> **PLG vs セールス主導の選択**: 市場に代替品が多い → PLGを優先。パイオニア製品 → セールス＋プロダクト主導のハイブリッドで市場を押さえる。
