# ノー推敲嫌われ文章パターン集

推敲を怠った文章に現れる5つの典型パターンと、
推敲の手間そのものを減らすための3つのテクニック、
推敲に詰まったときの3つの突破口を体系化する。

本リファレンスは、ビジネス文章技術に関する公開知見を体系化したものである。

---

## 1. ノー推敲嫌われ文章 5パターン

推敲なしで提出・公開された文章に繰り返し現れる問題パターン。
各パターンを検出・修正するための手順を示す。

---

### パターン1: 接続詞が多すぎる文章

#### なにが問題か

「だから」「しかし」「そして」「すると」を多用すると、文章全体がくどい印象になる。
接続詞は本来、読み手に「ここで話が変わる」「前の内容の原因がここにある」と示す標識だ。
使わなくても意味が通じる接続詞は不要であり、残すべきではない。

#### 接続詞の11種類と役割

| 種類 | 役割 | 代表例 |
|------|------|-------|
| 順接 | 前文を原因とし、結果をつなぐ | だから、したがって、すると |
| 逆説 | 前文と異なる内容を続ける | しかし、だが、ところが |
| 添加 | 前文に別の内容を加える | しかも、加えて、それに |
| 並列 | 前文と同等の内容を並べる | また、かつ、および |
| 対比 | 前後の文章を対比させる | 一方、逆に、反対に |
| 選択 | 前文と選択関係にある | あるいは、もしくは |
| 言換 | 前文の内容をわかりやすく言い換える | つまり、すなわち、要するに |
| 説明 | 前文の内容を説明する | なぜなら、というのは |
| 補足 | 前文に補足内容を続ける | ただし、なお、ちなみに |
| 例示 | 前文の具体例を示す | たとえば |
| 転換 | 話題を切り替える | ところで、では、さて |

11種類のうち、同じ種類が短い間隔で繰り返し登場する場合は削除か言い換えを検討する。

#### 検出方法

1. 書いた文章を読み返し、接続詞に下線を引く
2. その接続詞を削除しても意味が通じるか確認する
3. 通じるなら削除する

#### 修正例

❌ 推敲前
> その年の健康診断では、過去最悪の結果が出てしまった。だから、それから毎日ジョギングに励んだ。すると、ウエストまわりだけでなく脚や二の腕なども引き締まったような気がした。だが、空腹を覚える時間が増えてしまった。そうして食事量が増えたのだが、しかし、その分揚げ物や炭水化物もたっぷりと摂ってしまったため、その結果として体重は以前よりも数キロ増えてしまった。

✅ 推敲後
> その年の健康診断では、過去最悪の結果が出た。毎日ジョギングに励んだところ、ウエストまわりだけではなく脚や二の腕なども引き締まった気がした。だが、空腹を覚える時間が増え、食事量も増えた。揚げ物や炭水化物もたっぷりと摂ったため、体重は以前よりも数キロ増えてしまった。

接続詞を極限まで削っても意味は通じる。意味が通じれば削除が正解だ。

### パターン2: 主語と述語がおかしな文章

#### なにが問題か

文章のコアは「主語＋述語」の対応関係だ。この対応が崩れると、読み手は意味を推測させられ、不快感を覚える。「てにをは」の誤用・主語の多重化・主述のねじれが主な原因となる。

#### 典型的なミス

| 問題の種類 | 悪い例 | 正しい例 |
|----------|--------|--------|
| てにをは の誤り | 私にプレゼンを依頼された | 私はプレゼンを依頼された |
| 主語の多重化 | 私に上司にプレゼンを依頼した | 上司は私にプレゼンを依頼した |
| 受動と能動のねじれ | 葉と茎を選別されず | 葉と茎は選別されず |
| 授受表現の混乱 | 飲まれたのですが | 飲ませていただいたのですが |

#### 検出方法

1. 文章を書いたら、まず主語を特定する（「誰が・何が」）
2. 次に述語を特定する（「どうした・どうだ」）
3. 主語と述語だけを取り出して声に出す
4. つじつまが合うか確認する

#### 修正例

❌ 推敲前（問題箇所が複数混在）
> 今回ご紹介する商品は、生産者さんに自家用に保存しているお茶。葉と茎を選別されず、ミックスになっています。実際に生産者さんを訪れたときに飲まれたのですが、葉と茎を選別していないためか香りがよく、味も濃厚。

✅ 推敲後
> 今回ご紹介する商品は、生産者さんが自家用に保存しているお茶。葉と茎は選別されず、ミックスになっています。実際に茶どころの生産者さんを訪れたときに飲ませていただいたのですが、葉と茎を選別していないためか香りがよく、味も濃厚。

主語・助詞・授受表現の修正で、すっと頭に入る文章になる。

### パターン3: 無駄が多く長いだけの文章

#### なにが問題か

書き手が思いついた順に書き綴ると、文章が脱線する。
読み手は「結局、何が言いたいの？」と感じ、途中で読むのをやめる。

#### 原因

- 書く前に構成を決めていない
- 「伝えたいこと」と「書いてしまったこと」の区別ができていない
- 推敲で不要情報を削除していない

#### 検出方法：情報の3分類

書き終えた文章の各要素を次の3つに分類する。

| 分類 | 定義 | 扱い |
|------|------|------|
| 絶対必要な情報 | 伝えたい内容の核。これがなければ成立しない | 残す |
| まあまあ必要な情報 | 核を補足・強調するが、なくても概ね伝わる | 判断して残すか削除 |
| 不要な情報 | 脱線・雑談・重複。核と無関係 | 削除する |

#### 修正例

❌ 推敲前（ジョギング会の告知メール）
> お世話になっております。落ち葉がフカフカの絨毯のようで、歩道をジョギングしていても足にやさしい季節となりました。でも、雨が降ると葉が湿ってツルっと滑りやすいので注意が必要ですね。……（中略）……このカフェのランチ、先月からハマっておりまして、特にパン食べ放題なのがとても良いです。最近、妻がある絵本を読んでからパンには目がないようで……

✅ 推敲後
> 今月最後の日曜日にジョギング会を開催します。軽くジョギングしながら郊外の自然を楽しみましょう。終了後は、集合場所近くのカフェでランチを予定しています（パン食べ放題あり）。参加費無料・ランチ代のみ実費です。先着四名で締切りますのでお申し込みはお早めに。

「妻がパンが好き」という情報は3分類の「不要な情報」。削除することで伝えたいことが明確になる。

### パターン4: 専門用語ばかりでチンプンカンプンの文章

#### なにが問題か

業界内で当たり前の言葉も、業界外の読み手には意味不明な暗号と同じだ。
読み手は理解するために調べる手間を強いられ、途中で読むのをやめる。
「専門用語を使いすぎていないか」は推敲時に必ず確認する。

#### 専門用語・略称の使用判断

| 状況 | 判断 |
|------|------|
| 読み手が同業者・専門家 | 使用可 |
| 読み手が一般顧客・不特定多数 | 使用を控える。平易な表現に置き換える |
| 一般的か判断できない場合 | 使用を控えるか、「○○（説明）」の形式で補足 |
| 初出のみ略称を使いたい場合 | 「正式名称（略称）」と表記してから略称を使う |

#### 修正例

❌ 推敲前
> UUにいかにリーチできるかを考慮。さらに、クロスセリングを向上させカゴ落ちを防ぐ。

✅ 推敲後
> サイト訪問者にいかに広告を届けられるかを考慮。さらに、客単価を上げるため別商品との「あわせ買い」をうながし、ショッピングカートに商品を入れたままサイトを離れてしまうのを防ぐ。

文字数は増えるが、読み手の理解度は大幅に向上する。

#### 読み手の属性に合わせたチェック項目

- [ ] 読み手が知らない可能性のある略称を使っていないか
- [ ] 業界固有の言葉を一般読者向けに使っていないか
- [ ] 略称を初出で「正式名称（略称）」の形で紹介しているか
- [ ] 難しい漢字に読み仮名（ルビ）を振っているか

---

### パターン5: ウラがとれていない文章

#### なにが問題か

憶測・伝聞・引用を事実のように書くと、読み手に不快感を与える。
特にヘルスケア・医学・法律・金融など、誤情報が実害につながる領域では致命的だ。
「これは事実か、それとも憶測か」を推敲時に問い直す必要がある。

#### 検出方法

1. 文章中の数値・統計・引用を抽出する
2. 各情報の出どころを確認する
3. 出典を明記していない情報があれば補足するか削除する

#### 出典の書き方

| 種類 | 書き方の例 |
|------|----------|
| 書籍 | 「山田太郎『文章術入門』（〇〇出版、2023年）p.45」 |
| Webサイト | 「〇〇 https://example.com（2026年5月現在）」 |
| 取材・聞き取り | 「〇〇社 山田部長へのインタビュー（2026年3月）」 |
| 公的統計 | 「総務省統計局『国勢調査』（2020年）」 |

#### 注意すべき表現

| 問題のある表現 | 対処方法 |
|-------------|--------|
| 「〜と言われています」（出典不明） | 出典を探して明記。見つからなければ削除 |
| 「〜の可能性があります」（根拠なし） | 根拠を示すか、断言しないことを明示する |
| 「多くの専門家は〜」（誰かを確認できない） | 具体的な専門家名・機関名を挙げる |

## 2. 推敲不要に近づける3テク

推敲そのものをゼロにすることはできない。
しかし、事前の意識次第で推敲の手間を大幅に減らせる。

---

### テク1: 自分のクセを把握する

誰でも文章に書き言葉のクセがある。
クセを把握し、書くときから意識的に避けることで、推敲の手間が減る。

#### クセの見つけ方

1. 自分が書いた文章を3〜5本集める
2. 特定の言い回しが繰り返し登場していないか確認する
3. 検索機能で頻出語句をカウントする

#### よくある文章クセの例

| クセの種類 | 例 |
|----------|-----|
| 特定の文末の繰り返し | 「〜でしょうか」「〜のです」が何度も連続 |
| 特定の接続詞の乱用 | 「しかし」を3文に1回以上使う |
| 特定のフレーズの繰り返し | 「要するに〜だ」「そして〜した」が頻出 |
| 「こと」「もの」の多用 | 「〜ということは〜ということ」 |

#### クセのリスト化

見つけたクセは「自分の頻出NGワードリスト」として書き留める。
執筆・推敲の際に手元に置いておくと、クセの再発を防げる。

---

### テク2: 「正しい言葉」を意識する

推敲で修正する前に、書きながら「この言葉は正しいか」を確認する習慣が推敲コストを下げる。

#### 誤用しやすい言葉

| 語句 | よくある誤用 | 正しい意味・用法 |
|------|------------|--------------|
| 了解しました | 目上の人に使う | 同僚・部下への返答。目上には「承知しました」 |
| させていただく | あらゆる場面で使う | 相手の許可を受け、かつ恩恵を受ける場合のみ |
| しずらい | 「〜しにくい」の意味で | 正しくは「しづらい」 |
| どおりで | 「道理で」として | 正しくは「どうりで」 |
| 役不足 | 「本人には重すぎる仕事」の意味で | 「本人の実力に対して仕事が軽すぎる」 |
| 確信犯 | 「悪いと知ってやること」の意味で | 「正しいと信じてやること」 |

#### 重複表現に注意する

| 重複例 | 修正後 |
|--------|--------|
| 約半数ほど | 約半数 / 半数ほど |
| 全職員総出で | 職員総出で |
| 毎日が仕事に追われ余裕がない日々 | 毎日仕事に追われ余裕がない / 仕事に追われ余裕がない日々 |
| 日記を日々綴る | 日記を綴る |
| 議決を決める | 議決する |

#### 同音異義語の選択

同じ読みでも意味が異なる語は、文脈に合った漢字を選ぶ。

| 読み | 候補 | 意味の違い |
|------|------|---------|
| ついきゅう | 追及 | 追いつめる |
| ついきゅう | 追求 | どこまでも追い求める |
| ついきゅう | 追究 | 真実を明らかにしようとする |

---

### テク3: 構成で脱線を防ぐ

書き始める前に「何をどの順番で書くか」を決めておけば、脱線が大幅に減る。
構成を決めずに書き始めると、思いついたことを順番に書き連ねてしまい、無駄な情報が混入する。

#### 基本構成テンプレート

```
① イントロダクション（背景・問題提起）
② 伝えたい内容 その1（最重要）
③ 伝えたい内容 その2
④ 伝えたい内容 その3（必要に応じて）
⑤ まとめ（結論の再提示）
```

#### 構成を決めたあとの確認

- 「この段落で書くのは何か」を1文で言えるか
- イントロで提示したテーマと、本文の内容が一致しているか
- まとめが「問いへの回答」になっているか

#### 主な文章の型

| 型 | 用途 | 構造 |
|----|------|------|
| PREP法 | ビジネス文書・説明文 | 結論→理由→具体例→結論の繰り返し |
| 主張型 | 論文・意見文 | 問題提起→主張→根拠→結論 |
| ストーリー型 | エッセイ・事例紹介 | 状況設定→展開→結末 |
| 直感型 | エッセイ・随筆 | 体験→感想→深い考察 |

型に沿って構成を決めてから書けば、不要な情報が入り込む余地が減り、推敲の手間が大幅に下がる。

## 3. 推敲に困ったら（First Aid）

推敲していても「これでいいのか」「どこを直せばよいかわからない」という状態に陥ることがある。
そのときに有効な3つのアプローチを紹介する。

---

### First Aid 1: 意地悪な読者になりきる（敵対的読み）

「物わかりのいい友人として読む」のは推敲にならない。
「どれだけ手を尽くせば、読み手にわかってもらえるだろうか」というスタンスで文章を見直す。

#### 敵対的読みの実践手順

1. 「これは本当に伝わるか？」と疑問を持ちながら読む
2. 「意味が分からない」と感じた箇所に印をつける
3. 「別の解釈ができる」箇所を探す
4. 印をつけた箇所を書き直す

#### 敵対的読みのチェック視点

| 視点 | 確認内容 |
|------|--------|
| 子ども視点 | 中学生が読んでも意味がわかるか |
| 専門外視点 | 業界外の人が読んで理解できるか |
| 批判者視点 | 反論できる箇所はないか |
| 「だから何？」視点 | 読み終えたあとに「それで？」となっていないか |

> 推敲の心得: 書き上げたら、意地悪な読み手になったつもりで読み返すこと。わかりのいい友人のような読み方では、よい推敲はできない。

---

### First Aid 2: 身近な読者にチェックしてもらう

自分の文章を自分で推敲するには限界がある。
書き手は内容を知りすぎているため、客観的に読めない。
身近な人に初見で読んでもらうことで、読み手の視点を得られる。

#### 依頼する人の選び方

| 文章の種類 | 適した確認者 |
|---------|------------|
| 仕事の書類 | 同僚・上司（内容を知らない人が望ましい）|
| ブログ・SNS | 家族・友人（対象読者に近い属性の人） |
| 技術文書 | 同分野の専門家（専門性を確認する場合） |
| 一般向け文章 | 専門外の人（難易度確認に最適） |

#### 依頼時のポイント

1. チェックリストを渡して「特にここを確認してほしい」と伝える
2. 「わかりにくい」「引っかかった」箇所を具体的に挙げてもらう
3. 「なぜ気になったか」「どう直すとよいか」まで聞く
4. 修正指示を受け取ったら、感情的にならず「何はともあれ書き直す」
5. 同じ人に何度も依頼すると慣れが生じるため、時折確認者を変える

#### 確認後の振り返り

- 自分が見落とした箇所と他者が指摘した箇所を比較する
- 「なぜ自分は気づかなかったか」を分析する
- 次回の推敲に活かす

---

### First Aid 3: プロの文章と自分の差分を分析する

「いい文章のお手本」を設定し、自分の文章と比較することで、不足点が見える。

#### 差分分析の手順

1. 「これはいい文章だ」と感じる書籍・記事を1本選ぶ
2. 「どこがいいと感じたか」を具体的に言語化する（リズム？構成？語彙？）
3. 自分の文章と同じ主題で1段落書いてみる
4. 2で挙げた「いい点」が自分の文章にあるか確認する
5. 不足する点を書き留め、次回の執筆・推敲に活かす

#### 比較する観点

| 観点 | 確認内容 |
|------|--------|
| 文章の長さ | 一文の長さ・段落の長さがどう設計されているか |
| 語彙の質 | 平易な言葉か、専門語か。言い換えの工夫は |
| リズム | 長文と短文のバランス。文末表現のバリエーション |
| 構成 | 情報の順番。導入→展開→結論の流れ |
| 削り方 | 何を削り、何を残しているか |

> プロライターの定石: 自分が「この文章はいいなぁ」と感じるビジネス書を数冊、模範として手元に置いて参考にすること。

## Quick Reference

### 5パターン早見表

| パターン | 検出の合言葉 | 即効対処 |
|---------|-----------|--------|
| 接続詞過多 | 「これ、消しても意味通じる？」 | 不要な接続詞を削除 |
| 主述のねじれ | 「主語と述語だけ取り出す」 | 主語を確認→述語を対応させる |
| 無駄に長い | 「絶対必要・まあまあ・不要の3分類」 | 不要情報を削除 |
| 専門用語 | 「外部の人に伝わるか」 | 平易な言葉に置き換え |
| ウラなし | 「出典はどこ？」 | 出典を明記 or 削除 |

### 3テク早見表

| テク | 実践のポイント |
|-----|-------------|
| クセ把握 | 自分の頻出NGワードをリスト化して手元に置く |
| 正しい言葉 | 「了解」「させていただく」などの誤用を意識する |
| 構成先行 | 書き始める前に5段落構成を書き出す |

### First Aid 早見表

| 詰まったとき | 取るべきアクション |
|----------|----------------|
| 「これで伝わるか自信がない」 | 意地悪な読者になりきる（敵対的読み） |
| 「自分では直しきれない」 | 身近な読者に初見で読んでもらう |
| 「どこを直せばいいかわからない」 | プロの文章と並べて差分を言語化する |
