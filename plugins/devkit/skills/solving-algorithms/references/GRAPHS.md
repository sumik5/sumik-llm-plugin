# グラフアルゴリズム リファレンス

グラフ（基本走査・最小全域木・最短経路・高度アルゴリズム）の実装ガイド。

---

## グラフ表現

グラフ G = (V, E) を表現する方法は主に2つある。問題の性質（密なグラフか疎なグラフか）で使い分ける。

### 隣接行列 (Adjacency Matrix)

`adj[i][j]` に辺の有無（または重み）を記録する |V| × |V| の2次元配列。

```
adj[i][j] = 1        // 辺あり（無向グラフ）
adj[i][j] = w        // 重みw（重み付きグラフ）
adj[i][j] = ∞ or 0  // 辺なし
```

| 操作 | 計算量 |
|------|--------|
| 辺 (u, v) の存在確認 | O(1) |
| 辺の追加・削除 | O(1) |
| メモリ使用量 | O(\|V\|²) |
| 全隣接頂点の列挙 | O(\|V\|) |

**向いている場面**: 頂点数が少ない（≤ 1,000）密なグラフ、辺の存在確認が頻繁な場合

**向いていない場面**: 頂点数が多い、辺数が少ない疎なグラフ

### 隣接リスト (Adjacency List)

各頂点 u に隣接する頂点（と重み）のリストを配列で管理する。

```
G[u] = [(v1, w1), (v2, w2), ...]  // 頂点uに隣接する頂点と重みのリスト
```

| 操作 | 計算量 |
|------|--------|
| 辺 (u, v) の存在確認 | O(deg(u)) |
| 辺の追加 | O(1) |
| メモリ使用量 | O(\|V\| + \|E\|) |
| 全隣接頂点の列挙 | O(deg(u)) |

**向いている場面**: 頂点数・辺数が多い疎なグラフ、多くのグラフアルゴリズムのデフォルト選択

## 基本走査

### 深さ優先探索 (DFS: Depth First Search)

「行けるところまで進み、行き止まりで戻る」戦略のグラフ探索。

**核心アイデア**: スタック（または再帰呼び出し）を使い、未訪問の隣接頂点を優先的に掘り下げる。

```
DFS(G, s):
    全頂点の状態を WHITE（未訪問）に初期化

    dfs_visit(u):
        u の状態を GRAY（訪問中）に設定
        for u に隣接する各頂点 v:
            if v が WHITE:
                dfs_visit(v)
        u の状態を BLACK（完了）に設定
```

**計算量**:
- 隣接行列使用時: O(|V|²)
- 隣接リスト使用時: O(|V| + |E|)

**タイムスタンプ**: DFS では各頂点に発見時刻 `d[v]` と完了時刻 `f[v]` を記録でき、辺の分類（木辺・後退辺・横断辺）に使える。

**実装上の注意**: 大きなグラフでは再帰によるスタックオーバーフローに注意。明示的スタックを使った反復実装を検討する。

### 幅優先探索 (BFS: Breadth First Search)

始点から距離の小さい順に頂点を訪問する探索。始点からの最短距離（辺数ベース）を求められる。

**核心アイデア**: キューを使い、発見した頂点を順番に処理する。

```
BFS(G, s):
    全頂点 u に d[u] = ∞ を設定
    d[s] = 0
    Q.enqueue(s)

    while Q が空でない:
        u = Q.dequeue()
        for u に隣接する各頂点 v:
            if d[v] == ∞:
                d[v] = d[u] + 1
                Q.enqueue(v)
```

**計算量**:
- 隣接行列使用時: O(|V|²)
- 隣接リスト使用時: O(|V| + |E|)

**BFS の重要な性質**: 辺の重みが均一（または無重みグラフ）の場合、BFS で求めた距離が最短経路の辺数に等しい。

## 連結成分 (Connected Components)

無向グラフの連結成分（互いに到達可能な頂点の極大集合）を求める。

**核心アイデア**: 未訪問頂点を始点として DFS/BFS を繰り返し、各探索ごとに異なるラベル（ID）を割り当てる。

```
assign_components(G):
    全頂点の label = -1（未割り当て）に初期化
    id = 0
    for u が 0 から |V|-1:
        if label[u] == -1:
            dfs(u, id)  // u に到達できる全頂点に id を割り当て
            id++

    // 2頂点 s, t が同じ連結成分かどうかは label[s] == label[t] で O(1) 判定
```

**計算量**: O(|V| + |E|)（隣接リスト使用時）

**代替手段**: Union-Find を使った連結成分管理は辺を動的に追加する場合に有効（参照: DATA-STRUCTURES.md）。

## 最小全域木 (Minimum Spanning Tree: MST)

重み付き無向連結グラフの全域木のうち、辺の重みの総和が最小のもの。

### Kruskal のアルゴリズム

辺を重みの昇順にソートし、閉路を作らない辺を貪欲に追加していく。Union-Find を使って閉路判定を行う。

```
kruskal(V, E):
    E を辺の重みの昇順にソート
    Union-Find S を初期化（各頂点が独立した集合）
    MST = 空集合

    for 各辺 e = (u, v, w) in ソート済み E:
        if S.find(u) != S.find(v):  // u と v が異なる連結成分
            MST.add(e)
            S.unite(u, v)
        if |MST| == |V| - 1: break

    return MST
```

**計算量**: O(|E| log|E|)（辺のソートが支配的）

**向いている場面**: 疎なグラフ（|E| が小さい場合）

### Prim のアルゴリズム

MST に属する頂点集合 T を1頂点から出発して拡張していく。T と V-T を繋ぐ辺のうち最小重みのものを選ぶ。

```
prim(G, root):
    全頂点 u に key[u] = ∞ を設定
    key[root] = 0
    MST_set = 空集合

    while MST_set != V:
        u = key が最小かつ MST_set に属さない頂点
        MST_set.add(u)
        for u に隣接する各頂点 v（MST_set 外）:
            if w(u, v) < key[v]:
                key[v] = w(u, v)
                parent[v] = u
```

**計算量**:
- 隣接行列 + 線形スキャン: O(|V|²)
- 隣接リスト + 優先度付きキュー: O((|V| + |E|) log|V|) → O(|E| log|V|)

**向いている場面**: 密なグラフ（|E| が大きい場合）では O(|V|²) 版が有利

## 最短経路

### Dijkstra のアルゴリズム（単一始点、非負辺）

始点 s から全頂点への最短経路を求める。**負の辺に使用不可**。

```
dijkstra(G, s):
    全頂点 u に d[u] = ∞ を設定
    d[s] = 0
    優先度付きキュー PQ に (0, s) を挿入

    while PQ が空でない:
        (cost, u) = PQ.pop_min()
        if cost > d[u]: continue  // 古い情報は無視
        for u に隣接する各辺 (v, w):
            if d[u] + w < d[v]:
                d[v] = d[u] + w
                PQ.push((d[v], v))
```

**計算量**:
| 実装方式 | 計算量 |
|---------|--------|
| 隣接行列 + 線形スキャン | O(\|V\|²) |
| 隣接リスト + 優先度付きキュー | O((\|V\| + \|E\|) log\|V\|) |

**使用条件**: 辺の重みが非負（w ≥ 0）のグラフのみ

### Bellman-Ford のアルゴリズム（単一始点、負辺あり）

始点 s から全頂点への最短経路を求める。**負の辺も扱える**。負の閉路の検出も可能。

```
bellman_ford(G, s):
    全頂点 u に d[u] = ∞ を設定
    d[s] = 0

    for i = 1 to |V| - 1:        // |V|-1 回繰り返す
        for 各辺 (u, v, w) in E:
            if d[u] + w < d[v]:
                d[v] = d[u] + w

    // 負の閉路検出
    for 各辺 (u, v, w) in E:
        if d[u] + w < d[v]: return "NEGATIVE CYCLE"
```

**計算量**: O(|V| × |E|)

**負の閉路検出**: |V| 回目の緩和でさらに更新できる頂点が存在する場合、負の閉路がある。

### Floyd-Warshall のアルゴリズム（全点対間）

全頂点ペア間の最短経路を求める動的計画法。**負辺に対応、負の閉路も検出可能**。

**核心アイデア**: 中継頂点 k を 1 から |V| まで増やしながら、k のみを経由した場合の最短距離を更新する。

```
floyd_warshall(G):
    // 初期化: 直接の辺の重み、辺がない場合は ∞、自己ループは 0
    d[i][j] = w(i, j) if 辺あり
    d[i][j] = ∞       if 辺なし
    d[i][i] = 0

    for k = 0 to |V| - 1:
        for i = 0 to |V| - 1:
            for j = 0 to |V| - 1:
                d[i][j] = min(d[i][j], d[i][k] + d[k][j])

    // 負の閉路検出: d[i][i] < 0 の頂点が存在すれば負の閉路あり
```

**計算量**: O(|V|³)

**実装の注意**: `d[i][k] + d[k][j]` のオーバーフローに注意。`∞ + ∞` にならないよう `d[i][k] == ∞` のケースをスキップする。

## 高度なアルゴリズム

### トポロジカルソート（DAG）

閉路のない有向グラフ（DAG）の頂点を、すべての辺 (u, v) において u が v より先になるよう並べる。

**BFS による実装（カーン法）**:
```
topological_sort_bfs(G):
    全頂点の入次数 indeg[u] を計算
    indeg[u] == 0 の頂点をキュー Q に挿入

    while Q が空でない:
        u = Q.dequeue()
        result.append(u)
        for u に隣接する各頂点 v:
            indeg[v]--
            if indeg[v] == 0: Q.enqueue(v)
```

**DFS による実装**:
```
topological_sort_dfs(G):
    全頂点 u について color[u] = WHITE に初期化
    for 各頂点 s:
        if color[s] == WHITE: dfs(s)

    dfs(u):
        color[u] = GRAY
        for u に隣接する各頂点 v:
            if color[v] == WHITE: dfs(v)
        result.prepend(u)  // 完了した頂点をリスト先頭に追加
```

**計算量**: O(|V| + |E|)

**使用条件**: グラフが DAG（閉路なし）であること。閉路があるとソート不可能。

### 関節点（二重連結性）

無向グラフから除去するとグラフが非連結になる頂点。一度の DFS で全関節点を検出できる。

**核心アイデア**:
- `prenum[u]`: DFS での u の訪問順
- `lowest[u]`: u またはその子孫から back-edge（後退辺）で到達できる最小の prenum

**関節点の判定条件**:
1. DFS 木のルート r が子を **2つ以上** 持つ → r は関節点
2. ルート以外の頂点 p において、p の子 u に対して `prenum[p] ≤ lowest[u]` → p は関節点

```
find_articulation_points(G):
    dfs(u, parent):
        prenum[u] = lowest[u] = timer++
        child_count = 0
        for u に隣接する各頂点 v:
            if 未訪問:
                child_count++
                dfs(v, u)
                lowest[u] = min(lowest[u], lowest[v])
                // ルートでなく prenum[u] <= lowest[v] なら u は関節点
                if parent != -1 and prenum[u] <= lowest[v]:
                    articulation_points.add(u)
            elif v != parent:       // back-edge
                lowest[u] = min(lowest[u], prenum[v])
        if parent == -1 and child_count >= 2:
            articulation_points.add(u)  // ルートの特別処理
```

**計算量**: O(|V| + |E|)

### 木の直径

重み付き木（閉路なし連結グラフ）における最遠節点間の距離。

**アルゴリズム（2回 BFS/DFS）**:
1. 任意の頂点 s から最も遠い頂点 x を求める
2. x から最も遠い頂点 y を求める
3. d(x, y) が木の直径

```
tree_diameter(T):
    x = 任意の頂点 s から BFS で最遠頂点を求める
    y = x から BFS で最遠頂点を求める
    return d(x, y)
```

**計算量**: O(|V| + |E|)（BFS を2回）

**正当性**: x が木の直径の一端であることは、非負重みの木の性質から証明できる。

## アルゴリズム選択ガイド

| 問題の性質 | 推奨アルゴリズム |
|-----------|----------------|
| グラフ全探索 / 到達可能性 | DFS または BFS |
| 辺数ベースの最短距離（無重み） | BFS |
| 連結成分の特定 | DFS/BFS + ラベリング、または Union-Find |
| 最小全域木（疎なグラフ） | Kruskal（O(\|E\| log\|E\|)） |
| 最小全域木（密なグラフ） | Prim（O(\|V\|²) または O(\|E\| log\|V\|)） |
| 単一始点最短経路（非負辺） | Dijkstra |
| 単一始点最短経路（負辺あり） | Bellman-Ford |
| 全点対最短経路 | Floyd-Warshall |
| 全点対最短経路（負辺なし、疎なグラフ） | 各頂点から Dijkstra を \|V\| 回 |
| DAG の処理順序 | トポロジカルソート |
| グラフの脆弱点（切断頂点） | 関節点（DFS 応用） |
| 木の端点間の最長距離 | 木の直径（BFS 2回） |
| 辺を動的追加しながら連結判定 | Union-Find |

### グラフ表現の選択

| 状況 | 推奨表現 |
|------|---------|
| \|V\| ≤ 1,000 かつ辺が密 | 隣接行列 |
| \|V\| が大きい または辺が疎 | 隣接リスト |
| Floyd-Warshall を使う | 隣接行列 |
| DFS/BFS + 大規模グラフ | 隣接リスト |

## よくある落とし穴

| 状況 | 注意点 |
|------|--------|
| Dijkstra を負辺グラフに使う | 誤った結果になる。Bellman-Ford を使うこと |
| Floyd-Warshall の初期化 | `d[i][i] = 0`、辺なしは `∞`、オーバーフローに注意 |
| BFS で最短経路（重み付き） | BFS は無重みグラフのみ。重みがあれば Dijkstra |
| 疎なグラフに隣接行列 | O(\|V\|²) のメモリ消費で TLE/MLE のリスク |
| トポロジカルソートの前提 | 閉路が存在する場合は不可能（検出は indeg が 0 にならない頂点の存在で確認） |
| 負の閉路がある最短経路 | 最短経路が定義できない。Floyd-Warshall で `d[i][i] < 0` を確認 |
