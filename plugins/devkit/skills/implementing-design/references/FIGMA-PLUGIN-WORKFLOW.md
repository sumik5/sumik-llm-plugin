# プラグイン活用ワークフロー

Figmaのプラグインを活用して、リアルなダミーデータをデザインに組み込むパターン集。

---

## 1. プラグイン活用の原則

デザインの完成度を高めるためには、**実際のデータに近いダミーデータ**を早い段階から使用することが重要。

**なぜリアルなデータが必要か:**
- 「ユーザー名」「プロフィール写真」などのプレースホルダー表示ではレイアウトの問題が見えにくい
- テキストの長さや折り返しの挙動は、実データに近い内容でないと発見できない
- ステークホルダーへのレビューや確認で説得力が増す
- ハンドオフ前に実装上の課題（テキスト分割の必要性など）を発見できる

**活用タイミング:**
- ワイヤーフレームが完成し、コンポーネントへのデータ流し込みが始まる段階
- プロトタイプを関係者にレビューする前
- ハンドオフ前の最終確認フェーズ

## 2. 写真素材プラグイン（Unsplash）

フリー写真素材をFigmaのオブジェクトに直接配置できるプラグイン。

### 基本的な使い方

1. 写真を配置したいオブジェクトを選択（フレームまたは画像オブジェクト）
2. リソースパネル（`Shift` + `1`）でプラグインを検索・実行
3. **Searchタブ**でキーワード検索 → 結果をクリックして配置

```
[Unsplash] Search タブ
 → キーワード入力（例: "face", "nature", "food"）
 → 検索結果クリック → 選択オブジェクトの塗りに写真が設定される
```

### Presets による一括配置

複数のオブジェクトを選択した状態でPresetsを使うと、一括でランダム写真を配置できる。

```
[Unsplash] Preset タブ
 → カテゴリ選択（例: "portrait"）→ クリック
 → 選択中のすべてのオブジェクトにランダム写真が配置される
```

**ポイント:**
- 複数選択は `⌘` + クリック（深い階層のオブジェクトは `⌘` + `Shift` + クリック）
- 元画像は解像度が高くデータ容量が大きいが、**プラグインが自動的に適切なサイズに縮小**して読み込むため容量を気にせず使える
- 配置後は塗りの設定（フィット/フィル/クロップ等）を確認して調整する

## 3. ダミーデータプラグイン（Content Reel）

名前・住所・電話番号・メールアドレスなどのテキストデータをテキストオブジェクトに自動挿入するプラグイン。

### 基本的な使い方

1. データを挿入したいテキストオブジェクトを選択（複数選択可）
2. リソースパネルからContent Reelを起動
3. **Textタブ**でデータカテゴリを検索・選択
4. **[Apply All]** をクリックして選択中のすべてのテキストに適用

```
[Content Reel]
 ├── Text タブ: テキストデータ（名前/住所/電話番号/メールなど）
 └── Image タブ: 画像データ（アバター画像など）
```

### 主要なデータカテゴリ

| カテゴリ | 用途例 |
|---------|-------|
| First Name / Full Name | ユーザー名、投稿者名 |
| Address | 住所表示 |
| Phone | 電話番号フィールド |
| Email | メールアドレスフィールド |
| Company | 企業名 |

**国籍・性別対応:**
データによっては「Japanese Female Name」のように属性を絞った検索が可能。写真（Unsplash）と組み合わせて整合性を取ることで、より自然なデモになる。

**ログイン機能:**
[Sign In] からサービスにログインすると、自分でカスタムデータセットを作成してコミュニティと共有できる。チーム固有のデータ（商品名リスト、架空のユーザー名など）を共有する場合に活用する。

## 4. 「貼り付けて置換」パターン

インスタンスの上書き内容（写真・テキストの差し替え）は**個別インスタンスごとに独立**している。同じコンポーネントのインスタンスが複数箇所に存在する場合、片方を編集しても他方には反映されない。

### 使い方

1. 上書き済みのインスタンスをコピー（`⌘` + `C`）
2. 置き換え先のインスタンスを右クリック → **「貼り付けて置換」** を選択

| 操作 | Mac | Windows |
|------|-----|---------|
| コピー | `⌘` + `C` | `Ctrl` + `C` |
| 貼り付けて置換 | `⌘` + `Shift` + `R` | `Ctrl` + `Shift` + `R` |

### ユースケース

- **Flowをまたいだデザイン同期**: Flow 1で編集したインスタンスを、Flow 2の対応するインスタンスに貼り付けて置換することで、複数Flowのデザインを一致させる
- **部分的なリセット**: 上書きの少ないインスタンスをコピーして、上書きが多すぎるインスタンスをリセットする

## 5. データ挿入の注意点

### テキストオブジェクト分割の問題

Content Reelはテキストオブジェクト全体を置き換えるため、**固定テキストとダミーデータが混在するオブジェクト**に使用すると意図しない上書きが発生する。

**例:** "○○さんにLIKEされました" というテキストオブジェクト
- Content Reelで名前を挿入 → "田中 太郎" のように「さんにLIKEされました」が消える

**解決方法:** テキストオブジェクトを分割する

```
NG: [○○さんにLIKEされました] ← 1つのオブジェクト

OK: [○○] + [さんにLIKEされました] ← 2つのオブジェクトに分割
        ↑ Content Reel適用対象    ↑ 固定テキスト（変更しない）
```

**設計上の考慮点:**
- テキスト分割はAuto Layoutで横並びにすることで自然なレイアウトを保てる
- 分割対象が多い場合はハンドオフ前に優先度を決め、重要な画面だけ対応する
- 完全な対応が困難な場合は、ハンドオフドキュメントにその旨を明記しておく

## 6. その他の推奨プラグイン

### Artboard Studio Mockups

デザインをデバイスフレームや実写シーンにはめ込んだ**モックアップを自動生成**するプラグイン。

- **用途**: プレゼン資料、ポートフォリオ、クライアント向けレビュー
- **手順**: フレームを選択してプラグインを実行 → テンプレートを選ぶだけで完成
- モックアップ上の画像がリンクされているため、デザイン更新時に再実行するだけで最新状態に反映

### Figmotion

Figma上で**キーフレームアニメーション**を作成するプラグイン。

- **用途**: マイクロインタラクションの動作確認、アニメーションGIFの生成
- Figmaのプロトタイプでは表現しにくい細かいイージング・タイミングを調整できる
- アニメーションGIFとして書き出して、Slackやドキュメントで共有可能
- 長時間・複雑なアニメーションよりも、UIの小さなアクション（ボタン押下、ローディング）向き

### Google Sheets Sync

Googleスプレッドシートのデータを**セルとレイヤーを対応付けて**Figmaに挿入するプラグイン。

- **用途**: 商品一覧、ユーザーリスト、価格表など構造化されたデータの流し込み
- Content Reelとの違い: 任意のデータセットを自分で定義でき、カラム名でレイヤーと紐付けるためより精密
- データが用意できるプロジェクトではContent Reelよりも再利用性が高い
- スプレッドシートを更新して再同期するだけでFigmaのデザインも更新される

## 7. プラグイン自作の可能性

FigmaはAPIドキュメントが充実しており、TypeScript/JavaScriptの知識があれば独自プラグインを作成できる。

**自作が有効なケース:**
- チーム固有のデザインルールを自動チェックしたい（カラースタイル未適用の検出など）
- 社内システムやAPIとFigmaを連携させたい
- 繰り返し行う作業（命名規則の一括変更、スタイルの移行など）を自動化したい

**進め方の目安:**
1. Figmaコミュニティで似たプラグインを探して参考にする
2. 開発工数が高い場合はエンジニアやマネージャーへの相談を先行させる
3. 小さな機能から始めてチーム内で試用し、フィードバックを得てから公開を検討する
