## クラウド & オーケストレーション（Kubernetes）

Kubernetesクラスターとワークロードの堅牢化: アイデンティティ、ポリシー、ネットワーキング、機密情報、サプライチェーンのコントロール。

### コントロール
- アイデンティティ & RBAC: ユーザーとサービスアカウントへの最小権限。ネームスペースを分離する。必要なロールのみをバインドする。
- ポリシー: イメージソース、ケーパビリティ、root実行、ネットワークポリシー、必須ラベル/アノテーションのためのアドミッションコントロール（OPA/Gatekeeper/Kyverno）。
- ネットワーキング: ネットワークポリシーによるデフォルト拒否。明示的なエグレスアローリスト。該当する場合はメッシュ内のサービスアイデンティティ/mTLS。
- 機密情報: KMSプロバイダーを使用する。マニフェストへの平文記述を避ける。定期的にローテーションする。機密情報のマウントパスを制限する。
- ノード: 堅牢化されたOS、自動更新、最小限の攻撃面。テイント/トレランションと専用ノードでセンシティブなワークロードを分離する。
- サプライチェーン: イメージ署名を検証する。アドミッションでの証明（SLSA/Sigstore）を強制する。

### チェックリスト
- チーム/アプリごとのネームスペース。スコープされたRBACロール。監査ロギングを有効化する。
- アドミッションポリシーがイメージプロビナンス、非root実行、ドロップされたケーパビリティ、読み取り専用ルートFS、ネットワークポリシーの存在を強制する。
- イングレス/エグレスのネットワークポリシーを配置する。サービスアカウントはデプロイメントごとにスコープする。

### 検証
- クラスター準拠とCISベンチマークスキャン。
- マニフェストのCI内でのポリシーテスト（OPAユニットテスト）。定期的なアドミッションドライラン。

### インシデント対応準備
- 監査ログを有効化して一元化する。etcdへのアクセスを制限する。バックアップ/リストアをテストする。
- MFAと時間制限付き承認を持つブレークグラスロールを定義する。
