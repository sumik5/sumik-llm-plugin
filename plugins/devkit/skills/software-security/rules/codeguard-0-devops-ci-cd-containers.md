## DevOps、CI/CD、コンテナ

ビルド・パッケージング・デプロイのサプライチェーンを保護する: パイプラインと成果物を保護し、コンテナを堅牢化し、必要に応じて仮想パッチとツールチェーンフラグを使用する。

### CI/CDパイプライン セキュリティ
- リポジトリ: 保護ブランチ、必須レビュー、署名付きコミットを設定する。
- 機密情報: 絶対にハードコードしない。ランタイムでvault/KMSから取得し、ログではマスクする。
- 最小権限: 最小限のパーミッションを持つ使い捨て・分離されたランナーを使用する。
- CIのセキュリティゲート: SAST、SCA、DAST、IaCスキャンを設け、クリティカルな問題ではビルドをブロックする。
- 依存関係: ロックファイルで固定し、整合性を検証し、プライベートレジストリを使用する。
- すべてに署名する: コミットと成果物（コンテナ/jar）に署名し、デプロイ前に検証する。SLSAプロベナンスを採用する。

### DockerとコンテナHardening
- ユーザー: 非rootで実行し、Dockerfileに`USER`を設定する
- `--security-opt=no-new-privileges`を使用して権限昇格を防止する。
- Capabilities: `--cap-drop all`を設定し、必要なものだけを追加する。`--privileged`は絶対に使用しない。
- デーモンソケット: `/var/run/docker.sock`を絶対にマウントしない
- TLSなしでTCP Dockerデーモンソケット（`-H tcp://0.0.0.0:XXX`）を有効化してはならない。
- docker-composeファイルで`- "/var/run/docker.sock:/var/run/docker.sock"`の使用を避ける。
- ファイルシステム: 読み取り専用ルート、一時書き込みにはtmpfsを使用し、リソース制限（CPU/メモリ）を設定する。
- ネットワーク: ホストネットワークを避け、カスタムネットワークを定義し、公開ポートを最小化する。
- イメージ: 最小ベース（distroless/alpine）を使用し、タグとダイジェストを固定し、最終イメージからパッケージマネージャーとツールを除去し、`HEALTHCHECK`を追加する。
- 機密情報: Docker/Kubernetesシークレットを使用し、レイヤー/環境変数には絶対に含めず、ランタイムシークレットでマウントする。
- スキャン: ビルド時とアドミッション時にイメージをスキャンし、高深刻度の脆弱性をブロックする。

### コンテナ内のNode.js
- 決定論的ビルド: `npm ci --omit=dev`を使用し、ダイジェストでベースイメージを固定する。
- 本番環境: `ENV NODE_ENV=production`を設定する。
- 非root: 正しい所有権でコピーし、`USER node`に降格する。
- シグナル: initプロセス（例: `dumb-init`）を使用し、グレースフルシャットダウンハンドラーを実装する。
- マルチステージビルド: ビルドとランタイムを分離し、BuildKit経由で機密情報をマウントし、`.dockerignore`を使用する。

### 仮想パッチ（一時的な緩和策）
- コード修正がまだできない場合に即時保護として WAF/IPS/ModSecurity を使用する。
- 精度のために肯定的なセキュリティルール（許可リスト）を優先し、エクスプロイト固有のシグネチャは避ける。
- プロセス: 事前にツールを準備し、CVEを分析し、最初はログのみモードでパッチを実装してから強制し、コード修正後に追跡・廃止する。

### C/C++ ツールチェーン堅牢化（該当する場合）
- コンパイラ: `-Wall -Wextra -Wconversion`、`-fstack-protector-all`、PIE（`-fPIE`/`-pie`）、`_FORTIFY_SOURCE=2`、CFI（`-fsanitize=cfi`とLTO）を使用する。
- リンカ: RELRO/now、noexecstack、NX/DEP、ASLRを設定する。
- デバッグ vs リリース: デバッグでサニタイザーを有効化し、リリースで堅牢化フラグを有効化し、assertはデバッグのみとする。
- CIチェック: フラグを検証し（`checksec`）、保護が不足している場合はビルドを失敗させる。

### 実装チェックリスト
- パイプライン: vaultに機密情報、使い捨てランナー、セキュリティスキャン、プロベナンス付き署名済み成果物を設定する。
- コンテナ: 非root、最小権限、読み取り専用FS、リソース制限を設定し、デーモンソケットをマウントしない。
- イメージ: 最小構成、固定、スキャン済みにし、ヘルスチェックを設定し、`.dockerignore`を維持する。
- Nodeイメージ: `npm ci`、`NODE_ENV=production`、適切なinitとシャットダウンを設定する。
- 仮想パッチ: 定義済みプロセス、正確なルール、ログ、修正後の廃止を設定する。
- ネイティブビルド: 堅牢化フラグを有効化してCIで検証する。
