## 依存関係・サプライチェーンセキュリティ

選定とピン留めから出所証明、スキャン、迅速な対応まで、エコシステム全体にわたるサードパーティリスクを管理する。

### ポリシーとガバナンス
- 許可リスト化されたレジストリとスコープを維持し、信頼できないソースからの直接インストールを禁止する。
- ロックファイルとバージョンのピン留めを必須とし、イメージとベンダー管理アセットにはダイジェストピン留めを優先する。
- アプリ/イメージのSBOMを生成してアーティファクトと共に保管し、出所証明を行う（SLSA、Sigstore）。

### パッケージの衛生管理（npmを中心に他のエコシステムにも適用）
- 定期的に監査（`npm audit`、エコシステムのSCA）してパッチを適用し、深刻度別にSLAを設定して強制する。
- 決定論的ビルドを使用する：CI/CDでは `npm install` ではなく `npm ci` を使用し、ロックファイルの一貫性を維持する。
- 可能な限りインストール時に実行されるインストールスクリプトを避け、リスクを評価する。
- `.npmrc` を使用してプライベートレジストリのスコープを設定し、ワイルドカードレジストリを避け、整合性検証を有効にする。
- 公開アカウントの2FAを有効にする

### 開発プラクティス
- 依存関係のフットプリントを最小化し、未使用パッケージを削除し、些細なタスクには標準ライブラリ/ファーストパーティを優先する。
- タイポスクワッティングとプロテストウェアへの対策：メンテナーをピン留めし、リリースを監視し、出所証明チェックを使用する。
- ハーメティックビルド：必要でない限りコンパイル/パッケージングステージではネットワークを遮断し、認証チェックを伴うキャッシュを使用する。

### CI/CD統合
- SCА、SAST、IaCスキャンをゲートに組み込み、クリティカル検出時は失敗させ、補完的コントロールを伴う承認を必須とするオーバーライドを要求する。
- アーティファクトに署名し、デプロイ時に署名を検証し、アドミッションでポリシーを適用する。

### 脆弱性管理
- パッチ済み脆弱性の場合：アップデートをテストしてデプロイし、APIの破壊的変更をドキュメント化する。
- 未パッチ脆弱性の場合：CVEの種類に基づいて補完的コントロール（入力検証、ラッパー）を実装し、推移的な回避策より直接依存関係の修正を優先する。
- リスク判断をドキュメント化し、ビジネス上の正当性を添えて適切な権限者へのリスク受容を申請する。

### インシデント対応
- 迅速なロールバックを維持し、侵害されたパッケージを隔離し、ロールアウトを絞り込み、ステークホルダーに通知する。
- 脅威インテリジェンスフィード（例：npmアドバイザリー）を監視し、クリティカルなCVEに対してチケットを自動作成する。

### 実装チェックリスト
- ロックファイルが存在すること。整合性チェックが有効であること。プライベートレジストリが設定されていること。
- SBOM＋出所証明が保管されていること。デプロイ前に署名が検証されていること。
- テストとレビューゲートを伴う依存関係の自動更新が設定されていること。
- 高深刻度の脆弱性がSLA内に修正されているか、軽減策が講じられてドキュメント化されていること。
