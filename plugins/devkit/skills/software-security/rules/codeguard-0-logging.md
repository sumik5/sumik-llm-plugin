## ロギングと監視

機密情報を露出させることなく、検知・対応・フォレンジックを支援する、構造化されたプライバシー配慮のテレメトリを生成する。

### ログに記録すべき内容
- 認証/認可イベント、管理者操作、設定変更、機密データへのアクセス、入力検証の失敗、セキュリティエラー。
- 相関ID/リクエストID、ユーザー/セッションID（非PII）、ソースIP、ユーザーエージェント、タイムスタンプ（UTC、RFC3339）を含める。

### ログの記録方法
- 安定したフィールド名を持つ構造化ログ（JSON）を使用し、重要なシグナルに自由形式テキストを避ける。
- ログインジェクションを防ぐためすべてのログ入力をサニタイズする（CR/LF/デリミタを除去）。他の信頼ゾーンからのデータを検証する。
- 機密情報と機密フィールドをリダクション/トークン化する。認証情報、トークン、リカバリーコード、生のセッションIDは絶対にログに記録してはならない。
- 完全性を確保する：追記専用またはWORMストレージ、改ざん検知、集中型集約、アクセス制御と保持ポリシー。

### 検知とアラート
- 認証異常（認証情報スタッフィングパターン、不可能な移動）、権限変更、過剰な失敗、SSRFの指標、データ漏洩パターンに対するアラートを構築する。
- しきい値を調整し、ランブックを提供し、オンコール対応を確保し、アラートフローをテストする。

### ストレージと保護
- ログストレージを隔離する（別パーティション/データベース）。ファイル/ディレクトリのアクセス権限を厳格に設定する。Webアクセス可能な場所の外に保管する。
- システム間で時刻を同期する。送信には安全なプロトコルを使用する。改ざん検知と監視を実装する。

### プライバシーとコンプライアンス
- データインベントリと分類を維持し、ログ内の個人データを最小化し、保持・削除ポリシーを遵守する。
- ポリシーで要求される場合、ユーザーに紐づくログデータの追跡・削除を行うためのメカニズムを提供する。

### 実装チェックリスト
- JSONロギングが有効であること。ログインジェクションのサニタイズが有効であること。リダクションフィルターが有効であること。すべてのリクエストに相関IDが付与されていること。
- 改ざん検知を伴う隔離されたログストレージ。完全性保護を備えた集中型ログパイプライン。保持期間が設定されていること。
- セキュリティアラートが定義されてテストされていること。ダッシュボードとレポートが整備されていること。

### 検証
- ユニット/インテグレーションテストで主要フィールドの存在/不在を検証する。リダクションのユニットテストを実施する。
- 機密情報/PII漏洩の定期監査を行う。インシデントワークフローのテーブルトップ演習を実施する。
