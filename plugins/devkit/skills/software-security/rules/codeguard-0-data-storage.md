## データベースセキュリティガイドライン

このルールは、データ侵害と不正アクセスから保護するために SQL および NoSQL データベースを安全に設定する方法を示す：

- バックエンドデータベース保護
  - データベースサーバーを他のシステムから分離し、ホスト接続を制限する。
  - 可能な場合はネットワーク（TCP）アクセスを無効化し、ローカルソケットファイルまたは名前付きパイプを使用する。
  - 必要に応じてデータベースを localhost のみにバインドするよう設定する。
  - ファイアウォールルールで特定のホストへのネットワークポートアクセスを制限する。
  - アプリケーションサーバーから分離された別の DMZ にデータベースサーバーを配置する。
  - シッククライアントからバックエンドデータベースへの直接接続を絶対に許可しない。

- トランスポート層セキュリティ
  - 暗号化された接続のみを許可するようデータベースを設定する。
  - データベースサーバーに信頼できるデジタル証明書をインストールする。
  - クライアント接続には TLSv1.2+ と最新の暗号スイート（AES-GCM、ChaCha20）を使用する。
  - クライアントアプリケーションでデジタル証明書の有効性を検証する。
  - 初期認証のみならず、すべてのデータベーストラフィックが暗号化されていることを確認する。

- 安全な認証設定
  - ローカルサーバー接続を含め、常に認証を要求する。
  - 強力でユニークなパスワードでアカウントを保護する。
  - アプリケーションまたはサービスごとに専用アカウントを使用する。
  - 最小限必要な権限のみを設定する。
  - アカウントと権限を定期的にレビューする。
  - アプリケーションが廃止された際はアカウントを削除する。
  - スタッフが離職した場合や侵害が疑われる場合はパスワードを変更する。

- データベース認証情報の保管
  - アプリケーションのソースコードに認証情報を絶対に保管しない。
  - Web ルート外の設定ファイルに認証情報を保管する。
  - 認証情報アクセスに適切なファイルパーミッションを設定する。
  - 認証情報ファイルをソースコードリポジトリに絶対にチェックインしない。
  - 利用可能な場合は組み込み機能を使用して認証情報ストレージを暗号化する。
  - 環境変数または機密情報管理ソリューションを使用する。

- 安全な権限管理
  - すべてのデータベースアカウントに最小権限の原則を適用する。
  - 組み込みの root、sa、SYS アカウントを使用しない。
  - アプリケーションアカウントに管理者権限を付与しない。
  - アカウント接続を許可されたホストのみに制限する。
  - 開発、UAT、本番環境で別々のデータベースとアカウントを使用する。
  - 必要な権限のみを付与する（必要に応じて SELECT、UPDATE、DELETE）。
  - 権限昇格を防ぐため、アカウントをデータベースオーナーにすることを避ける。
  - 必要に応じてテーブルレベル、カラムレベル、行レベルの権限を実装する。

- データベース設定とハードニング
  - 必要なセキュリティアップデートとパッチを定期的にインストールする。
  - 低権限ユーザーアカウントでデータベースサービスを実行する。
  - デフォルトアカウントとサンプルデータベースを削除する。
  - トランザクションログをメインデータベースファイルとは別のディスクに保存する。
  - 適切な権限で定期的な暗号化データベースバックアップを設定する。
  - 不要なストアドプロシージャと危険な機能を無効化する。
  - データベースアクティビティの監視とアラートを実装する。

- プラットフォーム固有のハードニング
  - SQL Server：xp_cmdshell、CLR 実行、SQL Browser サービス、混合モード認証（必要な場合を除く）を無効化する。
  - MySQL/MariaDB：mysql_secure_installation を実行し、ユーザーの FILE 権限を無効化する。
  - PostgreSQL：PostgreSQL セキュリティドキュメントのガイドラインに従う。
  - MongoDB：MongoDB セキュリティチェックリストの要件を実装する。
  - Redis：Redis セキュリティガイドの推奨事項に従う。

まとめ：  
データベースシステムを分離し、暗号化された接続を強制し、強力な認証を実装し、機密情報管理を使用して認証情報を安全に保管し、最小権限を適用し、データベース設定をハードニングし、定期的なセキュリティアップデートと監視を維持する。
